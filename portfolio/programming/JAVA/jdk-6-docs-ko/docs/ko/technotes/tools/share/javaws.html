javaws 커멘드행

 

NAME

javaws - Java Web Start 기동 커멘드

형식

javaws [run-options] <jnlp>

javaws [control-options]

파라미터

[run-options]

커멘드 유키자네행 옵션. 실행 옵션은 임의의 순서로 지정할 수 있습니다. 각종 옵션에 대해서는, 아래의 「실행 옵션」을 참조해 주세요.

<jnlp>

JNLP (Java Network Launching Protocol) 파일의 패스 또는 URL (Uniform Resource Locator)의 어느 쪽인지입니다.

[control-options]

커멘드행 제어 옵션. 제어 옵션은 임의의 순서로 지정할 수 있습니다. 각종 옵션에 대해서는, 아래의 「제어 옵션」을 참조해 주세요.

설명

javaws 커멘드는, JNLP (Java Network Launching Protocol)의 레퍼런스 구현인 Java Web Start 를 기동합니다. Java Web Start 는, 네트워크상에서 동작하는 Java 어플리케이션 또는 애플릿을 기동합니다.

JNLP 파일을 지정하면(자),javaws 는 JNLP 파일로 지정된 Java 어플리케이션 또는 애플릿을 기동합니다.

javaws 기동 툴에는, 현재의 릴리스로 지원되고 있는 1 조의 옵션이 있습니다. 다만, 이러한 옵션은 장래의 릴리스에서는 삭제될 가능성이 있습니다.

실행 옵션

-offline

Java Web Start 를 오프 라인 모드로 실행합니다.

-Xnosplash

최초의 스플래쉬 화면의 표시를 무효로 합니다.

-open <arguments>

이 옵션을 지정하면(자), JNLP 파일내의 인수가 -open <arguments> 에 옮겨집니다.

-print <arguments>

이 옵션을 지정하면(자), JNLP 파일내의 인수가 -print <arguments> 에 옮겨집니다.

-online

온라인 모드로 실행합니다 (디폴트의 동작).

-wait

이 옵션을 지정하면(자), 어플리케이션이 종료할 때까지 javaws 프로세스는 종료하지 않습니다.

-verbose

추가의 출력을 표시합니다.

-J<option>

vm 에 옵션을 추가합니다.

-system

어플리케이션을 시스템 캐쉬로부터 마셔 실행합니다.

제어 옵션

-viewer

Java 컨트롤 패널로 캐쉬 viewer를 표시합니다.

-userConfig <property name>

지정된 배치 프로퍼티을 클리어 합니다.

-userConfig <property name> <property value>

지정된 배치 프로퍼티이 지정된 값으로 설정합니다.

-uninstall

캐쉬로부터 모든 어플리케이션을 삭제합니다.

-uninstall <jnlp>

캐쉬로부터 어플리케이션을 삭제합니다.

-import [import-options] <jnlp>

캐쉬에 어플리케이션을 임포트 합니다.

임포트 옵션

-silent

사일런트 모드로 임포트 합니다 (UI 는 표시되지 않습니다).

-system

시스템 캐쉬에 어플리케이션을 임포트 합니다.

-codebase <url>

임의의 codebase 로부터 자원을 가져옵니다.

-shortcut

사용자 허용 prompt와 같이 쇼트 컷을 인스톨 합니다. 이 옵션은,-silent 옵션과 함께 사용하지 않으면 효과가 없습니다.

-association

사용자 허용 prompt와 같이 관련짓고를 인스톨 합니다. 이 옵션은,-silent 옵션과 함께 사용하지 않으면 효과가 없습니다.

파일

사용자 캐쉬, 시스템 캐쉬, 및 deployment.properties 파일에 대해서는,시스템 레벨 및 사용자 레벨의 프로퍼티을 참조해 주세요.

상세 정보

Java Web Start 의 자세한 것은,「Java Web Start」를 참조해 주세요.
