serialver - 시리얼 버젼 커멘드

serialver 커멘드는 serialVersionUID 를 돌려줍니다.

형식

serialver [ options ] [ classnames ]
options
이 문서로 설명되고 있는 커멘드행 옵션입니다.
classnames
1 개(살) 이상의 클래스명입니다.

설명

serialver 는, 1 개(살) 이상의 클래스의 serialVersionUID 를, 전개하고 있는 클래스에 카피하는데 적합한 형식에서 돌려줍니다. 인수를 지정하지 않고 호출하면(자), 사용 방법이 표시됩니다.

옵션

-classpath <: 로 단락지어진 디렉토리와 zip 나 jar 파일 >
어플리케이션의 클래스 및 자원의 검색 패스를 설정합니다.
-show
간단한 사용자 인터페이스를 표시합니다. 완전 지정의 클래스명을 입력해, Enter 키나 [Show] 버튼을 눌러, 시리얼 버젼 UID 를 표시합니다.
-Joption
Java 가상 머신에 option 를 건네줍니다. option 에는,「Java 어플리케이션 기동 툴」의 레퍼런스 페이지에 기재되어 있는 옵션을 1 개 지정합니다. 예를 들어,-J-Xms48m 와 지정하면(자), 스타트 업 메모리는 48M 바이트로 설정됩니다.

주

serialver 커멘드는, 지정된 클래스를 그 가상 머신내에 읽어들여 초기화합니다만, 디폴트에서는 시큐리티 매니저의 설정은 실시하지 않습니다. 신뢰할 수 없는 클래스와 함께 serialver 를 실행하는 경우에는, 다음의 옵션을 사용해 시큐리티 매니저를 설정할 수 있습니다.

-J-Djava.security.manager

또, 필요하면, 다음의 옵션을 사용해 시큐리티 정책를 지정할 수 있습니다.

-J-Djava.security.policy=<policy file>
