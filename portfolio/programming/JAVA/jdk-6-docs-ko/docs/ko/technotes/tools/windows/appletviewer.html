appletviewer - Java 애플릿 viewer

appletviewer 커멘드는 Web 브라우저의 밖에서 애플릿을 실행시킵니다.

형식

appletviewer [ options ] urls ...

설명

appletviewer 커멘드는 urls 로 지정된 문서 혹은 자원과 접속해, 그 문서가 참조하는 각각의 애플릿을 독자적인 윈도우로 표시합니다. 주:urls 에 의해 참조된 문서가,OBJECT,EMBED, 또는 APPLET 태그로 어느 애플릿도 참조하고 있지 않는 경우,appletviewer 는 아무것도 실시하지 않습니다. appletviewer 로 지원되는 HTML 태그의 상세한 것에 대하여는,「애플릿 viewer의 태그」를 참조해 주세요.

Java 2 JDK 에는, 다음의 2 개의 Java 가상 머신 구현이 짜넣어지고 있습니다.

주:appletviewer 는, RFC2396 로 규정된 이스케이프 기구에 따라 encode 된 URL 를 필요로 합니다. 지원되는 것은, encode 된 URL 만입니다. 다만, 파일명에 대해서는, RFC2396 로 지정된 방법으로 encode를 해제해 둘 필요가 있습니다.

  • Java HotSpotTM Client VM
    Java HotSpot Client VM 는, 디폴트의 가상 머신입니다. Java HotSpot 테크놀러지를 사용하는 것으로써, Classic VM 에 비해, 보다 뛰어난 퍼포먼스를 제공합니다. Classic VM 를 호출할 때는 특별한 커멘드행 옵션을 사용합니다만 (이후의 설명을 참조),appletviewer 에서는 항상 애플릿이 기동되어 Classic VM 상에서 동작합니다.
  • Java 2 Classic VM
    Java 2 Classic VM 는, Java 2 SDK 버젼 1.2 의 가상 머신의 구현과 기본적으로 같습니다. 다음의 예에와 같이,-classic 커멘드행 옵션을 사용해 호출합니다. 
    appletviewer -classic MyApplet.html

옵션

-debug
Java 디버거 jdb 로 애플릿 viewer를 개시합니다. 이것에 의해, 문서중의 애플릿을 디버그 할 수가 있습니다.
-encoding   encoding name
입력 HTML 파일의 인코딩명을 지정합니다.
-Jjavaoption
캐릭터 라인 javaoption 는, appletviewer 를 실행하는 Java interpreter에 1 개의 인수로서 건네받습니다. 인수에 스페이스를 포함해서는 안됩니다. 복수의 인수는, 각 인수의 모든 것을 접두사 -J 로 시작하는 것으로 구분할 필요가 있습니다. 이것은, 컴파일러의 실행 환경, 또는 메모리의 이용에 유효합니다.
