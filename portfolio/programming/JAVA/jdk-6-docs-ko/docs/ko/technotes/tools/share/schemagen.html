schemagen - XML 바인드를 위한 JavaTM 아키텍쳐(architecture) schema 제네레이터

스펙 버젼: 2.0
1.3 레퍼런스 구현 (RI) 버젼: 2.0 ea3

schemagen 의 기동

schema 제네레이터는, 사용자의 플랫폼의 bin 디렉토리에 있는 schemagen 셸 스크립트를 사용해 기동할 수 있습니다.

현재의 schema 제네레이터는, Java 원시 파일과 클래스 파일의 모두 처리할 수 있습니다.

또, schema 제네레이터를 실행하기 위한 Ant 태스크도 준비되어 있습니다. schemagen 를 Ant 와 함께 사용하기 위한 순서를 참조해 주세요.

Solaris/Linux 의 경우

      % path/to/jaxb/bin/schemagen.sh Foo.java Bar.java ...
      Note: Writing schema1.xsd

WindowsNT/2000/XP 의 경우

      > path\to\jaxb\bin\schemagen.bat Foo.java Bar.java ...
      Note: Writing schema1.xsd

사용자의 Java 소스/클래스가 다른 클래스를 참조하고 있는 경우, 시스템의 CLASSPATH 환경 변수 경유로 그러한 클래스에 액세스 할 수 있게 되어 있든가, 혹은 -classpath/-cp 옵션을 사용해 그러한 클래스를 툴로 지정할 필요가 있습니다. 그렇게 하지 않으면 schema의 생성시에 에러가 발생합니다.

커멘드행 옵션

사용 방법: schemagen [-options ...] <java files> 

옵션: 
    -d <path>         :  프로세서와 javac 생성의 클래스 파일을 두는 장소를 지정합니다.
    -cp <path>        :  사용자 지정 파일의 검색 장소를 지정합니다.
    -classpath <path> :  사용자 지정 파일의 검색 장소를 지정합니다.
    -help             :  이 사용 방법에 관한 메세지를 표시합니다.

생성되는 resource file

현재의 schema 제네레이터는 단순하게, Java 클래스내에서 참조되고 있는 이름 공간 마다 1 개의 schema 파일을 작성합니다. 생성되는 schema 파일의 이름을 제어하는 방법은, 현시점에서는 존재하지 않습니다. 그러한 목적으로는,schema 제네레이터의 ant 태스크를 사용해 주세요.
