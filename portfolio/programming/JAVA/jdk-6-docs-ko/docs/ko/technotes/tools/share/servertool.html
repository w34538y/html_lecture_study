servertool - Java(TM) IDL 서버 툴

servertool 는 어플리케이션 프로그래머가, 지속 서버의 등록, 등록 해제, 기동, 및 정지를 행하기 위한 커멘드행 인터페이스를 제공합니다.

형식

servertool -ORBInitialPort nameserverport options  [ commands  ]

커멘드를 입력하지 않고 servertool 를 기동하면(자), 커멘드행 툴로서 servertool > prompt가 표시됩니다. servertool > prompt에커멘드를 입력합니다.

커멘드를 입력해 servertool 를 기동하면(자), Java IDL 서버 툴이 기동해, 커멘드를 실행해 종료합니다.

-ORBInitialPort nameserverport 옵션은필수입니다. nameserverport 의 값에는,orbd 가 실행되어 착신 요구를 대기하고 있는 포트를 지정할 필요가 있습니다. Solaris 소프트웨어의 사용시는, 1024 보다 작은 포트로 프로세스를 개시하는 경우는, root 사용자가 될 필요가 있습니다. 이 때문에,nameserverport 로서 1024 또는 1024 보다 큰 포트를 사용하는 것을 추천합니다.

설명

servertool 는 어플리케이션 프로그래머가, 지속 서버의 등록, 등록 해제, 기동, 및 정지를 행하기 위한 커멘드행 인터페이스를 제공합니다. 그 외에, 서버에 관한 다양한 통계 정보를 취득하기 위한 커멘드도 제공합니다.

옵션

-ORBInitialHost nameserverhost
네임서버-가, 실행되어 착신 요구를 대기하고 있는 호스트 머신을 지정합니다. 이 옵션을 지정하지 않는 경우,nameserverhost 는 디폴트로 localhost 로 설정됩니다. orbd 와 servertool 가 다른 머신상에서 실행되고 있는 경우는,orbd 가 실행되고 있는 호스트의 이름과 IP 주소를 지정할 필요가 있습니다.

-Joption
Java 가상 머신에 option 를 건네줍니다. option 에는,「Java 어플리케이션 기동 툴」의 레퍼런스 페이지에 기재되어 있는 옵션을 1 개 지정합니다. 예를 들어,-J-Xms48m 와 지정하면(자), 스타트 업 메모리는 48M 바이트로 설정됩니다. -J 를 사용해 배후의 실행 환경에 옵션을 건네주는 것은 잘 행해지고 있습니다.

커멘드

register -server <server class name>  -classpath <classpath to server> [ -applicationName <application name> -args <args to server> -vmargs <flags to be passed to Java VM>  ]
Object Request Broker Daemon (ORBD)에 신규 지속 서버를 등록합니다. 서버가 미등록의 경우, 등록해 기동합니다. 이 커멘드에 의해,-server 옵션으로 식별되는 서버의 메인 클래스내에서 인스톨 메소드가 불려 갑니다. 이 인스톨 메소드는,public static void install(org.omg.CORBA.ORB) 가 되어 있을 필요가 있습니다. 인스톨 메소드는, 옵션이며, 데이타베이스 schema의 작성등의 독자적인 서버 인스톨 동작을 개발자가 지정할 수 있습니다.

unregister -serverid <server id > | -applicationName <application name>
서버 ID 또는 어플리케이션명을 사용해, ORBD 의 서버의 등록을 해제합니다. 이 커멘드에 의해,-server 옵션으로 식별되는 서버의 메인 클래스내에서 언인스톨 메소드가 불려 갑니다. 이 언인스톨 메소드는,public static void uninstall(org.omg.CORBA.ORB) 가 되어 있을 필요가 있습니다. 언인스톨 메소드는, 옵션이며, 인스톨 메소드의 동작의 취소등의 독자적인 서버 언인스톨 동작을 개발자가 지정할 수 있습니다.

getserverid -applicationName <application name>
어플리케이션에 관련지을 수 있고 있는 서버 ID 를 돌려줍니다.

list
ORBD 에 등록되어 모든 지속 서버에 관한 정보를 일람표 가리킵니다.

listappnames
현재 ORBD 에 등록되어 모든 서버의 어플리케이션명을 일람표 가리킵니다.

listactive
ORDB 에 의해 기동되어 현재 실행 떠날 수 있어 모든 지속 서버에 관한 정보를 일람표 가리킵니다.

locate -serverid <server id > | -applicationName <application name> [-endpointType <endpointType> ]
등록된 서버로 작성한 모든 ORB 의 특정의 형태에 대해 단 점 (포트)을 검출합니다. 서버가 실행되어 있지 않은 경우, 기동됩니다. 단 점의 형태가 지정되어 있지 않은 경우, 서버의 ORB 마다 관련지을 수 있고 있는 plain 형 또는 non-protected 형의 단 점이 돌려주어집니다.

locateperorb -serverid <server id > | -applicationName <application name> [-orbid <ORB name> ]
등록된 서버의 특정의 ORB 로 등록된 단 점 (포트)을 검출합니다. 서버가 실행되어 있지 않은 경우, 기동됩니다. orbid 가 지정되어 있지 않은 경우는, 디폴트 값의 「""」이orbid에 할당할 수 있습니다. ORB 가 공문자열의 orbid 로 작성되고 있는 경우, 등록한 포트가 모두 돌려주어집니다.

orblist -serverid <server id > | -applicationName <application name>
서버상에 정의된 ORB 의 ORBId 를 일람표 가리킵니다. ORBId 는 서버로 작성된 ORB 의 캐릭터 라인명입니다. 서버가 실행되어 있지 않은 경우, 기동됩니다.

shutdown -serverid <server id > | -applicationName <application name>
ORBD 에 등록된 액티브한 서버를 정지합니다. 이 커멘드의 실행중에,-serverid 파라미터 또는 -applicationName 파라미터로 지정된 클래스내에 정의된 shutdown() 메소드도 불려 가 서버 프로세스를 올바르게 정지합니다.

startup -serverid <server id > | -applicationName <application name>
ORBD 에 등록된 서버를 기동합니다. 서버가 실행되어 있지 않은 경우는, 이 커멘드로 서버를 기동합니다. 서버가 벌써 실행되고 있는 경우는, 사용자에게 에러 메세지가 돌려주어집니다.

help
서버가 서버 툴로 사용할 수 있는 모든 커멘드를 표시합니다.

quit
서버 툴을 종료합니다.
