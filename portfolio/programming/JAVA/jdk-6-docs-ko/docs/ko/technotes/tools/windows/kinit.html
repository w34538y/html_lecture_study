kinit - Kerberos 티켓 인가 티켓의 취득과 캐쉬

kinit 는, Kerberos 티켓 인가 티켓 (Ticket Granting Ticket, TGT)의 취득과 캐쉬에 사용됩니다. 이 툴의 기능은, SEAM 나 MIT 레퍼런스 구현 등, 다른 Kerberos 구현에 일반적으로 볼 수 있는 kinit 툴 과 유사합니다.

kinit 를 실행하기 전에, Key Distribution Center (KDC)를 사용해 사용자를 주체로서 등록할 필요가 있습니다.

형식

kinit [ commands ] <principal name> [<password>]

설명

Windows 플랫폼에서는 디폴트로,<USER_HOME>\krb5cc_<USER_NAME> 라고 하는 캐쉬 파일이 생성됩니다. <uid> 는, 시스템에 로그인하고 있는 사용자의 사용자 ID 번호입니다. 모든 Unix 플랫폼에서는 디폴트로,/tmp/krb5cc_<uid> 라고 하는 캐쉬 파일이 생성됩니다.

<USER_HOME> 은,java.lang.System 프로퍼티 user.home 로부터 취득됩니다. <USER_NAME> 은,java.lang.System 프로퍼티 user.name 로부터 취득됩니다. <USER_HOME> 가 null 의 경우, 캐쉬 파일은, kinit 를 실행한 커런트 디렉토리에 보존됩니다. <USER_NAME> 는, operating system의 로그인 사용자명입니다. 이 사용자명은, 사용자의 주체명과 다른 이름으로 할 수도 있습니다. 예를 들어 Windows NT 에서는,c:\winnt\profiles\duke\krb5cc_duke 와 같이 됩니다. 이 경우,duke 는 <USER_NAME> 이며,c:\winnt\profiles\duke 는 <USER_HOME> 입니다.

디폴트에서는, 키 탭명은, Kerberos 설정 파일로부터 취득됩니다. Kerberos 구성 파일로 키 탭명이 지정되어 있지 않은 경우, 키 탭명은 <USER_HOME>\krb5.keytab 이다고 보여집니다.

커멘드행으로 password 옵션을 사용하는 경우, 패스워드를 지정하지 않으면 kinit 로부터 패스워드의 입력이 요구됩니다.

주 -password 는, 테스트 목적에서만 준비되어 있습니다. 스크립트에 패스워드를 기술하거나 커멘드행에 패스워드를 입력하거나 하지 말아 주세요. 이와 같이 했을 경우, 패스워드가 누설 하는 위험성이 있습니다.

자세한 것은,kinit 의 메뉴얼 페이지를 참조해 주세요.

커멘드

사용 방법: kinit [-fp] [-c <<cache_name>>] [-k] [-t <<keytab_filename>>] [<<principal>>] [<<password>>] [-help]

커멘드 옵션 설명
-f 전송 가능 티켓을 발행한다
-p 프록시화 가능 티켓을 발행한다
-c <<cache_name> 캐쉬명 (즉,FILE:d:\temp\mykrb5cc)
-k 키 탭을 사용
-t <<keytab_filename> 키 탭명 (즉,d:\winnt\profiles\duke\krb5.keytab)
<principal> 주체명 (즉,duke@java.sun.com)
<password> 주체의 Kerberos 패스워드.
커멘드행 또는 스크립트로 지정하지 않는 것
-help 설명을 표시한다

례

디폴트 서비스에 대해, 현재의 클라이언트 호스트로부터 인증에 대해서 유효한 자격을 요구해, 디폴트의 장소 (c:\winnt\profiles\duke\krb5cc_duke)에 자격 캐쉬를 보존합니다.

kinit duke@JAVA.SUN.COM

다른 주체 마다 프록시화 가능한 자격을 요구해, 지정한 파일 캐쉬에 이러한 자격을 보존합니다.

kinit -p -c FILE:c:\winnt\profiles\duke\credentials\krb5cc_cafebeef cafebeef@JAVA.SUN.COM

다른 주체 마다 프록시화 가능 및 전송 가능한 자격을 요구해, 지정한 파일 캐쉬에 이러한 자격을 보존합니다.

kinit -f -p -c FILE:c:\winnt\profiles\duke\credentials\krb5cc_cafebeef cafebeef@JAVA.SUN.COM

kinit 의 헬프 메뉴를 표시합니다.

kinit -help

시큐리티상의 주의

password 플래그는, 테스트용으로 준비되어 있을 뿐입니다. 커멘드행에 패스워드를 입력하지 말아 주세요. 패스워드를 입력하면(자) 시큐리티 홀이 생겨 예를 들어 Unix ps 커멘드를 실행하고 있을 때, 그 패스워드가 공격자가 찾아낼 수 있을 가능성이 있습니다.
