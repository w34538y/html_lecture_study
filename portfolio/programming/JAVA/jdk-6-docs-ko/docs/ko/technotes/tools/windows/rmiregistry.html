rmiregistry - Java 원격 객체 레지스트리

rmiregistry 커멘드는, 현재의 호스트의 지정한 포트상에 원격 객체 레지스트리를 개시합니다.

형식

rmiregistry [port]

설명

rmiregistry 커멘드는, 현재의 호스트의 지정 port 상에 원격 객체 레지스트리를 작성해, 개시합니다. port 의 지정을 생략 했을 경우, 레지스트리는 포트 1099 로 개시합니다. rmiregistry 커멘드에, 출력 기능은 없습니다. 일반적으로은, 이것은 백그라운드에서 실행됩니다. 예를 나타냅니다.

start rmiregistry

원격 객체 레지스트리는, bootstrap의 네임 서비스입니다. 동일 호스트의 RMI 서버가, 원격 객체를 이름에 바인드 하기 위해서 사용됩니다. 다음에, 로컬 및 원격 호스트의 클라이언트는 원격 객체를 검색해, 원격 메소드의 호출을 실시합니다.

레지스트리는, 일반적으로, 최초의 원격 객체의 위치를 지정합니다. 거기서, 어플리케이션은 메소드를 호출할 필요가 있습니다. 대신해, 그 객체는 어플리케이션 지정의 지원를 제공해, 다른 객체를 찾습니다.

java.rmi.registry.LocateRegistry 클래스의 메소드는, 로컬 호스트, 또는 로컬 호스트와 포트로 동작하는 레지스트리를 취득하기 위해서 사용됩니다.

java.rmi.Naming 클래스의 URL 베이스의 메소드는, 레지스트리로 동작해, 임의의 호스트 및 로컬 호스트상의 원격 객체의 검색에 사용됩니다. 원격 객체에 단순한 (캐릭터 라인) 이름을 바인드 하거나 새로운 이름을 리바인드 (낡은 바인드에 오버라이드(override)) 합니다. 또 원격 객체를 언바인드(unbind) 하거나 레지스트리에 바인드 된 URL 를 출력합니다.

옵션

-J
-J 의 뒤로 계속되는 옵션을 java interpreter에 인도합니다. java 옵션과 조합해 사용합니다 (-J 와 java 옵션의 사이에 스페이스는 들어갈 수 없다).

관련 항목

java ,java.rmi.registry.LocateRegistry 및 java.rmi.Naming
