형식 

jinfo [ option ] pid
jinfo [ option ] executable core
jinfo [ option ] [server-id@]remote-hostname-or-IP

파라미터

option
각 옵션은 서로 배타적입니다. 옵션을 사용하는 경우, 커멘드명의 직후에 기술합니다.

pid
출력하는 구성 정보의 프로세스 ID. 프로세스는 Java 프로세스일 필요가 있습니다. 머신상에서 실행하고 있는 Java 프로세스의 일람을 가져오려면 ,jps 를 사용합니다.
executable
코어덤프의 작성원의 Java 실행 가능 파일입니다.
core
출력하는 구성 정보의 코어 파일.
remote-hostname-or-IP
원격 디버그 서버 (jsadebugd 를 참조)의 호스트명 또는 IP 주소입니다.
server-id
복수의 디버그 서버가 동일한 원격 호스트로 실행하고 있는 경우의, 옵션 고유의 ID 입니다.

설명

jinfo 는, 지정된 Java 프로세스나 코어 파일 또는 원격 디버그 서버의 Java 구성 정보를 출력합니다. 구성 정보는, Java 시스템 프로퍼티과 Java 가상 머신의 커멘드행 플래그로부터 구성됩니다. 지정된 프로세스가 64 비트 VM 상에서 실행되고 있는 경우,-J-d64 옵션을 지정해야 하는 경우가 있습니다. 다음에 예를 나타냅니다.

jinfo -J-d64 -sysprops pid

주 - 이 유틸리티는 지원 대상외이며, 장래의 JDK 의 버젼에서는 이용할 수 없게 될 가능성이 있습니다. Windows 상의 jinfo 로 지원되고 있는 것은, 실행중의 프로세스의 -flag 옵션 뿐입니다.

옵션

<옵션 없음 >
커멘드행 플래그를, 시스템 프로퍼티명과 값의 페어와 함께 출력합니다.
-flag name
지정된 커멘드행 플래그의 이름과 값을 출력합니다.
-flag [+|-]name
지정된 불형의 커멘드행 플래그를 유효 또는 무효로 합니다.
-flag name=value
지정된 커멘드행 플래그가 지정된 값으로 설정합니다.
-flags
JVM 에게 건네지는 커멘드행 플래그를 페어로 출력합니다.
-sysprops
Java 시스템 프로퍼티을 이름과 값의 페어로서 출력합니다.
-h
헬프 메세지를 출력합니다.
-help
헬프 메세지를 출력합니다.
