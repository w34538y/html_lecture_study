ktab - Kerberos 열쇠 접대비 전기밥통

ktab 를 사용하면(자), 로컬의 열쇠 테이블에 보존된 주체명과 서비스열쇠를 관리할 수가 있습니다. 키 탭에 일람표 나타난 주체와 열쇠의 페어를 사용하는 것으로써, 호스트상에서 실행하고 있는 서비스를, Key Distribution Center (KDC)에 인증시킬 수가 있습니다. Kerberos 를 사용할 수 있도록(듯이) 서버를 설정하려면 , 서버를 실행하고 있는 호스트상에서 키 탭을 설정할 필요가 있습니다. ktab 를 사용해 키 탭을 갱신했을 경우, Kerberos 데이타베이스는 변경되지 않습니다. 키 탭내의 열쇠를 변경했을 경우, Kerberos 데이타베이스에도 같은 변경을 더하지 않으면 안됩니다.

형식

ktab [ commands ]

설명

ktab 는, 열쇠 테이블의 주체명과 열쇠의 페어를 관리합니다. 또, 사용자는 ktab 를 사용해, 열쇠 테이블내의 주체명과 키의 페어를 일람표시, 추가, 갱신, 또는 삭제할 수 있습니다. 어느 조작을 실시해도, Kerberos 데이타베이스에는 영향을 주지 않습니다.

키 탭
키 탭은, 호스트 자신의 열쇠 리스트를 카피한 것입니다. 열쇠 리스트는 사용자의 패스워드에 유사하고 있습니다. Key Distribution Center (KDC)에 인증받을 필요가 있는 어플리케이션 서버에서는, 서버 자신의 주체와 열쇠가 기술된 키 탭이 필요하게 됩니다. 사용자에게 있어 자신의 패스워드의 보호가 중요하도록, 호스트에게 있어 자신의 키 탭의 보호가 중요하게 됩니다. 키 탭 파일은, 반드시 로컬 디스크에 보존해, 수퍼 사용자 이외에는 읽어낼 수 없게 할 필요가 있습니다. 키 탭 파일을 평문인 채 네트워크로 송신하지 말아 주세요.

커멘드

사용 방법:커멘드행 옵션은, 대문자와 소문자를 구별하지 않습니다.

ktab -help

ktab -l [-k <keytab_name>]

ktab [-a <principal_name> <password>] [-k <keytab_name>]

ktab [-d <principal_name>] [-k <keytab_name>]

커멘드 옵션 설명
-l 키 탭명과 엔트리를 일람표시 한다
-a <principal_name> <password> 엔트리를 키 탭에 추가한다. Kerberos 데이타베이스는 변경되지 않는 커멘드행 또는 스크립트로 패스워드를 지정하지 않는 것
-d <principal_name> 엔트리를 키 탭으로부터 삭제한다. Kerberos 데이타베이스는 변경되지 않는다
-k <keytab_name> 접두사 FILE: 를 사용해 키 탭명과 패스를 지정한다
-help 설명을 표시한다

례

디폴트의 열쇠 테이블내의 엔트리를 모두 일람표시 하는 경우
ktab -l
열쇠 테이블에 새로운 주체를 추가하는 경우 (패스워드가 요구되는 것에 주의)
ktab -a duke@java.sun.com
열쇠 테이블에서 주체를 삭제하는 경우
ktab -d duke@java.sun.com

시큐리티상의 주의

커멘드행에 패스워드를 입력하지 말아 주세요. 패스워드를 입력하면(자), 시큐리티상의 위험이 생깁니다. 예를 들어, Unix ps 커멘드를 실행하고 있을 때, 공격자에게 패스워드가 발견될 가능성이 있습니다.
