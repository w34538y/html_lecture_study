JDK 및 JRE 의 파일 구조

 JDK 툴

이 페이지에서는, JDK 에 포함되는 디렉토리와 파일의 개요에 대해 설명합니다. JRE 의 파일 구조는, JDK 의 jre 디렉토리의 파일 구조와 동일합니다.

개발용 파일과 디렉토리

여기에서는, Java 플랫폼용의 어플리케이션 개발에 필요한, 가장 중요한 파일과 디렉토리에 도착해 설명합니다. 개발에 필요없는 디렉토리에는, 데모, Java 원시 코드, C 의 헤더 파일등이 있습니다. 이것들에 대해서는, 이후에의「그 외의 파일과 디렉토리」에서 설명합니다.

 

          jdk1. 6.0 
   ___________|_________________
  |           |                 |
 bin          lib                jre 
  |           |         ________|__________
java*     tools.jar    |                   |        
javac*    dt.jar      bin                  lib                 
javap*                 |       ________ ___|___ _________ ________ _______       
javah*               java*    |        |       |         |        |       |    
javadoc*                    rt.jar    ext    security    i386    applet   fonts 
                        charsets.jar   |              /    \
                                  localedata.jar  server   client

JDK 소프트웨어는 /jdk1. 6.0 에 인스톨 되고 있는 것으로 합니다. 다음에, 몇개의 중요한 디렉토리에 도착해 설명합니다.

/jdk1. 6.0
JDK 소프트웨어가 인스톨 되는 루트 디렉토리입니다. 저작권, 라이센스, README 의 파일이 들어가 있습니다. 또, Java 플랫폼의 원시 코드의 어카이브(archive)인 src.jar 도 들어가 있습니다.
/jdk1. 6.0/bin
JDK 에 포함되는 모든 개발툴의 실행 파일용 디렉토리입니다. PATH 환경 변수에는, 이 디렉토리의 엔트리를 포함하지 않으면 안됩니다. 개발툴의 자세한 것은,「JDK 툴」을 참조해 주세요.
/jdk1. 6.0/lib
개발툴에 의해 사용되는 파일용의 디렉토리입니다. JDK 의 툴과 유틸리티를 지원하는, 코어 이외의 클래스의 파일인 tools.jar 가 포함됩니다. 또, 이 디렉토리에는, BeanInfo 파일군의 DesignTime 어카이브(archive)인 dt.jar 도 들어가 있습니다. BeanInfo 파일은, 대화형 개발 환경 (IDE)에 대해서, Java 컴퍼넌트를 표시하는 방법이나, 개발자가 컴퍼넌트를 어플리케이션에 맞추어 커스터마이즈 하는 방법을 통지하기 위한의 것입니다.
/jdk1. 6.0/jre
JDK 개발툴에 의해 사용되는 Java Runtime Environment 의 루트 디렉토리입니다. Runtime Environment 는 Java 플랫폼의 구현입니다. 이것은,java.home 시스템 프로퍼티으로부터 참조되는 디렉토리입니다.
/jdk1. 6.0/jre/bin
Java 플랫폼이 사용하는 툴과 라이브러리용의, 실행 파일의 디렉토리입니다. 포함되어 있는 실행 파일은 /jdk1. 6.0/bin 내의 파일과 같습니다. JDK 의 버젼 1.1 에 부속되어 있던 jre 툴에 대신해,java 기동 툴이, 어플리케이션 기동 툴로서 기능합니다. 이 디렉토리는,PATH 환경 변수에 포함할 필요가 없습니다.
/jdk1. 6.0/jre/lib
Java Runtime Environment 가 사용하는 코드 라이브러리, 프로퍼티 설정, 및 resource file의 디렉토리입니다. 예를 들어, 다음의 파일이 포함됩니다.
  • rt.jar -- bootstrap 클래스 (Java 플랫폼의 코어 API 를 구성하는 런타임 클래스)
  • charsets.jar -- 문자 변환 클래스
이후에 설명하는 ext 서브 디렉토리 외에도, 여기에서는 설명하지 않는 자원 서브 디렉토리가 몇개인가 포함되어 있습니다.
/jdk1. 6.0/jre/lib/ext
Java 플랫폼에 대한확장 기능의 디폴트의 인스톨처 디렉토리입니다. 예를 들어 JavaHelp 의 JAR 파일을 인스톨 하면(자), 이 디렉토리에 놓여집니다.
  • localedata.jar -- java.text 및 java.util 용의 로컬 데이터
/jdk1. 6.0/jre/lib/security
시큐리티 관리에 사용되는 파일이 들어가 있습니다. 이것에는, 시큐리티 정책 (. java.policy)나 시큐리티 프로퍼티 (java.security)의 파일이 있습니다.
/jdk1. 6.0/jre/lib/i386/client
Java HotSpotTM 테크놀러지를 사용해 구현되는 Java HotSpotTM Client 가상 머신이 사용한다 . so 파일이 포함되어 있습니다. 이것은 디폴트의 VM 입니다.
/jdk1. 6.0/jre/lib/i386/server
Java HotSpotTM Server 가상 머신이 사용한다 . so 파일이 포함되어 있습니다.
/jdk1. 6.0/jre/lib/applet
애플릿의 지원 클래스가 들어간 JAR 파일은,lib/applet/ 디렉토리에 배치할 수 있습니다. 이와 같이 하면(자), 애플릿의 클래스가 애플릿 클래스 로더에 의해 로컬 파일 시스템으로부터 사전 로드 되기 (위해)때문에, 대규모 애플릿의 기동 시간을 단축할 수 있습니다. 또, 넷 경유로 다운로드했을 경우와 같은 보호가 제공됩니다.
/jdk1. 6.0/jre/lib/fonts
플랫폼이 사용하는 폰트 파일입니다.

그 외의 파일과 디렉토리

여기에서는, 데모, Java 원시 코드, 및 C 헤더 파일의 디렉토리 구조에 대해 설명합니다. 

                        jdk1. 6.0 
                 ___________|__________ _________
                |           |          |         |
              demo        include     src.jar      man 
             ___|___ _________ __________ ________
            |       |         |          |        |
          applets   jfc        jpda      plugin      jni

상기의 파일과 디렉토리에 도착해 설명합니다.

/jdk1. 6.0/src.jar
Java 플랫폼의 원시 코드를 포함한 어카이브(archive)입니다.
/jdk1. 6.0/demo
원시 코드 첨부의 샘플 파일입니다. Java 플랫폼용의 프로그래밍의 방법을 나타내고 있습니다.
/jdk1. 6.0/demo/applets
Web 페이지상에서 사용 가능한 애플릿입니다.
/jdk1. 6.0/demo/jfc
Java 2DTM 및 JFC/Swing 컴퍼넌트의 사용예입니다.
/jdk1. 6.0/demo/jpda
Java Platform Debugging Architecture 의 사용예입니다. javadt 및 jdb 유틸리티의 원시 코드도 포함되어 있습니다.
/jdk1. 6.0/demo/plugin
Java Plug-in 제품으로 사용하는 데모입니다.
/jdk1. 6.0/demo/jni
Java 플랫폼으로부터 poll(2) 기능에의 액세스 방법을 나타내는, 샘플 클래스와 C 코드입니다.
/jdk1. 6.0/include
Java Native Interface (JNI) 및 Java Virtual Machine Debugger Interface (JVMDI) 를 사용하는 native code의 프로그래밍을 지원하는, C 언어의 헤더 파일입니다.
/jdk1. 6.0/man
JDK 툴의 메뉴얼 페이지가 포함되어 있습니다.
