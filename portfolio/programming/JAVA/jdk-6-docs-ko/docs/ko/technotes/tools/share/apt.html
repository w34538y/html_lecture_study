NAME

apt - 주석 처리 툴

형식

apt [-classpath classpath] [-sourcepath sourcepath] [-d directory] [-s directory] [-factorypath path] [-factory class] [-print] [-nocompile] [-Akey[=val] ...][javac option] sourcefiles [@files]

파라미터

옵션은 무순서입니다. 특정의 옵션에 적용되는 파라미터에 대해서는, 아래와 같은 「옵션」을 참조해 주세요.

sourcefiles
제로, 1 개(살), 또는 복수의 처리 대상의 원시 파일
@files
원시 파일 또는 다른 옵션을 일람표시 하는 1 개(살) 또는 복수의 파일

설명

주석 처리 툴 apt 는, 새로운 리플렉트 API 와 지원 인프라스트럭쳐(infrastructure)로부터 구성되어 프로그램 주석을 처리합니다. apt 리플렉트 API 는, 구축시의 소스 베이스로, 프로그램 구조에 관한 읽기 전용 뷰를 제공합니다. 이러한 리플렉트 API 는, 총칭을 추가한 후에, JavaTM 프로그램 언어의 형태 시스템을 올바르고 모델화하도록(듯이) 설계되고 있습니다. 최초로,apt 는, 새로운 원시 코드와 다른 파일을 작성하는 주석 프로세서를 실행합니다. 다음에,apt 는, 원의 원시 파일과 생성한 원시 파일의 양쪽 모두를 컴파일 하기 위한(해), 개발이 편해집니다. 툴과의 인터페이스에 사용되는 리플렉트 API 등의 API 는,com.sun.mirror 의 서브 패키지입니다.

툴의 기능에 관한 상세와apt 를 사용한 개발 방법에 대해서는,「apt 입문」 을 참조해 주세요.

주:
apt 의 기능은,javac 툴의 일부가 된 주석 처리 인프라스트럭쳐(infrastructure)에 짜넣어져 모든 Java 컴파일러로 사용할 수 있도록(듯이) 표준화 되었습니다. 이 새로운 인프라스트럭쳐(infrastructure)는, 언어 모델 및 Java 플랫폼의 일부가 된 주석 처리 API 에 의존합니다. 주석 프로세서의 개발은, 새로운 API 및 javac 툴에 근거해 실시하도록(듯이) 추천합니다.

옵션

apt 고유의 옵션

-s dir
프로세서의 생성하는 원시 파일을 두는 디렉토리 루트를 지정합니다. 파일은, 패키지의 이름 공간에 근거해 서브 디렉토리에 놓여집니다.
-nocompile
원시 파일을 클래스 파일에 컴파일 하지 않습니다.
-print
지정한 타입의 텍스트 표현을 출력합니다. 주석 처리 또는 컴파일은 실시하지 않습니다.
-A[key[=val]]
주석 프로세서에 건네주는 옵션입니다. 이 옵션은,apt 가 직접 해석하는 것이 아니라, 각각의 프로세서에 의해 사용할 수 있도록(듯이) 바꿀 수 있습니다.
-factorypath path
주석 프로세서 팩토리를 검색하는 장소를 지정합니다. 이 옵션을 사용하는 경우, 클래스 패스의 팩토리는 검색되지 않습니다.
-factory classname
사용하는 주석 프로세서 팩토리의 이름입니다. 디폴트의 검출 프로세스를 생략 합니다.

javac 와 공용하는 옵션

-d dir
프로세서와 javac 생성의 클래스 파일을 두는 장소를 지정합니다.
-cp path or -classpath path
사용자 클래스 파일과 주석 프로세서 팩토리를 검색하는 장소를 지정합니다. -factorypath 가 지정되고 있는 경우, 클래스 패스의 팩토리는 검색되지 않습니다.

javac 옵션의 상세한 것에 대하여는, javac(1)의 메뉴얼 페이지를 참조해 주세요.

주

apt 의 기능은,javac 에 의해 제공되는 표준 주석 처리 인프라스트럭쳐(infrastructure)에 짜넣어졌습니다. 향후의 JDK 릴리스에서는,apt 및 관련하는 API 의 지원가 중지할 가능성이 있습니다.
