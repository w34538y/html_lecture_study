형식 

jstack [ option ] pid
jstack [ option ] executable core
jstack [ option ] [server-id@]remote-hostname-or-IP

파라미터

각 옵션은 서로 배타적입니다. 옵션을 사용하는 경우, 커멘드명의 직후에 기술합니다. 옵션을 참조해 주세요.

pid
인쇄하는 스택 트레이스의 프로세스 ID 입니다. 프로세스는 Java 프로세스일 필요가 있습니다. 머신상에서 실행하고 있는 Java 프로세스의 일람을 가져오려면 ,jps 를 사용합니다.
executable
코어덤프의 작성원의 Java 실행 가능 파일입니다.

core
인쇄하는 스택 트레이스의 코어 파일입니다.

remote-hostname-or-IP
원격 디버그 서버 (jsadebugd 를 참조)의 호스트명 또는 IP 주소입니다.

server-id
복수의 디버그 서버가 동일한 원격 호스트로 실행하고 있는 경우의, 옵션 고유의 ID 입니다.

설명

jstack 는, 지정된 Java 프로세스나 코어 파일 또는 원격 디버그 서버에 대한 Java thread의 Java 스택 트레이스를 인쇄합니다. Java 프레임 마다, 풀 클래스명, 메소드명, 「bci」(바이트 코드 인덱스), 및 행 번호 (이용 가능한 경우)가 인쇄됩니다. -m 옵션을 사용하면(자), jstack 는, 모든 thread의 Java 프레임과 네이티브 프레임의 양쪽 모두를, 「pc」(프로그램 카운터)과 함께 인쇄합니다. 네이티브 프레임 마다, 「pc」에 가장 가까운 네이티브 심볼 (이용 가능한 경우)이 인쇄됩니다. C++ 분해명은 분해 해제되지 않습니다. C++ 명을 분해 해제하는데는, 이 커멘드의 출력을 c++filt 에 파이프 합니다. 지정된 프로세스가 64 비트 VM 상에서 실행되고 있는 경우,-J-d64 옵션을 지정해야 하는 경우가 있습니다. 다음에 예를 나타냅니다. 

jstack -J-d64 -m pid

주 - 이 유틸리티는 지원 대상외이며, 장래의 JDK 의 버젼에서는 이용할 수 없게 될 가능성이 있습니다. Windows 플랫폼상에서 사용 가능한 유일한 jstack 형식은, 다음과 같습니다. 

    jstack [-l] pid

옵션

-F
「jstack [-l] pid」가 응답하지 않는 경우에 스택 덤프를 강제합니다.
-l
긴 표시. 소유 java.util.concurrent 의 소유할 수 있는 싱크로나이저 의 일람 등, 락에 대한 추가 정보를 인쇄합니다.
-m
혼합 모드 (Java 및 네이티브 C/C++ 프레임의 양쪽 모두)의 스택 트레이스를 인쇄합니다.
-h
헬프 메세지를 인쇄합니다.

-help
헬프 메세지를 인쇄합니다.

기존의 버그

혼합 모드의 스택 트레이스 (-m 옵션 사용)는, 원격 디버그 서버에서는 기능하지 않습니다.
