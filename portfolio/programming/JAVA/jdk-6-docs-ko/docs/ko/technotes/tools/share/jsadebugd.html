형식

jsadebugd pid [ server-id ]
jsadebugd executable core [ server-id ]

파라미터

pid
디버그 서버가 접속하는 프로세스의 프로세스 ID 입니다. 프로세스는 Java 프로세스일 필요가 있습니다. 머신상에서 실행하고 있는 Java 프로세스의 일람을 가져오려면 ,jps 를 사용합니다. 단일의 프로세스에 접속할 수 있는 디버그 서버의 인스턴스는, 1 개(살)에 제한됩니다.
executable
코어덤프의 작성 바탕으로 되는 Java 실행 가능 파일입니다.
core
디버그 서버를 접속하는 코어 파일입니다.
server-id
복수의 디버그 서버가 동일한 머신상에서 실행하고 있는 경우에 필요하게 되는, 옵션 고유의 ID 입니다. 이 ID 는, 원격 클라이언트가, 접속처의 디버그 서버를 특정하기 위해서 사용할 필요가 있습니다. 이 ID 는, 단일의 머신내에서 일의가 아니면 안됩니다.

설명

jsadebugd 는, Java 프로세스 또는 코어 파일에 접속해, 디버그 서버로서 기능합니다. jstack ,jmap , 및 jinfo 등의 원격 클라이언트는, Java Remote Method Invocation (RMI)를 사용하고 있는 서버에 접속할 수 있습니다. jsadebugd 를 기동하기 전에, 다음과 같이 지정해 rmiregistry 를 기동할 필요가 있습니다. 

rmiregistry -J-Xbootclasspath/p:$JAVA_HOME/lib/sajdi.jar

여기서,$JAVA_HOME 는, JDK 의 인스톨 디렉토리입니다. rmiregistry 가 기동하고 있지 않는 경우, jsadebugd 는 표준 (1099) 포트의 rmiregistry 를 내부에서 기동합니다. 디버그 서버는, SIGINT 를 송신하는 (Ctrl+C 를 누른다) 것에 따라 정지할 수 있습니다.

주 - 이 유틸리티는 지원 대상외이며, 장래의 JDK 의 버젼에서는 이용할 수 없게 될 가능성이 있습니다. jsadebugd 는, 현시점에서는, Windows 플랫폼에서는 사용할 수 없습니다.
