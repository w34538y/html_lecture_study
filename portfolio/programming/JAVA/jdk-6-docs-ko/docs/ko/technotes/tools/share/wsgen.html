wsgen - JAX-WS 2.0 베타

wsgen 툴은, JAX-WS Web 서비스로 사용되는 JAX-WS 휴대용 아티팩트를 생성합니다. 이 툴은, 서비스 엔드 포인트 구현 클래스 (SEI)를 읽어내, Web 서비스의 배치라고 불러 방편에 필요한 모든 아티팩트를 생성합니다.

형식

wsgen [options] <SEI>

옵션

다음의 겉(표)에,wsgen 의 옵션을 나타냅니다.

겉(표) 1-1 wsgen 의 옵션

옵션

설명

-classpath <path>

입력 클래스 파일의 검색 장소를 지정합니다

-cp <path>

-classpath <path> 와 같다

-d <directory>

생성되는 출력 파일을 포함하는 장소를 지정한다

-extension

벤더 확장 (스펙으로 규정되어 있지 않은 기능)을 허가합니다. 확장을 사용하면(자), 어플리케이션의 이식성이 없어지거나 다른 구현과의 상호 운용을 실시할 수 없게 되는 가성이 있습니다

-help

헬프를 표시합니다

-keep

생성된 파일을 보관 유지합니다

-r <directory>

항상 -wsdl 옵션과 조합해 사용합니다. WSDL 등의 생성된 resource file의 포함 장소를 지정합니다

-s <directory>

생성되는 원시 파일을 포함하는 장소를 지정한다

-verbose

컴파일러가 실행하고 있는 처리에 관한 메세지를 출력합니다

-version

버젼 정보를 출력합니다. 이 옵션을 사용했을 경우, 버젼 정보가 출력될 뿐입니다. 일반적으로의 처리는 실행되지 않습니다.

-wsdl[:protocol]
 wsgen 는 디폴트에서는 WSDL 파일을 생성하지 않습니다. 이 플래그는 생략 가능합니다만, 지정했을 경우는 wsgen 가 WSDL 파일을 생성하게 됩니다. 이 플래그는 일반적으로, 엔드 포인트의 배치전에 개발자가 WSDL 를 참조할 수 있도록(듯이) 하기 위한(해)인 만큼 사용됩니다. protocol 는 생략 가능하고,wsdl:binding 로 사용해야 할 프로토콜을 지정하기 위해서 사용됩니다. 유효한 프로토콜은 다음과 같습니다.   soap1. 1 으로 Xsoap1. 2. 디폴트는 soap1. 1 입니다. Xsoap1. 2 는 표준은 아니기 때문에,-extension 옵션과 조합하지 않으면 사용할 수 없습니다.

-servicename <name>

항상 -wsdl 옵션과 조합해 사용합니다. WSDL 내에서 생성되어야 할 wsdl:service 의 이름을 지정하기 위해서 사용합니다. 예 : -servicename "{http://mynamespace/}MyService"

-portname <name>

항상 -wsdl 옵션과 조합해 사용합니다. WSDL 내에서 생성되어야 할 wsdl:port 의 이름을 지정하기 위해서 사용합니다. 예 : -portname "{http://mynamespace/}MyPort"

Ant 태스크

wsgen 툴에는, 이 툴용의 Ant 태스크가 부속되어 있습니다. 이 Ant 태스크가 지원하는 속성이나 요소를, 다음에 나타냅니다. 

  
    <wsgen      
         sei="..."
         destdir="directory for generated class files"      
         classpath="classpath" | cp="classpath"
         resourcedestdir="directory for generated resource files such as WSDLs"
         sourcedestdir="directory for generated source files"      
         keep="true|false"
         verbose="true|false"
         genwsdl="true|false"      
         protocol="soap1. 1|Xsoap1. 2"
         servicename="..."
         portname="...">
         extension="true|false"
         <classpath refid="..."/>
     </wsgen>

-r

생성되는 원시 파일을 포함하는 장소를 지정한다

컴파일러가 실행하고 있는 처리에 관한 메세지를 출력합니다

-verbose

genwsdl

WSDL 파일을 생성해야할 것인가 제발을 지정합니다

-wsdl

protocol

genwsdl 와 조합해 사용합니다. wsdl:binding 로 사용하는 프로토콜을 지정합니다. 유효한 값은 「soap1. 1으로 「Xsoap1. 2」로, 디폴트는 「soap1. 1」입니다. 「Xsoap1. 2」는 표준은 아니기 때문에, -extension 옵션과 조합하지 않으면 사용할 수 없습니다.

-wsdl:soap11

servicename

genwsdl 옵션과 조합해 사용합니다. 생성되는 WSDL 의 wsdl:service 의 이름을 지정하기 위해서 사용합니다. 예 : servicename="{http://mynamespace/}MyService"

-servicename

portname

genwsdl 옵션과 조합해 사용합니다. 생성되는 WSDL 의 wsdl:portmame 의 이름을 지정하기 위해서 사용합니다. 예 : portname="{http://mynamespace/}MyPort"

-servicename

extension

벤더 확장 (스펙으로 규정되어 있지 않은 기능)을 허가합니다. 확장을 사용하면(자), 어플리케이션의 이식성이 없어지거나 다른 구현과의 상호 운용을 실시할 수 없게 되는 가성이 있습니다

-extension

classpath 속성은,path-like structure 이며, 상자가 된 <classpath> 요소를 사용해 설정할 수도 있습니다. 이 태스크를 사용하려면 , 다음의 <taskdef> 요소를 프로젝트에 추가해 둘 필요가 있습니다. 

  <taskdef name="wsgen" classname="com.sun.tools.ws.ant.WsGen">
      <classpath path="jaxws.classpath"/>
  </taskdef>

여기서,jaxws.classpath 는, JAX\-WS 툴이 필요로 하는 클래스의 리스트를 포함한, 빌드 환경내의 다른 장소에 정의된 path-like structure 에의 참조입니다.

례

<wsgen      
      resourcedestdir="${wsdl.dir}"
      sei="fromjava.server.AddNumbersImpl">
      <classpath refid="compile.classpath"/>
</wsgen>
