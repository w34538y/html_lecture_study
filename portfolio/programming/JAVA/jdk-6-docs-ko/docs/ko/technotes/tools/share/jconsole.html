형식 

jconsole [ options ] [ connection ... ]

파라미터

options
옵션을 사용하는 경우, 커멘드명의 직후에 기술합니다.

connection = pid | host:port | jmxUrl
pid   로컬의 Java VM 의 프로세스 ID. Java VM 는, jconsole 를 실행하고 있는 사용자 ID 와 같은 사용자 ID 를 사용해 실행할 필요가 있습니다. 상세한 것에 대하여는,「JMX 감시 및 관리」를 참조해 주세요.

host:port   Java VM 가 실행하고 있는 호스트 시스템의 이름과 JVM 를 기동했을 때에 시스템 프로퍼티 com.sun.management.jmxremote.port 로 지정한 포트 번호. 상세한 것에 대하여는,「JMX 감시 및 관리」를 참조해 주세요.

jmxUrl   접속처의 JMX 에이전트의 주소. 이것에 대해서는,JMXServiceURL 를 참조해 주세요.

설명

jconsole 커멘드는, 로컬 머신 또는 원격 머신상의 Java 어플리케이션과 가상 머신의 감시와 관리를 실시하는 그래피컬 콘솔 툴을 기동합니다.

Windows 상에서는,jconsole 는 콘솔 윈도우와 관련지을 수 있고 있지 않습니다. 다만, 하등의 이유로써 jconsole 커멘드가 실패하면(자), 에러 정보를 나타내는 다이알로그 박스가 표시됩니다.

옵션

-interval=n
갱신 간격을 n 초로 설정합니다 (디폴트는 4 초).
-notile 
최초로 윈도우를 타일링 하지 않습니다 (복수의 접속을 위해).
-pluginpath  plugins
JConsole 플러그 인의 검색처가 되는 디렉토리 또는 JAR 파일의 리스트를 지정합니다. plugins 패스에는, 다음의 이름의 프로바이더 구성 파일을 포함해 주세요. 
   META-INF/services/com.sun.tools.jconsole.JConsolePlugin

이것에는,com.sun.tools.jconsole.JConsolePlugin 클래스를 구현하는 클래스의 완전 수식 클래스명을 지정하는 행이, 플러그 인 마다 1 행씩 포함되어 있습니다.

-version 
버젼 정보를 출력해 종료합니다.
-help 
헬프 메세지를 출력해 종료합니다.
-J<flag> 
jconsole 가 실행되고 있는 Java 가상 머신에 <flag> 를 건네줍니다.
