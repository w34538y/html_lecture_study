extcheck - jar 의 경합 검출 유틸리티

extcheck 는, 타겟의 jar 파일과 현재 인스톨 되고 있는 확장 기능의 jar 파일간의 버젼의 경합을 검출합니다.

형식

extcheck [ -verbose ] targetfile.jar

설명

extcheck 유틸리티는, 지정된 Jar 파일의 타이틀 및 버젼이 JavaTM 2 SDK 에 인스톨 되고 있는 확장 기능과 경합 하고 있지 않는가를 체크합니다. 확장 기능을 인스톨 하기 전에, 이 유틸리티를 사용해, 버젼이 같은가, 보다 새로운 확장 기능이 벌써 인스톨되어 있지 않은지 어떤지를 조사할 수가 있습니다.

extcheck 유틸리티는,targetfile.jar 파일의 Manifest내의 헤더 Specification-title 및 Specification-version 를, 확장 기능 디렉토리내에 현재 인스톨 떠날 수 있어 모든 Jar 파일내의 대응하는 헤더와 비교합니다. 디폴트에서는, 확장 기능 디렉토리는,jre\lib\ext 입니다. extcheck 유틸리티는,java.lang.Package.isCompatibleWith 메소드와 같은 방법으로 버젼 번호를 비교합니다.

경합이 검출되지 않는 경우의 return code는 0 입니다.

확장 기능 디렉토리내의 jar 파일의 Manifest에, 동일한 Specification-title, 및 동일 또는보다 새로운 Specification-version 번호가 있는 경우는, 제로가 아닌 에러 코드가 돌려주어집니다. targetfile.jar 의 Manifest에 Specification-title 또는 Specification-version 속성이 없는 경우도, 제로가 아닌 에러 코드가 돌려주어집니다.

옵션

-verbose
확장 기능 디렉토리내의 Jar 파일을, 체크시에 일람표 가리킵니다. 또, 타겟 jar 파일의 Manifest의 속성, 및 경합 하는 jar 파일에 대해서도 보고합니다.
-Joption
Java 가상 머신에 option 를 건네줍니다. option 에는,「Java 어플리케이션 기동 툴」의 레퍼런스 페이지에 기재되어 있는 옵션을 1 개 지정합니다. 예를 들어,-J-Xms48m 와 지정하면(자), 스타트 업 메모리는 48M 바이트로 설정됩니다.
