형식 

jrunscript [ options ] [ arguments... ]

파라미터

options
옵션을 사용하는 경우, 커멘드명의 직후에 기술해 주세요.
arguments
인수를 사용하는 경우, 옵션 또는 커멘드명의 직후에 기술해 주세요.

설명

jrunscript 는 커멘드행 스크립트 쉘입니다. jrunscript 는, 대화형 (읽기 - 평가 - 출력) 모드와 배치 (-f 옵션) 모드의 양쪽 모두의 스크립트 실행을 지원합니다. 이것은 스크립트 언어에 의존하지 않는 쉘입니다. 디폴트의 사용 언어는 JavaScript 입니다만, -l 옵션을 사용하면 다른 언어도 지정할 수 있습니다. jrunscript 는, Java 와 스크립트 언어와의 통신에 의해 「탐구적인 프로그래밍」스타일을 지원합니다.

주:이 툴은시험적인 것이며, 장래의 JDK 의 버젼에서는이용할 수 없게 될가능성이 있습니다.

옵션

-classpath path
스크립트로부터의 액세스 대상이 되는 사용자의 . class 파일의 검색 장소를 지정합니다.
-cp path
-classpath path 와 동의입니다.
-Dname=value
Java 의 시스템 프로퍼티을 설정합니다.
-J<flag>
jrunscript 가 실행되고 있는 Java 가상 머신에 <flag> 를 건네줍니다.
-l language
지정된 스크립트 언어를 사용합니다. 디폴트에서는 JavaScript 가 사용됩니다. 다른 스크립트 언어를 사용하려면 , 대응하는 스크립트 엔진의 JAR 파일도 지정할 필요가 있습니다. 거기에는, -cp, -classpath 의 몇개의 옵션을 사용합니다.
-e script
지정된 스크립트를 평가합니다. 이 옵션을 사용하면, 커멘드행에 모든 것이 지정된 「1 행」스크립트를 실행할 수 있습니다.
-encoding encoding
스크립트 파일의 읽기시에 사용하는 문자 인코딩을 지정합니다.
-f script-file
지정된 스크립트 파일을 평가합니다 (배치 모드).
-f -
표준 입력으로부터 스크립트를 읽어내, 그것을 평가합니다 (대화형 모드).
-help 
헬프 메세지를 출력해 종료합니다.
-?  
헬프 메세지를 출력해 종료합니다.
-q 
이용 가능한 모든 스크립트 엔진을 일람표 가리킨 뒤, 종료합니다.

인수

[arguments...] 하지만 존재하고 있어, 한편 -e,-f 의 어느 옵션도 사용되지 않았던 경우, 최초의 인수가 스크립트 파일이 되어, 다른 인수가 존재하는 경우는 스크립트 인수로서 건네받습니다. [arguments..] 하지만 사용되어 한편 -e,-f 의 몇개의 옵션이 사용되었을 경우, 모든 [arguments..] 하지만 스크립트 인수로서 건네받습니다. [arguments..],-e,-f 가 어느 것도 존재하지 않았던 경우는, 대화형 모드가 사용됩니다. 스크립트로부터 스크립트 인수를 사용하려면 , 「arguments」라고 하는 이름의 String 배열형의 엔진 변수를 사용합니다.

례

인 라인 스크립트의 실행

jrunscript -e "print('hello world')"
jrunscript -e "cat('http://java.sun.com')"

지정된 언어를 사용해, 지정된 스크립트 파일을 평가한다

jrunscript -l js -f test.js

대화형 모드

jrunscript
js>print('hello world');
hello world
js>34 + 55
89
js> thread(function() { print('hello world'); }
hello world
js>

스크립트 인수를 지정해 스크립트 파일을 실행한다

jrunscript test.js arg1 arg2 arg3
test.js 가 실행 대상이 되는 스크립트 파일이며, arg1, arg2, 및 arg3 는 스크립트 인수로서 스크립트에게 건네집니다. 스크립트는 「arguments」배열을 사용해 이것들에 액세스 할 수 있습니다.

관련 항목

JavaScript 가 사용되는 경우, jrunscript 는, 최초의 사용자 정의 스크립트를 평가하기 전에, 몇개의 편입 함수나 짜넣어 객체를 초기화합니다. 이러한 JavaScript 편입 기능에 대해서는,jsdocs 를 참조해 주세요.
