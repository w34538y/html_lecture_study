JavaTM 2 SDK v1. 4
툴의 변경점

 SDK 툴
이 문서에서는, Java 2 SDK, Standard Edition 버젼 1.4 의 툴에 대해서 더해져 따분한가의 변경점에 대해 설명합니다. 모든 툴 및 유틸리티에 관한 레퍼런스 페이지로의 링크는,「SDK 툴과 유틸리티」 를 참조해 주세요.

Java 2 SDK 1.4. 1 에서의 툴의 변경점

  • Java 2 플랫폼의 1.4. 1 릴리스에서는,kinit,klist, 및 ktab 의 3 개의 새로운「시큐리티」 툴이 추가되었습니다. 이러한 툴은, Kerberos 티켓을 취득, 일람표시, 및 관리하는 경우에 사용합니다.
  • J2SE 1.4. 1 의 javac 바이트 코드 컴파일러에 더해진 변경에 대해서는,「online release notes」를 참조해 주세요.
  • javadoc 툴에서는, 약 24 의 버그를 수정했습니다. 자세한 것은,「Javadoc 1.4. 1 의 신기능」을 참조해 주세요.

Java 2 SDK 1.4. 0 에서의 툴의 변경점

이러한 변경점은, Java 2 SDK 1.4 패밀리의 1.4. 0 이후의 릴리스에 상속되고 있습니다.
  • Java 어플리케이션 기동 툴이, 새로운 assertion 기능을 위해서(때문에) 몇개의 커멘드행 옵션을 지원하게 되었습니다. 새로운 옵션 -Xconcgc 를 사용하면(자), 멀티 프로세서 플랫폼에서 병행 가베지 컬렉션을 호출해, 큰 heap에 대한 가베지 컬렉션에 의한 시스템의 일시정지를 줄일 수가 있습니다. SolarisTM Operating Environment (SPARCTM 플랫폼판)에서는, 64 비트 또는 32 비트의 조작을 지정하기 위한 새로운 커멘드행 옵션을 사용할 수 있습니다. 모든 플랫폼에서, 새로운 옵션 -Xloggc:file 를 지정하면(자), 각 가베지 컬렉션 이벤트가, 지정된 파일에 기록됩니다.
  • javac 바이트 코드 컴파일러에, 새롭고 -source 옵션이 추가되었습니다. 이 옵션을 지정하면(자), assertion를 포함한 원시 코드의 컴파일이 지원됩니다. 또, 디폴트의 컴파일은 -target 1.2 입니다. 이전의 디폴트는 1.1 이었습니다. 현재의 컴파일러는, 버그 번호 4083890 으로 설명되고 있는 도달 불가능한 공문을 검출합니다. 또, Javac 도, fall-through 하는 case 가 있는지 없는지 switch 블록을 검사하는 새로운 옵션 -Xswitchcheck 를 인식해, 발견되었을 경우는 경고 메세지를 표시합니다.
  • javadoc 툴은, 신규 태그, 다수의 신규 옵션, 향상한 다큐멘테이션 코멘트의 상속 기능, HTML 출력의 제어의 강화, 드크렛트 API 의 개선, 에러 메세지의 개선, 다수의 버그의 수정에 의해, 지금까지보다 실행하기 쉬워졌습니다. 자세한 것은,「Javadoc 1.4. 0 의 신기능」을 참조해 주세요.

    SDK 와는 별도로 다운로드되는 MIF 도크 렛 은 큰폭으로 업그레이드 되어 FrameMaker 및 PDF 형식에서 API 문서를 생성하게 되었습니다.

  • native2ascii 툴로 버그가 수정되었기 때문에, Windows NT 의 디폴트의 인코딩등과 같이, Unicode 로 encode 된 원시 파일을 올바르게 처리할 수 있게 되었습니다. 게다가 줄 끝이 2 바이트로 encode 되는 문자 인코딩 (가장 유명한 것은 Unicode)으로, 변환 코드가 줄 끝을 올바르게 처리하게 되었습니다. 이것들은, 버그 4191313 및 4218514 의 수정입니다.
  • 그래피컬한 Policy Tool 유틸리티가 확장되어 지정한 액세스 제어권을 어느 사용자에게 부여하는지를 나타내는 Principal 필드를 지정할 수 있게 되었습니다.
  • idlj 툴에서는, 디폴트로 생성되는 파일이 변경되어 새로운 커멘드행 옵션이 제공되게 되었습니다. 디폴트에서는,idlj 툴은, 「이식 가능 서번트 상속 모델」에 따른 서버측의 바인딩을 생성합니다. 새로운 커멘드행 옵션을 사용하면(자), 하위 호환성이 있는 서버측 바인딩을 생성할 수 있습니다. 이러한 변경은, Object Management Group (OMG)의 CORBA 스펙에 의해 한층 더 적합시키기 위해서(때문에) 더해진 것입니다.
  • orbd 툴 ( 「Object Request Broker Daemon」의 약어)이, 이 릴리스로 새롭게 추가되었습니다. ORBD 는 일시 네임 서비스 tnameserv 대신에 사용됩니다. ORBD 에는 일시 네임 서비스와 지속 네임 서비스의 양쪽 모두가 포함됩니다. 「orbd」 툴은, 클라이언트로부터 CORBA 환경의 서버에 있는 지속 객체를 투과적으로 검색해 호출하기 위해서(때문에) 사용합니다. orbd 툴에는, 서버 매니저, Interoperable Naming Service, 및 bootstrap 네임서버의 기능이 짜넣어지고 있습니다. 「s ervertool」 와 함께 사용하면(자), 서버 매니저는 클라이언트가 서버에의 액세스를 요구했을 때에 그 서버를 검색, 등록, 및 기동합니다.
  • servertool 가, 이 릴리스로 새롭게 추가되었습니다. servertool 는, 서버를 등록, 등록 해제, 기동, 및 종료하기 위한 , 어플리케이션 프로그래머전용의 사용하기 쉬운 인터페이스를 제공합니다.
  • rmic 컴파일러에, 원격 메소드 호출 (RMI)을 위한 Portable Object Adapter (POA) 지원를 가능하게 하는 새로운 옵션이 추가되었습니다. POA 는, 다른 벤더의 ORB 간이나, 다른 사용 방법과의 사이의 이식성을 실현합니다. POA 의 상세를 조사하려면 ,이 페이지를 참조해 주세요. rmic 컴파일러로 POA 지원를 유효하게 하려면 ,rmic -iiop -poa 라고 하는 인수를 지정해 컴파일러를 사용합니다.
  • 다음에 드는 툴에서는, 기능으로 변경은 없습니다만, 작은 버그의 수정을 했습니다. appletviewer,jar,jdb,javah,javap,extcheck, 및 unregbean
