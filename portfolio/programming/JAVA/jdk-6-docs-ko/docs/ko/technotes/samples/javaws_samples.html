Java Web Start 샘플

 문서의 목차


JavaTM 2 SDK 에는, 다음의 5 개의 샘플을 수록한 sample/jnlp 디렉토리가 있습니다.

  • CORBA
  • JRE 자동 다운로드 인스톨러
  • 랜덤 억세스 파일
  • 서브 렛
  • WebPad 어플리케이션

다음에, 각각의 샘플에 대해 간단하게 설명합니다. 상세한 것에 대하여는, JDK 의 samples/jnlp 디렉토리에 놓여진, 각 샘플의 README 파일을 참조해 주세요.

CORBA

이 샘플에서는, Java Web Start 내에서의 CORBA 의 사용 방법을 설명하고 있습니다. CORBA 를 사용한 단순한 Java 프로그램을 Java Web Start 로 패키지화, 다운로드, 및 JNLP 어플리케이션으로서 실행하는 방법을 나타내고 있습니다. RMI-IIOP 는, Java CORBA Object Request Broker (ORB) 및 IIOP 를 사용합니다. 이 때문에, Java 로 모든 코드를 기술해, rmic 컴파일러를 사용해, 어플리케이션을, CORBA 준거의 임의의 언어로 기술된 다른 어플리케이션에 Internet InterORB Protocol (IIOP)를 개입시켜 접속하기 위해서 필요한 코드를 생성할 수 있습니다. 이 샘플에서는, JNLP API 는 일절 사용하고 있습니다.

JRE 자동 다운로드 인스톨러

이 샘플에서는, Java Web Start JRE 자동 다운로드 인스톨러를 구축하는 방법을 설명하고 있습니다.

랜덤 억세스 파일

이 샘플에서는,JNLPRandomAccessFile API 의 기능에 대해 설명하고 있습니다. 사용자가 선택하는 파일은, 기입 및 읽기의 어느 쪽의 경우에서도, 랜덤 파일로서 열립니다. 이 어플리케이션은 우선, 몇개의 하드 코딩 된 캐릭터 라인을, 사용자가 선택한 파일에 기입합니다. 계속되어, 이 파일에 기입해지고 있는 것을 모두 읽어내,awt Container 및 TextArea 컴퍼넌트를 사용해, 화면상에 표시합니다.

서브 렛

Java Web Start 의 sample/jnlp/servlet 디렉토리에 포함되어 있는 서브 렛을 사용하면, JNLP 파일과 그 관련 자원을, Web 어카이브(archive) (. war) 파일내에 패키지화할 수 있습니다. 이 서브 렛의 목적은, JNLP 어플리케이션용의 간단해 편리성의 높은 패키징 형식을 제공하는 것에 의해, Tomcat 나 J2EE 준거의 어플리케이션 서버등의 Web 컨테이너에 JNLP 어플리케이션을 용이하게 배치할 수 있도록(듯이) 하는 것입니다.

WebPad 어플리케이션

이 단순한 워드 프로세서 어플리케이션을 사용하면, 파일의 작성, 수정, 보존, 및 인쇄를 실시할 수가 있습니다. Webpad 어플리케이션에서는, 신뢰할 수 없는 실행 환경에서 실행할 경우에도 로컬 자원에 액세스 하기 위해서,FileOpenService,FileContent,FileSaveService,ClipboardService,PrintService,DownloadService,BasicService, 및 PersistenceService 의 JNLP API 를 폭넓게 사용합니다.
