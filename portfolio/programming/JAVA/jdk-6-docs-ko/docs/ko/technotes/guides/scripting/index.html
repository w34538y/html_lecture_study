JavaTM 플랫폼의 스크립트

 문서의 목차

실제의 Java 프로그램과 같게, 프로그램이 Java 플랫폼에서 실행되도록(듯이), 스크립트 및 동적으로 형지를 무늬를 박아서 염색한 것 된 언어가 많은 구현이 Java 바이트 코드를 생성합니다. 이 방법에서의 (또는, 스크립트 언어의 Java interpreter 클래스로서의) 언어의 구현은, Java 플랫폼의 이점을 모두 제공합니다. 스크립트 구현에서는, Java 플랫폼의 바이너리 이식성, 시큐리티, 및 퍼포먼스의 높은 바이트 코드 실행의 이점을 얻을 수 있습니다.

Java SE 6 은,「JSR 223: Scripting for the JavaTM Platform」의 API 를 포함합니다. 이것은, Java 어플리케이션이 스크립트 엔진을 「호스트 한다」유익의 시스템입니다. Sun 에 의한 Java SE 6 의 구현은,「Mozilla Rhino:JavaScript for Java」에 근거하는 스크립트 엔진의 예를 포함하고 있습니다. 스크립트 시스템에서는, jar 「서비스 검출」기구에 의해, 서드 파티제의 스크립트 엔진이 지원되고 있습니다. CLASSPATH 내의 JSR-223 준거의 스크립트 엔진을 「드롭」해, Java 어플리케이션으로부터와 같이 액세스 하는 것이 가능합니다 (JDBC 드라이버와 같이, JNDI 구현이 액세스 됩니다).
