부록 1:
JNDI 표준 환경 프로퍼티

 

JNDI 환경 프로퍼티 1

환경 구성

java.naming.applet

(Context.APPLET)

java.applet.Applet 의 인스턴스. 그 애플릿 파라미터는, 환경 프로퍼티의 취득에 사용된다.
자세한 것은, 제 6 장을 참조. 디폴트는 존재하지 않는다

프로그램 구성 2

java.naming.factory.initial

(Context.INITIAL_CONTEXT_FACTORY)

사용하는 초기 문맥 팩토리의 클래스명

InitialContext 를 참조

디폴트는 존재하지 않는다

java.naming.factory.object

(Context.OBJECT_FACTORIES)

사용하는 객체 팩토리 클래스의 클래스명의 리스트. 구두점으로 단락짓는 것

NamingManager.getObjectInstance() 및 DirectoryManager.getObjectInstance() 를 참조

디폴트는 빈 상태(empty)의 리스트

java.naming.factory.state

(Context.STATE_FACTORIES)

사용하는 상태 팩토리 클래스의 클래스명의 리스트. 구두점으로 단락짓는 것

NamingManager.getStateToBind() 및 DirectoryManager.getStateToBind() 를 참조

디폴트는 빈 상태(empty)의 리스트

java.naming.factory.control

(LdapContext.CONTROL_FACTORIES)

사용하는 응답 제어 팩토리 클래스의 클래스명의 리스트. 구두점으로 단락짓는 것

ControlFactory.getControlInstance() 를 참조

디폴트는 빈 상태(empty)의 리스트

java.naming.factory.url.pkgs

(Context.URL_PKG_PREFIXES)

URL 문맥 팩토리에 로드할 때 사용하는 패키지의 접두사의 리스트. 구두점으로 단락짓는 것. com.sun.jndi.url 는 항상 리스트의 마지막에 추가된다

NamingManager.getURLContext() 를 참조

디폴트는 빈 상태(empty)의 리스트


액세스 구성 3

java.naming.provider.url

(Context.PROVIDER_URL)

프로바이더가 사용하는 구성 정보를 지정한다

디폴트는 프로바이더의 디폴트 값. 프로바이더 독자적인 구성 또는 발견 프로토콜이 사용된다

java.naming.dns.url

(Context.DNS_URL)

DNS 의 호스트명 및 도메인명을 지정한다

디폴트는 존재하지 않는다


서비스 관련

java.naming.authoritative

(Context.AUTHORITATIVE)

요구된 서비스의 권한을 지정한다. 「true」의 경우는, 가장 권한이 있는 소스를 사용하도록(듯이) 지정한다 (캐쉬를 모두 생략 하는, 시스템에 따라서는 복제를 생략 하는 등). 그렇지 않은 경우는, 권한이 있는 소스를 사용할 필요는 없다 (다만, 사용할 수도 있다)

디폴트 값는 「false」.

java.naming.batchsize

(Context.BATCHSIZE)

서비스의 프로토콜을 개입시켜 데이터를 돌려줄 때의, 적절한 배치 사이즈를 지정한다. 조작의 결과를 지정한 사이즈의 배치로 돌려주도록(듯이), 프로바이더에 통지된다. 프로바이더에서는, 성능 및 자원의 사용율이 최적화된다. 돌려주는 데이터의 합계수 또는 사이즈는 변경되지 않는다

디폴트는 프로바이더의 디폴트 값

java.naming.referral

(Context.REFERRAL)

서비스 프로바이더에 의해 검출된 소개가, 자동적으로 평가되도록(듯이) 지정한다. 「follow」의 경우는, 리페랄이 자동적으로 follow 된다. 「ignore」의 경우는, 검출된 리페랄이 ignore 된다. 「throw」의 경우는, 리페랄이 검출되었을 때에 ReferralException 가 throw 된다

디폴트는 프로바이더의 디폴트 값


시큐리티

java.naming.security.protocol

(Context.SECURITY_PROTOCOL)

서비스에 사용하는 시큐리티 프로토콜

디폴트는 프로바이더의 디폴트 값

java.naming.security.authentication

(Context.SECURITY_AUTHENTICATION)

「none」, 「simple」, 「strong」, 또는 프로바이더 고유의 캐릭터 라인 ( 「CRAM-MD5 DIGEST-MD5 EXTERNAL」등)을 지정할 수 있다

디폴트는 프로바이더의 디폴트 값

java.naming.security.principal

(Context.SECURITY_PRINCIPAL)

인증 schema의 주체 (사용자등)의 식별

디폴트는 프로바이더의 디폴트 값

java.naming.security.credentials

(Context.SECURITY_CREDENTIALS)

인증 schema의 주체의 자격

자격은, 선택한 인증 schema에 의해 결정된다. 자격에는, 패스워드, 키, 증명서등이 있다

디폴트는 프로바이더의 디폴트 값


국제화

java.naming.language

(Context.LANGUAGE)

이 서비스로 사용되는 우선 언어 ( 「en-US」, 「fr」, 「fr-CH」, 「ja-JP-kanji」등)를 구두점으로 단락지은 리스트를 지정한다.

언어는, RFC 1766 에 정의되고 있는 태그를 사용해 지정한다

디폴트는 프로바이더의 디폴트 값

 

1. Context 및 LdapContext 인터페이스에는, 이러한 프로퍼티명의 정수를 정의한다. 정수의 이름은, 프로퍼티의 캐릭터 라인명아래의 괄호내를 참조

2. 이러한 프로퍼티은, Java 실행 환경의 시스템 프로퍼티 또는 애플릿의 파라미터 리스트로 설정한다

3. 이러한 프로퍼티은, Java 실행 환경의 시스템 프로퍼티 또는 애플릿의 파라미터 리스트로 설정한다
