3. 아키텍쳐(architecture)의 개요

JNDI 아키텍쳐(architecture)는, JNDI API 및 JNDI SPI 로 구성되어 있습니다. JNDI API 를 사용해, Java 어플리케이션으로부터 다양한 네임 서비스 및 디렉토리 서비스에 액세스 할 수가 있습니다. JNDI SPI 는, 디렉토리 서비스 프로바이더등의 임의의 서비스 프로바이더용으로 설계되고 있습니다. 이것에 의해, 다양한 디렉토리 서비스 및 네임 서비스가, JNDI API 만을 사용하는 Java 어플리케이션에 투과적으로 접속됩니다. 다음의 JNDI 아키텍쳐(architecture)의 그림에는, 디렉토리 문맥 및 네이밍 문맥의 서비스 프로바이더의 예를 몇개인가 가리키고 있습니다.
