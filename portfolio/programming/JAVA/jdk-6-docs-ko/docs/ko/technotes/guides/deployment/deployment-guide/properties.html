구성 파일 및 프로퍼티의 배치

구성 파일 및 프로퍼티의 배치에는 다음의 내용이 포함됩니다.

  • 개요
  • 구성 파일의 배치
  • 구성 프로퍼티의 배치

개요

deployment.properties 파일은, Java 컨트롤 패널에서의 구성 프로퍼티의 배치를 포함 및 취득하기 위해서 사용합니다. 또, Java Plug-in 와 Java Web Start 양쪽 모두의 실행시의 동작을 커스터마이즈 하기 위해서도 사용합니다.

사용자 레벨의 deployment.properties 파일은 필수입니다. 그 위치는 후술 합니다만, 변경은 할 수 없습니다. 또, 시스템 레벨의 deployment.properties 파일 (생략 가능)도 있습니다. 존재하는 경우, 그 위치는, 시스템 관리 책임자가 후술 하는 위치에 있는 deployment.config 파일로 정의합니다.

구성 파일의 배치 (deployment.properties)

사용자 레벨

사용자 레벨의 deployment.properties 파일은 다음의 위치에 있습니다.

operating system
위치
Windows <User Application Data Folder>\Sun\Java\Deployment\deployment.properties
Unix ${user.home}/. java/deployment/deployment.properties

(${user.home} 는, 일반적으로 home/<username>)

례:

Windows 2000/XP 상에서 실행하고 있는 사용자 jsmith 의 경우,deployment.properties 파일은 다음의 위치에 있습니다.

C:\Documents and Settings\jsmith\Application Data\Sun\Java\Deployment\deployment.properties

Unix 상에서 실행하고 있는 사용자 bjones 의 경우,deployment.properties 파일은 다음의 위치에 있습니다.

/home/bjones/.java/deployment/deployment.properties

시스템 레벨

deployment.config 파일은, 사용의 인프라스트럭쳐(infrastructure)에 있어서의, 시스템 레벨의 deployment.properties 를 지정하기 위해서 사용됩니다. 디폴트에서는 deployment.config 파일은 존재하지 않습니다. 그 때문에, 시스템 레벨의 deployment.properties 파일도 존재하지 않습니다. deployment.config 가 존재하는 경우, 그 위치는 다음과 같습니다.

operating system
위치
Windows <Windows Directory>\Sun\Java\Deployment\deployment.config
Unix /etc/.java/deployment/deployment.config

또는

operating system
위치
Windows ${deployment.java.home}\lib\deployment.config
Unix ${deployment.java.home}/lib/deployment.config

(${deployment.java.home} 는, Java Web Start, Java Plug-in, Java Control Panel 등의 배치 대상 제품이 실행되고 있는 위치로부터의 jre 의 위치)

deployment.config 이 존재하는 경우,deployment.system.config 와 deployment.system.config.mandatory 의 2 개의 프로퍼티이 있습니다.

deployment.system.config 는, 시스템 (기업 전체)의 deployment.properties 파일에 대한 URL 입니다. 시스템 관리 책임자가 사용자 고유의 구성 설정을 중앙 관리 또는 엄중 관리하기 위해서 사용할 수 있습니다.

deployment.system.config.mandatory 는 불리언 값입니다. 디폴트 값는 false 입니다. false 의 경우,deployment.system.config 의 URL 로 나타나는 deployment.properties 파일을 로드하려고 합니다. URL 가 발견되어, 로드할 수 있는 경우는, 이 파일이 사용됩니다. true 의 경우,deployment.system.config 의 URL 로 나타나는 시스템 레벨의 deployment.properties 파일도 이와 같이 로드하려고 합니다. URL 가 발견되어, 로드할 수 있는 경우는, 이 파일이 사용됩니다. 발견되지 않는 경우나, 로드할 수 없는 경우는, 아무것도 실행할 수 없습니다.

구성 프로퍼티의 배치

deployment.properties 파일로 설정 가능한 프로퍼티을 다음의 겉(표)에 나타냅니다.

주:

어느 시스템 배치 프로퍼티 (즉 SomeKey=SomeValue)도, 다른 키 SomeKey.locked 를 포함하는 것으로 잠글 수 있습니다. 키 SomeKey.locked 는, 값을 설정할 수 있는 경우도 할 수 없는 경우도 있습니다만, 어느 쪽의 경우도 SomeKey=SomeValue 는 락 되기 (위해)때문에, 사용자가 변경할 수 없습니다. 시스템 배치 프로퍼티을 잠그지 않는 경우, 사용자에 의한 변경이 가능하게 됩니다.

deployment.security.authenticator
String true 일반적으로, Java Plug-in 와 Java Web Start 에서는, Web 페이지의 인증이나 프록시의 인증을 실시할 필요가 있는 통신을 처리하기 위해서, Authenticator 를 인스톨 한다. 이것은 디폴트의 동작이다 (true). 이 옵션은, 일반적으로의 동작을 멈추는 경우에 사용한다. 예를 들어, 어플리케이션이 Web 페이지의 인증을 실시하면서 자신과 통신하기 위해서, 독자적인 Authenticator 를 인스톨 할 필요가 있는 경우 등

브라우저의 선택과 패스
