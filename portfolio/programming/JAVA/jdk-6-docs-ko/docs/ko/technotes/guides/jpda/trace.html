Java Platform Debugger Architecture

트레이스는, 인수로서 건네받은 Java 언어 프로그램을 실행해, 그 실행의 트레이스를 생성합니다. 트레이스는, Java Debug Interface (JDI)를 사용하는 간단하고 쉬운 커멘드행 툴입니다. 디버그의 경우, 이 정보는 사용되지 않기 때문에, 프로그램을 컴파일 할 필요는 없습니다.

트레이스는 다음과 같이 해 호출할 수가 있습니다. 

        java Trace options class args
클래스 패스에는, JDI 라이브러리 (set-up 를 참조)를 포함한 Java 2 SDK 의 lib/tools.jar 파일, 컴파일 된트레이스클래스 파일에의 패스, 및 감시되는 어플리케이션의 패스가 포함되어 있을 필요가 있습니다.

사용 가능한 옵션은 다음과 같습니다. 
  -output filename
        트레이스의 출력처를 설정합니다. 디폴트에서는
        터미널에 출력합니다. 

  -all
        출력에 시스템 클래스를 포함합니다. 디폴트에서는
        java. *, javax. *, sun. * and com.sun. * 의 각 이벤트를
        표시하지 않습니다. 

  -fields
        필드에의 대입도 표시합니다. 

  -help
        help 메세지를 출력합니다.

class 는 감시하는 프로그램으로,args 는 class 의 인수입니다.

트레이스의 소스

트레이스의 완전한 원시 코드는,examples.jar 의 trace 디렉토리에 있습니다. 이러한 샘플 어플리케이션의 원시 코드는, 디버거 개발자를 위해서(때문에) 구체적인 예를 나타내기 위한 것입니다. 디버거의 개발자에 의한 코드 예의 사용, 변경, 재배포는, 저작권 표시의 준수를 조건으로서 인정됩니다.

트레이스의 구축

제공된 trace 디렉토리의 원시 파일로부터 trace 클래스를 완전하게 재구축 하는 경우, 필요한 작업은 원시 파일의 컴파일 뿐입니다. 클래스 패스에 lib/tools.jar 의 JDI 라이브러리 를 포함하는 것 이외, 특별한 옵션은 필요 없습니다.
