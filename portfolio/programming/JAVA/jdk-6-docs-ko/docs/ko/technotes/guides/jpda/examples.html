Java Platform Debugger Architecture - JDI 어플리케이션의 예

v1. 3.0 이후의 릴리스의 경우, Java 2 SDK 의 demo/jpda 디렉토리에는,Java Platform Debugger Architecture (JPDA)의 Java Debug Interface (JDI)를 사용해 기술된 3 개의 어플리케이션의 원시 코드와 문서가 포함되어 있습니다. 이것들은 디버거를 개발하기 위한 교육용 툴로서 준비되어 그 출발점이 됩니다.

복잡도가 더하는 순으로 열거합니다.

  • 트레이스 는, 프로그램 실행의 트레이스를 표시합니다. JDI 의 기본 기능의 대부분은, 매우 간략 (600 행 미만)입니다. 출발점에는 적격인 것입니다.
  • Jdb 는 J2SE SDK v1. 3 이후의 릴리스로 배포되고 있는 커멘드행 디버거입니다.
  • Javadt 는 GUI 디버거의 선두입니다.

트레이스는 trace 디렉토리에 있습니다. Jdb 와 Javadt 는 패키지를 공유해,debuggers 디렉토리에 들어가 있습니다.

필수 설정

클래스 패스에는 JDI 라이브러리 코드가 포함되어 있을 필요가 있습니다. JDI 라이브러리 코드는 Java 2 SDK 의 lib/tools.jar 디렉토리에 있습니다 (tools.jar 는 Java Runtime Environment 에는 포함되어 있지 않습니다. Java 2 SDK 에게만 포함되어 있습니다). 이것은, 샘플 코드의 컴파일과 실행의 양쪽 모두를 위해서(때문에) 필요합니다. 클래스 패스의 설정 방법법에 대해서는,「Java SE 툴의 문서」를 참조해 주세요.
