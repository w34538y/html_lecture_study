처음에

「Java Management Extensions (JMX) 테크놀러지의 튜토리얼」에서는, Java Platform, Standard Edition (Java SE)로 제공되는 JMX 테크놀러지의 주요한 기능의 사용법의 예를 나타냅니다.

본서를 읽기 전에

「Java Management Extensions (JMX) 테크놀러지의 튜토리얼」에서는, 「Java Management Extensions (JMX) 테크놀러지의 개요」로 소개한 개념과 테크놀러지를 예를 들어 설명합니다. 따라서, 이 튜토리얼의 작업에 들어가기 전에, 개요를 읽어 둘 필요가 있습니다. 이 튜토리얼의 정보를 충분히 활용하기 위해서는, 다음의 프로토콜 및 스펙을 숙지하고 있을 필요가 있습니다.

  • Remote Method Invocation (RMI)
  • Lightweight Directory Access Protocol (LDAP)
  • Service Location Protocol (SLP)
  • JiniTM 네트워크 테크놀러지
  • Java Naming and Directory InterfaceTM (JNDI) API
  • Java Secure Socket Extension (JSSE)
  • Java Authentication and Authorization Service (JAAS)

본서의 구성

이 튜토리얼은, 다음의 장으로 폭넓은 카테고리의 예를 제공합니다.

  • 제 1 장 「입문」 에는, 초기설정 정보의 일부를 기재합니다.
  • 제 2 장 「JMX API 의 기본 요소」 에서는, JMX 스펙의 코어 개념을 소개합니다.
  • 제 3 장 「JMX 연결기」 에서는, 표준 타입과 동적 타입의 MBean 를 구현해, 양타입의 MBean 상에서 조작을 로컬 및 원격에 실행하는 방법의 예를 나타냅니다.
  • 제 4 장 「검색 서비스」 에서는, JMX 테크놀러지와 아울러 사용할 수 있는 검색 서비스를 예증 합니다.
  • 제 5 장 「시큐리티」 에서는, 시큐리티 설정의 몇개의 예를 나타냅니다.

이 튜토리얼에 대해

벌써 말한 것처럼, 이 튜토리얼에서는 「Java Management Extensions (JMX) 테크놀러지 개요」로 말한 개념의 몇개인가를 예를 들어 가리킵니다. 이 튜토리얼은, 예제에 따라 작업을 진행시키면서 최초부터 차례로 읽는 구성을 취하고 있습니다. 튜토리얼의 첫머리에서 실시하는 작업이, 이후의 예로 빠뜨릴 수 없는 원가요소가 되어 있는 경우가 있습니다. 따라서, 튜토리얼의 도중부터 시작했을 경우, 특정의 예제로 요구되는 작업이 빠져 버릴 가능성이 있습니다.
