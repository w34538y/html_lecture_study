  • 트레이스

  • 그 외의 옵션

트레이스

트레이스란, Java 콘솔에서의 임의의 출력을 트레이스 파일에 리다이렉트(redirect) 하는 기능입니다.

Java Plug-in

트레이스는, 프로퍼티 javaplugin.trace 를 유효하게 하면(자) 온이 됩니다. 다만, 이것에 의해 Java Plug-in 내부의 모든 트레이스 기능이 온이 됩니다. 한층 더 상세한 트레이스를 유효하게 하는 경우는, javaplugin.trace.option 를 사용합니다. 전장에서 가리킨 것처럼,Java 콘솔로 트레이스 레벨 옵션 (0 ? 5)를 설정할 수 있습니다. 이러한 옵션에는 다음의 의미가 있습니다.

  • 0 — 오프
  • 1 — 기본
  • 2 — 네트워크, 캐쉬, 및 기본
  • 3 — 시큐리티, 네트워크 및 기본
  • 4 — 확장 기능, 시큐리티, 네트워크, 및 기본
  • 5 — LiveConnect, 확장 기능, 시큐리티, 네트워크, temp, 및 기본

이것으로, 즉시 트레이스를 유효하게 할 수가 있습니다.

상세하게 트레이스를 설정하는 이제(벌써) 1 개의 방법은,Java 컨트롤 패널을 사용하는 것입니다. 예를 들어, 모든 트레이스를 유효하게 하는 (상기의 옵션 5) 경우,[Java 런타임 파라미터] 텍스트 필드에 다음과 같이 입력합니다.

-Djavaplugin.trace=true
-Djavaplugin.trace.option=basic|net|cache|security|ext|liveconnect|temp

컨트롤 패널로 트레이스를 설정했을 경우, Plug-in 의 기동시에 트레이스가 유효하게 됩니다만, Plug-in 의 실행중으로 컨트롤 패널로 변경을 더해도, 재기동할 때까지 트레이스는 유효하게는 되지 않습니다.

Java Web Start

Java Web Start 어플리케이션의 첫회의 트레이스 레벨을 설정하려면 , 배치 프로퍼티에 deployment.trace.level 을 설정합니다. deployment.trace.level 프로퍼티에는, 다음의 값의 어느쪽이든을 사용할 수 있습니다.

  • basic
  • cache
  • net
  • security
  • ext
  • liveconnect

모든 트레이스 레벨을 사용하려면 ,deployment.trace.level 를 all 로 설정합니다.

로그 기능

트레이스와 같게, 로그와는 Java 콘솔에서의 임의의 출력을, Java Logging API 를 사용해 로그 파일에 리다이렉트(redirect) 하는 기능입니다. 로그 기능을 유효하게 하려면 , 다음의 작업을 실시합니다.

  • Java 컨트롤 패널을 엽니다.
  • 「상세 설정」 탭을 클릭합니다.
  • 「디버그」옵션의「로그를 유효하게 한다」를 선택합니다.

 

그 외의 옵션

파일명

트레이스 및 로그 파일명은 다음과 같이 됩니다.

plugin<modified version number>.trace
plugin<modified version number>.log

<modified version number> 에는, 메이저, 마이너, 및 팟치파젼 번호가 포함됩니다 (plugin142_06.trace 또는 plugin150.log 등).

파일의 위치

트레이스와 로그 파일의 디폴트 위치 (디렉토리)를 다음에 나타냅니다.

  • <user.home>/.java/deployment/log (Unix 또는 Linux 의 경우)
  • <User Application Data Folder>\Sun\Java\Deployment\log (Windows 의 경우)

<User Application Data Folder> 는여기에 정의되고 있습니다.

환경 변수 USER_JPI_PROFILE 가 <user plugin home> 로 설정되어 있는 경우, 트레이스 및 로그 파일은 다음의 장소에 기입해집니다.

  • <user plugin home>/.java/deployment/log (Unix 또는 Linux 의 경우)
  • <user plugin home>\Sun\Java\Deployment\log (Windows 의 경우)
