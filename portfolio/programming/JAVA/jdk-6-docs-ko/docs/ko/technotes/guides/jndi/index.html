Java Naming and Directory Interface

 문서의 목차
Java Naming and Directory Interface (JNDI)는, Java 프로그램 언어로 기술된 어플리케이션에, 네이밍 및 디렉토리 기능을 제공합니다. 다른 네임 및 디렉토리 서비스의 구현과 독립해 동작하도록(듯이) 설계되고 있습니다. 새로운 서비스, 전개되고 있는 서비스, 벌써 전개되고 있는 서비스 등, 다양한 서비스에 대해서 공통의 방법으로 액세스 할 수가 있습니다.

JNDI 아키텍쳐(architecture)는, API (Application Programming Interface) 및 SPI (Service Provider Interface)로 구성됩니다. 이 API 를 사용해, Java 어플리케이션으로부터 다양한 네임 및 디렉토리 서비스에 액세스 할 수가 있습니다. SPI 에 의해, 다양한 네임 및 디렉토리 서비스가 투과적으로 플러그 인 됩니다. 이러한 서비스에는, JNDI 테크놀러지의 API 를 사용해 Java 어플리케이션으로부터 액세스 할 수가 있습니다.

  • JNDI 서비스 프로바이더：
    JNDI 로 특정의 네임 및 디렉토리 서비스를 사용할 때는, JNDI 의 「서비스 프로바이더」가 필요합니다. 서비스 프로바이더는, 네임 및 디렉토리 서비스에 액세스 하기 위해서, JNDI API 의 하위에 플러그 인 되는 모듈입니다. JNDI 용의서비스 프로바이더의 리스트는,JNDI 의 Web 사이트로부터 입수할 수 있습니다.
