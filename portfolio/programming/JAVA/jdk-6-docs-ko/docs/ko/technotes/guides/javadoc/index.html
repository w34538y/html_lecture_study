Javadoc 테크놀러지

 문서의 목차
Javadoc 는, 원시 파일 세트내의 선언 및 다큐멘테이션 코멘트를 해석해, 클래스, 인터페이스, 생성자 , 메소드, 및 필드에 도착해 기술한 HTML 페이지 세트를 생성하는 툴입니다.

Javadoc 의 도크 렛을 사용해 Javadoc 의 출력을 커스터마이즈 할 수 있습니다. 도크 렛이란, 도크 렛 API 를 사용해 쓰여진 프로그램으로, Javadoc 툴로 생성해야 할 출력의 내용과 형식을 지정하고 있습니다. HTML, SGML, XML, RTF, MIF 등, 임의의 종류의 텍스트 파일 출력을 생성하기 위한 도크 렛을 작성할 수 있습니다. SunTM 는, HTML 형식의 API 문서를 생성하는 「표준」도크 렛과 MIF, PDF, FrameMaker, 및 그 외의 형식을 생성하는 실험적인 MIF 도크 렛을 제공하고 있습니다. 게다가 도크 렛을 사용하면(자), API 문서의 생성과는 관계가 없는 특별한 처리를 실시할 수도 있습니다.

태그 렛은, -tag 옵션을 사용해 작성된 커스텀 태그보다 유연성이 있는 커스텀 태그를 작성해 사용할 수 있는 프로그램입니다. 커스텀 태그는, Javadoc 태그로부터 텍스트의 서식 설정과 표시를 실시하기 위해서(때문에) 도크 렛으로 사용됩니다. 태그 렛은, 태그 렛 인터페이스를 구현할 필요가 있습니다.

API 의 스펙

  • 「도크 렛 API」 - 도크 렛 API 를 구성하는 com.sun.javadoc 패키지에 대해 설명하고 있습니다.
  • 「인터페이스 태그 렛」 - 태그 렛 API 를 구성하는 com.sun.tools.doclets.Taglet 클래스에 도착해 설명하고 있습니다.

툴

이 페이지에서는, Javadoc 툴을 실행하기 위한 모든 javadoc 태그와 커멘드행 옵션을, Solaris 및 Microsoft Windows 로 사용하는 경우의 예와 함께 일람표 가리킵니다.
  • Javadoc 툴의 레퍼런스 페이지 (Solaris)
  • Javadoc 툴의 레퍼런스 페이지 (Microsoft Windows)

튜토리얼 및 개발자 안내

도크 렛의 정보:
  • 「도크 렛의 개요」 - 도크 렛의 개요를 도크 렛 API 의 사용법의 간단한 실례와 함께 소개합니다.
  • 표준 도크 렛의 실행 - 디폴트의 HTML 형식의 API 문서를 생성하는 도크 렛. Javadoc 의 -doclet 옵션으로 도크 렛을 아무것도 지정하지 않으면 이 표준 도크 렛이 사용된다
  • 「5.0 도크 렛 API 에의 이행」 - 5.0 의 새로운 언어 기능을 지원할 수 있도록(듯이) 종래의 도크 렛을 확장하기 위한 힌트.
태그 렛의 정보:
  • 「태그 렛의 개요」 - 태그 렛의 개요를 태그 렛 API 의 사용법의 간단한 실례와 함께 소개합니다.
