Java SE 6 감시 및 관리의 확장 기능

 문서의 목차

감시 및 관리 API 의 확장 기능

Java Platform, Standard Edition (Java SE)의 패키지 java.lang.management 에는, Java 가상 머신 (Java VM)과 다른 어플리케이션의 감시 및 관리를 행하기 위한 API 가 포함되어 있습니다.

Java SE 6 플랫폼에는, 이 API 의 확장 기능으로서 다음의 것이 포함되어 있습니다.

  • java.util.concurrent 락의 지원가 추가되었습니다.

    ThreadMXBean.getThreadInfo 메소드가 확장되어 thread로 취득 대기가 되고 있는 java.util.concurrent.locks.AbstractOwnableSynchronizer 를 보고합니다. Java 2 Platform, Standard Edition (J2SE) 5.0 에서는, thread에 의한 취득 개시 또는 취득 대기가 블록 되었다고 하는 객체 모니터를 보고하는 것만으로 했다.

    그 외 ,ThreadMXBean 인터페이스에, 다음의 새로운 메소드가 추가되었습니다.

    • isObjectMonitorUsageSupported()
    • isSynchronizerUsageSupported()
    • findDeadlockedThreads()
  • thread가 소유하는 락의 리스트를 취득해, 모니터를 잠근 스택 프레임을 보고하기 위한 새로운 클래스와 메소드가,java.lang.management 패키지에 추가되었습니다.
    • 새로운 LockInfo 클래스.
    • 새로운 MonitorInfo 클래스.
    • 새로운 ThreadInfo 메소드 getLockInfo(),getLockedMonitors(), 및 getLockedSynchronizers() .
    • 새로운 ThreadMXBean 메소드 dumpAllThreads() 및 getThreadInfo(long[] ids, boolean lockedMonitor, boolean lockedSynchronizers)
  • 새로운 메소드 getSystemLoadAverage() 가 OperatingSystemMXBean 에 추가되어 시스템의 로드 평균이 돌려주어지게 되었습니다.

JConsole 의 확장 기능

Java SE 플랫폼에는, JConsole 로 불리는 감시용 그래피컬 툴이 준비되어 있습니다. JConsole 툴은, Java Management Extensions (JMX) API 를 구현해, Java VM 와 계측 된 어플리케이션의 퍼포먼스 감시를 가능하게 합니다. JConsole 는 J2SE 5.0 으로부터 도입되어 Java SE 6 으로 정식적 지원의 대상이 되었습니다.

J2SE 5.0 으로부터 Java SE 6 에의 이행에 수반해, JConsole 툴에 베풀어진 확장 기능의 일부를 다음에 나타냅니다.

  • JConsole 플러그 인의 지원: 어플리케이션의 MBean 에 액세스 할 때에 필요한 커스텀 탭을 추가하는 등, JConsole 로 실행되는 독자적인 플러그 인 구축이 가능해집니다.
  • 동적 접속 기능: 동적 Attach API (이것도 Java SE 6 에 추가되었다)를 지원하는 임의의 어플리케이션에, JConsole 를 접속할 수 있습니다.
  • 강화된 사용자 인터페이스: 데이터 액세스를 한층 더 간단하게 합니다.
  • 새로운 「개요」탭 및 「VM 의 개요」탭: 사용중의 Java VM 에 관한 전체 정보를 보다 알기 쉽게 표시합니다.
  • HotSpot 진단 MBean: 실행시에 heap 덤프를 요구해, 또 일부의 VM 옵션의 설정 변경을 실시하는 API 가 준비되어 있습니다.
  • 강화된 MBean 프레젠테이션: MBean 의 처리나 속성에 대한 액세스를 간단하게 합니다.

아웃 오브 박스의 remote administration의 확장 기능

Java VM 에는 Instrumentation이 짜넣어지고 있어 이것에 의해 JMX 테크놀러지를 사용한 감시 및 관리를 실시할 수가 있습니다. 이러한 편입 관리 유틸리티는 일반적으로, Java VM 의 「아웃 오브 박스의」관리툴로 불립니다. 이 릴리스에서는, 아웃 오브 박스의 관리 유틸리티에 다음의 확장이 베풀어졌습니다.

  • SSL (secure sockets layer)를 사용하는 것으로, 원격 메소드 호출 (RMI: remote method invocation) 레지스트리의 시큐리티가 확보되었습니다. 새로운 관리 프로퍼티 com.sun.management.jmxremote.registry.ssl 가 추가되었습니다. 이것에 의해, 적절한 SSL 증명서를 가지는 정당한 클라이언트만이 연결기 Stub를 취득할 수 있도록(듯이), SSL 유효화 RMI 레지스트리를 사용하는 것이 가능하게 되었습니다.
