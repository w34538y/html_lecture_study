NullPointerException 가 java.awt.Graphics.drawImage() 로 throw 된다


증상

애플릿을 Sun JRE 를 사용한 브라우저로 실행하고 있는 경우,NullPointerException 가 java.awt.Graphics.drawImage 메소드로 throw 됩니다. 같은 애플릿이 Microsoft VM 에서는 실행할 수 있습니다.

원인

이 예외는, Sun JRE 로 null 이미지가 drawImage 메소드에게 건네지기 위해서(때문에) 일어납니다.

Sun JRE 의 Java 클래스 라이브러리는 시간의 경과와 함께 변화해 왔습니다. 상세하게 된 API 도 있으면, 폐지된 API 도 있습니다. 또, 구현이 변경된 API 도 있습니다.

null 이미지를 Graphics.drawImage 메소드에 건네주었을 경우의 결과는 상세하게 정의되어 있지 않기 때문에, Microsoft VM 에서는 null 이미지를 무조작으로서 취급합니다. 그러나, Sun JRE 의 대부분의 버젼에서는 null 를 유효한 이미지로서 허용 하지 않기 때문에,NullPointerException 가 throw 됩니다. 5.0 에서는 스펙이 상세화 되어 null 이미지의 인수는 무조작으로서 다루어집니다.

해결 방법

5.0 이전의 버젼의 JRE 에서는, null 이외의 이미지만이 drawImage 메소드에게 건네지도록(듯이) 대책을 강의(강구)한 코드를 기술합니다. 예를 나타냅니다.

        Graphics g = getGraphics();
    g.drawImage(img, 100, 100, this);

        The code should be changed to:

        Graphics g = getGraphics();
        if (img ! = null)
            g.drawImage(img, 100, 100, this);
