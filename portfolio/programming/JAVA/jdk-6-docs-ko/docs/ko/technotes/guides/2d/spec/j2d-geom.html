제 3 장  

기하학적 도형

Java 2D™ API 에서는, 점, 직선, 곡선, 구형 등, 기본적인 기하학 객체를 정의하는 클래스가 몇개인가 제공되고 있습니다. 새로운 기하학 클래스는,java.awt.geom 패키지에 포함되어 있습니다. 하위 호환이기 때문에,Rectangle,Point,Polygon 등, 구버젼의 JDK 로 제공되고 있던 기하학 클래스도,java.awt 패키지에 남아 있습니다.

GeneralPath,Arc2D,Rectangle2D 등 Java 2D API 의 기하 도형은,java.awt 로 정의되고 있는 Shape 인터페이스를 구현하고 있습니다. Shape 에서는, 기하학적인 패스를 가지는 객체를 기술하거나 검사하거나하기 위한 , 공통의 프로토콜이 제공되고 있습니다. 새로운 인터페이스의 PathIterator 는, 기하학 도형으로부터 요소를 추출하기 위한 메소드를 정의하고 있습니다.

기하학 클래스를 사용하면(자), 거의 모든 2 차원 객체를, 간단하게 정의해 조작할 수 있습니다.

3.2 기하학적 도형의 컨셉

Shape 는,GeneralPath 나 Rectangle2D.Float 등,Shape 인터페이스를 구현하는 모든 클래스의 인스턴스입니다. Shape 의 외형 (윤곽)을 「패스」라고 부릅니다.

Shape 를 draw 하는 경우는,Graphics2D 문맥의 Stroke 객체로 정의되고 있는 펜의 스타일이,Shape 의 패스에 적용됩니다. Shape 를 전부 칠하는 경우는,Graphics2D 문맥의 Paint 가, 패스의 안쪽의 영역에 적용됩니다. 상세한 것에 대하여는,「Graphics2D 를 사용한 렌더링」을 참조해 주세요.

Shape 의 패스는, 「클리핑 패스」의 정의에도 사용할 수 있습니다. 클리핑 패스에 의해, 렌더링 되는 픽셀이 정해집니다. 클리핑 패스로 정의되는 영역의 안쪽에 있는 픽셀만이, 렌더링 됩니다. 클리핑 패스는,Graphics2D 문맥의 일부입니다. 상세한 것에 대하여는,「클리핑 패스의 설정」을 참조해 주세요.

GeneralPath 를 사용하면(자), 직선, 2 차 곡선, 및 3 차 곡선을 사용해 구성되는 임의의 2 차원 객체를 나타낼 수가 있습니다. java.awt.geom 에서는, 구형, 타원, 원호, 곡선등이 일반적인 기하 도형 객체를 간단하게 표현할 수 있도록, 전용의 Shape 인터페이스의 구현이 별도로 제공되고 있습니다. 또, Java2D™ API 에서는, 구성 영역 지오메트리를 지원하는 특별한 종류의 Shape 도 제공되고 있습니다.

3.2. 1 구성 영역 지오메트리

구성 영역 지오메트리 (CAG)는, 기존의 객체에 불 연산을 행해 새로운 기하학 객체를 만드는 처리입니다. Java 2D API 에서는,Area 라는 이름의 특별한 종류의 Shape 가, 불 연산을 지원하고 있습니다. 임의의 Shape 로부터 Area 를 구성할 수 있습니다.

Area 에서는, 다음의 불 연산을 실행할 수 있습니다.

  • 논리합
  • 논리적
  • 논리차이
  • 배타적 논리합 (XOR)

3.2. 2 도형의 범위와 히트 검사

「바운딩 박스」는, Shape 의 도형을 완전하게 둘러싸는 구형입니다. 바운딩 박스를 사용해, 사용자가 객체를 선택했는지 어떠했는지, 즉 「히트」했는지의 여부를 판정합니다.

Shape 인터페이스에서는, 도형의 바운딩 박스를 취득하는 메소드로서getBounds 와 getBounds2D 의 2 개가 정의되고 있습니다. getBounds2D 로부터는 Rectangle 는 아니고 Rectangle2D 가 돌려주어지므로, 도형의 바운딩 박스에 대해 한층 더 높은 정밀도의 기술을 취득할 수 있습니다.

Shape 에서는, 다음의 판정을 실시하는 메소드도 제공되고 있습니다.

  • 지정한 점이 도형의 범위내에 있을지 어떨지 (contains)
  • 지정한 구형이 도형의 범위에 완전하게 포함될지 어떨지 (contains)
  • 지정한 구형이 도형과 교차하고 있을지 어떨지 (intersects)

3.3 Area 의 결합에 의한 새로운 Shape 의 작성

Area 를 사용하면(자), 엔이나 정방형과 같이 단순한 도형으로부터 복잡한 Shape 를 간단하게 만들 수가 있습니다. 다음은,Area 를 결합해 새롭고 복잡한 Shape 를 만드는 순서입니다.

  1. Shape 를 사용해, 결합하는 Area 를 작성한다
  2. 불 연산자 add,subtract,intersect,exclusiveOr 로부터 적절한 것을 호출한다

예를 들어, CAG 를 사용하면(자),그림 3-2 에 나타내는 것 같은 양없음형을 만들 수가 있습니다.

그림 3-2 복수의 엔으로부터 만들어진 양없음

양없음의 본체는, 서로 겹치는 엔과 타원의 2 개의 Area 에 논리합연산을 행해 만듭니다. 각각의 잎은, 겹치는 2 살의 엔에 논리곱연산을 행해 만들어, 그것을 논리합연산으로 1 개의 Shape 에 결합하고 있습니다. 줄기의 부분도, 서로 겹치는 엔에 논리차이 연산을 2 회 행해 만들어지고 있습니다.

3.4 독자적인 Shape 의 작성

Shape 인터페이스를 구현해, 새로운 종류의 형상을 정의하는 클래스를 작성할 수 있습니다. Shape 인터페이스의 메소드를 구현할 수 있다면, 내부적으로 도형을 어떻게 표현해도 괜찮습니다. Shape 에서는, 윤곽을 지정하는 패스를 생성하지 않으면 안됩니다.

예를 들어, 점의 배열로서 다각형을 나타내는 Shape 의 간단한 구현을 작성할 수 있습니다. 이와 같이 해 다각형을 작성되어 있으면,draw 나 setClip 등, 인수로서 Shape 객체를 받는 임의의 메소드에 다각형을 건네줄 수가 있습니다.

PolygonPath 클래스에서는, 다음의 Shape 인터페이스 메소드를 구현할 필요가 있습니다.

  • contains
  • getBounds
  • getBounds2D
  • getPathIterator
  • intersects
