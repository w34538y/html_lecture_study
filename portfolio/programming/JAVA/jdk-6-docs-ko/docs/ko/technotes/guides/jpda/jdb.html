jdb 의 호출

jdb 샘플은, 다음을 실행하는 것에 의해 실행할 수 있습니다. 

   java tty.TTY <options>.. <class-name>
이 경우, <class-name> 는 일반적으로,java 커멘드행에 배치하는 이름입니다. -help 옵션은, 옵션에 관한 정보를 제공합니다. 주:Java 2 SDK 의 lib/tools.jar 파일은, 반드시 tty.TTY 를 호출하기 위한 클래스 패스로 설정해 주세요. tools.jar 파일에는,JDI 라이브러리와 컴파일 된 jdb 클래스 파일이 포함됩니다.

호출과 접속의 상세한 것에 대하여는, JPDA 문서의 「접속 및 호출의 상세」섹션의, 특히 jdb 의 섹션을 참조해 주세요.

jdb 의 소스

jdb 의 완전한 원시 코드는, Java 2 SDK 의 lib/jpda 디렉토리의 코드 예의 일부로서examples.jar 에 포함되어 있습니다 (jar xvf examples.jar 를 사용해 examples.jar 의 내용을 추출합니다). 주:이 파일에는,javadt 의 소스도 들어가 있습니다. 이러한 샘플 어플리케이션의 원시 코드는, 디버거 개발자를 위해서(때문에) 구체적인 예를 나타내기 위한 것입니다. 디버거의 개발자에 의한 코드 예의 사용, 변경, 재배포는, 저작권 표시의 준수를 조건으로서 인정됩니다.

jdb 는 다음의 패키지 (examples.jar 파일에 있다)를 사용합니다.

tty
어플리케이션 코드
expr
식 처리 코드

jdb 의 구축

제공된 원시 파일로부터 jdb 클래스를 완전하게 재구축 하는데 필요한 작업은, 컴파일 뿐입니다. 클래스 패스에 JDI 라이브러리 를 포함하는 것 이외, 특별한 옵션은 필요 없습니다. 다만, 파일 Expr.jj 의 표현 퍼서를 변경하는 경우는,JavaCC 퍼서 제네레이터 가 필요합니다.
