제 3 장

Image I/O 어플리케이션의 작성

3.1 이미지의 read와 기입해

javax.imageio.ImageIO 클래스는, 단순한 이미지 입출력 조작의 대부분을 실행하는 정적인 편리한 메소드 일식을 제공하고 있습니다.

표준 형식 (GIF, PNG, 또는 JPEG)의 이미지의 read는, 다음의 예의 같게 간단합니다. 

File f = new File("c:\images\myimage.gif");
BufferedImage bi = ImageIO.read(f);
이미지의 형식은, 파일의 내용에 근거해 API 가 자동적으로 검출합니다. 대부분의 이미지 파일에는, 선두의 수바이트에, 파일 형식을 식별하는 「magic number-」가 포함되어 있습니다. magic number-가 없는 이미지 형식에 대해서는, 자동 검출이 실패하기 위해(때문에), 한층 더 복잡한 어플리케이션 코드가 필요하게 됩니다.

추가의 이미지 형식의 처리는, 적절한 플러그 인이 들어간 JAR 파일을 인스톨 하면 가능하게 됩니다. 상세한 것에 대하여는, 다음의 장으로 설명합니다. 플러그 인을 인스톨 한 뒤는, 어플리케이션 코드를 전혀 변경하지 않고 구라고도, 새로운 이미지 형식을 자동적으로 인식할 수 있게 됩니다.

읽어들여 가능한 이미지 형식의 리스트는,ImageIO.getReaderFormatNames 를 호출하면 입수할 수 있습니다. 이 메소드로부터는, 이용할 수 있는 이미지 형식의 이름 ( 「gif」나 「png」등)이 들어간 String 의 배열이 돌려주어집니다.

지원되고 있는 형식에서 이미지를 기입하는 조작도, 똑같이 간단합니다. 

BufferedImage bi;
File f = new File("c:\images\myimage.png");
ImageIO.write(im, "png", f);
지원되고 있는 형식의 리스트는,ImageIO.getWriterFormatNames 를 호출하면 입수할 수 있습니다.
