주제, 법칙, 인증, 및 쿠레덴샤르

자원에의 액세스를 승인하는 경우, 최초로, 어플리케이션이 요구원을 인증할 필요가 있습니다. JAAS 시스템에서는, 요구원을 「주제」라고 하는 말로 나타냅니다. 주제는, 사용자나 서비스등의 임의의 엔티티입니다. 주제는,javax.security.auth.Subject 클래스에서 나타내집니다.

「인증」은, 주제의 식별 정보를 검증하는 프로세스를 나타냅니다. 인증은, 시큐리티 보호된 방법으로 실행할 필요가 있습니다. 그렇지 않으면, 악의가 있는 사용자가 다른 사용자를 가장해 시스템에의 액세스를 시도할 가능성이 있습니다. 일반적으로, 인증에는, 식별 정보를 증명하는 어떠한 증거를 나타내는 주제가 포함됩니다. 이런 종류의 증거에는, 주제만이 파악하는 또는 보관 유지 할 수 있는 정보 (패스워드나 지문 등), 또는 주제만이 작성 가능한 정보 (비공개키를 사용한 서명 첨부 데이터 등)가 있습니다.

인증이 완료하면(자), 주제는 관련지을 수 있었던 식별 정보, 즉 「법칙」(java.security.Principal 타입의 것)을 사용해 생성됩니다 주제에는, 다수의 법칙을 포함할 수가 있습니다. 예를 들어, 사용자는, 다른 주제와 명확하게 구별되는 이름 법칙 ("John Doe") 및 SSN 법칙 ("123-45-6789")을 보관 유지할 수 있습니다.

주제는, 관련지을 수 있었던 법칙에 가세해 「쿠레덴샤르」라고 불리는 독자적인 시큐리티 관련 속성을 보관 유지할 수 있습니다. 쿠레덴샤르에는, 신규 서비스에 대해서 주제를 인증할 때에 사용 가능한 정보가 포함됩니다. 이런 종류의 쿠레덴샤르에는, 패스워드, Kerberos 티켓, 및 공개키 증명서가 포함됩니다. 쿠레덴샤르에는, 주제에 의한 특정 조작의 실행을 가능하게 하는 데이터가 포함되는 경우도 있습니다. 예를 들어, 암호화열쇠는, 주제에 의한 데이터에의 서명 또는 암호화를 가능하게 하는 자격을 나타냅니다. 공개 및 비공개 쿠레덴샤르크라스는, 코어 JAAS 클래스 라이브러리의 일부에서는 없습니다. 따라서, 모든 클래스가 쿠레덴샤르를 나타낼 수가 있습니다.
