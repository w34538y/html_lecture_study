JavaServer Pages

이 장에서는, 다음의 토픽에 대해 설명합니다.

  • JSP Java Plug-in 의 액션 요소 :<jsp:plugin>
  • JSP 를 사용한 Java Plug-in 의 배치

JavaServer Pages (JSP)는, Web/어플리케이션 서버로부터 동적인 Web 페이지 컨텐츠를 생성하는 기술입니다. JSP 는, Java Plug-in 의 HTML 요소를 지원합니다. 이 문서에서는, JSP Plug-in 의 액션 요소를 사용해, Java Plug-in 로 애플릿을 배치하는 방법을 나타냅니다.

JSP Java Plug-in 의 액션 요소:<jsp:plugin>

JSP 페이지 작성자는,<jsp:plugin> 액션 요소를 사용해 HTML 를 생성하는 것으로써, Java Plug-in 의 다운로드 (인스톨되어 있지 않은 경우) 및 애플릿의 실행을 가능하게 할 수 있습니다. 특히, 액션 요소는, Internet Explorer 의 경우는 OBJECT 요소를, Netscape Navigator 의 경우는 EMBED 요소를 생성합니다.

JSP 의 실행시에,<jsp:plugin> 액션 요소는, 브라우저에 응해 <OBJECT> 또는 <EMBED> HTML 태그로 옮겨집니다.

예를 들어, 애플릿을 JSP 내에서 다음과 같이 지정할 수 있습니다. 

<jsp:plugin type=applet code="Molecule" codebase="/html" >
<jsp:params>
    <jsp:param name="molecule" value="molecules/benzene.mol"/> 
</jsp:params>
<jsp:fallback>
       <p> Unable to start Plug-in.  </p>
</jsp:fallback>
</jsp:plugin>

위의 예에서는,<jsp:plugin> 요소의 속성은, 애플릿 자체의 구성 데이터를 제공합니다. <jsp:param> 요소는, 애플릿에 건네주는 파라미터를 지정합니다. <jsp:fallback> 요소는, Java Plug-in 의 기동에 실패했을 경우에 표시하는 브라우저의 컨텐츠를 지정합니다.

JSP 페이지를 요구하는 사용자 에이전트에 응해, Web 서버는 EMBED 또는 OBJECT 태그를 생성합니다.

예를 들어, Netscape Navigator 가 JSP 페이지를 요구하는 사용자 에이전트의 경우, 서버에 의해 다음의 EMBED 태그가 생성됩니다. 

<EMBED type="application/x-java-applet" 
        code="Molecule" codebase="/html" molecule="molecules/benzene.mol">
<NOEMBED>
<p> Unable to start plug-in </p>
</NOEMBED>

주:

위의 예는, Netscape 4 에서는 EMBED 태그가 지원되고 있으므로 기능합니다만, Netscape 6. x 에서는 현재 EMBED 태그가 지원되어 있지 않기 때문에 기능하지 않습니다.

<jsp:plugin> 요소의 자세한 것은,「Java Server Pages」를 참조해 주세요.

JSP 를 사용한 Java Plug-in 의 배치

JSP 는, Java Plug-in 를 이용해 애플릿을 배치하는 대체 방법을 제공합니다. JSP 가 가지는 동적인 성질을 위해서(때문에), 개발자는 HTML 페이지를 순간에 생성할 수 있습니다. 이 때문에, Java Plug-in 를 지금까지 이상으로 용이하게 이용할 수 있습니다.

<jsp:plugin> 태그의 지원는 JSP 스펙의 일부에서는 있습니다만, 다양한 Web 또는 어플리케이션 서버 벤더가, 이 기능의 독자적인 구현이나 독자적인 제어 레벨을 제공할 가능성이 있습니다. 자세한 것은, 서버 벤더에 문의해 주세요.
