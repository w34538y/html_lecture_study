J2SE 5.0 에 있어서의 Java Web Start 의 확장 기능

 Java Web Start 의 컨텐츠
  • 어플리케이션 매니저의 종료 :J2SE 5.0 릴리스로부터, 어플리케이션 매니저는 짜넣어지고 없어졌습니다. 이 주요했던 기능은, 현재 Java 컨트롤 패널 (다음의 항목 3 을 참조)과 JNLP 캐쉬 viewer (다음의 항목 4 를 참조)에 탑재되고 있습니다. 나머지의 기능 (JNLP 링크를 포함한 Web 페이지의 표시)은, Web 브라우저로 한층 더 효과적으로 실행할 수 있습니다.

  • 개발자전용의 번들의 종료:현재, Java Web Start 는 완전하게 JRE 와 JDK 에 통합되고 있습니다. 또, 종래, 다른 개발자전용의 번들에 포함되어 있던 항목은, 현재 JDK 에 짜넣어지고 있습니다. 같이 종래, 개발자전용의 번들에 포함되어 있던 문서는, 현재 온라인으로 이용할 수 있습니다. 현행의 JDK 에는 샘플용의 디렉토리가 준비되어 있어 Java Web Start 어플리케이션의 샘플 원시 코드가 4 개, 샘플 JRE 인스톨러가 1 개, 한층 더 거기에 대응한 JNLP 다운로드 서브 렛이 포함되어 있습니다. 또, 현행의 JDK 에는,jardiff.jar,jnlp.jar,jnlp-servlet.jar 도 준비되어 있습니다.

  • JNLP 캐쉬 viewer :JNLP 캐쉬 viewer는, 어플리케이션 매니저의 「다운로드한 어플리케이션」뷰 기능의 대신입니다. 캐쉬 viewer를 사용하는 것으로, 모든 어플리케이션, 애플릿, 라이브러리, 및 인스톨러가 표시됩니다. 사용할 수 있는 어플리케이션은 온라인 또는 오프 라인으로 기동할 수 있습니다. 또, Web 로부터 삭제하거나 복원하기도 할 수 있습니다. 현재, 모든 JNLP 파일은 href 파라미터에 관계없이 캐쉬에 보존되고 있습니다.

  • 시스템 캐쉬:현재, Java Web Start 에는 시스템 캐쉬와 사용자 캐쉬의 양쪽 모두가 준비되어 있습니다. 이것에 의해, 시스템 관리 책임자는 어플리케이션과 공유 라이브러리를 시스템 캐쉬에 사전에 로드할 수 있기 (위해)때문에, 복수의 사용자가 같은 자원을 공유할 수 있습니다.

  • 임포트 기능:현재, Java Web Start 에는 임포트 기능이 준비되어 있습니다. 최초의 코드가 1 개의 장소에서 로드 된 후, 다른 장소로부터 그것을 갱신하는 것 같은 상황에 있어, 이 기능을 CD 인스톨용으로 활용할 수 있습니다. 또, 사용자 캐쉬내나 시스템 캐쉬내의 어플리케이션과 라이브러리를 어플리케이션을 실행하는 일 없이 사전 인스톨 하는 경우에도 사용할 수 있습니다.

  • 데스크탑의 통합:(1) 어플리케이션의 데스크탑 통합은 GNOME 에 확장되었습니다. 또, 어플리케이션의 JNLP 파일이 통합해야 할 내용을 지정할 수 있도록(듯이)도 확장되었습니다. 이것에 의해, JNLP 파일로부터 데스크탑 및 메뉴의 양쪽 모두 또는 어느쪽이든 한편을 추천 해, 부메뉴명도 지정할 수 있게 되었습니다. (2) Windows 나 Unix 상에서 GNOME 2.0 이상을 실행하고 있는 경우, Java Web Start 어플리케이션을, 데스크탑상 또는 파일 시스템상의 특정의 MIME 타입이나 확장 기능에 대한 primary 핸들러로서 등록할 수 있게 되었습니다 (Web 페이지의 링크로서는 불가). (3) 다른 관련 컨텐츠 (HTML 링크, 네이티브 컨텐츠, JNLP 어플리케이션 관련)는, 이번으로부터 어플리케이션의 데스크탑의 통합에 포함하는 것이로.

  • API 의 확장 :JNLP API 는 확장되어 Extended Service 와 Single Instance Service 가 준비되었습니다. Extended Service 에는, 원의 스펙에서는 사용할 수 없는 파일의 IO API 가 포함되어 있습니다. Single Instance Service 는, 어플리케이션이, 같은 어플리케이션을 복수 호출하는 것으로, 원의 인스턴스에 새로운 인수를 확실히 건네줄 수가 있는 것 같은 메카니즘을 제공합니다. 이것을 사용하는 것으로, Java 의 재기동의 오버헤드를 회피할 수 있습니다. 어플리케이션이 JNLP 어플리케이션의 Single Instance API 를 호출한 후는, Single Instance 의 동작만이 유효하게 되는 것에 주의해 주세요.

  • VM 의 인수 :JNLP 파일은, 임의의 인수를 가상 머신에 요구할 수 있게 되었습니다. 이전의 JNLP 어플리케이션은, J2SE 요소에 대해서 initial-heap-size 및 max-heap-size 인수만 사용할 수 있었습니다만, 이번으로부터 java-vm-args 인수가 추가되었습니다. Java Web Start 는, 「안전」이라고 생각되는 일반적으로의 Java 와 -X 인수의 요구를 우선합니다. 이러한 인수의 모든 리스트가 각 릴리스와 함께 포함되어 있기 (위해)때문에, JNLP 의 스펙은 바꿀 필요가 없습니다.

  • Java Printing API 를 사용하는 것으로, 샌드 박스로 실행하고 있는 JNLP 어플리케이션의 인쇄를 완전하게 지원할 수 있게 되었습니다. JNLP 어플리케이션이 인쇄하려고 하면(자), Java Web Start 가 시큐리티 다이얼로그를 매회 표시해, 어플리케이션의 인쇄를 허가할지 어떨지를 사용자에게 통지합니다. 이것에 의해, JNLP 어플리케이션의 인쇄를 완전하게 지원할 수 있습니다. 종래는, JNLP 인쇄 API 밖에 사용할 수 없었습니다. 이 기능은, Java Plug-in 에 있어서의 애플릿의 인쇄 동작에 유사하고 있습니다.
