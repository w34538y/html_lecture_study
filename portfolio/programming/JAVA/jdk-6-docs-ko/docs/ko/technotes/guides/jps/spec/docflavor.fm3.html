서비스 지정 서식 첨부 인쇄 데이터

클라이언트에서는, MIME 타입을 사용해 인쇄 데이터 서식을 지정하지 않고 , Java 객체를 인쇄 서비스에 건네줄 수도 있습니다. 인쇄 서비스에서는, Java 객체로부터 인쇄 데이터 서식을 결정합니다. 예를 들어, Java 객체에는, PostScriptTM 문서를 캡슐화할 수 있습니다. 클라이언트에서는, MIME 타입을 사용해 문서를 PostScriptTM 로서 명시적으로 지정하지 않고 , Java 객체내에 문서에의 참조를 랩 할 수가 있습니다. 인쇄 서비스에서는, 이 참조를 검사해, 서식이 PostScriptTM 이다고 판단합니다. 인쇄 데이터는 Java 객체 경유로 인쇄 서비스에게 건네지기 (위해)때문에, Java 객체가 구현하는 클래스는 그 표현 클래스가 됩니다.

DocFlavor 클래스에는, DocFlavor.SERVICE_FORMATTED 로 불리는 내부 클래스가 있습니다. 이 내부 클래스에는, 서비스가 서식을 설정하는 인쇄 데이터를 표현하는 DocFlavor 객체 정수가 짜넣어지고 있습니다. 이러한 정수의 MIME 타입은, 「application/x-java-jvm-local-objectref」입니다. 표현 클래스로서 이름을 붙일 수 있었던 인터페이스를 구현하는 Java 객체에의 참조를, 클라이언트가 공급하는 것을 나타냅니다.

DocFlavor.SERVICE_FORMATTED 에 포함되는 다음의 3 개의 DocFlavor 객체 정수는, 서비스가 서식을 설정하는 인쇄 데이터와 함께 사용되는 일반적인 표현 클래스명을 나타내고 있습니다.

  • DocFlavor.SERVICE_FORMATTED.PAGEABLE - 클라이언트가 java.awt.print.Pageable 인터페이스를 구현하는 객체를 공급한다. 이 객체는, 인쇄하는 페이지를 표현한다. 프린터는, 인쇄하는 페이지를 1 페이지씩 취득하기 위해서, 이 인터페이스내에서 메소드를 호출한다. 페이지 마다 프린터는 그래픽스 문맥을 공급해, 그래픽스 문맥내에 클라이언트가 draw 한 것을 모두 인쇄한다.
  • DocFlavor.SERVICE_FORMATTED.PRINTABLE - 클라이언트가 java.awt.print.Pageable 인터페이스를 구현하는 객체를 공급한다. 이 객체는, 각 페이지의 컨텐츠를 draw 한다. 프린터는, 인쇄하는 페이지를 1 페이지씩 취득하기 위해서, 이 인터페이스내에서 메소드를 호출한다. 페이지 마다 프린터는 그래픽스 문맥을 공급해, 그래픽스 문맥내에 클라이언트가 draw 한 것을 모두 인쇄한다.
  • DocFlavor.SERVICE_FORMATTED.RENDERABLE_IMAGE - 클라이언트가 java.awt.image.renderable.RenderableImage 인터페이스를 구현하는 객체를 공급한다. 이 객체는, 렌더링과는 독립한 방법으로 취급할 수 있는 이미지를 표현해, 품질에 대해 타협하지 않고 , 프린터등이 다양한 문맥에 렌더링 할 수가 있다. 프린터는, 인쇄하는 이미지를 취득하기 위해서, 이 인터페이스내에서 메소드를 호출한다.
    • 「처음에」 로 설명한 것처럼, 2D 그래픽스를 Java 객체에 랩 하는 어플리케이션에서는, 서비스가 서식을 설정하는 인쇄 데이터의 DocFlavor 정수를 사용할 수 있습니다. 자세한 것은,「2D 그래픽스의 인쇄와 스트리밍」을 참조해 주세요.
