애플릿의 시큐리티의 기본

이하는, 애플릿의 시큐리티와 Java Plug-in 에 관한 기본 사항입니다. 한층 더 상세한 설명은, 다음의 장의「Java Plug-in 에서의 RSA 서명 첨부 애플릿의 검증」을 참조해 주세요.

  • 서명이 없는 애플릿은 모두, 표준의 애플릿 시큐리티 모델로 실행됩니다.
  • java.policy 파일에 usePolicy 가정의되어 있지 않은경우, Java Plug-in 가 서명자를 확인할 수가 있어 표시되는 prompt로 사용자가 AllPermission 액세스권의 부여에 동의 했을 경우에게만,서명 첨부 애플릿은 AllPermission 액세스권을 가집니다.
  • usePolicy 가정의되고 있는경우,서명 첨부 애플릿은 java.policy 로 정의된 액세스권 밖에 가지지 않고, prompt는 표시되지 않습니다.

게다가 Java Plug-in 가 증명서의 관리도 실시하게 되었습니다. 즉, 증명서의 검증 태스크가 브라우저에게 건네지지 않게 되었습니다.

Windows Vista 에서의 서명 첨부 애플릿의 실행

Windows XP Home 나 Windows XP Professional 등의 다른 Windows operating system상에서 실행되는 애플릿과 비교했을 경우, Windows Vista 상의 서명 첨부 애플릿의 특권은 한정되고 있습니다. 이것은, 브라우저 프로세스의 무결성의 레벨이 낮아지고 있기 (위해)때문에입니다. 저레벨의 무결성과는, 관리자보다 특권이 낮은 일을 의미합니다. 이 때문에, 서명 첨부 애플릿은, 특정의 미디어나 고레벨의 무결성을 가지는 디렉토리에의 파일의 기입 및 삭제를 실시할 수가 없습니다.

Windows Vista 로 서명 첨부 애플릿을 실행하면(자), 시큐리티 경고에 관한 다이알로그 박스가 표시됩니다. 로컬 드라이브상의 파일에의 기입해 또는 삭제 액세스권 이외의 모든 액세스권을 부여해 애플릿을 실행하려면 ,「Run」를 클릭합니다.

안전한 버젼 관리

Java SE 5.0 Update 6 또는 그 이전의 버젼을 사용하는 Java 애플릿은, 그 애플릿을 실행 가능한 JRE 의 버젼을 지정할 수가 있었습니다. 이 때문에, Java 애플릿은, 최신의 시큐리티 수정이 적용되지 않을 가능성이 있는 JRE 를 사용해 실행할 수가 있었습니다.

Java SE 6 을 사용하면(자), 모든 애플릿은, 인스톨 되고 있는 최신의 JRE 버젼을 사용해 실행할 수 있습니다. 이 새로운 확장 기능에 의해 시큐리티가 강화됩니다.
