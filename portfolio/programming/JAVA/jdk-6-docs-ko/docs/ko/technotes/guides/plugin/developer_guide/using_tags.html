처음에

이 장에서는, HTML 작성자에게,applet,object, 및 embed 태그를 언제 어떻게 사용해 Java 애플릿을 Web 페이지에 추가할까에 임해서 설명합니다. 또, 인터넷이나 인트라넷에서의 애플릿의 배치에 관한 가이드 라인과 다른 브라우저로 사용하기 위한 가이드 라인을 나타냅니다.

전반적인 고려사항

애플릿의 배치 방법은, 사용자가 Web 페이지에 인터넷 또는 인트라넷의 머지않아로 액세스 하는지, 및 사용자가 사용하는 브라우저의 타입에 따라서 다릅니다. 사용자에 관한 이 정보에 주의해, 다음의 전반적인 가이드 라인에 따라 주세요.

인터넷과 인트라넷에서의 애플릿의 배치

애플릿을 다음과 같이 배치합니다.

  • 인터넷으로부터 Web 페이지에 액세스 하는 경우는,applet 태그를 사용합니다.

  • 인트라넷으로부터 Web 페이지에 액세스 하는 경우는,object 또는 embed 태그를 사용합니다.

특정의 브라우저에 대한 애플릿의 배치

애플릿을 다음과 같이 배치합니다.

  • Internet Explorer 만의 경우,object 태그를 사용합니다.

  • Mozilla 패밀리의 브라우저만의 경우,embed 태그를 사용합니다.

브라우저가 혼재하는 환경에서 애플릿을 배치하는 경우는, 섹션「브라우저가 혼재하는 환경에서의 애플릿의 배치」의 가이드 라인에 따라 주세요.

applet 태그의 사용

applet 태그를 사용해, 멀티 브라우저 환경에 애플릿을 배치합니다.

applet 태그의 자세한 것은,「W3 HTML specification」를 참조해 주세요.

주:HTML 스펙에서는,applet 태그는 추천 되지 않고, 대신에 object 태그를 사용하는 것이 추천 되고 있습니다. 다만, 브라우저가 object 태그를 구현해 Java 애플릿을 지원하는 방법에 관해서 이 스펙은 명확하지 않고, 현재로서는 일관성이 있는 브라우저 지원는 행해지고 있지 않습니다. 이 때문에, 모든 플랫폼의 브라우저에 Java 애플릿을 배치하기 위한 일관성이 있는 방법으로서applet 태그를 계속해 사용하는 것을 추천합니다.

applet 태그의 예를 다음에 나타냅니다. 

    <applet code=Applet1.class width="200" height="200">
    Your browser does not support the <code>applet</code> tag.
    </applet>

Internet Explorer 와 Mozilla 패밀리의 브라우저의 양쪽 모두에 대해, Java Plug-in 가 인스톨 되고 있는 경우 (버젼 1.3. 1_01a 이후), 인스톨 되고 있는 가장 상위의 버젼의 Java Plug-in 가, 애플릿을 실행하기 위해서 불려 갑니다.

주:

JRE 가 로컬에 인스톨되어 있지 않은 경우에 JRE 를 자동적으로 다운로드하기 위해서,applet 태그를 사용할 수 없습니다.

object 태그의 사용

object 태그를 사용해, Internet Explorer 에서만 사용되는 애플릿을 배치합니다. object 태그의 자세한 것은,「W3 HTML specification」를 참조해 주세요. object 태그의 예를 다음에 나타냅니다. 

<OBJECT 
  classid="clsid:8AD9C840-044E-11D1-B3E9-00805F499D93"
  width="200" height="200">
  <PARAM name="code" value="Applet1.class">
</OBJECT>

classid 속성

classid 속성은, 사용하는 Java Plug-in 의 최소 버젼을 식별합니다.

다음에 나타내는 예는,classid 속성의 가장 일반적으로 사용되는 형식입니다. 이 예에서는, 인스톨 되고 있는 가장 상위의 버젼의 Java Plug-in 를 사용하도록(듯이) Internet Explorer 에 지시합니다.

classid="clsid:8AD9C840-044E-11D1-B3E9-00805F499D93"

classid 속성의 다른 형식을 다음에 나타냅니다.

classid="clsid:CAFEEFAC-xxxx-yyyy-zzzz-ABCDEFFEDCBA"

이 형식에서는, 「xxxx」, 「yyyy」, 및 「zzzz」는, 사용하는 Java Plug-in 의 최소 버젼을 식별하는 4 자리수의 숫자입니다. 예를 들어, Java Plug-in 버젼 1.6. 0 을 사용하려면 , 다음과 같이 지정합니다.

classid="clsid:CAFEEFAC-0016-0000-0000-ABCDEFFEDCBA"

codebase 속성

옵션의 codebase 속성을 사용해, JRE 를 다운로드할지 어떨지, 및 그방법을 지정합니다.

codebase 속성에는, 다음의 2 개의 형식이 있습니다.

  • codebase=<URL>

    이 형식에서는,classid 속성에 의해 지정된 JRE 가 로컬에 인스톨되어 있지 않은 경우에, 사용자는 지정된 URL 로부터 JRE 를 다운로드하는 것이 요구됩니다.
  • codebase=<URL>#Version=major, minor, micro, update

    이 형식에서는, codebase 속성으로 지정된 JRE 의 버젼이 classid 속성으로 지정된 JRE 의 버젼보다 상위의 경우에, 사용자는 지정된 URL 로부터 JRE 를 다운로드하는 것이 요구됩니다.

codebase 속성을 사용해, Sun 의 Java 의 Web 사이트로부터의 자동 다운로드를 설정하는 방법의 예를, 다음에 나타냅니다. 

<object 
  classid="clsid:8AD9C840-044E-11D1-B3E9-00805F499D93"
  width="200" height="200"
  codebase="http://java.sun.com/products/plugin/autodl/
jinstall-1_5_0-windows-i586.cab#Version=1, 5,0,0">
  <param name="code" value="Applet1.class">
</object>

주:이 예에서는,codebase=http://java.sun.com ... 의 행은 읽기 쉽게 하기 위해서 분할되고 있습니다. 실제의 HTML 파일에서는 긴 1 행이 됩니다.

Sun 에서는, JRE 인스톨러의 각 버젼을 Microsoft 캐비넷 (. cab) 파일 형식에 패키지 하고 있습니다. 이러한 릴리스 및 대응한다 . cab 파일명의 리스트를 표시할 수 있습니다.

embed 태그의 사용

embed 태그를 사용해, Mozilla 패밀리의 브라우저에서만 사용되는 애플릿을 배치합니다.

embed 태그의 예를 다음에 나타냅니다. 

<embed code="Applet1.class"
  width="200" height="200"
  type="application/x-java-applet;version=1. 6.0"
  pluginspage="http://java.sun.com/javase/downloads/ea.jsp"/>

type 속성은, 다음의 2 개의 형식의 머지않아인가입니다.

  • type="application/x-java-applet;version=1. 6.0"

    이 형식에서는, MIME 타입을 지원하는, 인스톨 되고 있는 최상정도의 JRE, 즉 application/x-java-applet;version=1. 6.0 이 애플릿을 실행하기 위해서 불려 갑니다. 지정된 버젼 번호 이상의 버젼의 JRE 가 로컬에 인스톨 되고 있는 경우는, 그 JRE 가 불려 갑니다. 그 이외의 경우는,pluginspage 속성의 값으로 해서 지정된 URL 에 액세스 합니다.
  • type="application/x-java-applet;jpi-version=1. 5.0_07

    이 형식에서는, 적어도 jpi-version 의 값 (이 예에서는, 1.5. 0_07)에 의해 지정된 갱신 버젼의 JRE 가, 애플릿을 실행하기 위해서 불려 갑니다. 그 이외의 경우는,pluginspage 속성의 값으로 해서 지정된 URL 에 액세스 합니다.

브라우저가 혼재하는 환경에서의 애플릿의 배치

다음의 2 개의 방법의 어느쪽이든으로, Internet Explorer 와 Mozilla 패밀리의 브라우저의 양쪽 모두의 사용자에게 애플릿을 배치할 수 있습니다.

  • Pure HTML 의 사용

  • JavaScript 의 사용

Pure HTML 를 사용한 애플릿의 배치

Pure HTML 어프로치를 사용해 브라우저가 혼재하는 환경에서 애플릿을 배치하는 경우는, 다음 일에 주의해 주세요.

  • Internet Explorer

    • object 태그를 인식한다
    • comment 태그의 내용을 무시한다
  • Mozilla 브라우저
    • classid 속성이 붙은 object 태그를 무시한다
    • comment 태그의 내용을 해석한다

HTML 페이지의 다음의 코드예를 검토해 주세요. 

<object 
  classid="clsid:CAFEEFAC-0016-0000-0000-ABCDEFFEDCBA"
  <param name="code" value="Applet1.class">
    <comment>
      <embed code="Applet1.class"
        type="application/x-java-applet;jpi-version=1. 6.0">
        <noembed>
          No Java Support.
        </noembed>
      </embed>
    </comment>
  </object>

JavaScript 를 사용한 애플릿의 배치

전술한 Pure HTML 어프로치를 사용하는 대신에, JavaScript 를 사용해 브라우저가 혼재하는 환경에서 애플릿을 배치할 수 있습니다.

JavaScript 를 사용해, 다음의 순서를 실행합니다.

  1. appName 변수를 사용해, 사용자의 브라우저를 검출합니다.

  2. document.write() 메소드를 사용해,appName 변수의 값에 근거해 태그를 기술합니다.

    1. 브라우저명이 「Mozilla」의 경우,embed 태그를 기술합니다.

    2. 브라우저명이 「Microsoft Internet Explorer」의 경우,object 태그를 기술합니다.

다음의 예에서는,document.write() 메소드는, 각 사용자가 실행할 때마다 embed 또는 object 태그를 출력합니다. 

<html>
<script language="Javascript">

  var _app = navigator.appName;

  if (_app == 'Mozilla') {
    document.write('<embed code="Applet1.class"',
                   'width="200"',
                   'height="200"',
                   'type="application/x-java-applet;version=1. 5.0">');
    }
  else if (_app == 'Microsoft Internet Explorer') {
    document.write('<OBJECT ',
                   'classid="clsid:8AD9C840-044E-11D1-B3E9-00805F499D93"',
                   'width="200"',
                   'height="200">',
                   '<PARAM name="code" value="Applet1.class">',
                   '</OBJECT>');
    }
  else {
    document.write('<p>Sorry, unsupported browser. </p>');
    }

</script>
</html>
