XML 처리용의 JavaTM API 릴리스 노트

스펙 버젼: 1.4
레퍼런스 구현 (RI) 버젼: 1.4. 0

이 문서에서는, 인스톨 순서와 이 소프트웨어 라이브러리를 보다 효과적으로 사용하기 위한 주의가 설명되고 있습니다. 자세한 것은,JAXP FAQ 를 참조해 주세요.

주:
이 페이지를 온라인으로 표시하고 있는 경우는, 릴리스 노트의 최신판을 보시고 있습니다. 이 페이지를 릴리스 번들의 일부로서 다운로드했을 경우는, 최신판의 릴리스 노트에 대해,JAXP Documentation 페이지를 참조해 주세요.

목차

  • 기능 변경
  • gMonth
  • JAXP 1.3 이후의 스펙 변경

기능 변경

JAXP 1.4 에서의 기술적으로 중요한 변경점은, Streaming API for XML(StAX) (JSR 173)의 javax.xml.stream. * 패키지가 추가된 것입니다. 이 추가에 의해, JAXP 1.4 는, 스트림 API 를 사용해 문서의 검증 및 변환을 지원합니다. JAXP 1.4 는 Java SE 6 에 통합되고 있기 (위해)때문에, StAX 의 지원는 최신의 Java 가 실행되고 있는 임의의 시스템으로 투과적입니다.

gMonth

JAXP 1.4 에서는, gMonth 에 대한 XML schema 검증 논리가 변경되고 있어 XML Schema 1.0 Specification Errata: E2-12 Error 로 행해진 변경이 반영되고 있습니다. 구체적으로는, 자구 표현 「--MM--」를 사용한 gMonth 를 무효라고 해 거부합니다.

JAXP 1.3 이후의 스펙 변경

JAXP 1.4 스펙 의 제 9 장 「Changes Since JAXP 1.3」도 참조해 주세요.

다음의 섹션에는, JAXP 1.4 로 행해진 중요한 변경점이 정리되고 있습니다. 자세한 것은, 해당하는 JavaDoc 를 참조해 주세요.

1.  패키지 javax.xml.datatype

1.1.  클래스 DatatypeFactory

  • DATATYPEFACTORY_IMPLEMENTATION_CLASS 는 다음과 같이 변경되었습니다. com.sun.org.apache.xerces.internal.jaxp.datatype.DatatypeFactoryImpl

  • newInstance(String, ClassLoader) 메소드를 추가했습니다.

1.2.  클래스 Duration

  • equals() 에 대한 인수가 null 인 경우,NullPointerException 를 throw 하는 대신에 false 를 돌려줍니다.

1.3.  클래스 XMLGregorianCalendar

  • equals() 에 대한 인수가 null 인 경우,NullPointerException 를 throw 하는 대신에 false 를 돌려줍니다.

2.  패키지 javax.xml.parsers

2.1.  클래스 DocumentBuilderFactory

  • newInstance(String, ClassLoader) 메소드를 추가했습니다.

2.2.  클래스 SAXParserFactory

  • newInstance(String, ClassLoader) 메소드를 추가했습니다.

  • XMLStreamException

      에러:

  • FactoryConfigurationError

5.  패키지 javax.xml.transform

5.1.  클래스 ErrorListener

  • fatalError() 의 시멘틱스를 명시했습니다.

5.2.  클래스 Transformer

  • 스타일 시트로 setOutputProperty() 또는 xsl:output 를 지정해 명시적으로 설정되어 있지 않은 프로퍼티에 대해서,getOutputProperty() 의 시멘틱스를 명시했습니다.

5.3.  클래스 TransformerFactory

5.4.  패키지 javax.xml.transform.stax

  • TemplatesHandler

  • TransformerHandler

      클래스:

  • StAXResult

6.  클래스 javax.xml.validation

6.1.  클래스 Schema

  • SchemaFactory 상에서 설정되어 있는 기능은 newValidator() 를 사용해 작성된 Validator 에게 건네질 필요가 있는 것을 명시했습니다.

  • SchemaFactory 상에서 설정되어 있는 기능은 newValidatorHandler() 를 사용해 작성된 ValidatorHandler 에게 건네질 필요가 있는 것을 명시했습니다.

6.2.  클래스 SchemaFactory

  • newInstance(String, ClassLoader) 메소드를 추가했습니다.

  • SchemaFactory 상에서 설정되어 있는 기능은 newSchema() 를 사용해 작성된 Schema 에게 건네질 필요가 있는 것을 명시했습니다. 이 문맥으로 newSchema() 에 관련하는 미묘한 점에 관한, 구현자와 개발자에게로의 주의를 포함했습니다.

  • newSchema(Source[]) 에의 입력은 XML 문서 또는 요소가 상정되고 있는 것을 명시했습니다.

  • SchemaFactory 상에서 설정되어 있는 기능은 이 팩토리를 사용해 작성된 Schema 에게 건네질 필요가 있어, 한층 더 그 Schema 로부터 작성된 Validator 와 ValidatorHandler 에게 건네질 필요가 있는 것을 명시하기 위해서,setFeature() 의 문서를 갱신했습니다.

6.3.  클래스 SchemaFactoryLoader

SchemaFactoryLoader 클래스는, JAXP 1.3 의 개발 프로세스중에 작성되었습니다. JAXP 1.3 이 완료하기 직전에, 검증에 관련하는 팩토리 기구가 변경되어 이 클래스는 삭제되었습니다. 다만, 파일은 리포지터리(repository)에 남아, Java 2 Platform Standard Edition 5.0 의 API 스펙에 기재되었습니다.

이 클래스는, 하위 호환성 문제이기 때문에 JAXP 1.3 에 추가할 수도 있지 않고, 또 Java 5.0 API 스펙으로부터 삭제하는 일도 할 수 없기 때문에, JAXP 1.4 에 추가되었습니다. 이 클래스는 효과가 없기 때문에, 사용하지 말아 주세요. 이 클래스가 추가된 것은, 이 클래스가 플랫폼에 정의되고 있지만 JAXP 에는 정의되어 있지 않은 것에 개발자가 눈치챘을 때에 발생하는 혼란을 피하기 (위해)때문에입니다.

이 클래스는 사용하지 말아 주세요.

6.4.  클래스 TypeInfoProvider

  • getElementTypeInfo() 의 시멘틱스가 확장되어 startElement 이벤트또는 endElement 이벤트의 어느쪽이든들도 호출할 수가 있게 되었습니다. 이것에 의해, API 로 W3C XML schema 공용 체형을 보다 효율적으로 지원할 수 있습니다.

    W3C XML schema 검증이 실행되고 있을 때, 요소에 공용 체형이 포함되어 있는 경우, startElement 이벤트로 getElementTypeInfo() 의 호출에 의해 반환되는 TypeInfo 는 공용 체형이 됩니다. endElement 이벤트의 호출에 의해 반환되는 TypeInfo 는, 요소를 검증할 경우에 사용되는 실제의 멤버형이 됩니다.

6.5.  클래스 Validator

  • Validator 는,DOMSource 와 SAXSource 이외의 Source (StAXSource 등)를 받아들일 수가 있습니다.

  • validate() 에의 입력은 XML 문서 또는 요소가 상정되고 있는 것을 명시했습니다.

7.  패키지 javax.xml.xpath

7.1.  클래스 XPath

  • 표현내의 변수를 해결하기 위해서 사용되는 것은, 컴파일시에 유효한 XPathVariableResolver 인 것을 명시했습니다.

7.2.  클래스 XPathFactory

  • newInstance(String, ClassLoader) 메소드를 추가했습니다.
