신기능과 확장 기능

 문서의 목차

IPv6 (Internet Protocol version 6)

J2SETM 1.4 는, TCP 및 UDP 베이스의 어플리케이션으로 IPv6 를 지원합니다. 멀티 캐스트의 지원도 포함되어 있어 현재 사용되고 있는 대부분의 최종 사용자 어플리케이션을 지원합니다. 「IPv6 사용자 가이드」 를 참조해 주세요.

HTTP 다이제스트 인증

HTTP 다이제스트 인증의 구현이 갱신되어 프록시와 오리진 서버를 지원해, 게다가 RFC2617 에 정의된, auth-int 를 제외하는 모든 기능을 제공하게 되었습니다.

다이제스트 인증 기구의 동작을 변경하는 몇개의 시스템 프로퍼티이 추가되었습니다. 이러한 프로퍼티에 대해서는,「네트워크의 프로퍼티」 문서를 참조해 주세요.

접속되어 있지 않은/바인드되어 있지 않은 소켓의 지원

소켓의 작성, 바인드, 및 접속을, 한층 더 유연하게 실시할 수가 있습니다. 접속을 확립하거나 받아들이거나 하기 전에, 소켓의 옵션을 조작하는 것이 가능합니다. 게다가 접속을 확립할 때에 타임 아웃을 지정할 수 있습니다.

새로운 클래스 javax.net.ssl.SSLSocket 는,java.net.Socket 의 서브 클래스에서, 암호화에 의한 시큐리티를 제공합니다. 이 기구는 Java Secure Socket Extension (JSSE)로 불려 소켓 경유로 송신되는 데이터의 암호화를 제공합니다. 자세한 것은,「JSSE 레퍼런스 가이드」 를 참조해 주세요.

접속된 UDP 소켓의 지원

DatagramSocket.connect 메소드가, 주소가 관련짓고를 네이티브 레벨로 확립하게 되었습니다. 이 때문에, 지원되고 있는 경우, 어플리케이션은 ICMP 포트 도달 불능 메세지를 봐, 원격 어플리케이션을 이용할 수 없는 것을 식별할 수 있습니다.

Uniform Resource Indentifier (URI)

J2SE 1.4 에서는, 새로운 클래스 java.net.URI 가 추가되었습니다. 이 클래스를 사용하면(자), 프로토콜 핸들러가 존재하지 않아도, URI 의 구축과 해석이 가능하게 됩니다. 그 처리는, URL 클래스에서는 실행할 수 없습니다. URI 클래스는, RFC 2396 및 RFC 2732 에 준거하고 있습니다.

FTP 프로토콜 핸들러

현재의 표준 규격 (RFC 1738 및 959)에 가능한 한 적합하도록(듯이), 기능이 철저하게 재검토되었습니다. 이 확장에 의해, FTP 기능에 관련하고 싶은 구두인가의 버그가 수정되었습니다. 예를 들어, J2SE 1.4 에서는, FTP 의 패시브 모드가 기능합니다.

InetAddress 에서의 JNDI DNS SP 의 지원

클래스 java.net.InetAddress 의 기능이 확장되었기 때문에, JDNI 를 사용해 DNS 네임 서비스 프로바이더를 이용하는 것으로써, pure Java 네임 서비스 프로바이더를 어플리케이션을 구성할 수 있게 되었습니다. 이 지원의 유효화에 대해서는, 「네트워크의 프로퍼티」문서의「JNDI DNS 서비스 프로바이더의 설정」 을 참조해 주세요.

URLEncoder 및 URLDecoder 의 개선

Methods URLEncoder.encode(String s,  String enc) and URLDecoder.decode(String s,  String enc) have been added to enable applications to use other character encoding/decoding schemes.

TCP 의 아웃 밴드 데이터

클래스 java.net.Socket 의 2 개의 새로운 메소드 sendUrgentData() 및 getOOBInline() 에 의해, 특정의 구래의 어플리케이션을 지원하기 위한 TCP 의 긴급 데이터가 한정적으로 지원됩니다. 긴급 데이터의 송신은, 임의의 TCP 소켓으로 실행할 수 있습니다. 다만, 긴급 데이터의 수신에 대해서는, 부분적인 지원만이 제공됩니다.

SOCKS

TCP 의 V5 (RFC 1928) 및 V4 가 완전하게 지원되고 있습니다. 어느 버젼을 사용할까에 관한 프록시와의 자동 협상도 지원됩니다. SOCKS 프록시 서버-를 지정하는 프로퍼티의 사용법에 대해서는, 「네트워크의 프로퍼티」문서의「SOCKS 프로토콜 지원의 설정」 을 참조해 주세요.

NetworkInterface

J2SE 1.4 에서는, 새로운 클래스 java.net.NetworkInterface 가 추가되었습니다. 이 클래스를 사용하면(자), 인터페이스와 주소를 열거할 수 있습니다. 이 새로운 클래스는, multi-homed상에 배치된 어플리케이션이나, 복수의 네트워크 인터페이스를 가지는 머신의 경우에 편리합니다.

그 외의 큰 개선점

J2SE 1.4 에서는, 한층 더 다음의 점이 개선되었습니다.
  • HTTP 스트리밍의 퍼포먼스가 개선되었다
  • 요구 헤더와 응답 헤더를 처리하는 기능이,java.net.URLConnection 클래스의 다음과 같은 새로운 메소드의 형태로 제공되게 되었다
    addRequestProperty
    getRequestProperties
    getHeaderFields
  • 메소드 HttpURLConnection.getErrorStream() 의 에러 처리의 구현이 개선되었다
