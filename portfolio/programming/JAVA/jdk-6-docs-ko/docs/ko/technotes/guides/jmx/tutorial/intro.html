제 1 장

입문

이 장에서는, Java Management Extensions (JMX)의 예제를 사용한, 입문에 필요한 순서에 대해 설명합니다. 이후의 장으로 설명하는모든예에 적용되는 순서를 설명합니다.

1.1 플랫폼 정보

이 튜토리얼의 예로 사용되는 모든 변수 할당 및 커멘드는, UNIX Korn 쉘 구문을 사용해 정의하고 있습니다. UNIX 또는 Linux 플랫폼에서 Korn 쉘 이외의 쉘을 실행하고 있는 경우는, 이러한 커멘드를 현재사용의 쉘 환경에 맞추어 변경할 필요가 있습니다.

Microsoft Windows 오퍼레이팅(operating) 환경을 실행하고 있는 경우, 변경 작업은 대부분의 경우, 패스의 slash (/)를 backslash (\)에 옮겨놓아 구두점 (:)을 세미콜론 (;)에 옮겨놓는 것만으로 끝납니다. 특정의 Microsoft Windows 의 커멘드가 지정되는 것은, 제시된 UNIX/Linux 커멘드와 크게 다른 경우만입니다.

1.2 예제 디렉토리

이 튜토리얼의 기초를 이루는 예제에는, 다음의 URL 에 있는 Java Platform, Standard Edition (Java SE) 문서 번들의 JMX 의 페이지로부터 액세스 할 수 있습니다.

http://java.sun.com/javase/6/docs/technotes/guides/jmx/examples.html

이 튜토리얼로 인용되는 코드의 발췌는, 이러한 예제 프로그램에 근거하고 있습니다. 다만, 일부의 코드 발췌는 새로 고쳐 되어 코멘트가 변경되고 있는 경우가 있습니다. 일반적으로, 튜토리얼로 리스트 되는 프로그램에서는, 스페이스의 형편상, 코멘트가 간소화되어 출력문이 생략 되고 있습니다. 특별한 주의를 필요로 하는 코드의 클래스 또는 섹션은, 이 튜토리얼의 목적에 의해, 굵은 글씨로 강조 표시되고 있는 경우가 있습니다.

1.3 작업 디렉토리

원의 예제 파일을 수정하지 않고 카피한 것을 보존하기 위해서, 예제를 컴파일 해 실행하는 디렉토리를 신규에 작성합니다.

  1. 새로운 작업 디렉토리를 작성합니다.

    2. 이 작업 디렉토리는, 이 튜토리얼을 통해서 work_dir 라고 부릅니다.

  2. 다음의 URL 로부터 예제의 ZIP 번들을 다운로드합니다.

    3. http://java.sun.com/javase/6/docs/technotes/guides/jmx/examples/jmx_examples.zip

  3. 모든 예제를 work_dir 디렉토리에 해동합니다.

모든 예제는 work_dir/jmx_examples 에 놓여집니다.
