JavaTM 가상 머신 테크놀러지

 문서의 목차

JDKTM 에는, 1 개(살) 또는 복수의 JavaTM 가상 머신 (VM) 구현이 짜넣어지고 있습니다.

  • 일반적으로, 클라이언트 어플리케이션용으로 사용되는 플랫폼에서는, JDK 는 Java HotSpotTM Client VM ( 「클라이언트 VM」)로 불리는 VM 구현에 부속되어 있습니다. 클라이언트 VM 는 기동 시간을 단축해, 메모리 사이즈를 축소하도록(듯이) 조정되고 있습니다. 클라이언트 VM 는, 어플리케이션의 기동시에,-client 커멘드행 옵션을 사용해 호출합니다.

  • 모든 플랫폼에서, SDK 는 Java HotSpot Server VM (서버 VM)로 불리는 Java 가상 머신 (VM) 구현에 부속되어 있습니다. 서버 VM 는 프로그램 실행 속도가 최대가 되도록(듯이) 설계되고 있습니다. 서버 VM 는, 어플리케이션의 기동시에,-server 커멘드행 옵션을 사용해 호출합니다.

어느 쪽의 VM 구현에도 공통되는 Java HotSpot 테크놀러지의 기능을 다음에 나타냅니다.

  • 「적응형 컴파일러」 - 어플리케이션은 표준의 interpreter로 기동합니다만, 실행시에는 코드를 해석해 퍼포먼스의 보틀 넥, 즉 「핫 스포트」를 검출합니다. Java HotSpot VM 는, 퍼포먼스를 향상시키기 (위해)때문에, 코드의 이러한 퍼포먼스-위기인 부분을 컴파일 합니다만, 거의 사용되지 않는 코드 (프로그램의 대부분)를 불필요하게 컴파일 할 것은 없습니다. 또, Java HotSpot VM 는, 적응형 컴파일러를 사용해, 인 라인화 등, 컴파일 한 코드를 최적화하기 위한 최선의 방법을 순간에 판단합니다. 컴파일러가 실행시 해석을 실시하므로, 최대의 퍼포먼스를 낳는 최적화 처리를 추측으로 판단하지 않아도 됩니다.

  • 「고속 기억 장치 할당하고」와「가베지 컬렉션」 - Java HotSpot 테크놀러지에서는, 객체에 대해서 고속으로 메모리의 할당을 합니다. 또, 고속으로 효율적인 최첨단의 가비지 컬렉터가 갖춰지고 있습니다.

  • 「thread 동기」 - Java 프로그램 언어는, 프로그램 실행의 복수의 병렬 패스 ( 「thread」라고 불린다)의 사용을 고려에 넣고 있습니다. Java HotSpot 테크놀러지는, thread 처리 기능을 실현하고 있습니다. 이 기능은, 대규모 메모리 공유형의 멀티 프로세서 서버에서의 사용을 용이하게 슬캘링 할 수 있도록(듯이) 설계되었습니다.

툴

Java HotSpot VM 의 조작 특징의 1 개는, 커멘드행 플래그를 사용해 제어할 수 있다고 하는 것입니다.

  • 표준 HotSpot VM 옵션 (Microsoft Windows  | Solaris and Linux Operating Environments  ) - Java HotSpot VM 로 지원되고 있는 커멘드행 옵션에 대해서는, Java 어플리케이션 기동 툴의 참조 페이지에 설명되고 있습니다.
  • 비표준 Java HotSpot VM 옵션 - 현재의 VM 구현에서는 인식되고 있어도, 장래의, 또는 Sun 이외의 구현으로 인식된다고는 할 수 없는 비표준 옵션에 대해서는, 이쪽의 Web 페이지로 설명되고 있습니다.

  • DTrace 프로브
  • 가베지 컬렉션의 확장 기능

    • 병행 Mark Sweep 수집가의 확장 기능

  • 클래스 데이터의 공유

  • 가베지 컬렉션 인간공학
  • thread 우선 순위의 변경
