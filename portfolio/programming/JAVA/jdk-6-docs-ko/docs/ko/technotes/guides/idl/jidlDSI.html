상급자전용의 토픽 :Dynamic Skeleton Interface (DSI)

Dynamic Skeleton Interface (DSI)를 사용하면(자), 객체의 인터페이스가 사전에 (컴파일시에) 모르는 경우에서도, 서버는 서번트 객체를 제공할 수가 있습니다. 서버는, IDL 인터페이스 정의로부터 컴파일 된 스켈리턴 코드를 사용하는 대신에, 오퍼레이션의 호출을 동적으로 구축합니다.

DSI 를 사용하는 이점

DSI 는, CORBA 환경과 CORBA 이외의 환경과의 중개역으로서 사용할 수 있습니다. DSI 를 사용하면(자), CORBA 클라이언트는, COM 서비스등으로 해서 구현되고 있는 CORBA 객체상의 메소드를 호출할 수 있게 됩니다.

CORBA 클라이언트의 요구를 COM 서버를 이해할 수 있는 형식으로 변환하려면 , 동적 서번트를 구현합니다. 이 변환 처리를 실시하는 코드는 모두 기술할 필요가 있습니다. 이 처리를 일반적인 정적 객체의 호출과 비교해 보세요. 서버는, 불려 가는 인터페이스용의 스켈리턴 (컴파일 된 것)에 액세스 할 수 있습니다. 이 스켈리턴은, IDL 인터페이스 정의를 idlj 컴파일러로 컴파일 하는 것으로써 생성됩니다. ORB 는, 요구를 받아들이면(자), 이 스켈리턴 코드를 사용해 서버측에서 오퍼레이션 인수를 조립해 결과를 돌려 보냅니다.
