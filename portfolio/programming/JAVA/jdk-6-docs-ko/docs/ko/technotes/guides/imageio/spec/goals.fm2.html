2.2 목표로 하지 않는 사항

thread에 대한 안전성
ImageReader 또는 ImageWriter 의 특정의 인스턴스에 대해서는, 메소드의 재입가능 (동시) 호출을 지원할 필요는 없습니다 (예외는, 현재의 읽어들여 또는 기입을 정지하도록(듯이) 요구하는 abort 메소드입니다). 그러나, 동일한 플러그 인의 복수의 인스턴스를 동시에 가동할 수 있도록(듯이) 할 필요는 있습니다. 간결하게 하기 위한(해), read 플러그 인의 경우에 대한 보고, 이후에 설명합니다.

완전한 재입가능성을 지원한다고 하면(자), read 플러그 인은, 자신 상태 정보 (예를 들어, 현재의 입력 소스등)를 모두, 별개 상태 객체에 넣어 둘 필요가 있습니다. 그러면, 실행중의 메소드는 그 개시시에 유효했던 설정에 따라 작업을 계속할 수가 있는 것과 동시에, 별개의 thread가 상태를 변경해 다음번의 조작으로 사용하는 일도 가능하게 됩니다.

좀 더 간단한 방법은, 각 ImageReader 가 자신 상태를 이 방법으로 보관 유지하는 것을 강제하는 것이 아니라, multi-thread의 처리를 실행하고 싶은 어플리케이션이 같은 ImageReader 클래스의 복수의 인스턴스를 생성하도록(듯이) 정하는 것입니다. 이것은,ImageReader 상태를 반드시 static 이외의 인스턴스 변수를 사용해 보관 유지해야 하는 것을 의미하고 있습니다만, 플러그 인의 개발자에게 있어 이 제약은 무거운 짐은 아닐 것입니다.
