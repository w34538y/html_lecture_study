네트워크 기능 및 확장 기능
대상: Java SE version 6.0

 문서의 목차

NetworkInterface 의 확장 기능

시스템의 네트워크 어댑터에 관한 상태 정보나 구성 정보에의 액세스에 사용하는 수많은 메소드를 제공합니다. 이것에는, 브로드캐스트 어드레스, subnet mask, MAC 주소, MTU 의 사이즈라고 하는 정보가 포함됩니다. java.net.NetworkInterface 를 참조해 주세요.

새로운 클래스 java.net.InterfaceAddress 는, NetworkInterface 의 IP 주소 (브로드캐스트 어드레스를 포함한다) 및 서스네트워크의 접두사의 길이 (subnet mask)에 관한 모든 정보를 캡슐화합니다.

국제화 된 도메인명의 지원

RFC 3490, 3491, 3492, 3454 에서는, US ASCII 이외의 알파벳 문자로 구성된 도메인명을 DNS 내에 보존할 수 있는 기구가 정의되고 있습니다. 임의의 Unicode 캐릭터 라인과 「분해되었다」ASCII 형식과의 상호 변환을 실현하는 알고리즘이 정의되고 있습니다. 이 기능이 제공하는 새로운 클래스 java.net.IDN 에 의해, 어플리케이션으로 이러한 변환을 실시해,java.net.InetAddress 로 IDN 를 검출하는 것이 가능해집니다.

HTTP 협상 인증

이것에 의해, Microsoft 의 정의하는 「협상」HTTP 인증 방식에 관한 지원가 추가됩니다. 협상에 의해, 기본이 되는 인증 기구를 복수 지원하는 것이 가능해집니다. 그 경우, 각 인스턴스에서는 몇개의 인증 기구를 선택해 사용합니다. 현시점에서 지원되고 있는 기구는 Kerberos 뿐입니다. 상세한 것에 대하여는,HTTP 인증의 가이드를 참조해 주세요.

크로스 플랫폼 NTLM

종래, Microsoft 정의의 HTTP 용 NTLM 인증 프로토콜은 Windows 상에서 밖에 지원되지 않았습니다. 현재는, 모든 플랫폼상에서 이 프로토콜이 지원되고 있습니다.

CookieManager 의 디폴트 구현

CookieHandler 클래스는 1.5 에 추가되었습니다만, java.net 에는 구현되지 않았습니다. 이 기능에서는, cookie 의 받아들여에 관한 정책로부터 cookie 가 보존되고 있는 영역을 분리하는 단순해 확장성의 높은 구현을 제공합니다. 이 매니저는, 디폴트에서도 다수의 표준 정책와 단순한 메모리내의 기억 기구를 제공합니다. 사용자 정의의 정책 및 기억 기구는, 이 구현으로 사용할 수가 있습니다. 다음에, 이 기능에 관련하는 새로운 클래스를 나타냅니다.
  • java.net.CookieManager
  • java.net.CookiePolicy
  • java.net.CookieStore
  • java.net.HttpCookie

경량 HTTP 서버

경량 HTTP 서버 어플리케이션을 구축하기 위한 단순한 클래스 및 인터페이스 일식을 제공합니다. 이 기능에 대해, com.sun.net.httpserver 와 com.sun.net.httpserver.spi 의 2 살의 신패키지가 정의되고 있습니다. 서버는 HTTP 와 HTTPS 를 지원합니다.
