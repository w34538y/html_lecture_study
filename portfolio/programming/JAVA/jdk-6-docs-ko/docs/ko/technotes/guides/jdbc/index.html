JDBC

 문서의 목차

JDBC (Java Database Connectivity) API 는, Java 프로그램 언어로부터의 범용적인 데이터 액세스를 제공합니다. JDBC 3.0 API 를 사용해, RDB, 스프레드쉬트, 플랫 파일 등, 실질상 모든 데이터 소스에 액세스 할 수가 있습니다. JDBC 테크놀러지는, 툴과 대체 인터페이스를 구축할 수 있는 공통의 베이스도 제공하고 있습니다.

JDBC 3.0 API 에는, 다음의 2 개의 패키지가 포함됩니다.

  1. java.sql 패키지
  2. 서버측의 기능을 추가한 javax.sql 패키지

이러한 패키지는, 다운로드한 Java Platform Standard Edition (Java SE) 6 에 양쪽 모두 자동적으로 포함되어 있습니다.

특정의 데이타베이스 관리 시스템으로 JDBC API 를 사용하려면 , JDBC 테크놀러지와 데이타베이스를 중개하는 JDBC 테크놀러지 베이스의 드라이버가 필요합니다. 드라이버는, 다양한 요인에 근거해, Java 프로그램 언어만으로 작성하거나 Java 프로그램 언어와 Java Native Interface (JNI) 네이티브 메소드를 조합해 작성하거나 합니다. JDBC 의 Web 사이트에는,드라이버를 현재 입수 가능 또는 개발중의 벤더의 일람이 있습니다.

최신의 JDK 에는, JDBC-ODBC 브릿지 드라이버가 포함되어 있습니다. 이 JDBC 테크놀러지 베이스의 드라이버를 사용하면, 프로그래머는 JDBC API 를 사용해 대부분의 Open Database Connectivity (ODBC) 드라이버를 이용할 수 있습니다. 이 소프트웨어의 현재의 상태에 대해서는,「JDBC-ODBC 브릿지 드라이버」를 참조해 주세요. Java Platform Standard Edition (Java SE) 6 에 포함되는 브릿지 드라이버는, 시험용으로서 또는 그 밖에 대신이 되는 드라이버를 이용할 수 없는 경우에만 사용해 주세요.
