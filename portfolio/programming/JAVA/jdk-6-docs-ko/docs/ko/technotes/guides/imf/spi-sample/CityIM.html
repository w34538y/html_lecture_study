1. City 입력 메소드에 대해

City 입력 메소드에서는, 도시명을 입력할 수 있습니다. 3 문자의 공항 코드를 입력하는 것으로, 대응하는 도시명에 변환됩니다. 사용자가 복수의 공항 코드를 입력하면(자) (한 번에 3 개(살)까지 입력 가능), 입력 메소드는 예를 들어 "From San Francisco to Beijing via Tokyo" 라고 한 짧은 구를 작성합니다. 입력 메소드에서는, 도시 마다 복수의 언어 (정확하게는 로케일)로 도시명 및 구가 인식되고 있습니다. 임의의 입력 조작으로, 지원하는 모든 언어의 도시명 및 구를 자유롭게 조합할 수가 있습니다. 입력 메소드는, 입력 메소드를 지원하는 모든 텍스트 컴퍼넌트 (Swing 텍스트 컴퍼넌트등)에의 텍스트 입력에 사용할 수 있습니다.

2. 입력 메소드의 인스톨과 선택

어플리케이션으로부터 입력 메소드를 이용 가능하게 하려면 , Java 2 Runtime Environment 의 확장 기능 디렉토리에 jar 파일을 인스톨 할 필요가 있습니다. 입력 메소드를 지원하는 텍스트 컴퍼넌트를 가지는 Java 어플리케이션을 기동 후에, Java Runtime Environment 의 제공하는 입력 메소드 선택 사용자 인터페이스를 사용해, 입력 메소드를 선택할 수 있습니다. 우선, 텍스트 컴퍼넌트에 포커스가 있는 것을 확인합니다. 일반적으로, 텍스트 컴퍼넌트에 포커스가 있으면(자), 삽입 포인트가 표시됩니다. 다음에, Sun 의 제공하는 Java 2 Runtime Environment 의 경우, 윈도우의 좌상의 풀다운 메뉴로 「입력 방식의 변환」메뉴를 선택해 (Windows 의 경우는 「시스템」메뉴, Solaris 의 경우는 「윈도우」메뉴), 다음에 「City Input Method」를 선택합니다. 표시되는 pop-up menu로 로케일을 선택합니다. 언어에 따라서는, 텍스트가 적절히 표시될지 어떨지는 폰트의 설정에 의존합니다. 이 때문에, 호스트 operating system의 로케일을 선택하는지, 대부분의 폰트가 지원하는 언어 (영어나 독일어등)를 선택하는 것이 일반적입니다.

3. 인스톨 메소드를 처음으로 사용한다

입력 메소드를 선택하면(자), 샌프란시스코 국제 공항의 코드인 「SFO」를 입력합니다. 텍스트가 텍스트 컴퍼넌트에 입력됩니다. 다만, 변환 대기인 것을 나타내는 강조 표시인 채입니다 (Sun 의 Java 2 Runtime Environment 의 경우, 텍스트는 Solaris 상에서는 반전 표시, Windows 상에서는 밑줄 첨부로 표시됩니다). 다음에 스페이스 키를 누르면(자), 변환이 개시됩니다. 공항 코드가, 선택한 언어로 샌프란시스코의 도시명으로 옮겨집니다. 이 시점에서, 텍스트는 아직 강조 표시된대로 입니다 (강조 표시의 표시 형식이 얼마인가 다릅니다). 한번 더 스페이스 키를 누릅니다. 룩 업 윈도우가 열려, 원의 도시명이 선택된 상태로, 이용 가능한 모든 언어로 도시명이 표시됩니다. 계속해 스페이스 키를 누릅니다. 스페이스 키를 누를 때마다 선택하는 이름이 변화합니다. 동시에, 선택된 이름이 텍스트 컴퍼넌트에 표시됩니다. Return 키를 누릅니다. 룩 업 윈도우가 닫혀져 마지막에 선택한 이름이, 강조 표시 없음 (확정한 것을 나타낸다)으로 텍스트 컴퍼넌트에 표시됩니다.

4. 입력 메소드의 사용 방법

raw 텍스트의 입력

raw 텍스트는, 입력 메소드를 개입시켜 입력되어 아직 변환되어 있지 않은 임의의 텍스트입니다. 공항의 최초의 문자를 입력하고 나서 변환을 개시할 때까지, 입력 메소드는 raw 텍스트 모드가 됩니다. 이 모드에서는, 「a」로부터 「z」까지의 문자 또는 그 대문자 (입력 메소드는 대문자 소문자를 구별한다)가 raw 텍스트 입력에 추가됩니다. 문자를 삭제하려면 백스페이스키를 사용합니다. 또, 텍스트를 마우스로 클릭하는 것으로써, 삽입 포인트를 변경할 수 있습니다.

도시명에의 변환

raw 텍스트에 1 ? 3 개의 공항 코드 (즉, 3, 6, 또는 9 문자)를 입력해, 도시명에의 변환 처리를 개시할 수 있습니다. 변환을 개시하려면 , 스페이스 키를 누릅니다. 1 개의 공항 코드는, 1 개의 도시명에 변환됩니다. 2 개의 공항 코드의 경우는 「From San Francisco to Frankfurt」등의 구에, 3 개의 공항 코드의 경우는 「From San Francisco to Frankfurt via Chicago」등의 구에 변환됩니다. 각 도시명은, 별개의 세그먼트(segment)로서 고려됩니다. 세그먼트(segment)의 1 개가 선택되면(자), 그 세그먼트(segment)는 룩 업 윈도우로 한층 더 변환이 가능하게 됩니다. 복수의 세그먼트(segment)가 존재하는 경우, 세그먼트(segment)의 선택은, 세그먼트(segment)를 클릭하는지, 오른쪽 커서 키와 왼쪽 커서 키를 사용해 실시합니다.

룩 업 윈도우를 사용해 한층 더 변환한다

룩 업 윈도우는, 선택된 세그먼트(segment)의 공항 코드에 대응하는 도시명을 이용 가능한 모든 언어로 표시하는 일시적인 윈도우입니다. 사용자는 룩 업 윈도우를 사용해 특정의 도시명을 선택합니다. 룩 업 윈도우를 표시하려면, 세그먼트(segment)를 선택하고 나서 스페이스 키를 누릅니다. 세그먼트(segment) 텍스트아래에 윈도우가 열립니다. 아래에 충분한 표시 영역이 없는 경우는 세그먼트(segment) 텍스트에 가까운 다른 장소에 표시됩니다. 선택중의 도시명이 강조 표시되고 있습니다. 선택하는 도시명을 변경하려면 , 스페이스 키, 하 커서 키, 상 커서 키를 사용합니다. 선택한 이름의 확정 방법에 대해서는, 다음의 섹션으로 설명합니다. 이름을 선택해도 확정하지 않고 룩 업 윈도우를 닫으려면 , 이름의 전의 숫자를 입력합니다.

입력의 확정

입력한 텍스트를 확정하는 것은, 입력 메소드의 처리를 종료해, 텍스트를 텍스트 컴퍼넌트의 일반적으로의 일부로 하는 것을 의미합니다. 텍스트를 확정하면(자), 입력 메소드에 의한 강조 표시는 없어집니다. Return 키를 누르는 것으로, 입력 텍스트 전체를 언제라도 확정할 수 있습니다. 복수의 공항 코드를 입력해 변환을 개시했을 경우, 부분 확정을 실행할 수 있습니다. 컨트롤 키를 누르면서 하 커서 키를 누르는 것으로, 선택중의 세그먼트(segment)까지를 확정해, 나머지의 세그먼트(segment)를 변환 가능하게 할 수 있습니다.

변환은, 공백 문자를 사용해 요구됩니다. 변환중에, 입력 메소드에 의해, 선택한 언어의 도시명이 최초로 제공됩니다 (그 이름이 사용 가능한 경우). 사용자가 그 선택을 받아들이지 않았던 경우는, 선택 윈도우가 표시되어 그 도시의 사용 가능한 이름이, 대응하는 로케일명으로 모두 제공됩니다.

5. 상태 윈도우의 제어

입력 메소드는, 선택중의 로케일을 나타내는 작은 상태 윈도우를 표시합니다. 기동 직후, 이 윈도우는 화면의 우하부에 배치됩니다만, 위치를 자유롭게 변경할 수 있습니다. 포커스를 가지는 텍스트 컴퍼넌트를 포함한 윈도우아래에 상태 윈도우가 자동적으로 표시되도록(듯이) 하려면 , 상태 윈도우내를 더블 클릭 합니다.

6. 지원되는 도시와 로케일

입력 메소드의 지역 정보는 한정되어 있습니다. 인식되고 있는 공항 코드는, SFO, NYC, JFK, LGA, ORD, FRA, MUC, TYO, NRT, HND, KIX, ITM, PEK, BJS, SHA, HKG, 및 TPE 입니다. 현재로서는, 지원 대상의 언어는, 영어, 독일어, 일본어 (일반적으로 및 읽어), 중공에서 정식으로 쓰는 약자체�중국어, 번체자 중국어입니다.
