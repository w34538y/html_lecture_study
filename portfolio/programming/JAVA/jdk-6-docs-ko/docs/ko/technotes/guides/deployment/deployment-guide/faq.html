질문과 그 답

Q: Java 컨트롤 패널로 「업데이트를 자동적으로 체크」를 체크했습니다만, 스케줄 한 자동 갱신이 실행되어 있지 않은 것 같습니다. 무엇이 문제입니까.

A:자동 갱신 기능이 동작하는 것은 Windows 2000 (SP3)으로 Windows XP 뿐입니다. 지원되고 있는 그 외의 operating system를 실행하고 있는 경우, 이 기능은 동작하지 않습니다. 그 경우에 갱신을 실시하려면 ,[금방 업데이트] 버튼을 사용해 Java 를 수동으로 갱신해 주세요.

Q:인스톨러를 다운로드했습니다만, 200 KB 밖에 없습니다. 이렇게 작은 것은 왜입니까.

A:Java 인스톨러는, 인스톨 On Demand용으로 설계되고 있습니다. 사용자의 선택 내용이나 시스템 구성에 맞추어, 인스톨러 파일이 추가로 다운로드됩니다. 이 인스톨러를 사용하면(자), 불필요한 파일을 대량으로 다운로드하는 것을 막을 수가 있습니다.

Q:Java Update 로 재기동을 요구하는 것은 왜입니까.

A:Windows Installer 2.0 (Java 인스톨러로 사용하는 Microsoft 테크놀러지)을 인스톨 했을 경우에 재기동이 필요하게 되기 (위해)때문에, 재기동이 요구되는 경우가 있습니다.

주:이번, Java 인스톨러에서는, 재기동을 회피할 수 있도록(듯이) 하기 위한(해), Java 대응의 Mozilla 브라우저의 락 파일을 검출해, 브라우저를 닫도록 유발 (prompt)하는 메세지를 사용자에게 표시합니다.

Q:Java Update 를 위해서(때문에) Java 컨트롤 패널을 열었는데, 「버젼 정보」탭에 버젼 x.x.x 라고 표시되었습니다. 그 후 Java Update 를 실행해도, 여전히 버젼 x.x.x 라고 표시됩니다. 이것은 왜입니까.

A:갱신된 컨트롤 패널을 가져오려면 , Java 컨트롤 패널을 일단 닫고 나서 개시할 필요가 있습니다.

Q:Java Plug-in 를 사용하면 Netscape/Mozilla 가 올바르게 동작하지 않습니다. 왜입니까.

A:우선, 모든 브라우저 세션을 닫아 주세요. 이것으로 해결할 수 없는 경우는, 시스템을 재기동하고 나서 한번 더 조작해 보세요.

Q:Java Update 로 갱신 파일을 D:\ 드라이브에 인스톨 하려고 했는데, 여전히 C:\ 드라이브에 인스톨 됩니다. 왜입니까.

A:다른 타겟 디렉토리가 선택되었는지 어떠했는지에 관계없이, Java Update 에서는 일부의 갱신 파일을 Windows 의 시스템 드라이브에 인스톨 할 필요가 있습니다. 이와 같이 하는 것으로, 최신 버젼의 Java 에 장래 갱신할 때의 다운로드 시간을 절약합니다.

Q:인스톨 한 버젼의 Java Update 를 언인스톨하려면 어떻게 합니까.

이하의 2 방법의 방법이 있습니다.

  1. JRE 를 언인스톨하려면 , Microsoft Windows 의 컨트롤 패널로 「어플리케이션의 추가와 삭제」유틸리티 ( 「스타트」> 「설정」> 「컨트롤 패널」로부터 「어플리케이션의 추가와 삭제」)를 사용합니다.
  2. 또는, JRE 를 인스톨 하는데 사용한 원의 인스톨 프로그램이 남아 있는 경우는, 그 프로그램을 더블 클릭 합니다. 그러자(면), 「변경」과「삭제」옵션이 있는 화면이 표시됩니다. [삭제] 를 선택해, JRE 를 언인스톨합니다. 이 조작에서는, 인터넷에 접속하고 있을 필요가 있는 것에 유의해 주세요.

Q: JRE bootstrap 인스톨러를 다운로드해 실행하면(자), "This installer cannot proceed with the current Internet Connection settings of your system.In your Windows Control Panel, please check Internet Options -> Connections to make sure the settings and proxy information are correct. " 라고 하는 메세지가 표시됩니다. 왜입니까.

A:JRE bootstrap 인스톨러에서는, 다른 파일을 다운로드하기 위해서 Web 에 접속할 때에, 시스템의 인터넷 접속 설정을 사용합니다. 방화벽(fire wall)의 배후에 있어 프록시 설정이 필요한 경우는,[인터넷 옵션] 또는 [인터넷의 프로퍼티] ([스타트] > [컨트롤 패널] > [인터넷 옵션] 또는 [인터넷의 프로퍼티] > [접속] > [근거리 통신망 (LAN)의 설정])의 프록시 설정이 올바르게 설정되어 있는 것을 확인해 주세요.

외부의 Web 사이트 (즉 방화벽(fire wall)의 외측)를 Internet Explorer 로 브라우즈 할 수 있는 경우, 프록시 설정은 올바르게 설정되어 있습니다. 인스톨러에서는, Netscape/Mozilla 로 지정한 프록시 설정은 인식할 수 없습니다.

Q:JRE 를 인스톨 했는데, 시스템의 백그라운드에서 실행하고 있는 2 개의 프로세스 jucheck.exe 와 jusched.exe 가 발견되었습니다. 종료 하는 방법은 있습니까.

A:jusched.exe 는 Java Update 의 스케쥴러 프로세스로,jucheck.exe 는 Java Update 에서의 갱신을 확인해 실행하는 프로세스입니다. 이러한 프로세스는 자동적, 한편 사용자에게 투과적으로 실행됩니다. 이러한 프로세스를 종료 하려면 , Java 컨트롤 패널의 「업데이트」탭에 있는 「업데이트를 자동적으로 체크」체크 박스를 클릭하는 것만입니다.


Q:Java Update 로 시스템을 갱신했지만, 브라우저에서는 애플릿의 실행에 Microsoft VM 또는 이전의 버젼의 Sun Java VM 가 사용되고 있습니다. 무엇이 원인입니까.

A:JRE 를 [커스텀] 옵션으로 인스톨 해, 브라우저의 선택을 해제했을 경우는, 이러한 상황이 됩니다. 문제를 해결하려면 , Java 컨트롤 패널의 「브라우저」탭으로 이동해, 새롭게 인스톨 한 Java 를 실행하는 브라우저를 선택합니다.

Q:Java 컨트롤 패널의 [금방 업데이트] 버튼을 클릭하면(자), 시스템이 「오프 라인」이다라는 메세지가 표시됩니다. 이것은 어떤 의미입니까.

A:Java Update 는, 시스템이 네트워크에 접속되고 있는 경우만 실행할 수 있습니다. 네트워크에 접속되어 있지 않은 시스템은 「오프 라인」이다고 하여 참조됩니다. [금방 업데이트] 버튼을 클릭하면(자), 시스템의 온라인/오프 라인 상태를 체크합니다. 시스템이 현시점에서 네트워크에 접속하고 있지 않는 경우나, 다이얼 업 네트워크가 절단 되고 있는 경우는, 에러 메세지가 표시됩니다. 시스템이 네트워크에 접속하고 있는 상태인 것을 확인해, 한번 더 조작해 보세요.

Q:지시에 따라 버젼 x.x.x 를 인스톨 했습니다. 인스톨 해 재기동했는데, 다운로드 가능한 갱신이 있는 것을 지메세지가 시스템 트레이로부터 pop-up 되었습니다. 어떻게 하면 좋습니까.

A:이 메세지는, Java 자동 갱신 기구의 일부로, 사용자의 로그인시에 새로운 버젼의 JRE 를 다운로드할 수 있는 것을 검지했습니다. 시스템 트레이의 Java Update 아이콘을 클릭하는 것만으로, 갱신을 다운로드해 인스톨 할 수 있습니다.

Q:Java Update 를 실행해 보고 싶습니다만, 컴퓨터에 문제가 일어나는 것은 아니어서 걱정입니다. 어떻게 하면 좋습니까.

A:Java Update 를 테스트하기 전에, Windows 의 레지스트리를 백업 할 수 있습니다. 필요한 레지스트리 키를 파일에 export 하려면 , 다음의 커멘드를 실행합니다.

regedit -e reg. 1 -t HKEY_LOCAL_MACHINE
regedit -e reg. 2 -t HKEY_CURRENT_USER
regedit -e reg. 3 -t HKEY_CLASSES_ROOT

레지스트리 키를 완전하게 복원하려면 , export 한 레지스트리의 값을 다음의 커멘드를 사용해 임포트 합니다.

regedit -i reg. 1
regedit -i reg. 2
regedit -i reg. 3

Q:Java SE 인스톨러를 실행중에,"Error 1606: Could not access network location file MSI/CAB and still fail after immediate retry. " 라고 하는 에러가 표시되었습니다. 이것은 어떤 의미입니까.

A:이 문제는, 서버가 Busy 지나는지, 네트워크가 혼잡하고 있어 파일을 제공할 수 없는 경우에 발생합니다. 인스톨러를 한번 더 실행해, 재시행해 주세요.

Q:Java SE 인스톨러를 실행중에,"This installation package could not be opened. Contact the application vendor to verify that this is a valid Windows Installer package. " 라고 하는 에러가 표시되었습니다.

A:몇개의 이유가 생각됩니다.

  • 네트워크 접속의 실패
  • GetRight 등의 다운로드 매니저 소프트웨어가 다운로드 프로세스를 중단했다.
  • Norton AntiVirus 와 같은 TSR (Terminate and Stay Resident) 프로그램이 인스톨 프로세스를 방해했다.

이 문제를 해결하려면 , 서드 파티제의 다운로드 /TSR 프로그램이 무효가 되어 있어, 네트워크 접속이 올바르게 설정되어 있는 것을 확인해 주세요. 프록시를 사용중의 경우는, 프록시 인증이 무효가 되어 있는 일도 확인합니다.


Q:Error 1311:Source File Not Found.Could not find C:\Downloads\ja555000.cab.

A:MSI 엔진의 기존의 문제입니다. 다음의 Web 사이트를 참조해 주세요.

  • http://support.microsoft.com/default.aspx? scid=kb;[LN];Q290106
  • http://support.microsoft.com/default.aspx? scid=kb;[LN];Q307346
  • http://support.microsoft.com/default.aspx? scid=kb;[LN];290896

Q:Error 1722. There is a problem with this windows installer package.A program run as part of the setup did not finish as expected.Contact your support personnel or package vender.

A:이 에러 메세지가 표시되었을 경우는, 피드백 앨리어스(alias) (java-deployment-feedback@sun.com )에 물음합 끓는이다 차이.
