지원되고 있는 폰트

지원되는 폰트는, Java 플랫폼의 구현에 따라서 다릅니다. 여기에서는, Sun 의 JDK (Java SE Development Kit) 6 및 JRE (Java SE Runtime Environment) 6 으로 지원되는 폰트에 대해 설명합니다. 여기서 사용되고 있는 용어에 대해서는,Font 클래스의 설명을 참조해 주세요.

  • 물리 폰트의 지원
  • 논리 폰트의 지원
  • Lucida 폰트

물리 폰트의 지원

Sun 의 Java SE Runtime Environment 는, TrueType 폰트와 PostScript Type 1 폰트를 지원하고 있습니다.

물리 폰트는, Java 실행 환경에서 인식되고 있는 장소에 인스톨 되고 있을 필요가 있습니다. Sun 의 Java SE Runtime Environment 에서는, JRE 내의 lib/fonts 디렉토리와 호스트 operating system로 정의되고 있는 일반적으로의 폰트 포함처가 검색됩니다. 양쪽 모두의 장소에 같은 이름의 폰트가 존재하는 경우는,lib/fonts 의 폰트가 사용됩니다.

지원되고 있는 폰트 기술을 사용하고 있는 물리 폰트이면, JRE 내의 lib/fonts 디렉토리에 인스톨 하는지, 호스트 operating system로 지원되고 있는 방법으로 인스톨 할 수가 있습니다.

논리 폰트의 지원

논리 폰트명은, 구현에 의존한 방법으로 물리 폰트에 맵 됩니다. 일반적으로, 1 개의 논리 폰트명은, 광범위의 문자를 커버하기 위해서 복수의 물리 폰트명에 할당할 수 있습니다. Sun 의 Java SE Runtime Environment 에서는,폰트 구성 파일을 사용해 매핑을 정의합니다.

Java 2D 렌더링으로 사용하는 논리 폰트의 대체 폰트로서 물리 폰트를 추가하려면 , 그 물리 폰트를 JRE 내의 lib/fonts/fallback 디렉토리에 인스톨 합니다.

Lucida 폰트

Sun 의 Java SE Development Kit 와 Java SE Runtime Environment 에는, Lucida 디자인 패밀리의 물리 폰트가 몇개인가 번들 되고 있습니다. 이러한 폰트는, Java 플랫폼의 다른 구현에서도 사용할 수 있도록(듯이) 라이센스 부여되고 있습니다. 이러한 폰트는 물리 폰트입니다만, Java SE Runtime 에 부속되어 있기 (위해)때문에, 호스트 operating system에는 의존하지 않습니다. 이러한 폰트를 사용하면(자), 다양한 플랫폼이나 구현, 광범위한 언어에 건너, 일관한 Look&Feel 와 조작성을 얻을 수 있다고 하는 이점이 있습니다.

타입 패밀리로서는,「Lucida Sans」,「Lucida Sans Typewriter」, 및 「Lucida Bright」 의 3 개가 있습니다. 각 패밀리에게, 프레인, 볼드, 이탤릭, 및 볼드 이탤릭의 스타일이 있습니다. 이러한 폰트 모든 것이 Java SE 의 모든 구현에 존재하는 것은 아닙니다. 예를 들어, Sun 의 Windows 판 Java SE Runtime Environment 의 디폴트의 다운로드 번들에는, Lucida Sans 프레인 폰트만이 포함되어 있습니다 (다만, 어플리케이션 개발자는,어플리케이션과 함께 재배포하는 JRE 에 모든 Lucida 폰트를 포함할 수도 있습니다).
