문자 인코딩의 중요성

바이트 인쇄 데이터의 MIME 타입에, 인코딩을 나타내는 캐릭터 세트 파라미터가 지정되어 있지 않은 경우, Java 인쇄 서비스에서는 US-ASCII 의 캐릭터 세트라고 보여집니다. 이 동작은, Java 실행시의 동작과는 다릅니다. Java 실행시에는, 부하의 operating system상으로 설정되어 있는 디폴트의 인코딩이 사용자 로케일이라고 보여집니다. 이 인코딩은, US-ASCII 와 다른 일이 있습니다.

Java TM 가상 머신* 의 각 인스턴스에는, 디폴트의 문자 인코딩을 할당할 수 있습니다. 이 인코딩은, 가상 머신이 기동했을 때에 결정되어 일반적으로은 부하의 operating system로 사용되는 로케일과 캐릭터 세트에 의존합니다. 분산환경의 2 개의 JVM 가, 같은 디폴트의 인코딩을 공유하지 않는 것이 있습니다. 즉, 호스트 플랫폼이 encode 한 텍스트 데이터를, 클라이언트가 Java 인쇄 서비스의 인스턴스에 스트림 전달할 때는, 디폴트의 인코딩에 의존하지 않고, 캐릭터 세트를 명시적으로 선언할 필요가 있습니다.

이 때문에, 텍스트 데이터를 스트림 전달하는 어플리케이션에서는, MIME 타입에 캐릭터 세트를 항상 지정할 필요가 있습니다. MIME 타입을 지정하려면 , 어플리케이션으로부터 DocFlavor.hostEncoding 메소드를 호출해, 호스트 플랫폼의 인코딩을 조사할 필요가 있습니다. 이 메소드로부터 반환된 MIME 타입은, 현재의 JVM 상에서 반드시 인식됩니다.

Java 플랫폼상에서 지원되는 문자 인코딩의 상세한 것에 대하여는,「문자 인코딩」 을 참조해 주세요.


*이 Web 사이트에서는, 「Java 가상 머신」또는 「JVM」라고 하는 용어는, Java 플랫폼용의 가상 머신을 나타냅니다.
