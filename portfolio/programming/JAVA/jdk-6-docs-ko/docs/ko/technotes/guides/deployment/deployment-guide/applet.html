APPLET 태그

 문서의 목차

APPLET 태그의 간단한 예를 나타냅니다. 

  <applet code="MyApplet.class" width=100 height=140></applet>
이것은, 컴파일 된 코드가 MyApplet.class (현재의 HTML 문서와 같은 디렉토리내)에 있는 애플릿을 로드해, 애플릿의 초기 사이즈를 옆 100 픽셀, 세로 140 픽셀로 설정하도록(듯이) viewer 또는 브라우저에 통지합니다.

다음에, APPLET 태그의 것보다 복잡한 예를 나타냅니다. 

    <applet codebase="http://java.sun.com/applets/NervousText/1. 1"
	code="NervousText.class" width=400 height=75>
    <param name="text" value="Welcome to HotJava! ">
    <hr>
    If you were using a Java-enabled browser such as HotJava,
    you would see dancing text instead of this paragraph.
    <hr>
    </applet>
이것은, 컴파일 된 코드가 URL http://java.sun.com/applets/NervousText/1. 1/NervousText.class 에 있는 애플릿을 로드해, 그 애플릿의 초기 사이즈를 400x75 픽셀로 설정하도록(듯이), viewer 또는 브라우저에 통지합니다. viewer 또는 브라우저는, 애플릿의 text 속성 (이 애플릿이 표시하는 텍스트를 커스터마이즈 하는 속성)을 "Welcome to HotJava! "(으)로 설정할 필요가 있습니다. Java 애플릿을 실행할 수 없는 브라우저를 사용해 이 페이지를 표시하면(자), 브라우저는 APPLET 태그와 PARAM 태그를 무시해,<param> 태그와 </applet> 태그의 사이의 HTML (대체 HTML)만을 표시합니다.

다음은, 전술의 예를 HTML 파일에 기술한 결과입니다 (페이지를 최초로 열 때는, 애플릿이 로드 될 때까지 기다릴 필요가 있다).
If you were using a Java-enabled browser such as HotJava, you would see dancing text instead of this paragraph.

다음에, APPLET 태그의 다른 예를 나타냅니다. 

     <applet code=A21 width=256 height=256 archive="toir.jar">
     <param name=img value=test.gif>
     <hr>
     We need to convert some of the standard applets to use
     archive.   Any volunteers?
     <hr>
     </applet>
이 예에서는, 애플릿 클래스는 A21 입니다. 그 바이트 코드는, 어카이브(archive) "toir.jar" 에 들어가 있어 또 이 어카이브(archive)에는,test.gif 라는 이름의 화상 자원 (자원의 문서를 참조)이 들어가 있습니다.

다음에, APPLET 태그의 완전한 구문을 나타냅니다. 필수 요소는굵은 글씨, 생략 가능한 요소는 보통 서체, 사용자 지정의 요소는이탤릭으로 가리킵니다. 

    <APPLET
        CODEBASE = codebaseURL
        ARCHIVE = archiveList
        CODE = appletFile ...또는...  OBJECT = serializedApplet
        ALT = alternateText
        NAME = appletInstanceName
        WIDTH = pixels  HEIGHT = pixels
        ALIGN = alignment
        VSPACE = pixels  HSPACE = pixels
    >
    <PARAM NAME = appletAttribute1 VALUE = value>
    <PARAM NAME = appletAttribute2 VALUE = value>
    .  .  .
    alternateHTML
    </APPLET>

CODE, CODEBASE 등은 애플릿 태그의 속성으로, 브라우저에 애플릿에 관한 정보를 전합니다. 필수의 속성은 CODE, WIDTH, HEIGHT 뿐입니다. 각 속성에 대해, 다음에 설명합니다.

CODEBASE = codebaseURL
생략 가능한 속성입니다. 애플릿의 베이스 URL (애플릿의 코드를 포함한 디렉토리)를 지정합니다. 이 속성을 지정하지 않는 경우는, 문서의 URL 가 사용됩니다.
ARCHIVE = archiveList
생략 가능한 속성입니다. 「프리로드」되는 클래스등의 자원을 포함하고 있는 어카이브(archive) (복수가능)를 지정합니다. 이러한 클래스는, 지정된 CODEBASE 로 AppletClassLoder 의 인스턴스를 사용해 로드 됩니다. archiveList 내의 어카이브(archive)는 「,」그리고 단락지어집니다. 주:JDK 1.1 에서는, 같은 CODEBASE 를 가지는 복수의 APPLET 태그는, ClassLoader 의 같은 인스턴스를 공유합니다. 클라이언트 코드에는, 이 방법을 사용해 애플릿 상호의 통신을 구현하는 것이 있습니다. 장래의 JDK 에서는, 애플릿간 통신의 다른 기구가 제공될 가능성이 있습니다. 시큐리티상의 이유로부터, 애플릿의 클래스 로더는, 그 애플릿이 기동된 것과 같은 코드 베이스로부터 밖에 읽어낼 수가 없습니다. 이것은,archiveList 내의 어카이브(archive)가 코드 베이스와 같은 디렉토리인가, 코드 베이스의 서브 디렉토리내에 없으면 안 되는 것을 의미합니다. ../a/b.jar 의 형식의 archiveList 내의 엔트리는, 시큐리티 정책 파일내에서 명시적으로 허가되어 있지 않은 한, 기능하지 않습니다 (http 코드 베이스의 경우는 예외로,archiveList 내의 어카이브(archive)는 코드 베이스와 같은 호스트의 것이 아니면 안 되지만, 패스에 「..」(이)가 포함되어 있어도 상관없다).
CODE = appletFile
필수의 속성입니다. 애플릿의 컴파일 된 Applet 서브 클래스를 포함하고 있는 파일의, 이름을 지정합니다. 이 파일명은, 애플릿의 베이스 URL 로부터의 상대 패스로 지정합니다. 절대 패스로는 지정할 수 없습니다. CODE 와 OBJECT 의 어느 쪽인지 1 개가 필요합니다. appletFile 의 값은,classname. class 또는 packagename. classname. class 의 어느 쪽인가의 형식이 됩니다.
OBJECT = serializedApplet
이 속성은, 애플릿의 직렬화 표현을 포함하고 있는 파일의, 이름을 지정합니다. 애플릿은 직렬화 해제됩니다. init() 메소드는 불려 가지 않습니다만, start() 메소드가 불려 갑니다. 원의 객체의 직렬화시에 유효했던 속성은, 복원되지 않습니다. 이 APPLET 의 인스턴스에게 건네진 인수는 모두, 이 애플릿으로 이용할 수 있습니다. 이 기능의 사용에는, 강한 제한을 걸쳐야 합니다. 애플릿은, 직렬화하기 전에 정지하지 않으면 안됩니다. CODE 와 OBJECT 의 어느 쪽인지 1 개가 필요합니다.
ALT = alternateText
생략 가능한 속성입니다. 브라우저가 APPLET 태그를 이해할 수 있지만, Java 애플릿을 실행할 수 없는 경우에 표시되는 텍스트를 지정합니다.
NAME = appletInstanceName
생략 가능한 속성입니다. 애플릿의 인스턴스의 이름을 지정합니다. 이것에 의해, 동일 페이지상의 애플릿이 서로를 검지 ( 및 통신)할 수가 있습니다.
WIDTH = pixels HEIGHT = pixels
필수의 속성입니다. 애플릿의 표시 영역의 폭과 높이 (픽셀수)의 초기치를 지정합니다. 애플릿이 표시하는 윈도우나 다이얼로그는 고려하지 않습니다.
ALIGN = alignment
생략 가능한 속성입니다. 애플릿의 배치 방법을 지정합니다. 애플릿의 배치 방법을 지정합니다. 이 속성으로 설정할 수 있는 값은 IMG 태그와 같아, left, right, top, texttop, middle, absmiddle, baseline, bottom, absbottom 입니다.
VSPACE = pixels HSPACE = pixels
생략 가능한 속성입니다. 애플릿의 외측의 상하 (VSPACE) 및 좌우 (HSPACE)의 픽셀수를 지정합니다. IMG 태그의 VSPACE 와 HSPACE 속성과 같게 다루어집니다.
<PARAM NAME = appletAttribute1 VALUE = value>
<PARAM NAME = appletAttribute2 VALUE = value> . . .
이 태그는, 애플릿의 독자적인 속성을 지정하는 유일한 방법입니다. 애플릿은, getParameter() 메소드를 사용해 이러한 속성에 액세스 합니다.
