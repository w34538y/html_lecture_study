JavaTM Platform Debugger Architecture (JPDA)

 문서의 목차
Java Platform Debugger Architecture (JPDA)는, 3 개의 인터페이스로 구성되어 있어 데스크탑 시스템의 개발 환경에서 디버거를 사용하는 것을 전제로 해서 설계되고 있습니다. Java Virtual Machine Tools Interface (JVM TI)에서는, 디버그시에 VM 로부터 제공되는 서비스가 정의되고 있습니다. JVM TI 는, 폐지한 Java Virtual Machine Debug Interface (JVMDI)의 대신이 되는 것입니다. Java Debug Wire Protocol (JDWP)에서는, 디버그중의 프로세스와 디버거의 프론트엔드와의 사이에 전송 되는 정보 및 요구의 형식이 정의되고 있습니다. 프론트엔드에는, Java Debug Interface (JDI)가 구현되고 있습니다. Java Debug Interface 에서는, 사용자 코드 레벨에서의 정보 및 요구가 정의되고 있습니다.
