java.math 의 확장 기능

 Math 목차

JDK 5.0

  • 고정 정밀도의 부동 소수 계산을 지원하는BigDecimal 클래스를 추가했습니다. JSR 13 」을 참조해 주세요.

  • Math 및 StrictMath 라이브러리에 쌍곡선 초월 함수 (sinh, cosh, tanh), 입방근, 상용대수등이 포함됩니다.

  • 16 진수의 부동 소수의 지원 - 특정의 부동 소수치로 정확하고 예측 가능한 스펙을 허가하기 위해서,Float 와 Double 로, 부동 소수 리터럴과 부동 소수 변환 메소드에 대한 캐릭터 라인에, 16 진표기를 사용할 수 있습니다.

Java 2 SDK 1.4

  • BigIntenger 의 소수 생성의 퍼포먼스 향상
    BigInteger 클래스에, 소수를 생성하기 위한 새로운 static 메소드 probablePrime 가 추가되었습니다. 종래는,BigInteger 의 소수를 생성하는 방법은 1 개 밖에 없고, 메소드의 호출측으로부터, 「확실성」(호출측을 허용 할 수 있는, 결과가 소수가 아닌 확률의 대수)을 제공할 필요가 있었습니다. 그러나, 제공하는 확실성의 값이 부적절하다면, 퍼포먼스가 저하하거나 (확실성의 값이 너무 비싼 경우), 중대한 버그가 발생하거나 (값이 너무 낮은 경우) 하는 일이 있었습니다. 이 새로운 소수 생성 메소드는, ANSI 의 드래프트 스펙에 근거해 자동적으로 판별되는 적절한 확실성의 값을 사용하는, 소수 생성을 위한 효율적인 알고리즘을 제공합니다.

Java 2 SDK 1.3

  • BigIntenger 의 퍼포먼스의 향상
    java.math.BigInteger 클래스가, Pure Java 프로그램 언어의 코드에 재구현되었습니다. 종래의 BigInteger 의 구현은, 기초 C 라이브러리를 기본으로 해 구현되고 있었습니다. 고품질의 VM 로 새로운 구현을 사용하면(자), 낡은 구현과 비교해 모든 조작의 퍼포먼스가 향상합니다. 퍼포먼스는 5 배 이상 향상하고 있습니다만, 실행하는 조작 및 오퍼랜드의 길이에 따라서 다릅니다.

  • 보다 견뢰한 BigDecimal(String)생성자
    BigDecimal(String) 생성자 이 확장되어 부호 첨부 캐릭터 라인 ("+2. 71828" 등) 및 지수 표기법 ("1.23E-23" 등)을 입력으로서 사용할 수 있게 되었습니다.
