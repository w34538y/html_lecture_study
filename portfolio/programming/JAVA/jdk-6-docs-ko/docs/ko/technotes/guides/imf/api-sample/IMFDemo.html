Input Method Framework 의 데모

alt="입력 메소드와 대화하기 위한 다양한 방법을 나타내는 애플릿"
이 애플릿에는, Java 2 플랫폼에서 입력 메소드와 대화하기 위한 다양한 방법을 나타내는, 4 개의 텍스트 컴퍼넌트가 포함되어 있습니다. 이 애플릿은 다음의 2 개의 방법으로 실행할 수 있습니다.
  • 네이티브 입력 메소드를 사용하는 플랫폼/로케일로 실행되는 Java 2 Platform Standard Edition 버젼 1.2 이후의 구현을 사용한다. 예:네이티브 입력 메소드를 사용하는 플랫폼/로케일은, 일본어판 Windows NT, 중국어 킷을 사용하는 Macintosh, 한국어 로케일을 사용하는 Solaris 등
  • Java 2 Platform Standard Edition 버젼　1.3 이후의 구현을 사용해, Java 프로그램 언어로 구현되고 있는 입력 메소드를 인스톨 한다. Java 실행 환경은, 네이티브 입력 메소드를 지원하고 있지 않는 플랫폼/로케일로 실행되는 경우가 있는 최초의 3 개의 컴퍼넌트는 경량 컴퍼넌트입니다.

즉, 이러한 컴퍼넌트는 Java API 를 사용해 이벤트를 처리해, 텍스트를 draw 합니다. 마지막 1 개(살)은, 피어 컴퍼넌트입니다. 즉, 이벤트의 처리와 텍스트의 draw는, 기반 플랫폼이 제공하는 텍스트 편집 엔진에 의해 처리됩니다. 피어 컴퍼넌트는 JDK 1.0 AWT 로 자주(잘) 사용됩니다만, 경량 컴퍼넌트는 JDK 1.1 으로 도입된 것으로, Swing 클래스 라이브러리의 기초입니다.

각 경량 컴퍼넌트의 기능은, 다음과 같이 다릅니다.

  • 액티브 클라이언트는, Iinput Method Framework API 를 사용해 온더 스포트 (인 라인) 텍스트 변환을 구현합니다.
  • 패시브 클라이언트는, 가장 간단한 방법으로 입력 메소드를 사용합니다. Input Method Framework 에 의존해 모든 변환을 처리해, 확정 텍스트만을 받아들입니다.
  • 비클라이언트는, 입력 메소드의 사용을 거부합니다.

네이티브 일본어 입력 메소드로 다양한 컴퍼넌트를 테스트하려면 , 컴퍼넌트를 클릭해 포커스를 맞혀 입력 메소드를 온으로 하고 나서 (Windows 의 경우는 Alt +`, Macintosh 의 경우는 키보드 메뉴로부터 선택, Solaris 의 경우는 Control + 스페이스 키), 「nihongo」라고 입력합니다. 다음과 같은 동작이 일어납니다.

  • 액티브 클라이언트:단어는, 컴퍼넌트에 직접 입력되어 입력과 동시에 4 문자의 히라가나에 변환된다
  • 패시브 클라이언트:단어는, 표시된 다른 윈도우에 입력되어 입력과 동시에 4 문자의 히라가나에 변환된다
  • 비클라이언트:텍스트 「nihongo」는 컴퍼넌트에 입력된다
  • 피어 컴퍼넌트:동작은 플랫폼에 따라서 다르지만, 아마 액티브 클라이언트에 유사한다 (텍스트는 기존의 텍스트 위에 draw 되는지, 기존의 텍스트가 오른쪽으로 이동된다)

비클라이언트를 제외한 모든 컴퍼넌트의 경우, 텍스트를 변환해 (Windows 와 Macintosh 의 경우는 스페이스 키, Solaris 의 경우는 Control + N 또는 스페이스 키), 확정합니다 (이러한 플랫폼에서는 모두 Enter - 어째서 그렇게 되겠지요). 변환에 의해, 3 개의 한자가 됩니다. 이것은 확정 후에 컴퍼넌트에 강조 표시없이 표시됩니다.

Java 프로그램 언어에 구현되고 있는 같은 입력 메소드, 「City Input Method」로 다양한 컴퍼넌트를 테스트하려면 , 애플릿을 실행하기 전에 입력 메소드의 JAR 파일을 Java 실행 환경의 확장 기능 디렉토리에 인스톨 합니다. 다음에, 컴퍼넌트를 클릭해, 포커스를 맞혀 입력 메소드를 선택합니다. 선택 기구는 구현에 의존합니다. Sun 의 Java 실행 환경의 경우, 윈도우의 좌상구석의 시스템 메뉴로부터 「입력 방식의 변환」메뉴 항목을 선택해, 표시되는 pop-up menu로부터 입력 메소드를 선택합니다. 다음에, 「TYO」라고 입력합니다. 다음과 같은 동작이 일어납니다.

  • 액티브 클라이언트:텍스트는, 컴퍼넌트에 직접 입력되어 강조 표시된다
  • 패시브 클라이언트:텍스트는, 표시되는 다른 윈도우에 입력되어 강조 표시된다
  • 비클라이언트:텍스트는, 컴퍼넌트에 직접 입력되지만, 강조 표시되지 않는다
  • 피어 컴퍼넌트:동작은 플랫폼에 따라서 다릅니다.

비클라이언트 및 피어 컴퍼넌트 (일부의 플랫폼의 경우)를 제외하는, 이러한 모든 컴퍼넌트의 경우, 스페이스 키를 사용해 텍스트를 변환할 수 있습니다. 변환의 결과는 디폴트 언어 (일본어의 「Tokyo」, 일본어의 읽어, 영어, 독일어, 중공에서 정식으로 쓰는 약자체�중국어, 또는 번체자 중국어)에 따라서 다릅니다. 모든 옵션을 표시하려면 , 스페이스 키를 한번 더 누릅니다. 이번은 룩 업 윈도우가 표시되어 모든 후보가 나타납니다. 스페이스 키를 사용해 모든 선택사항을 조사하는지, 목적의 변환의 번호를 입력할 수가 있습니다. Enter 키를 누르면(자), 선택한 변환이 확정되어 강조 표시가 해제됩니다.
