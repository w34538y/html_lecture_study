JavaTM RMI 에서의 SSL 의 사용

JavaTM Remote Method Invocation (Java RMI) 어플리케이션으로 안전한 SSL 접속을 사용해 원격 호출을 실시하는 방법에 대해, 많은 질문이 전해지고 있습니다. Java RMI 에서의 SSL 의 사용에 관한 질문과 회답의 몇개인가를 다음에 나타냅니다.

Q1:Java RMI 로 SSL 를 사용할 수 있습니까.

A1:네. 커스텀 소켓 팩토리를 사용하면 가능합니다. 어플리케이션으로, 원격 객체를 export 해, 필요한 타입의 소켓 (SSL 소켓등)을 작성하는 커스텀 소켓 팩토리를 사용할 수 있습니다. 이 수법을 사용하면(자), 어플리케이션은 디폴트의 소켓 통신 대신에, SSL 소켓 통신을 사용할 수 있습니다. 커스텀 소켓 팩토리의 사용법의 자세한 것은,「Java RMI 에 의한 커스텀 소켓 팩토리의 사용」튜토리얼을 참조해 주세요.

Q2:SSL 의 Pure Java 프로그램 언어의 구현 제품은 어디에 있습니까.

A2: JDK 에는, SSL 소켓의 구현을 제공하는 Java Secure Socket Extension (JSSE) API 가 포함되어 있습니다. JSSE 의 기능 및 이점의 자세한 것은,「JSSE 레퍼런스 가이드」를 참조해 주세요.

Q3:Java RMI 로 SSL 를 사용하고 있는 예는 어디에 있습니까.

A3:원격 객체의 호출용으로 SSL 베이스의 접속을 사용하는 방법을 설명한「JSSE 샘플」에 예가 번들 되고 있습니다. 이것은, 원격 객체를 export 해, SSL 소켓을 작성하는 커스텀 소켓 팩토리를 사용하는 것에 의해 실현됩니다. J2SE 5.0 으로부터, JDK 에서는 몇개의 표준 SSL 베이스의 커스텀 소켓 팩토리 클래스를 제공하고 있습니다. 상세한 것에 대하여는 javax.rmi.ssl 패키지를 참조해 주세요.
