Java Plug-in 가 HTTPS 클라이언트 인증에 브라우저 증명서 스토어내의 증명서를 사용하지 않는다


증상

클라이언트의 인증에 브라우저로부터의 증명서를 사용할 수 없다

원인

Java Plug-in 에서는, 클라이언트를 인증하기 위해서, JRE 전용의 증명서 스토어로부터의 증명서를 사용합니다. 그러나, 브라우저 증명서 스토어로부터의 증명서를 사용하지 않습니다.

해결 방법

Java Plug-in 에서의 클라이언트 인증에는, 사용자의 디스크상에 있는 클라이언트 증명서 스토어, 키스토어의 타입, 및 액세스하기 위한 패스워드가 필요합니다. 이것들 3 개의 파라미터는, 다음과 같이 해 시스템 프로퍼티으로 지정하지 않으면 안됩니다.

-Djavax.net.ssl.keyStore=<client_keystore_file_path>
-Djavax.net.ssl.keyStorePassword=<password to access the client keystore file>
-Djavax.net.ssl.keyStoreType=<keystore_type>

이러한 프로퍼티은, Java 컨트롤 패널의 「상세」탭으로 지정합니다.

디지털 증명서를 브라우저 증명서 스토어로부터 파일에 export 하는 순서를 다음에 나타냅니다.

  1. 브라우저 윈도우를 열어, 「툴」을 클릭해, 「인터넷 옵션」부메뉴 항목을 클릭합니다.
  2. 「컨텐츠」탭을 클릭합니다.
  3. 안정도의 증명서의 영역에서, 「증명서」버튼을 클릭합니다. 이용 가능한 증명서가 다이알로그 박스에 표시됩니다.
  4. 「목적」드롭 다운 메뉴로부터 「클라이언트 인증」을 선택합니다.
  5. export 하는 증명서를 선택합니다.
  6. 「export」버튼을 클릭합니다. 「증명서의 export 위저드」가 기동합니다.
  7. 「증명서의 export 위저드의 개시」윈도우로 「다음에」를 클릭합니다.
  8. Keep the default to "Yes, export the private key" and click Next.
  9. Keep the default to "Personal Information Exchange - PKCS #12(. PFX) " and click Next.
  10. 「Password」윈도우에 입력합니다.
  11. export 하는 파일의 패스와 파일명을 입력합니다. 자동적으로 확장자(extension) . pfx 가 부가됩니다. 「다음에」를 클릭합니다.
  12. 정보를 확인해, 「완료」를 클릭합니다.

례:

증명서를 파일 (패스를 포함한다) c:\certs\client.pfx 에 export 해, 패스워드 "test&guot; 를 사용해 . pfx 파일을 보호하는 경우는, 필요한 시스템 프롭퍼티의 값은 다음과 같이 됩니다.

-Djavax.net.ssl.keyStore=c:\certs\client.pfx
-Djavax.net.ssl.keyStorePassword=test
-Djavax.net.ssl.keyStoreType=pkcs12
