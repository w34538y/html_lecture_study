Java Plug-in 란

Java Plug-in 는, Web 브라우저의 기능을 확장해, 애플릿이나 Java Beans 를 Web 브라우저 부속의 Java 실행 환경은 아니고, Sun 의 Java Runtime Environment (JRE)로 실행 가능하게 합니다. Java Plug-in 는 Sun 의 JRE 의 일부입니다. Java Plug-in 는, JRE 와 함께 사용자의 컴퓨터에 인스톨 됩니다. 또, 지원되고 있는 Mozilla 브라우저 및 Internet Explorer 로 동작합니다.

브라우저는, 다음의 2 개의 방법으로 Java Plug-in 를 호출해 애플릿을 실행합니다.

  1. Web 페이지로 종래의 APPLET 태그를 사용한다.
  2. Internet Explorer 의 경우는 APPLET 태그를 OBJECT 태그에 옮겨놓아 지원되고 있는 Mozilla 브라우저의 경우는 APPLET 태그를 EMBED 태그에 옮겨놓는다. 다만,OBJECT 및 EMBED 태그는, 다음의 장「Java Plug-in 에서의 OBJECT, EMBED, 및 APPLET 태그의 사용」 으로 설명하는 특별한 형식에 준거하고 있을 필요가 있습니다.

OBJECT 태그 및 EMBED 태그는, Web 페이지로 수동으로 갱신할 수 있습니다.

여기까지로, Java Plug-in 의 핵심 부분에 도착해 설명했습니다만, 그 외에도 배울 수 있는 다수의 관련 토픽이 존재합니다. 그 중에는, Java Plug-in 로 프록시 구성을 동작시키는 방법, Java Plug-in 가 지원하는 프로토콜, cookie 지원나 캐쉬등이 있습니다. 이러한 토픽에 대해서는, 「 제 I 부: Java Plug-in 의 기본」으로 설명합니다.

Java Plug-in 의 동작의 몇개인가를 결정하거나 Java 컨트롤 패널을 사용해 옵션을 설정할 수 있습니다. 이러한 방법도, 제 I 부의 「Java 컨트롤 패널을 사용한 Plug-in 의 동작/옵션의 설정」이라고 하는 장으로 설명합니다.

Java Plug-in 는, 인터넷, 인트라넷, Java Server Pages 경유 등, 다양한 방법으로 배치할 수 있습니다. 다양한 배치 타입 및 방법에 대해서는, 「 제 II 부: 배치 schema」로 설명합니다.

애플릿은, 시큐리티 보호된 환경에서 실행할 필요가 있습니다. RSA 서명 첨부 애플릿의 검증을 포함한, 시큐리티 관련의 다양한 토픽에 대해서는, 「 제 III 부: 시큐리티」로 설명합니다. 애플릿에의 서명은, 애플릿 개발의 초심자에게는 어려운 조작입니다. 이 가이드의「RSA 서명 첨부 증명서를 사용한 애플릿의 서명」에서는, 애플릿에의 서명을 실시하는 방법을, 순서를 쫓아 설명합니다.

Java Plug-in 는, 다양한 애플릿의 디버그 지원를 제공합니다. 「 제 IV 부: 디버그의 지원」로 그 상세를 설명합니다. 그곳에서는, Java 디버거의 지원에 관해서 설명합니다. 또, Java Plug-in 콘솔에 대해서도 설명합니다. Java Plug-in 콘솔에는, 디버그용으로 설정 가능한 다양한 옵션 (트레이스, 로그 기록등)이 포함됩니다.

「 제 V 부:상급자전용의 토픽」에서는, 같은 환경에서 복수의 JRE 를 지원하는 방법, Java 로부터 JavaScript 에의 통신, 복수의 브라우저 세션간에 애플릿을 지속시키는 방법, 애플릿 윈도우의 백그라운드 칼라를 변경하기 위한 특별한 애플릿의 속성 등, 보다 상세한 토픽을 다룹니다.

부록에서는, 지원되고 있는 Mozilla 브라우저, FAQ, HTML 컨버터를 소개해, Microsoft VM 와 Java SE 의 호환성에 관한 문제를 설명합니다.
