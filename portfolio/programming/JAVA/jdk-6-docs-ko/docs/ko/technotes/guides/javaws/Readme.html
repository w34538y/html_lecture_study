처음에

JavaTM Web Start 는, Java 테크놀러지 베이스의 어플리케이션에 대한 개발 솔루션입니다. 컴퓨터와 인터넷간을 잇는 것으로, Web 로부터 직접 어플리케이션을 기동 및 관리할 수 있습니다. Java Web Start 를 사용하면(자), 어플리케이션을 간단하게 1 회의 클릭으로 기동할 수 있습니다. 또, 복잡한 인스톨이나 업그레이드 순서를 실시하지 않고 , 항상 최신의 버젼의 어플리케이션을 실행할 수 있습니다.

종래의 Web 를 개입시켜 소프트웨어를 전달하는 방법에서는, 사용자가 Web 상에서 인스톨러를 찾아내 시스템상에 다운로드해, 거기서 프로그램을 실행할 필요가 있습니다. 그 후, 인스톨러를 실행하면(자), 완전, 일반적으로, 최소등의 옵션과 인스톨에 관한 지시가 표시됩니다. 이것에는 시간이 걸리고, 작업도 복잡합니다. 또, 소프트웨어의 새로운 버젼이 나올 때마다 같은 작업을 반복하지 않으면 안됩니다.

한편, 마음에 드는 HTML 베이스의 전자 메일 클라이언트나 달력, 옥션 사이트등의 Web 배치 어플리케이션은, 인스톨도 사용법도 매우 간단합니다. 이것은 Web 브라우저가 전체의 처리를 자동화하고 있기 때문입니다. 복잡한 다운로드나 셋업, 구성 순서도 없습니다. 게다가 항상 최신 버젼이 실행되고 있다고 하는 일도 보증되고 있습니다.

Java Web Start 테크놀러지는, 전술의 HTML 베이스의 어플리케이션의 완전 기능판과 같은 메리트가 있습니다. Java Web Start 테크놀러지는, Web 에 대한 어플리케이션 개발 솔루션의 1 개입니다. HTML 베이스의 클라이언트 대신에 완전 기능판의 어플리케이션을 사용하는 것에는, 많은 메리트가 있습니다.

  • 워드 프로세서나 스프레드쉬트와 같은 종래의 어플리케이션과 비교해 고도의 대화식 사용자 인터페이스.
  • 필요한 대역폭이 적다. 클릭할 때마다 어플리케이션이 Web 서버에 다시 접속할 필요가 없고, 벌써 다운로드한 정보에 대해서는 캐쉬할 수 있기 (위해)때문에, 저속 접속에서도 고품질인 대화를 할 수 있다.
  • 오프 라인 사용의 지원.
물론, 그런데도 최초로 사용하려면 어플리케이션을 다운로드할 필요가 있습니다. 그렇게 말한 의미로, HTML 베이스의 어플리케이션에는 첫회 기동 코스트가 듭니다. 다만, 일반적으로, Web 페이지의 기동에 걸리는 시간은 몇 초입니다. 한편, 일반적인 모뎀 접속으로 JavaTM 테크놀러지 베이스의 어플리케이션을 다운로드하는 경우, 몇분 정도 시간이 걸립니다. Java Web Start 는 컴퓨터상에 모든 다운로드 파일을 로컬에 캐쉬합니다. 즉, 어플리케이션의 첫회 기동 코스트는 HTML 페이지보다 높아집니다만, 그 이후의 기동은, 미리 필요한 자원이 모두 로컬에 준비되어 있으므로 거의 순간중에 행해집니다.

Java Web Start 는 기동할 때마다 Web 서버를 체크해, 최신 버젼의 어플리케이션을 사용할 수 있을지 어떨지를 확인합니다. 사용할 수 있는 경우는 자동적으로 다운로드해, 기동합니다. 그 때문에, 어플리케이션은 최신의 것이 자동적으로 기동합니다. 복잡한 업그레이드 순서는 없습니다.

시큐리티

Java Web Start 의 기반으로서 사용되고 있는 Java 2 플랫폼에는, 포괄적인 시큐리티 아키텍쳐(architecture)가 짜넣어지고 있습니다. Java Web Start 로 기동된 어플리케이션은 디폴트로, 「샌드 박스」라고 불리는 제한된 환경내에서 실행됩니다. 이 환경에서는, 파일이나 네트워크에의 액세스는 제한됩니다. 따라서, Java Web Start 를 사용해 어플리케이션을 기동하면, 시스템의 시큐리티나 무결성을 유지할 수 있습니다.

어플리케이션은 무제한의 액세스를 시스템에 요구할 수 있습니다. 어플리케이션은, 시스템에 대한 무제한의 액세스권을 요구할 수 있습니다. 그 경우, 어플리케이션의 첫회 기동시에, Java Web Start 에 의해 [시큐리티 경고] 다이얼로그가 표시됩니다. 이 시큐리티 경고 다이얼로그에는, 어플리케이션을 개발한 벤더에 관한 정보가 표시됩니다. 그 벤더를 신뢰하는 것을 선택했을 경우, 어플리케이션이 기동됩니다. 어플리케이션의 개발 바탕으로 관한 정보는, 코드의 디지털 서명에 근거하고 있습니다.

인스톨 순서

Java Web Start 는 SDK/JRE 와 함께 인스톨 됩니다. 인스톨 순서에 대해서는, 인스톨 하는 SDK/JRE 의 표준의 인스톨 순서에 따라 주세요.

Java Web Start 소프트웨어의 사용법

Java Web Start 를 사용하면, Java 테크놀러지 베이스의 어플리케이션을 Web 로부터 직접 기동할 수 있습니다. 어플리케이션의 기동 방법에는, 다음의 3 방법이 있습니다.
  • Web 브라우저로부터 링크를 클릭하는 것으로 기동
  • Java Web Start 에 짜넣어지고 있는 어플리케이션 매니저로부터 기동 (최근 사용한 어플리케이션을 추적하기 위해(때문에), 마음에 드는 어플리케이션에 재빠르게 액세스 할 수 있다).
  • 데스크탑의 아이콘 또는 [스타트] 메뉴로부터 기동 (Microsoft Windows 의 경우만)
어느 방법을 사용했을 경우도, 어플리케이션이 기동될 때마다, Java Web Start 는 Web 서버에 접속해, 어플리케이션의 갱신판이 있을지 어떨지를 체크합니다.

Web 브라우저로부터의 기동

Java Web Start 의 데모 페이지 (http://java.sun.com/products/javawebstart/demos.html )에는, 마우스의 1 회의 클릭 조작으로 기동할 수 있는 많은 어플리케이션에의 링크가 준비되어 있습니다. 「Launch」버튼을 클릭해, 각종 어플리케이션을 기동해 보세요. Web 브라우저가 Java Web Start 를 기동합니다. 다음에, 지정의 어플리케이션이 다운로드된 후 캐쉬되어 실행됩니다. 다음번 기동할 때는, 벌써 어플리케이션이 로컬에 보존되고 있어 다운로드할 필요가 없기 때문에, 기동 시간이 극적으로 단축되고 있는 것을 알 수 있습니다.

페이지상이 많은 데모는, 사용자측에서 조작하는 일 없이 간단하게 다운로드 및 실행할 수 있습니다. 이러한 어플리케이션은, 로컬 디스크나 네트워크에 액세스 할 수 없는 제한된 환경에서 실행되기 (위해)때문에, 컴퓨터에 바이러스가 인스톨 될 가능성을 회피할 수 있습니다.

데모의 몇개인가는, 로컬 하드 디스크에의 액세스권 등, 특별한 권한을 필요로 합니다. 이러한 어플리케이션의 경우, 시큐리티 다이얼로그가 pop-up 해, 어플리케이션 코드에 누가 디지털 서명 했는지를 기초를 두어, 어플리케이션의 제공원에 관한 정보가 표시됩니다. 어플리케이션이 실행되는 것은, 그 벤더를 사용자가 신뢰한다고 결정했을 경우만입니다.

이상이 Java Web Start 의 사용법에 관한 방법이라고입니다만, 어떻게 동작하는 것일까요. 어플리케이션을 기동하는 HTML 링크는, 실제의 곳, 단순한 표준의 HTML 링크입니다. 다만, 다른 Web 페이지를 가리키는 대신에, JNLP 파일로 불리는 특수한 설정 파일을 가리키고 있습니다. Web 브라우저는, 그 파일의 파일 확장자(extension) 또는 MIME 타입, 혹은 그 양쪽 모두를 검사해, 그 파일이 Java Web Start 용의 것임을 인식합니다. 그 후에, 다운로드된 JNLP 파일을 인수로서 Java Web Start 를 기동합니다. Java Web Start 는, JNLP 파일의 지시에 따라, 어플리케이션을 다운로드, 캐쉬, 및 실행합니다.  

짜넣어지고 있는 어플리케이션 매니저로부터의 기동

어플리케이션 매니저는 Java Web Start 제품에 짜넣어지고 있습니다. 이것을 사용하면(자), 이전 Java Web Start 로 기동한 어플리케이션을 재빠르게 간단하게 기동할 수 있습니다. 이것은, Web 배치의 Java 테크놀러지 베이스의 어플리케이션의 「History」메뉴와 「Start/Programs」메뉴의 편성에 의해 가능해집니다. 또, 어플리케이션 매니저를 사용하는 것으로, 어플리케이션의 추가 정보를 어플리케이션의 홈페이지로의 링크 첨부로 표시할 수 있습니다.

어플리케이션은, 어플리케이션 매니저의 어플리케이션 아이콘을 더블 클릭 하거나 「launch」버튼을 클릭하는 것으로 기동할 수 있습니다.

어플리케이션 매니저의 다른 중요한 기능으로서는, Java Web Start 의 설정을 테스트하거나 변경하거나 할 수 있는 「Preferences」다이얼로그가 있습니다. 예를 들어, 이 다이얼로그에는 다음 일을 실행할 수 있는 탭이 준비되어 있습니다.

  • HTTP 프록시를 지정한다 (또는, Java Web Start 에 디폴트로 사용하는 브라우저의 설정을 통지한다).
  • 다운로드한 어플리케이션의 캐쉬를 클리어 한다.
  • Java 가 다른 버젼의 런타임 환경의 장소를 지정한다.
  • Java 콘솔을 표시시킬지 어떨지를 선택한다.
  • 루트의 시큐리티 증명서세트를 표시한다.
어플리케이션 매니저는, 데스크탑 또는 Microsoft Windows 의 「스타트」메뉴에 있는 Java Web Start 아이콘을 클릭하는 것으로 기동합니다. SolarisTM 의 오퍼레이팅(operating) 환경이나 Linux 의 경우, Java Web Start 인스톨 디렉토리의 javaws 커멘드를 호출하면(자) 기동합니다. 또, 어플리케이션 매니저도 Web 브라우저로부터 기동할 수 있습니다. 예로서 데모 페이지 (http://java.sun.com/products/javawebstart/demos.html )를 참조해 주세요.

데스크탑의 아이콘 및 「스타트」메뉴로부터의 기동 (Microsoft Windows 의 경우만)

Java Web Start 테크놀러지에서는, Microsoft Windows 의 데스크탑이나 「스타트」메뉴에, Web 배치의 Java 테크놀러지 베이스의 어플리케이션의 쇼트 컷을 자동 작성할 수 있습니다. 디폴트에서는, Java Web Start 에 의해, 다음번의 어플리케이션 기동시에 쇼트 컷을 작성할지 어떨지를 묻는 메세지가 표시됩니다. 이 설정은 「설정」패널로부터 변경할 수 있습니다.

또, 쇼트 컷은, 「Application/Create Shortcut」나 「Application/Remove Shortcut」메뉴 항목을 사용해, 어플리케이션 매니저로부터도 추가하거나 삭제하거나 할 수 있습니다.

프록시 서버-/방화벽(fire wall) 환경에서의 Java Web Start 소프트웨어의 사용

Java Web Start 소프트웨어의 프록시 설정이 올바르지 않았던 경우, 방화벽(fire wall)의 외측에 있는 어플리케이션을 기동할 수 없습니다. Java Web Start 소프트웨어는, 시스템상의 디폴트 브라우저 (Microsoft Windows 상에서는 Internet Explorer 또는 NetscapeTM 브라우저, Solaris 오퍼레이팅(operating) 환경 및 Linux 상에서는 Netscape 브라우저)로부터, 프록시 설정을 자동적으로 검출하려고 합니다. Java Web Start 테크놀러지는, Web 프록시 자동 설정 스크립트의 대부분을 지원하고 있습니다. 거의 모든 환경에서 프록시 설정을 검출할 수 있습니다.

프록시 설정이 자동적으로 검출되지 않는 경우, 최초의 Java Web Start 사용시에 프록시 설정을 지정하도록(듯이) 요구됩니다. 또, Java Web Start 에서는, 인증 프록시 서버-에의 액세스에 필요한 사용자명과 패스워드의 입력이 요구됩니다. 사용자명과 패스워드는, 현재 호출하고 있는 Java Web Start 에 보존됩니다. 다만, 시큐리티가 있는 Web 사이트에 액세스 해 새롭고 Java 가상 머신을 호출했을 경우, 사용자명과 패스워드는 Java 가상 머신의 인스턴스에 보존되고 있기 (위해)때문에, 이 정보가 요구됩니다.

또, Java Web Start 의 「설정」패널을 사용해, 프록시 설정을 표시 또는 편집할 수도 있습니다. 데스크탑의 아이콘을 클릭하는지 (Microsoft Windows), Java Web Start 인스톨 디렉토리에서 . /javaws 를 입력해 (Solaris 오퍼레이팅(operating) 환경, Linux) 어플리케이션 매니저를 기동해, 「Edit/Preferences」를 선택합니다. 프록시 서버-를 사용해 Web 에 액세스 하는 환경에 있는 경우, Java Web Start 의 「설정」패널을 사용해, 올바르게 설정되어 있는지 어떤지를 확인하는 것을 추천합니다.
