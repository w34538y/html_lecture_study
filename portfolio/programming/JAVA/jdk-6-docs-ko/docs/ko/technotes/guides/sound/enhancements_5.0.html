JDKTM 5.0 에서의 JavaTM Sound 의 확장 기능

JDK 5.0 에서는, 다음의 확장을 하고 있습니다.

  • 포트는 모든 플랫폼에서 사용 가능합니다 (RFE 4782900).
  • MIDI 디바이스의 입출력은, 모든 플랫폼에서 사용 가능합니다 (RFE 4812168, 4782924).
  • 최적화된 직접 오디오 액세스가 모든 플랫폼에 구현되고 있습니다 (RFE 4908240 및 4908879). 이것은, 네이티브 믹싱을 제공하고 있는 시스템 (하드웨어 믹싱을 사용하는 Linux ALSA, 유효하게 되어 있는 Solaris Mixer, Windows DirectSound)으로 디폴트로 유효하게 되어 있습니다.
  • 새로운 리얼타임의 시퀀서는 모든 MIDI 디바이스와 함께 동작해, 무제한의 트랜스미터가 허가되고 있습니다 (RFE 4773012).
  • sound.properties 구성 파일을 사용하면(자), 디폴트의 디바이스를 선택할 수 있습니다 (RFE 4776511). 자세한 것은,MidiSystem 및 AudioSystem 를 참조해 주세요.
  • MidiDevice 는 접속되고 있는 리시버 및 트랜스미터에 문의할 수가 있습니다 (RFE 4931387, 메소드 MidiDevice.getReceiver 및 MidiDevice.getTransmitter).
  • AudioFormat,AudioFileFormat, 및 MidiFileFormat 의 프로퍼티을 사용하면(자), 포맷에 기술이나 수식을 추가할 수 있습니다 (RFE 4925767 및 4666845).
  • 간단하게 이용할 수 있는 일련의 메소드를 사용하면(자), AudioSystem 로부터 라인을 간단하게 취득할 수 있습니다 (RFE 4896221).
  • MIDI 순서의 특정의 부분에서 심리스인 루프를 실시하기 위해서(때문에), 시퀀서의 인터페이스는 루프 메소드로 확장되고 있습니다 (RFE 4204105).
  • Java Sound 에 의해 VM 의 종료가 방해되지 않게 되었습니다 (버그 4735740).
