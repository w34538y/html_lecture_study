기본 정보

Q:Java Plug-in 소프트웨어란 무엇입니까.

A:Java Plug-in 소프트웨어는, Sun Microsystems, Inc. 하지만 제공하는 소프트웨어 제품으로, 엔터프라이즈 Web 매니저가, Sun 의 Java Runtime Environment (JRE)를 사용해, 인트라넷 Web 페이지상의 Java 애플릿 및 JavaBeansTM 컴퍼넌트를 실행하는 것을 가능하게 합니다. Java Plug-in 의 릴리스 1.4 에서는, 종래의 APPLET 태그가 Java 애플릿에 Sun 의 JRE 를 사용하도록 지시할 수도 있습니다. 이것은 특히, Internet Explorer 6 등 Java 를 지원하고 있지 않는 브라우저에 대해 유효합니다.

Q:Java Plug-in 소프트웨어는 어떠한 사용자를 대상으로 하고 있습니까.

A:Java Plug-in 소프트웨어의 대상 사용자는, Java SDK, Standard Edition 에 근거하는 애플릿을 인트라넷 Web 페이지에 배치해, 기업내에서 Microsoft Windows, Solaris, 및 Linux 베이스의 브라우저를 지원하는 기업 커스터머입니다. 릴리스 1.4 에서는, Java 애플릿을 Web 페이지상에서 실행하는 소비자도 대상으로 해 설계되고 있습니다.

Q:입수 가능한 Java Plug-in 소프트웨어의 문서는 있습니까.

A:Java Plug-in 소프트웨어의 배치를 지원하기 위해(때문에), Sun 는 다양한 기술 문서를 제공하고 있습니다. HTML 스펙에 관한 문서, 인트라넷 환경에서의 Java Plug-in 소프트웨어의 사용, 프록시 구성의 동작, 애플릿의 스크립트 작성 방법 등,「Java Plug-in software document 페이지」로부터 많은 정보를 입수할 수 있습니다. 종래의 APPLET 태그의 지원를 사용하는 소비자용의 제품에 대해서는,Get Java Technology 의 페이지를 열어, 거기로부터 문서를 입수하면(자) 좋을 것입니다.

Q:Java Plug-in 소프트웨어에는 어떤 기능이 있습니까.

A:Java Plug-in 소프트웨어는, Internet Explorer 또는 Netscape Navigator 의 사용자에 대해서, 얼마든지의 중요한 기능을 제공합니다.

  • Java SDK, Standard Edition 의 풀 버젼
  • Java CompatibleTM 의 풀 지원
  • 장래에도 대응한 아키텍쳐(architecture)
  • 프리 다운로드 및 간단한 인스톨 방법
  • 프리의 Java Plug-in 소프트웨어

Q:브라우저의 디폴트 JavaTM 가상 머신은 아니고, Java Plug-in 소프트웨어를 다운로드해 사용하는 이점은 무엇입니까.

A:Java Plug-in 소프트웨어를 사용하면(자), 기업이나 소비자에게는 이하와 같은 이점이 있습니다.

  1. JavaBeans, JNITM, 및 RMI 등의 Java SDK, Standard Edition 의 기능을 곧바로 최대한 활용할 수 있습니다.
  2. Internet Explorer 및 Netscape Navigator 브라우저로 가동하는,100% Pure JavaSM 애플릿을 개발 및 배치할 수 있어 게다가 어느 쪽의 브라우저에서도 신뢰성이 높고, 일관한 동작이 가능하게 됩니다.
  3. Internet Explorer 및 Navigator 로, Sun 의 Java 플랫폼의 최신 릴리스의 지원 (고성능의 Java HotSpotTM 가상 머신을 포함한다)를, Sun 에 의한 릴리스 다음에 올 뿐(만큼) 빨리, 확실히 받을 수가 있습니다.
  4. Java 를 이제 지원하고 있지 않는 Web 브라우저로 애플릿을 실행합니다.

Q:Java Software Development Kit (SDK), Standard Edition 에 Java Plug-in 소프트웨어는 동고 되고 있습니까.

A:네.

Q:Java Plug-in 소프트웨어는, Microsoft Windows 나 Solaris 플랫폼 이외 (Mac OS, AIX, Linux, HP-UX 등)의 IE 나 Navigator 에서도 동작합니까.

A: Sun 는, 모든 operating system 프로바이더에 대해, Java Plug-in 소프트웨어를 이식용으로 제공하고 있습니다. 다른 operating system에서의 Java Plug-in 소프트웨어 지원의 자세한 것은, operating system 프로바이더에 확인해 주십시오.

Q:버그를 http://java.sun.com/cgi-bin/bugreport.cgi 에 보고하는 경우, 어떤 정보를 포함할 수가 있습니까.

A:Java Plug-in 소프트웨어의 버그를 보고하는 경우, 다음의 정보를 항상 포함해 주세요.

  1. operating system (버젼 번호를 포함한다)
  2. Web 브라우저 (버젼 번호를 포함한다)
  3. JavaTM 콘솔 윈도우의 완전한 출력
  4. 네트워크 구성 정보 (프록시, 특수한 인트라넷 환경 등)
  5. 조우한 장해나 문제의 상세한 설명. 문제를 재현 하는데 필요한 순서도 포함해 주세요.

지원되는 기능

Q:Java Plug-in 소프트웨어는, Java SDK, Standard Edition 의 기능을 모두 지원하고 있습니까.

A:네. Java Plug-in 소프트웨어는, 사용자가 Sun 의 Web 사이트로부터 현재 다운로드 가능한 JRE 를 사용합니다 ( http://java.sun.com/j2se/ 를 참조).

Q:Java Plug-in 소프트웨어는, Java SDK, Standard Edition 용의 JCK (Java Compatibility Kit) 테스트 스위트하게 완전하게 준거하고 있습니까.

A:Java Plug-in 소프트웨어는, Java SDK, Standard Edition 용의 JCK 테스트 스위트하게 완전하게 준거하고 있습니다.

Q:Java Plug-in 를 지원하는 브라우저나 operating system에는 어떠한 것이 있습니까.  

A:「지원되는 operating system와 브라우저」를 참조해 주세요.

Q:Java Plug-in 소프트웨어는, 스크립트를 지원합니까.  

A:Java Plug-in 소프트웨어는, Internet Explorer와 Netscape 6에서의 애플릿의 스크립트를 지원합니다. 다만, Navigator 4 로 Plug-in 소프트웨어를 사용하는 경우, 스크립트는 지원되지 않습니다.

Q:Java Plug-in 소프트웨어는, 서명 첨부 애플릿을 지원합니까.

A:지원합니다. Java Plug-in 소프트웨어는, 표준 Java SDK, Standard Edition 의 서명 첨부 JAR, 및 Netscape 서명 툴을 사용해 서명된 애플릿을 지원합니다. 자세한 것은,「RSA 서명 첨부 증명서를 사용한 애플릿의 서명 방법」 을 참조해 주세요.

Q:Java Plug-in 소프트웨어는, 브라우저가 사용하고 있던 프록시 서버-구성을 자동적으로 인식 및 사용합니까.

A:합니다. 릴리스 1.4 로부터는 실제로 3 개의 옵션이 제공되고 있습니다. 자세한 것은,「프록시 구성」을 참조해 주세요.

Q:Java Plug-in 소프트웨어는, 프록시로 어떤 프로토콜을 지원합니까.

A:HTTP, HTTPS, FTP, Gopher, 및 SOCKS 를 지원합니다.

Q:Java Plug-in 소프트웨어는, SSL 를 지원합니까.

A:네. SSL 는 Java Secure Socket Extension (JSSE)를 통해서 지원되고 있습니다.

Q:브라우저의 기동시에 Java Plug-in 소프트웨어를 사전에 로드하는 방법이 있습니까.

A:Internet Explorer 및 Netscape Navigator 에는, 기동시에 Java Plug-in 소프트웨어를 사전 로드하는 기구는 존재하지 않습니다. 다만, Java Plug-in 소프트웨어를 사용해 더미의 애플릿을 기동하는, 독자적인 기동 페이지를 제공하는 것으로, 이 제한을 회피할 수 있습니다.

Q:Java Plug-in 가 Sun 의 Web 사이트로부터는 아니고, 인트라넷의 Web 서버 (방화벽(fire wall) 내부)로부터 다운로드되도록(듯이), 설정할 수 있습니까.  

A:네. 자세한 것은, 「 제 II 부: 배치 schema」의「OBJECT/EMBED 태그에 의한 인트라넷」을 참조해 주세요.

Q:Microsoft Windows 로 Java Plug-in 컨트롤 패널을 여는 방법을 가르쳐 주세요.  

A:[스타트] 메뉴로,[스타트] > [설정] > [컨트롤 패널] 을 선택해, Java Plug-in 라고 하는 라벨이 붙은 Java Coffee Cup 의 아이콘을 클릭합니다.

버젼 정보

Q:인스톨 끝난 Java Plug-in 소프트웨어의 버젼을 아는 방법을 가르쳐 주세요.

A:2 개의 방법이 있습니다.

  • C:\Windows\Downloaded Program Files\Java Runtime Environment 또는 C:\WINNT\Downloaded Program Files\ 로 버젼 번호를 확인합니다.
  • Netscape Navigator 를 엽니다. [헬프] > [Plug-ins 에 대해] 메뉴를 선택합니다. Java Plug-in 아래에 표시되는 MIME 타입을 확인합니다. 확인 방법을 다음에 나타냅니다.

    application/x-java-applet;version=1. 4 의 경우, Java Plug-in 소프트웨어 1.4

    application/x-java-applet;version=1. 3 의 경우, Java Plug-in 소프트웨어 1.3

    application/x-java-applet;version=1. 2 의 경우, Java Plug-in 소프트웨어 1.2

    application/x-java-applet;version=1. 1.2 의 경우, Java Plug-in 소프트웨어 1.1. 2

    application/x-java-applet;version=1. 1.1 의 경우, Java Plug-in 소프트웨어 1.1. 1

    application/x-java-applet;version=1. 1 의 경우, Java Plug-in 소프트웨어 1.1

    주:복수 버젼의 Java Plug-in 가 인스톨 되고 있는 경우, 복수 표시되는 일이 있습니다.

Q:버젼이 다른 복수의 Java Plug-in 소프트웨어를 1 개의 시스템에 공존시킬 수 있습니까.

A:할 수 있습니다. 복수 버젼의 지원를 통해 공존시킬 수가 있습니다. 자세한 것은,「복수 버젼의 지원」를 참조해 주세요.
