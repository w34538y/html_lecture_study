Java IDL 입문:캐릭터 라인화 된 객체 참조의 사용

이 문서는 J2SETM 릴리스용으로 갱신되고 있지 않습니다. 이 문서를 J2SETM 문서 다운로드 번들의 일부로서 입수했을 경우는, Web 사이트 http://java.sun.com/j2se/1. 5.0/docs/guide/idl/tutorial/GSstring.html 로 갱신판을 확인해 주세요.

클라이언트 어플리케이션이 CORBA 객체상의 오퍼레이션을 호출하려면 , 그 객체에의 참조가 필요합니다. 이 참조는 다양한 방법으로 취득할 수 있습니다. 예를 들어,ORB.resolve_initial_references() 를 호출하는 방법이나, 네임 서비스등의 다른 CORBA 객체를 사용하는 방법이 있습니다. 이 튜토리얼의 여기까지시에에서는, 이 양쪽 모두의 방법을 사용해 초기 객체 참조를 취득했습니다.

그러나, 분산환경으로 이용할 수 있는 네임 서비스가 없는 것이 자주 있습니다. 이 경우는, CORBA 클라이언트는「캐릭터 라인화 되었다」객체 참조를 사용해 최초의 객체를 검색합니다.

갱신 후의 이 레슨에서는, 서버의 기동 순서의 일부로서 캐릭터 라인화 된 객체 참조를 작성하는 방법, 및 클라이언트가 참조를 취득해 실제의 객체 참조로서 사용하기 위해서(때문에) 캐릭터 라인화를 해제하는 방법을 배웁니다.
