속성의 카테고리와 값

각 프린터에는, 다양한 용지 사이즈로 인쇄하거나 인쇄 부수를 지정하는 등, 다양한 기능이 부속되어 있습니다. 이러한 기능으로 지정하는 값에는, 각각 일정한 범위가 있습니다. 예를 들어, 프린터의 인쇄 방향을 지정하는 기능에는, 「landscape」또는 「portrait」를 지정합니다. 인쇄 요구를 송신할 때는, 값의 범위의 어느쪽이든을 설정합니다. Java 인쇄 서비스 API 에서는, 인쇄 방향 등, 프린터 기능에는 「속성 카테고리」라고 하는 용어를 사용합니다. 또, 「landscape」 등, 기능의 값에는 「속성치」라고 하는 용어를 사용합니다.

Java 인쇄 서비스 API 에서는, 속성 카테고리는 Attribute 인터페이스를 구현하고 있는 Java 클래스를 표현해, 속성치는 그러한 Java 클래스 또는 그 서브 클래스의 인스턴스를 나타냅니다. 예를 들어, 특정의 작업을 5 부 인쇄하는 경우는, 어플리케이션상에서 Copies 클래스의 인스턴스를 작성할 경우에 값을 5 로 설정해, 이 Copies 인스턴스를 사용해 인쇄 작업을 지정합니다. Copies 클래스는 속성 카테고리를 표현해, Copies 인스턴스는 속성치를 표현합니다. 속성치는, Java 객체로서 캡슐화됩니다. 속성치는, 속성 카테고리를 암묵적으로 가리키기 (위해)때문에, 일반적으로은 속성으로 불립니다. 카테고리와 값은, 키와 값의 페어로 구성됩니다. 키와 값의 페어는, 다른 속성과 함께 속성 세트에 추가할 수 있습니다. 상세한 것에 대하여는,「속성세트」를 참조해 주세요.

Attribute 인터페이스에는, getName 및 getCategory 의 2 개의 메소드만을 정의합니다. getName 메소드는, 속성 카테고리의 String 명을 돌려줍니다. getCategory 메소드는, 그 카테고리를 최초로 구현하는 클래스를 돌려줍니다. 특정의 카테고리의 모든 구현이 같은 이름과 클래스를 돌려주도록(듯이) 하기 위해서, 서브 클래스가 getName 또는 getCategory 메소드를 오버라이드(override) 하지 않게 해 주세요.

특정의 프린터로 지원되는 속성 카테고리와 값을 조사하려면 , PrintService 인터페이스에 정의되고 있는 조회 메소드를 사용합니다. 인쇄 요구에는, 목적의 프린터로 지원되는 모든 속성 카테고리의 속성치를 지정할 필요는 없습니다. 지원되는 각 속성 카테고리에는, 디폴트 값가 설정되어 있습니다. 예를 들어, 특정의 프린터의 최대 인쇄 부수가 999 부이다고 합니다. 인쇄 요구에 부수를 지정하지 않았던 경우는, 디폴트의 1 부만이 인쇄됩니다.
