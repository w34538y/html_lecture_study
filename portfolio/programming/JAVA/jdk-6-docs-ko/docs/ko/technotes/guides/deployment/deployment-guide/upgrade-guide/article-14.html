java.lang.InstantiationException:「code」와「object」의 양쪽 모두는 아니고, 어느 쪽인지 다른 한쪽만을 지정한다.


증상

애플릿을 Sun JRE 를 사용한 브라우저로 실행하고 있는 경우,InstantiationException 가 throw 됩니다. 같은 애플릿이 Microsoft VM 에서는 실행할 수 있습니다.

원인

이하의 2 개의 원인이 있습니다.

  1. code 와 object 속성의 양쪽 모두를 <APPLET> 태그로 지정하고 있습니다.

    <APPLET code=MyApplet object=MyApplet.ser width=100 height=100>
    </APPLET>

    Sun JRE 는 code 또는 object 어느 쪽인지 다른 한쪽의 속성에 액세스 할 수 있습니다만, 양쪽 모두에는 액세스 할 수 없습니다.
     
  2. code 속성이 <APPLET> 태그로 지정되어object 속성이 <PARAM> 태그로 지정되고 있습니다.

    <APPLET code=MyApplet width=100 height=100>
    <PARAM name="object" value="someValue">
    </APPLET>

    public class MyApplet extends java.applet.Applet
    {
            public void init()
            {
                   String value = getParameter("object");
            }
            ....
    }

    Sun JRE 는 파라미터명으로서 object 를 지원하지 않습니다.

해결 방법

        최초의 예의 경우는,code 속성을 <APPLET> 태그로부터 삭제합니다.

             <APPLET object=MyApplet.ser width=100 height=100>
             </APPLET>

        2 번째의 예의 경우는, 파라미터명을 다른 이름으로 변경합니다.

             <APPLET code=MyApplet width=100 height=100>
             <PARAM name="property1" value="someValue">
             </APPLET>

             public class MyApplet extends java.applet.Applet
             {
                 public void init()
                 {
                         String value = getParameter("property1");
                 }
                 ....
             }
