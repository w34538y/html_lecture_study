Web 사이트의 설정


처음에

Java Web Start 는, HTTP 프로토콜이나 Web 서버라고 하는 기존의 인터넷 테크놀러지를 이용합니다. 이 때문에, Java Web Start 를 사용해 Java 테크놀러지 베이스의 어플리케이션을 배치할 때에, HTML 베이스의 컨텐츠를 배치하는 기존 인프라스트럭쳐(infrastructure)를 재사용할 수가 있습니다.

어플리케이션을 클라이언트 머신에 배치할 수 있도록(듯이) 하려면 , 어플리케이션을 구성하는 모든 파일에 Web 서버로부터 액세스 할 수 있을 필요가 있습니다. 이것은 일반적으로, 1 개(살) 또는 복수의 JAR 파일과 1 개의 JNLP 파일을, Web 서버의 디렉토리내에 카피하는 것을 의미합니다. Web 사이트상에서 Java Web Start 를 지원하기 위해서 필요한 설정은, HTML 베이스의 컨텐츠를 배치할 때의 설정과 매우 자주(잘) 닮았습니다. 다만 1 개(살) 주의가 필요한 것은, 새로운 MIME 타입을 Web 서버로 설정할 필요가 있는 점입니다.

기본적인 순서

1. Java Web Start 의 MIME 타입을 사용할 수 있도록(듯이) Web 서버를 설정한다

Web 서버를 설정해, 확장자(extension) . jnlp 를 가지는 모든 파일이 application/x-java-jnlp-file MIME 타입으로서 설정되도록(듯이) 합니다.

대부분의 Web 브라우저는, Web 서버로부터 컨텐츠와 함께 보내져 오는 MIME 타입에 근거해, 각 컨텐츠의 처리 방법을 판단합니다. 서버가 JNLP 파일에 대해서 application/x-java-jnlp-file MIME 타입을 돌려주지 않았던 경우, Java Web Start 는 기동되지 않습니다.

MIME 타입을 추가하는 방법은, Web 서버 마다 다릅니다. 예를 들어, Apache Web 서버의 경우,. mime.types 설정 파일에 다음의 1 행을 추가할 필요가 있습니다.

application/x-java-jnlp-file JNLP

사용의 Web 서버에서의 방법에 대해서는, 부속의 메뉴얼을 참조해 주세요.

2. 어플리케이션에 대한 JNLP 파일을 작성한다

이 파일을 작성하는 가장 간단한 방법은, 기존의 JNLP 파일을 필요에 따라서 변경하는 것입니다.

JNLP 파일의 구문과 형식에 대해서는, 나머지의섹션으로 설명합니다.

3. Web 서버상에서 어플리케이션을 액세스 할 수 있도록(듯이) 한다

JNLP 파일내에 기재된 URL 를 사용해, 어플리케이션의 JAR 파일과 JNLP 파일에 액세스 할 수 있는 것을 확인해 주세요.
