사일런트 인스톨

이 마디에서는, Windows 32 비트 플랫폼에 있어서의 Java Runtime Environment (JRE)의 사일런트 인스톨을 설명합니다. 대상이 되는 독자는 다음과 같습니다.

  • 인트라넷에 배치된 복수의 PC 에, 사용자에 의한 조작없이 Java Plug-in 및 JRE 를 배치하는 시스템 관리 책임자
  • JRE 를 필요로 하는 제품을 보유하고 있는 기업. 이러한 기업의 제품에서는, JRE 를 사일런트 인스톨 할 수 있습니다.

사일런트 인스톨의 마디에서는, 다음의 토픽에 대해 설명합니다.

  • 처음에
  • 사일런트 모드에서의 인스톨
  • 사일런트 모드에서의 언인스톨
  • 로그 파일의 작성

처음에

JavaTM SE Runtime Environment (JRE) 인스톨은, Microsoft Window Installer (MSI) 2.0 테크놀러지를 사용해 구축되고 있습니다. MSI 에는, 사일런트 (자동적인) 인스톨 지원가 짜넣어지고 있습니다. 이 문서에서는, MSI 인스톨을 실행한다 . exe 를 사용해 JRE 를 사일런트 인스톨 하는 방법에 대해 설명합니다.

사일런트 모드에서의 인스톨

Java SE 6 에서는 Windows 의 오프 라인 인스톨 커멘드는, Windows 인스톨의 구문과 같습니다. 커멘드는 다음과 같습니다. 

 <jre>.exe   [/lang=<language ID>] /s [ADDLOCAL=jrecore[, extra][, other_US] | ALL] 
      		[IEXPLORER=1] [MOZILLA=1] [INSTALLDIR=<drive>:\<install_path>] 
             [REBOOT=Suppress]

이 예에서는,

  • <jre>.exe 는, Java Runtime Environment (JRE)의 단일의 실행 가능 인스톨러입니다.
  • /lang=<language ID> 를 지정하면(자), 인스톨러에 로컬라이즈 된 언어를 설정합니다 (「언어 ID」 를 참조).
  • ADDLOCAL 를 지정하는 경우는,jrecore[, extra][, other_US] 또는 ALL 의 어딘가에 됩니다.
  • IEXPLORER=1 를 지정하면(자), Plug-in 가 Internet Explorer 브라우저에 등록됩니다.
  • MOZILLA=1 를 지정하면(자), Plug-in 가 Mozilla 1.1 이후 및 Netscape 의 브라우저에 등록됩니다 (5.0 보다 전의 릴리스용).
  • INSTALLDIR 를 지정하면(자), 인스톨 하는 드라이브와 패스가 지정됩니다.
  • REBOOT=Suppress 를 지정하면(자), 락 된 파일이 있는 경우는 컴퓨터를 재기동하지 않습니다.
  • ADDLOCAL=ALL 를 지정하면(자), 모든 기능이 인스톨 됩니다.
  • ADDLOCAL=jrecore[, extra][, other_US] 에서는, 다음의 옵션을 지정할 수 있습니다.
    • jrecore 는 JRE 의 코어가 인스톨 됩니다
    • extra (생략 가능)는, 추가의 폰트, 칼라, 사운드 뱅크가 인스톨 됩니다
    • other (생략 가능)는 로케일 고유의 . jar 파일이 인스톨 된다
  • ADDLOCAL 를 지정하지 않는 경우, 다음의 구성만이 인스톨 됩니다.
    • jrecore 가 인스톨 됩니다
    • extra 가 인스톨 됩니다
    • 영어 이외의 로컬라이즈 지원가 인스톨 되고 있는 경우에만,other 가 인스톨 됩니다.
  • INSTALLDIR 를 지정하지 않는 경우,C:\Program Files\java\jre1. 6.0 (디폴트의 위치)에 인스톨 됩니다.

주:

커멘드에서는 대문자와 소문자를 구별합니다. 또 ADDLOCAL 로 기능을 지정하는 경우는 공백 문자를 포함하지 않습니다.

인스톨 구성

프로그램예 1

JRE 인스톨러가 jre-6-rc-windows-i586.exe 로, 다음의 구성을 인스톨 한다고 합니다.

  • Windows 인스톨을 실행한다
  • JRE 코어, 추가의 폰트, 칼라, 사운드 뱅크를 인스톨 한다
  • Netscape 7 및 Mozilla 1.3 의 Plug-in 를 등록한다

상기의 구성을 인스톨 하는 커멘드는 다음과 같습니다.

jre-6-rc-windows-i586.exe /s ADDLOCAL=jrecore, extra MOZILLA=1

프로그램예 2

JRE 인스톨러가 jre-6-rc-windows-i586.exe 로, 다음의 구성을 인스톨 한다고 합니다.

  • Windows 오프 라인 인스톨을 실행해,D 드라이브의 java\jre 에 JRE 를 인스톨 한다
  • JRE 의 모든 기능을 인스톨 한다
  • Internet Explorer 에 Java Plug-in 를 등록한다

상기의 구성을 인스톨 하는 커멘드는 다음과 같습니다.

jre-6-rc-windows-i586.exe /s ADDLOCAL=ALL IEXPLORER=1　INSTALLDIR="D:\java\jre"

주:

JRE 의 인스톨이 완료할 때까지 MS-DOS 윈도우를 연 채로 하려면 ,start /w 커멘드를 다음과 같이 사용합니다.

start /w jre-6-rc-windows-i586.exe /s ADDLOCAL=jrecore, extra MOZILLA=1

사일런트 모드에서의 언인스톨

Implementation-Version n1. n2. n3_n4n5 를 사일런트 모드로 언인스톨하는 커멘드행은 다음과 같이 됩니다.

msiexec.exe /qn /x {3248F0A8-6813-11D6-A77B-00B0D0n1n2n3n4n50}

설명

  1. 이 커멘드는 어느 위치로부터에서도 실행 가능합니다.
  2. msiexec.exe 의 실행 가능 파일은, Windows 의 시스템 디렉토리에 포함되고 있습니다.
  3. 인스톨로 설명한 REBOOT=Suppress 옵션을 언인스톨 커멘드에서도 사용할 수 있습니다.
  4. 버젼의 서식에 대해 자세한 것은,「부록 2:Sun 의 지원하는 Specification-Version 및 Implementation-Version 의 형식」을 참조해 주세요.

버젼 1.6. 0 의 예

msiexec.exe /qn /x {3248F0A8-6813-11D6-A77B-00B0D0160000}

버젼 1.6. 0_02 에서의 예

msiexec.exe /qn /x {3248F0A8-6813-11D6-A77B-00B0D0151020}

로그 파일의 작성

로그 파일은, 인스톨 또는 언인스톨이 정상적으로 종료했는지 어떠했는지의 확인에 사용합니다. 인스톨 또는 언인스톨에 대한 로그 파일을 작성하는 경우는, 인스톨이나 언인스톨의 커멘드의 말미에 /L C:\<path>setup.log 를 추가합니다. 확인하려면 , 로그 파일의 말미까지 스크롤 합니다.

인스톨의 예

다음은 인스톨의 예입니다.

jre-6-rc-windows-i586.exe /s /L C:\<path>setup.log

상기의 커멘드에 의해, 로그가 setup.log 파일에 기록됩니다.

언인스톨의 예

다음은 언인스톨의 예입니다.

msiexec.exe /qn /x {3248F0A8-6813-11D6-A77B-00B0D0160000} /L C:\<path>setup.log
