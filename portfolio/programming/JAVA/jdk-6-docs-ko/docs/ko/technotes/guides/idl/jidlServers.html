여기에서는, Java IDL 에 의한, 일시 CORBA 객체 및 지속 CORBA 객체의 서버 지원에 대해 설명합니다.

일시 객체와 지속 객체

지속 CORBA 객체는, 명시적으로 파기될 때까지 존재합니다. 클라이언트가 지속 CORBA 객체를 참조하고 있는 경우, 그 객체의 서버가 실행중이 아니라도 참조는 사용 가능합니다. ORB 가 객체의 호출을 받으면(자), ORB demon (ORDB)가 서버를 기동하기 때문입니다.

J2SE 1.4 로부터, Java IDL 에 휴대용 객체 어댑터 (POA), Object Request Broker Daemon (ORBD), 및 servertool 가 포함되게 되었습니다. 이러한 기능을 사용하면(자), Java IDL 를 사용해,지속 객체 서버를 기술할 수 있습니다.

대조적으로,일시 CORBA 객체의 라이프 타임은, 객체를 작성한 서버 프로세스의 실행 기간과 같습니다. 서버가 정지하면(자), 그 일시 객체도 소실해, 클라이언트로부터의 객체 참조는 모두 무효가 됩니다. 일시 객체는, 어플리케이션과 객체간의 비동기 처리 및 콜백 통신에 사용됩니다. 예를 들어, 타이밍 서비스가 인터벌의 기한이 지난 것을 클라이언트 어플리케이션에 통지하는 경우, 이하의 순서가 실행됩니다.

  • 타이밍 서비스는,interval_expired() 오퍼레이션을 포함한 통지 객체의 인터페이스를 정의한다

  • 타이밍 서비스는, 클라이언트가 타이밍 서비스의 set_interval() 오퍼레이션에 통지 객체를 건네주는 것을 요구한다

  • 타이밍 서비스의 클라이언트 어플리케이션은, 통지 인터페이스를 구현한 일시 객체를 구현한다

  • 클라이언트 어플리케이션은, 통지 객체를 작성해, 타이밍 서비스에 건네준다

  • 인터벌의 기한이 지나면(자), 타이밍 서비스는 통지 객체의 interval_expired() 오퍼레이션을 호출한다

이 처리 방법은, 수속형 프로그래밍의 콜백을 닮아 있습니다.
