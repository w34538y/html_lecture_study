Java 플랫폼상에서의 인쇄의 역사

Java 플랫폼에 대한 기본적인 인쇄 지원는, 1997 년의 Java Development Kit, version 1.1 으로 처음으로 도입되었습니다. JDK 1.1 의 인쇄 API 는, 클라이언트 어플리케이션으로부터 사용자 인터페이스의 내용을 인쇄하기 위한 , 기본적인 시스템를 개발자에게 제공했습니다. JDK 1.1 의 인쇄 API 는, AWT 인쇄 API 라고도 불려 인쇄 요구가 캡슐화된 java.awt.PrintJob 클래스를 중심으로 설계되었습니다. PrintJob 클래스는, Graphics 의 서브 클래스를 작성합니다. 이 서브 클래스에는, 페이지를 draw하기 위한 렌더링 호출이 구현되고 있습니다.

1998 년의 JDK 1.2 에서는, java.awt.print 패키지를 도입해, Java 플랫폼상에서의 인쇄 성능이 향상했습니다. 2D 그래픽스, 텍스트, 및 이미지 등, 모든 Java 2D 그래픽스를 어플리케이션으로부터 인쇄할 수 있게 되었습니다.

JDK version 1.3 에서는, JobAttributes 및 PageAttributes 클래스가 AWT 인쇄에 도입되어 클라이언트 어플리케이션으로부터 인쇄 작업의 프로퍼티과 페이지의 속성을 지정할 수 있게 되었습니다.

이 2 개의 API 의 주요한 기능은, 페이지의 draw입니다. 즉, 이러한 API 는, 페이지의 렌더링과 서식 설정을 실시하는, 인쇄 하부조직의 컴퍼넌트의 1 개입니다. 클라이언트에서는, 인쇄 다이얼로그로부터 프린터를 선택해, JobAttributes 및 PageAttributes 를 사용해 인쇄 속성을 지정할 수가 있습니다. 다만, AWT 및 Java 2D 인쇄 API 에서는, 프린터의 기능에 근거해 프린터를 검색할 수 없습니다. 이 검색 기능은, 다른 인쇄 하부조직에서는, 다른 주요 컴퍼넌트에 할당할 수 있고 있습니다. Java 플랫폼상에서의 인쇄의 역사에 대해서는,「Printing on the Java Platform」를 참조해 주세요.
