Jar 및 Zip 의 확장 기능
이 페이지는, JDK 6 으로 지원되는 jar 및 zip 의 확장 기능을 집계한 것입니다.

API 의 변경

다음의 2 개의 새로운 압축 스트림이 추가되었습니다.

  • java.util.zip.DeflaterInputStream:이 스트림로부터 읽어내진 데이터는 압축됩니다.
  • java.util.zip.InflaterOutputStream:이 스트림에 기입해진 데이터는 압축 해제됩니다.
예를 들어, 클라이언트가 네트워크 경유로 압축 데이터를 송신하는 경우에, 이러한 스트림이 도움이 됩니다. 데이터는 DeflaterInputStream 로 패킷에 압축되어 그 패킷이 네트워크 접속상에 기입해집니다. 수신 단말은 네트워크 접속으로부터 압축 패킷을 읽어내,InflaterOutputStream 에 기입하는 것으로, 그 패킷을 압축 해제합니다.

구현의 변경점

모든 플랫폼에서 zip 파일의 용량은 64k 엔트리 이상입니다.

Windows 에서는 일부의 제한이 해제되었습니다.

  • 256 문자보다 긴 파일명도 지원됩니다.
  • 동시에 열 수 있는 zip 파일을 2,000 나머지로 하는 제한이 해제되었습니다.

jar 커멘드의 변경점

jar 커멘드의 동작에 대한 변경점은 다음의 2 개입니다.

  • 추출된 파일의 타임 스탬프는, 추출 시각은 아니고, 어카이브(archive)에 기록되고 있는 타임 스탬프입니다.
  • jar 를 작성하는 경우, 실행 가능 jar 파일에 번들 된 스탠드얼론형 어플리케이션의 엔트리 포인트를 지정할 수가 있습니다. 「e」옵션에서는, jar 파일의 Manifest의 Main-Class 속성치를 작성 혹은 오버라이드(override) 하는 것에 의해, 어플리케이션의 엔트리 포인트를 지정합니다.
