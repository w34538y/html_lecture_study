Java Accessibility 의 개요

 문서의 목차

Java Accessibility API

Java Accessibility API (JAAPI)는, Java Foundation Classes (JFC)의 코어 부분의 일부입니다. JFC 는, 그래피컬 사용자 인터페이스 컴퍼넌트의 통합 세트로, 인터넷, 인트라넷, 및 데스크탑 어플리케이션의 배치를 용이하게 하기 위해서 설계된 기간 서비스입니다. JAAPI 를 사용하면(자), GUI 컴퍼넌트의 정보를 사용자 보조 기능으로부터 이용할 수 있게 되기 (위해)때문에, 사용자는 Java 어플리케이션을 다른 방법으로 표현 및 제어할 수 있습니다.

Java Accessibility 유틸리티

Java Accessibility 유틸리티는, 사용자 보조 기술에 의한 Java Accessibility API 의 이용을 지원하는 기능입니다. Java Accessibility 유틸리티는, UI 컴퍼넌트에 관련한 이벤트를 감시합니다. 또, 사용자 보조 기능이, GUI 에 관한 보완적인 정보 (마우스의 현재 위치, 현재 포커스를 보관 유지하고 있는 윈도우등)를 취득하는 것을 지원합니다.

이 유틸리티는, JFC (와)는 별개에 릴리스 되고 있습니다만,JFC 홈 페이지로부터 다운로드할 수 있습니다.

플러그 인 가능한 Look & Feel

JFC (Java Foundation Classes)에 의해, 플러그 인 가능한 Look & Feel 아키텍쳐(architecture)가 구현됩니다. Look & Feel 아키텍쳐(architecture)를 이용하면(자), 시각 표시되지 않는 사용자 인터페이스를, 특정의 어플리케이션의 시각 표시에 치환하거나 확장하거나 할 수가 있습니다. 사용자 인터페이스의 표시는, 각 컴퍼넌트의 기본적인 구조 및 데이터에는 의존하지 않습니다. 이 비의존성은, 컴퍼넌트의 사용자 인터페이스를 그 모델로부터 독립시키는 것으로 실현됩니다. 컴퍼넌트의 모델과는, 사용자 인터페이스에 의해 사용자에게 표시되는 상태와 정보가 캡슐화된 구조입니다. 이 아키텍쳐(architecture)의 자세한 것은,「The Swing Connection」를 참조해 주세요.
