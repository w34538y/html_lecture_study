버젼 5.0 에서의 확장 기능

NameClassPair 클래스의 절대 나토리 이득의 지원
이 기능에 의해, SearchResult 로부터 LDAP 식별명을 취득할 수 있습니다. 이 기능을 지원하기 위해서, javax.naming.NameClassPair 메소드에 다음의 2 개의 새로운 메소드가 추가되고 있습니다.
  •  getNameInNamespace()
  •  setNameInNamespace()

표준 LDAP 컨트롤의 지원
필요한 LDAP 컨트롤은, 벌써, JNDI/LDAP 서비스 프로바이더가 이용 가능한 LDAP Booster Pack 확장 기능 패키지내에서 지원되고 있습니다. LDAP 컨트롤은, 최근 RFC 로 표준화 되었습니다. 이것들은 J2SE 의 javax.naming.ldap 패키지의 다음의 클래스를 통해서 이용할 수 있습니다.
  • ManageReferralControl (RFC 3296 )
  • PagedResultsControl (RFC 2696 )
  • PagedResultsResponseControl
  • SortControl (RFC 2891 )
  • SortKey
  • SortResponseControl
  • BasicControl

LDAP 명 조작의 지원
이 기능에 의해, 식별명 또는 상대 식별명을 메소드를 사용해 구축하는 경우의 LDAP 명의 조작이 간단하게 됩니다. 또, RFC 2253 의 정의에 따른 캐릭터 라인에 의한 표현에, 이러한 식별명을 포맷 하는 것이 가능하게 됩니다.
  • 이 기능을 지원하기 위해서, javax.naming.ldap 패키지에 다음의 2 개의 새로운 클래스가 추가되고 있습니다.
  • LdapName Rdn

버젼 1.3 에서의 확장 기능

DNS 서비스 프로바이더
Internet Domain Naming System (DNS) 서비스 프로바이더가 추가되었습니다. 이 컴퍼넌트에 의해, DNS 에 포함된 데이터를 어플리케이션으로 읽어낼 수가 있게 됩니다.

LDAP 서비스 프로바이더의 시큐리티 기능의 확장
이 기능확장에 의해, LDAP 서비스 프로바이더가 LDAP 의 표준적인 시큐리티 요건 (RFC 2829 )에 적합하게 되어, 넓게 배치되고 있는 LDAP 서버와의 제휴가 촉진됩니다.

  • 「Start TLS (트랜스폴트층 시큐리티)」의 확장 기능 (RFC 2830 ) 및 외부 SASL 인증 기구의 지원. 기존의 LDAP 접속 경유로 어플리케이션이 안전한 세션을 확립할 수 있게 됩니다. 이 기능을 위해서(때문에), 다음의 새로운 클래스가 도입되었습니다.
    • javax.naming.ldap.StartTlsRequest
    • javax.naming.ldap.StartTlsResponse
  • GSS-API/Kerberos v5 SASL 인증 기구 (RFC 2222 )의 지원. 이 기능에 의해, Windows 2000 및 XP Active Directory 서버에의, 안전해 인증된 액세스가 생기게 됩니다.
  • Digest-MD5 SASL 인증 기구 (RFC 2831 )의 지원.

상세 및 예에 대해서는,「LDAP 서비스 프로바이더」 및「JNDI 튜토리얼」을 참조해 주세요.

corbaname 및 corbaloc URL 의 지원
COS 네임 서비스 프로바이더에서는,Interoperable Naming Service (INS) 스펙 (99-12-03) 이 지원되고 있습니다. 이 기능을 사용하면(자), 어플리케이션으로, 초기 문맥으로부터 「corbaname」의 URL 를 사용하거나 「corbaname」 및 「corbaloc」의 URL 를 사용해 프로바이더를 구성하거나 할 수 있게 됩니다.
