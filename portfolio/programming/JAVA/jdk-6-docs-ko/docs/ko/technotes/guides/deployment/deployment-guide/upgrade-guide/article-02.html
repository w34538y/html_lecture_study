. cab 파일로서 패키지 된 애플릿을 실행하면(자),java.lang.ClassNotFoundException 가 throw 된다


증상

애플릿을 Sun JRE 를 사용한 브라우저로 실행하고 있는 경우,ClassNotFoundException 가 ClassLoader 로 throw 됩니다. 같은 애플릿이 Microsoft VM 에서는 실행할 수 있습니다.

원인

. cab 파일로서 패키지 된 애플릿이 원인입니다. . cab 파일은 Microsoft Windows 전용의 어카이브(archive) 형식이며, Sun JRE 에서는 지원하고 있습니다.

해결 방법

애플릿 클래스와 자원을 . cab 파일로부터 추출해, JDK 의 jar 툴을 사용해 . jar 파일로서 재패키지 합니다.

        jar cvf <jar_file> <input_files>

또, 이 . jar 파일을 지정하도록(듯이), HTML 페이지의<APPLET> 태그의 archive 속성을 변경할 필요도 있습니다. 다음에 예를 나타냅니다.

        <APPLET code="MyApplet" width=100 height=100>
        <PARAM name="cabbase" value="package1.cab, package2.cab"> 
    </APPLET>

        다음과 같이 변경합니다.

<APPLET code="MyApplet" archive="package1.jar, package2.jar" width=100 height=100>
</APPLET>
