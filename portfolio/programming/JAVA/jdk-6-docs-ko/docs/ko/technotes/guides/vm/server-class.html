서버 클래스 머신의 검출

J2SE 5.0 보다, 기동하는 어플리케이션이 서버 클래스 머신상에서 실행되고 있는지 어떤지를 기동 툴로 검출할 수 있게 되었습니다. 서버 클래스 머신상에서 실행되고 있으면, Java HotSpot Client Virtual Machine (클라이언트 VM)는 아니고, Java HotSpot Server Virtual Machine (서버 VM)가 사용됩니다. 이것으로, 실행하고 있는 어플리케이션을 반영하도록(듯이) VM 가 설정되어 있지 않아도, 퍼포먼스를 향상 당합니다. 일반적으로, 클라이언트 VM 보다 서버 VM 의 기동이 저속입니다만, 서서히 빠르게 동작하게 됩니다.

Note:Java SE 6 의 경우, 「서버 클래스 머신」이란, 적어도 2개의 CPU 와 최소에서도 2GB 의 물리 메모리를 갖추고 있는 머신을 가리킵니다.

Java SE 6 에서는, Solaris 또는 Linux 를 실행하고 있는 i586 또는 Sparc 32 비트 머신상에서 어플리케이션을 기동할 경우에,-server 와 -client 의 어느쪽이나 지정되어 있지 않은 경우에, 서버 클래스의 검출을 합니다. 이하의 겉(표)로부터도 알 수 있듯이, i586 Microsoft Windows 플랫폼에서는 디폴트로 클라이언트 VM 를 사용합니다. Sun 가 지원하고 있는 나머지의 플랫폼에서는, 서버 VM 만을 사용합니다.
