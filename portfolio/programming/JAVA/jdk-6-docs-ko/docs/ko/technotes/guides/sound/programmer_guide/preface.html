대상 독자

이 문서는, 다음의 3 개의 그룹의 독자를 대상으로 하고 있습니다.

  • 어플리케이션 개발자:오디오나 MIDI 를 사용하는 어플리케이션 또는 애플릿을 JavaTM 로 기술하는 소프트웨어 프로그래머. 독자의 대부분이 이 카테고리에 해당합니다.
  • 서비스 프로바이더 :Java Sound API (Application Programming Interface) 구현의 기능을 확장한 소프트웨어 모듈의 개발자. 예를 들어, 새로운 오디오 믹서 또는 MIDI 신디사이저를 제공하거나 새로운 파일 형식의 read 및 기입해 기능을 제공하는 벤더는, 이 카테고리에 해당합니다. Java Sound API 의 설계상의 목적은, 특정의 시스템으로 이용 가능한 사운드 관련의 모든 「플러그 인」모듈에 대해서, 프로그램으로부터의 자동 액세스를 가능하게 하는 것입니다.
  • API 의 구현자 :Java Sound API 의 신규 구현을 작성하는 개발자.

이 문서에서는, 독자가, Java 언어에 의한 프로그래밍의 기본적인 지식을 가지고 있는 것을 전제로 하고 있습니다. 오디오나 MIDI 에 관한 지식은 도움이 됩니다만, 전제로 하고는 있지 않습니다.

이 문서의 내용

대부분은 Java Sound API 의 개념의 설명이며, 프로그래밍예로 하고 있고 구두인가의 코드를 나타내고 있습니다. Java Sound API 는, 응용 프로그램에 의한 오디오와 MIDI 엔진과의 통신을 가능하게 하는 소프트웨어층을 지정합니다. Java Sound API 는, JavaTM 2 Platform, Standard Edition (J2SE) 버젼 1.3 의 일부이며, 이 문서로 설명되고 있는 것은 이 버젼입니다. Java Sound API 는, Java 2 Software Development Kit (SDK), Standard Edition 및 Java 2 Runtime Environment, Standard Edition 의 양쪽 모두에 포함되어 있습니다. Java Sound API 의 이전의 구현은, 별개의 제품으로서 제공되고 있었습니다. 그리고, 그 프로그래밍 인터페이스는 여기서 설명하는 것과는 다릅니다.

주:

이 문서는, Java Sound API 의 특정의 구현에 대해 설명하는 것이 아닙니다. 특히, Sun Microsystems, Inc 의 작성한 Java Sound API 의 레퍼런스 구현에 대해 설명하는 문서가 아닙니다. 예를 들어, 레퍼런스 구현으로 지원되고 있는 사운드 파일 형식의 완전한 리스트 등은 여기에는 포함되어 있지 않습니다. Java Sound API 에서는 「플러그 인」서비스가 가능하게 되기 (위해)때문에, API 의 구현자나 서드 파티는 새로운 형식이나 특별한 기능을 가지는 포트등에의 지원를 추가할 수 있습니다. 즉, 이 가이드에서는, 특정의 구현에 고유한 기능, 확장 기능, 제한, 버그 등은 취급하지 않습니다.

개발자의 편의를 위해서(때문에), Sun 의 레퍼런스 구현에 있어서의 몇개의 제한 사항에 대해서도 설명하고 있습니다. 문제가 있는 경우는, 이하를 참조하도록(듯이) 추천합니다.
