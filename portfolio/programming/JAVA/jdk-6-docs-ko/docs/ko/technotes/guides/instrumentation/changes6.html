JDK 6
java.lang.instrument 의 확장 기능

 목차
JDK 5.0 의 초기 릴리스로부터 JDK 6 에의 이행에 수반해,java.lang.instrument 패키지에 다음의 기능이 추가되었습니다.
  • 클래스 파일재변환의 지원 - 벌써 로드 되고 있는 클래스의 동적 변환을 편리화하기 (위해)때문에, 다음의 메소드가 추가되었습니다.
    • Instrumentation.retransformClasses(Class...)
    • Instrumentation.addTransformer(ClassFileTransformer, boolean)
    • Instrumentation.isModifiableClass(Class)
    • Instrumentation.isRetransformClassesSupported()
    이러한 메소드는, 벌써 클래스가 로드 되고 있는 메소드에 Instrumentation을 추가하는 것 같은 조작에 적절하고 있습니다. 재변환의 지원에 의해, 로드 된 클래스를 계측 하기 위해서 원의 클래스 파일에 액세스 할 필요가 없어집니다. 이것에 의해, 적용된 재변환의 삭제가 용이하게 되어, 멀티 에이전트 환경에서 기능합니다.

  • 네이티브 메소드의 계측의 지원 - 다음의 메소드를 사용하면, JVM 에 대응하는 기구로 네이티브 메소드의 랩핑으로 하는 것으로, 네이티브 메소드를 계측 할 수가 있습니다.
    • Instrumentation.setNativeMethodPrefix(ClassFileTransformer, String)
    • Instrumentation.isNativeMethodPrefixSupported()

  • 클래스 로더 검색에의 추가의 지원 - 다음의 메소드에 의해, 해당하는 클래스 로더로 Instrumentation사포트크라스를 정의할 수가 있습니다.
    • Instrumentation.appendToBootstrapClassLoaderSearch(JarFile)
    • Instrumentation.appendToSystemClassLoaderSearch(JarFile)
