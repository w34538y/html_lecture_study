JSP 속성의 사용

Java 인쇄 서비스 API 의 속성은, 종래의 Java 인쇄 API 의 속성과 달리, 프로그램과 사용자 다이얼로그의 양쪽 모두로 사용할 수 있습니다. 어플리케이션에서는, 작업의 인쇄 방법은 가능한 한 지정하지 않고, 최종 사용자가 사용자 다이얼로그로부터 지정한 선택을 검사합니다.

JSP 의 속성은, 확장할 수 있도록(듯이) 설계되고 있습니다. Attribute 인터페이스를 적절히 구현할 수 있는 속성은, 모두 AttributeSet 에 포함할 수가 있습니다. 즉, 표준의 속성 세트를 확장하면서, 향후의 JDK 버젼과의 호환성을 보관 유지할 수가 있습니다. Java 인쇄 서비스를 구현할 때는, 다음의 2 개의 방법으로 속성을 확장할 수 있습니다.

  • 표준 속성 카테고리에 독자적인 서브 클래스를 작성해, 그 카테고리에 추가한 값에 대응한다
  • 새로운 속성 카테고리를 작성한다
사용자 다이얼로그로부터 속성을 사용하려면 , 사용자 인터페이스를 작성할 필요가 있습니다만, 거기에 붙어 이 사용자 가이드에서는 설명하지 않습니다. 같이 JSP 속성의 확장은 서비스 프로바이더가 실시하기 (위해)때문에, 이 사용자 가이드의 예에서는 채택하고 있지 않습니다. 다만, 「문서의 인쇄」에서는, 몇개의 JSP 속성의 기본적인 사용법을 설명해, 그것들을 사용한 간단한 인쇄 어플리케이션에 대해 설명하고 있습니다.
