Java 승인 끝난 표준 오버라이드(override) 기구

 문서의 목차

처음에

승인이 끝난 표준은,Java Community ProcessSM (JCP) 이외의 표준화 과정에서 정의된 Java API 입니다. 승인이 끝난 표준은 JCP 의 외부에서 정의되기 (위해)때문에, Java 2 플랫폼의 릴리스의 사이에 개정될 가능성이 있습니다. 개발자나 소프트웨어 벤더는 승인이 끝난 표준의 오버라이드(override) 기구를 사용해, Sun Microsystems 의 릴리스로 Java 2 플랫폼에 짜넣어지고 있는 것보다 새로운, 개정된 승인이 끝난 표준을 이용할 수 있습니다.

승인이 끝난 표준의 클래스의 보관 장소

개정된 승인이 끝난 표준을 구현하는 클래스는, JAR 파일에 보존할 필요가 있습니다. java.endorsed.dirs 시스템 프로퍼티을 사용해, Java 실행 환경에 대해, 이러한 JAR 파일의 검색처 디렉토리를 1 개 이상 지정합니다. java.endorsed.dirs 로 복수의 디렉토리를 지정하는 경우는, 각 디렉토리의 사이를 File.pathSeparatorChar 로 단락짓습니다. java.endorsed.dirs 에 값을 지정하지 않는 경우, Sun Microsystem 에 의한 Java 2 플랫폼의 구현에서는, 다음의 디폴트의 디렉토리에서 JAR 파일이 검색됩니다. 
<java-home>\lib\endorsed          [Microsoft Windows]
<java-home>/lib/endorsed          [Solaris 또는 Linux]
여기서,<java-home> 는, 런타임 소프트웨어의 인스톨처 디렉토리 (Java SE Runtime Environment 의 톱 레벨 디렉토리 또는 JDK 의 jre 디렉토리)를 가리킵니다.

Java SE 실행 환경에서는, Sun 의 Java 플랫폼에 짜넣어지고 있는 클래스 대신에, 이러한 JAR 파일에 있는 대응하는 클래스를 사용합니다.

승인이 끝난 표준의 API

Java SE 의 승인이 끝난 표준은, 아래와 같은 패키지로 정의 떠날 수 있어 모든 클래스와 인터페이스입니다. 아래와 같은 패키지의 서브 패키지로 정의되고 있는 클래스와 인터페이스는, 서브 패키지 자체가 여기 로 거론되고 있는 경우를 제외해, 승인이 끝난 표준이 아닙니다. 승인이 끝난 표준의 오버라이드(override) 기구를 사용해, 아래와 같은 Java SE 플랫폼의 패키지를 오버라이드(override) 할 수 있습니다. 또, 이러한 패키지를 오버라이드(override) 할 수 있는 것은, Sun 의 릴리스 한 Java 플랫폼이 짜넣어지고 있던 것보다 새로운 승인이 끝난 표준에 한정합니다. 스탠드얼론 테크놀로지

Java SE 의 스탠드얼론 테크놀로지는, 이 섹션에 기재한 테크놀로지에 정의 및 구현되고 있는 클래스 및 인터페이스의 모든 것을 구성합니다. 승인 끝난 표준 오버라이드(override) 기구는 이 일람에 있는 Java 테크놀로지를 오버라이드(override) 하기 위해서 사용되는 경우가 있어, 이러한 테크놀로지는 Java 플랫폼의 원의 구현으로 제공되고 있는 스탠드얼론 테크놀로지의 구현보다 완전하고 새로운 구현을 제공하는 것에 의해서만 오버라이드(override) 됩니다. 이 일람의 테크놀로지와 상기의 「승인이 끝난 표준의 API」에 기재되어 있는 패키지는 예외로, Java SE 플랫폼 API 스펙외의 패키지는 오버라이드(override) 되지 않습니다.
