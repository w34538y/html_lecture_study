Netscape 6


여기에서는, Netscape 6 에 있어서의 Java Runtime 의 주된 기능을 설명합니다. Java Runtime 는 Java Plug-in 에 의해 유효하게 됩니다. 여기서 채택하는 내용은 이하대로입니다.

  • APPLET ,EMBED 및 OBJECT 태그의 지원
  • Java 와 JavaScript 의 사이의 쌍방향 통신
  • RSA 서명 첨부 애플릿의 검증
  • Java 콘솔의 표시
  • Java 의 유효화와 무효화
  • 애플릿의 실행 기간의 변경
  • 프록시 및 Cookie 의 지원
  • HTTPS
  • 자동 다운로드
  • 하위 호환성의 문제에 대해

APPLET 태그의 지원

종래의 APPLET 태그가 사용되고 있는 경우, Java Plug-in 에 의해 애플릿이 로드 됩니다.

주:

현재,OBJECT 요소는 Netscape 6 의 Java Plug-in 에서는 기능하지 않습니다. Netscape 6 에서는 APPLET 또는 EMBED 요소를 사용해 주세요.

Java 와 JavaScript 의 사이의 쌍방향 통신

JavaScript 는 애플릿의 메소드에 액세스 할 수 있어 애플릿은 JavaScript 를 개입시켜 Document Object Model (DOM)에 액세스 할 수 있습니다.

「Java 로부터 JavaScript 에의 통신」 및「JavaScript 로부터 Java 에의 통신」을 참조해 주세요. 시큐리티에 관한 섹션을 반드시 읽어 주세요.

RSA 서명 첨부 애플릿의 검증

RSA 서명 첨부 애플릿의 검증은, 지원되고 있습니다.

Java 콘솔의 표시

Java 콘솔은 Netscape 6 의 브라우저 메뉴 ([Tasks]>[Tools]>[Java Console])를 개입시켜 표시됩니다.

Java 의 유효화와 무효화

Java 의 유효화와 무효화는 Netscape 6 의 브라우저 메뉴 ([Edit]>[Preferences]>[Advanced])를 개입시켜 실시합니다. 이 조작을 유효하게 하려면 , 브라우저를 재기동할 필요가 있는 것에 주의해 주세요.

애플릿의 실행 기간의 변경

페이지가 참조되면(자), 애플릿의 init() 및 start() 메소드가 불려 가 페이지의 참조가 종료하면(자),stop() 및 destroy() 메소드가 불려 갑니다.

프록시 및 Cookie 의 지원

종래, 프록시 및 Cookie 는 Java Plug-in 에 의해 단독으로 지원되고 있었습니다. Netscape 6 에서는 이 지원가 브라우저로 이행했습니다.

HTTPS

HTTPS 는 J2SE 의 Java Secure Socket Extension (JSSE)를 개입시켜 지원되고 있습니다.

자동 다운로드

Netscape 6 에서는, XPInstall 기구를 통해서 Java Plug-in (JRE)의 자동 다운로드를 지원합니다 (Java Plug-in 가 존재하지 않는 경우).

하위 호환성의 문제에 대해

Sun 에서는, Java 2 로 Netscape VM 의 사이의 하위 호환성에 대해 가능한 한의 확인을 실시했습니다만, 완전하게 확인이 끝난 상태가 아닙니다. 애플릿안에는, 그대로 실행 가능한 것이나, 재컴파일이 필요한 만큼의 것도 있습니다만, Java 2 에의 이식이 필요한 것도 있습니다.
