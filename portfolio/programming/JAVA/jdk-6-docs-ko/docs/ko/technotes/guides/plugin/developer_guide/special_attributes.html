특별한 애플릿 속성

애플릿에는, 그 자체에 관련지을 수 있고 있는 디폴트의 속성이 있습니다. image 나 boxmessage 등이 특별한 속성은, 애플릿에 관련지어, 애플릿의 다운로드시의 애플릿 윈도우를 커스터마이즈 할 경우에 사용할 수 있습니다. 이러한 특별한 속성을 사용하면(자), JavaBean 컴퍼넌트를 커스터마이즈 할 수도 있습니다. 이 섹션에서는, 다음의 토픽에 대해 설명합니다.

  • 애플릿의 디폴트의 외관
  • 애플릿의 특별한 속성
  • 애플릿 속성의 우선 순위
  • 트러블 슈팅 시나리오

애플릿의 디폴트의 외관

특별한 속성을 사용하고 있지 않는 경우의 애플릿의 디폴트의 외관은, 다음과 같습니다.

  • Java 의 로고가 애플릿의 영역의 중앙에 표시됩니다.
  • Java 로고 마크를 둘러싸도록(듯이) 희게 빛나는 부분 (인연의 확실치 않은 엔)이 있습니다.
  • 김이 움직이는 애니메이션은, 처리를 하고 있는 것을 나타냅니다.
  • Java 의 로고의 하단에는 진행 상황 바가 있어, 애플릿이 다운로드되는 진행 상황을 나타냅니다.
  • 애플릿의 표시 영역의 사이즈에 의해, 디폴트의 외관은 변화합니다.
  • 애니메이션의 디자인은, 다양한 폭과 높이의 편성에 응해, 확대 축소됩니다.
  • 애플릿의 로드안에 마우스를 애플릿의 영역으로 이동하면(자), 커서가 일반적으로의 하이퍼 텍스트(hyper text) 커서에 변화해, 이 요소가 Web 페이지의 대화적 요소인 것이 나타납니다.
  • 회색의 박스내의 임의의 장소를 클릭 또는 선택하면(자), 다른 브라우저 윈도우가 열려,www.java.com Web 페이지가 표시됩니다.

애플릿의 특별한 속성

image

image 속성을 사용하면(자), 디폴트의 애니메이션을 커스텀 도형으로 옮겨놓을 수가 있습니다. 표준의 APPLET 요소에서는, 다음의 형식에서 이 속성을 사용합니다.

<APPLET ...>
<PARAM name="image" value="my_image.gif">
</APPLET>

OBJECT 태그나 EMBED 태그에서의 사용법에 대해서는,「Java Plug-in 에서의 OBJECT,EMBED, 및 APPLET 태그의 사용」을 참조해 주세요.

커스텀 도형을 지정하는 경우, 그 사이즈는 애플릿 윈도우의 영역과 일치할 필요가 있습니다. 사이즈가 일치하지 않는 경우, 애플릿용으로 지정되고 있는 영역의 좌상구석에 도형이 배치됩니다. 도형이 애플릿 윈도우보다 큰 경우는, 그 일부가 잘라 떨어뜨려집니다. 도형이 애플릿 윈도우보다 작은 경우는, 그 주위에 흰색 (또는 boxbgcolor 로 지정되고 있는 색)의 영역이 생깁니다.

GIF 또는 JPEG 의 이미지를 사용할 수 있습니다. 또, 애플릿의 다른 자원과 같은 디렉토리에 둘 필요가 있습니다. 즉, 애플릿으로 codebase 속성이 사용되고 있는 경우는, 이 이미지를 codebase 디렉토리에 둘 필요가 있습니다.

주: 패키지화된 JAR 파일에는, 애플릿의 다른 자원이 포함됩니다만, 이미지 파일은 자원의 다운로드중에 표시할 필요가 있으므로, JAR 파일에는 넣지 말아 주세요.

마우스로 애플릿 윈도우를 포인트 하면(자), 브라우저의 상태 바에 「Loading Java Applet ...」라고 표시됩니다.

boxmessage

boxmessage 속성에서는, 브라우저의 상태 바에 표시되는 텍스트를 커스터마이즈 할 수 있습니다. 이 속성은,image 속성이 사용되고 있는 경우에 유효하게 됩니다. 표준의 APPLET 요소에서는, 다음의 형식에서 이 속성을 사용합니다.

<APPLET ...>
<PARAM name="boxmessage" value="<your custom message goes here>">
</APPLET>

boxbgcolor, boxfgcolor

이러한 속성을 사용하면(자), 애플릿 윈도우의 배색을 커스터마이즈 할 수 있습니다.

이러한 속성은,image 속성이 사용되고 있는 경우에 유효하게 됩니다. 애플릿 표시 영역의 디폴트의 외관으로서 커스텀 칼라는 지정할 수 없습니다.

디폴트에서는 애플릿 윈도우의 백그라운드 칼라는 흰색입니다. boxbgcolor 속성을 사용하면(자), 다른 백그라운드 칼라를 지정할 수 있습니다. 표준의 APPLET 요소에서는, 다음의 형식에서 이 속성을 사용합니다.

<APPLET ...>
<PARAM name="boxbgcolor" value="<value>">
</APPLET>

<value> 로 지정할 수 있는 값은 이하대로입니다.

  • java.awt.Color 에 의한 임의의 Color
  • r, g, b. r,g, 및 b 는 0-255 의 범위의 정수로,Color 생성자 의 Color(int r, int g, int b) 에 대해 불투명한 표준 RGB (sRGB) 색을 draw 합니다.
  • HTML 의 표준색:silver, green, maroon, purple, navy, teal, 및 olive
  • 16 진형식

각 형식의 예: value="cyan", value="111, 222,145", value="silver", value="#33FF33"

디폴트에서는 애플릿 윈도우의 foreground색은 흑입니다. boxfgcolor 속성을 사용하면(자), 다른 foreground 칼라를 지정할 수 있습니다. 칼라치는 상술한 것 것과 같습니다. 표준의 APPLET 요소에서는, 다음의 형식에서 이 속성을 사용합니다.

<APPLET ...>
<PARAM name="boxfgcolor" value="<value>">
</APPLET>

애플릿 속성의 우선 순위

애플릿의 이러한 속성의 우선 순위는, 다음과 같습니다.

1) 어느 파라미터도 지정되어 있지 않은 경우는, 디폴트의 외관이 사용됩니다 (상기의「디폴트의 외관」을 참조).
image 속성으로 커스텀 도형이 지정되고 있는 경우는, 그 커스텀 도형이 표시됩니다 (상기의「image」를 참조).
boxmessage 속성으로 커스텀 텍스트가 지정되고 있는 경우는, 그 커스텀 텍스트가 표시됩니다 (상기의「boxmessage」를 참조).

image 및 boxmessage 의 각 속성을 어떻게 조합하는 경우에서도,boxbgcolor 및 boxfgcolor 의 각 속성으로 커스텀 칼라를 지정할 수 있습니다.

트러블 슈팅 시나리오

애플릿이 로드에 실패하면(자), 다음의 액션이 실행됩니다.

  • 애플릿 윈도우에 에러 정보가 표시됩니다.
  • 애플릿 윈도우는, foreground 칼라에 흰색을 사용한 1 픽셀폭의 경계를 가져, 좌상구석에는 작은 「망가졌다」도형이 표시됩니다.

애플릿 윈도우를 오른쪽 클릭하면(자), pop-up menu가 표시되어 다음의 옵션의 어느쪽이든을 열 수가 있습니다.

    • Java 콘솔을 연다
    • 윈도우를 열어, Java VM 및 Java Plug-in 에 관한 정보를 표시한다
