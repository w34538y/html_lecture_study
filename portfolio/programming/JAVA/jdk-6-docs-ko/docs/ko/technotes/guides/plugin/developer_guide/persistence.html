애플릿 지속성 API


JDK 1.4 로부터, 새롭게 2 개의 메소드가 java.Applet.AppletContext 인터페이스에 추가되었기 때문에, 복수의 브라우저 세션간에 애플릿의 지속성을 유지할 수 있게 되었습니다. 이러한 메소드를 다음에 듭니다.

  • setStream(String key, InputStream stream)
  • getStream(String key)

애플릿 개발자는, 이러한 신규 메소드를 사용해, 어느 브라우저 세션의 데이터나 객체를 스트림 처리하는 것으로써, 그 후의 브라우저 세션으로 데이터나 객체를 재사용할 수 있게 됩니다. 이것에 의해 애플릿의 지속성이 유지되기 (위해)때문에, 같은 목적으로 애플릿내의 static 객체를 사용할 필요가 없어집니다.

setStream(key, stream) 는, 키를 스트림에 매핑 합니다. getStream(key) 는, 키에 매핑 된 스트림을 돌려줍니다. setStream(key, stream) 의 호출시에, AppletContext 가 키에 매핑 된 스트림을 벌써 보관 유지하고 있는 경우, 매핑은 갱신됩니다.

자세한 것은,「API 스펙」을 참조해 주세요.
