지원되고 있는 인코딩

java.io.InputStreamReader, java.io.OutputStreamWriter, java.lang.String, 및 java.nio.charset 패키지의 클래스를 사용하면(자), Unicode 와 그 외의 몇개의 문자 인코딩간의 변환을 실시할 수가 있습니다. 지원되는 인코딩은, Java 플랫폼의 구현에 따라서 다릅니다. Java Platform Standard Edition 6 의 각 구현에 의한 지원가 필요한 인코딩의 일람은,「java.nio.charset.Charset 클래스의 해설」을 참조해 주세요.

Sun 의 Java SE Development Kit 6 의 전플랫폼 (SolarisTM 오퍼레이팅(operating) 환경, Linux, Microsoft Windows) 판과 Java SE Runtime Environment 6 의 Solaris 및 Linux 판은, 이 페이지에 게재 떠날 수 있어 모든 인코딩을 지원하고 있습니다. Sun 의 Java SE Runtime Environment 6 의 Windows 판은, 완전한 국제판과 유럽 언어판의 어느 쪽의 버젼에서도 인스톨 할 수가 있습니다. JRE 의 인스톨 프로그램은, 호스트 operating system가 유럽 언어만을 지원하고 있는 것을 식별하면(자), 디폴트로 유럽 언어판을 인스톨 합니다. 인스톨 프로그램이 다른 언어의 필요성을 식별했을 경우, 또는 커스터마이즈 된 인스톨로 사용자가 유럽 언어 이외의 지원를 요구했을 경우는, 완전한 국제화판이 인스톨 됩니다. 유럽 언어판에서는, 최초의 겉(표)에 게재되고 있는 인코딩만이 지원됩니다. lib/charsets.jar 파일을 포함한 국제화판에서는, 이 페이지에 나타내는 인코딩이 모두 지원되고 있습니다.

다음의 각 표는, Java SE 6 으로 지원되고 있는 인코딩 세트를 나타냅니다. 새로운 java.nio API 로 사용되는 정 준명은, 많은 경우, java.io API 및 java.lang API 로 사용되는 이름과는 다릅니다.
