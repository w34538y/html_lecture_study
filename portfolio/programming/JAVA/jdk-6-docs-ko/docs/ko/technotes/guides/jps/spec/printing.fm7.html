인쇄 서비스 프로바이더

인쇄 서비스 프로바이더는, Sun 이외의 벤더가 작성하는 기능으로, SPI JAR 파일 스펙에 준거해 추상 클래스 PrintServiceLookup 의 구현을 작성 및 인스톨 합니다.

이 클래스에 의해, 인쇄 서비스의 인스턴스를 검색하는 표준화 된 API 가 제공됩니다. 예를 들어, IPP 인쇄 서비스 프로바이더, JINI 룩 업 서비스, 또는 JNDI 베이스의 디렉토리 서비스는, 이 클래스를 구현하고 있어, PrintServices 를 돌려줍니다.

이 API 는, 비교적 간단하게 사용할 수 있습니다. 일반적으로은, PrintServiceLookup 클래스, PrintService 인터페이스, 및 DocPrintJob 를 구현할 필요가 있습니다. PrintService 에서는, Java 인쇄 서비스 API 에 정의된 이벤트도 지원할 필요가 있습니다.

추가하는 코드량은, 지원되는 문서의 종류에 따라서 다릅니다. 프린터로부터 JPEG 이미지를 인쇄할 경우에, 그 프린터의 하드웨어에 벌써 그 기능이 짜넣어지고 있는 경우는, 데이터를 적절히 spool 하는 코드를 추가할 뿐입니다. 그러나, 2D 그래픽스를 인쇄할 때는, 그래피컬 인터페이스로부터 프린터 서식 설정이 끝난 라스터로 변환하는 클래스가 필요하게 되어, 많은 코드를 추가할 필요가 있습니다.
