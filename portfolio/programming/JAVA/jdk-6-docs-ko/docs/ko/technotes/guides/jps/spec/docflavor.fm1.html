클라이언트 지정 서식 첨부 인쇄 데이터

클라이언트에서는, DocFlavor 를 사용해, 인쇄 데이터 서식 및 인쇄 데이터를 인쇄 서비스에 전달하는 방법을 지정합니다. MIME 타입에는, 데이터 서식을 지정합니다. 표현 클래스명에는, 데이터를 프린터에 전달하는 방법을 지정합니다.

DocFlavor 클래스에는, 사용 빈도의 높은 표현 클래스명의 내부 클래스가 짜넣어지고 있습니다. 각 내부 클래스에는, DocFlavor 객체의 정수 세트가 짜넣어지고 있습니다. 이러한 정수는, 표현 클래스를 사용해 전달할 때의 서식을 표현합니다. 예를 들어, 입력 스트림은, GIF 나 PostScriptTM 등, 다양한 종류의 인쇄 데이터 서식을 스트림 전달할 수가 있습니다. 전자는 DocFlavor.INPUT_STREAM.GIF 로 표현되어 후자는 DocFlavor.INPUT_STREAM.POSTSCRIPT 로 표현됩니다. DocFlavor 객체 정수의 어느 쪽인가에 특정의 인쇄 데이터의 서식을 정확하게 지정해 두면, 클라이언트에서는 DocFlavor 를 구축하는 대신에 이 정수를 사용할 수 있습니다.

적절한 DocFlavor 가 JPS API 에 정의되어 있지 않은 경우는, 클라이언트상에서 DocFlavor 생성자 을 사용해 작성할 수가 있습니다. 클라이언트에서는, 지금부터 설명하는 MIME 타입과 표현 클래스명을 사용해, DocFlavor 를 구축할 수가 있습니다.


서식 설정이 끝난 데이터의 MIME 타입

사용 빈도의 높은 서식 설정이 끝난 데이터의 종류는, 텍스트 데이터, 페이지 기술 언어 문서, 이미지 데이터, 및 자동 감지 인쇄 데이터의 4 개입니다. 여기에서는, 이러한 종류의 데이터에 대해 설명해, 대응하는 MIME 타입의 일람을 나타냅니다.


텍스트 데이터

서식 설정이 끝난 텍스트 데이터는 일반적으로, 문자 배열, String, Reader 등의 문자 지향의 표현 클래스인가, 바이트 배열, 입력 스트림, URL 등의 바이트 지향의 표현 클래스로서 제공됩니다. 서식 설정이 끝난 텍스트 데이터의 일반적인 서식은, plaintext 및 HTML 입니다. DocFlavor 를 구축할 때는, 이하의 MIME 타입 캐릭터 라인을 사용해 데이터 서식을 표현할 수 있습니다.

MIME 타입 데이터 서식

"text/plain"

plaintext. 캐릭터 세트는 디폴트의 US-ASCII

"text/plain;charset=xxx"

plaintext. 캐릭터 세트는 xxx

"text/html"

HTML. 캐릭터 세트는 디폴트의 US-ASCII

"text/html;charset=xxx"

HTML. 캐릭터 세트는 xxx


페이지 기술 언어 문서

서식 설정 끝난 페이지 기술 언어 (PDL) 문서는 일반적으로, 바이트 배열, 입력 스트림, URL 등의 바이트 지향의 표현 클래스로서 제공됩니다. DocFlavor 를 구축할 때는, 이하의 MIME 타입 캐릭터 라인을 사용해 데이터 서식을 표현할 수 있습니다.

MIME 타입 데이터 서식

"application/pdf"

PDF 문서

"application/postscript"

PostScriptTM 문서

"application/vnd.hp-PCL"

PCL 문서


이미지 데이터

서식 설정 끝난 이미지 데이터는, 바이트 배열, InputStream, 또는 URL 등의 바이트 지향의 표현 클래스로서 제공됩니다. DocFlavor 를 구축할 때는, 이하의 MIME 타입 캐릭터 라인을 사용해 데이터 서식을 표현할 수 있습니다.

MIME 타입 데이터 서식
"image/gif"

GIF 이미지

"image/jpeg"

JPEG 이미지

"image/png"

PNG 이미지


자동 감지 인쇄 데이터

프린터는, 서식 설정 끝난 자동 감지 인쇄 데이터로부터 인쇄 데이터의 해석 방법을 판단할 수 있습니다. 이 종류의 데이터는 일반적으로, 바이트 지향의 표현 클래스로서 제공됩니다. DocFlavor 를 구축할 때는, 이하의 MIME 타입 캐릭터 라인을 사용해 데이터 서식을 표현할 수 있습니다.

"application/octet-stream"


표현 클래스

클라이언트 지정 서식 첨부 인쇄 데이터의 표현 클래스는 일반적으로, 이하의 머지않아인가입니다.

인쇄 데이터의 표현 클래스 설명

문자 배열 (char[])

인쇄 데이터는, 배열내의 Unicode 문자로부터 구성된다. 텍스트 데이터 및 PDL 데이터를 표현한다

java.lang.String

인쇄 데이터는, 캐릭터 라인내의 Unicode 문자로부터 구성된다. 텍스트 데이터를 표현한다

java.io.Reader 로 표현되는 문자 스트림

인쇄 데이터는, 스트림 전체로부터 읽힌 Unicode 문자로 구성된다. 텍스트 데이터를 표현한다

바이트 배열 (byte[])

인쇄 데이터는, 배열내의 바이트로부터 구성된다. 바이트는, 드큐먼트후레이바의 MIME 타입으로 지정된 캐릭터 세트에 encode 된다. MIME 타입에 캐릭터 세트가 지정되어 있지 않은 경우, 디폴트의 캐릭터 세트는 US-ASCII 이다. 텍스트 데이터, PDL 문서, 및 이미지 데이터를 표현한다

java.io.InputStream 로 표현되는 바이트 스트림

인쇄 데이터는, 스트림 전체로부터 읽힌 바이트로부터 구성된다. 바이트는, 드큐먼트후레이바의 MIME 타입으로 지정된 캐릭터 세트에 encode 된다. MIME 타입에 캐릭터 세트가 지정되어 있지 않은 경우, 디폴트의 캐릭터 세트는 US-ASCII 이다. 텍스트 데이터, PDL 문서, 및 이미지 데이터를 표현한다

URL, java.net.URL

인쇄 데이터는, URL 의 장소로부터 읽힌 바이트로부터 구성된다. 바이트는, 드큐먼트후레이바의 MIME 타입으로 지정된 캐릭터 세트에 encode 된다. MIME 타입에 캐릭터 세트가 지정되어 있지 않은 경우, 디폴트의 캐릭터 세트는 US-ASCII 이다. 텍스트 데이터, PDL 문서, 및 이미지 데이터를 표현하는 네트워크 인쇄 서비스에 문서를 출력할 경우에, 이 URL 에의 액세스권이 네트워크 인쇄 서비스에 할당할 수 있지 않은 경우는, 이 URL 상에서 입력 스트림을 열어, 입력 스트림 데이터의 후레이바를 대신에 사용한다
