속성 세트

일반적으로, 클라이언트로부터 인쇄 작업을 송신할 때는, 복수의 처리 명령을 건네줄 필요가 있습니다. 예를 들어, A4 매체를 사용해, 횡방향에 인쇄하는 것을 지정합니다. 클라이언트에서는, 복수의 처리 명령을 프린터에 송신하기 위해서, 그러한 정보를 나타내는 속성을 수집해 1 개의 속성 세트에 정리합니다. Java 인쇄 서비스 API 에서는, 속성 세트는 AttributeSet 인터페이스로서 표현됩니다.

AttributeSet 인터페이스는, java.util.Map 인터페이스와 닮아 있습니다. 즉, 값 및 키의 대응 관계를 지정해, 각 키는 일의로, 1 개의 값 밖에 지정할 수 없습니다. 다만, AttributeSet 인터페이스는, Java 인쇄 서비스 API 의 요건에 대응하기 위한 인터페이스입니다. AttributeSet 의 요건은 다음과 같습니다.

  • AttributeSet 내의 각 키는 특정의 카테고리에 대응해, 키의 값은, 그 키가 표현하는 카테고리에 속하고 있는 속성치 가운데 어느쪽이든으로 한다. 따라서 Map 와 달리, AttributeSet 의 키에는, 지정할 수 있는 값이 제한된다. 즉, 속성 카테고리에는, 그 카테고리에 속하지 않은 속성치를 설정할 수가 없다
  • 속성 세트에는, Attribute 인터페이스를 구현하고 있는 속성만을 추가할 수 있다
  • AttributeSet 에는, 특정의 카테고리의 특정의 속성만을 포함할 수가 있다. 예를 들어, 특정의 속성 세트에 Media 속성이 벌써 포함되어 있다고 한다. 이 속성 세트에 다른 Media 속성을 추가하면(자), 원의 Media 속성은 덧쓰기된다
속성 세트에는, 단일 속성과 같게, 기능을 할당할 수 있습니다. 이 기능은, AttributeSet 의 서브 인터페이스에 의해 정의됩니다. AttributeSet 의 서브 인터페이스는, Attribute 의 서브 인터페이스를 닮아 있습니다. 「속성의 기능」을 참조해 주세요. AttributeSet 의 서브 인터페이스는, PrintRequestAttributeSet, PrintJobAttributeSet, DocAttributeSet, 및 PrintServiceAttributeSet 로 구성됩니다. 속성 세트의 기능에 의해, 구성되는 속성의 종류가 정해집니다. 예를 들어, 인쇄 요구의 속성 세트를, 인쇄 요구의 속성만으로 구성할 수 있습니다.

Java 인쇄 서비스 API 에는, AttributeSet 인터페이스의 고정 구현으로서 HashAttributeSet 가 짜넣어지고 있습니다만, 개발자가 독자적인 구현을 짜넣을 수도 있습니다. 많은 어플리케이션에서는, HashAttributeSet 를 직접 사용하지 않고 , HashAttributeSet 의 서브 클래스 (HashPrintRequestAttributeSet, HashPrintJobAttributeSet, HashDocumentAttributeSet, 및 HashPrintServiceAttributeSet)를 사용합니다. 이러한 서브 클래스에는, AttributeSet 의 대응하는 서브 인터페이스가 구현됩니다. 예를 들어, PrintRequestAttributeSet 는 AttributeSet 를 상속해, PrintRequestAttributeSet 의 고정 구현은 HashPrintRequestAttributeSet 가 됩니다. 속성 세트를 작성할 경우에, HashPrintRequestAttributeSet, 또는 PrintRequestAttributeSet 의 임의의 구현을 사용했을 때는, 요구 속성만을 포함할 수가 있습니다.

PrintRequestAttributeSet 의 서브 클래스와 달리, HashAttributeSet 에는 임의의 종류의 속성을 포함할 수가 있습니다. 복수의 종류의 속성을 속성 세트에 포함할 때는, HashAttributeSet 를 어플리케이션으로부터 직접 사용합니다. 예를 들어, PrintService 의 getUnsupportedAttributes 메소드로부터 반환되는 속성 세트를 들 수가 있습니다. 지원되지 않는 속성이 돌려주어졌을 때에, 그 속성은 복수의 속성 기능에 속하고 있을 가능성이 있습니다.

많은 어플리케이션에서는, 단일 속성을 사용하는 경우와 같게, 요구 속성 세트를 작성합니다. 인쇄 요구의 지정은, 인쇄 어플리케이션의 주요한 기능이기 (위해)때문에입니다. 요구 속성 세트를 작성할 때는, HashPrintRequestAttributeSet 클래스의 몇개의 생성자 을 사용합니다. 

PrintRequestAttributeSet aset = new HashPrintRequestAttributeSet();
다음의 마디에서는, 속성 세트에 추가할 수 있는 속성 가운데, 사용 빈도의 높은 것에 임해서 설명합니다.
