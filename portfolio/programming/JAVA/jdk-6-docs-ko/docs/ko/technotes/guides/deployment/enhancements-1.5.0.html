J2SE 5.0 에 있어서의 일반적인 배치 기능

  • Java Plug-in 와 Java Web Start 에 공통되는 기능의 대부분이 결합되었습니다. 양쪽 모두를 설정할 수 있는 단일의 Java 컨트롤 패널이 되었습니다. 이 컨트롤 패널은, Java Plug-in 컨트롤 패널과 Java Web Start 어플리케이션 매니저의 「설정」패널의 양쪽 모두의 기능을 가집니다. 설정은 양제품으로 사용됩니다. (4667682 )

  • 시큐리티:(1) 이 릴리스에서는, 시큐리티 액세스와 시큐리티 제어가 향상했습니다. 사용자 및 시스템 레벨의 시큐리티 옵션과 배치 프로퍼티의 확장된 일람이 포함됩니다. (4667645) (2) 새로운 구성 프로퍼티이 몇개인가 추가되어 기업이 사용자의 시큐리티 설정을 커스터마이즈 할 수 있게 되었습니다. 이것에 의해, 사용자가 신뢰할 수 있는 어플리케이션이나, 신뢰할 수 있는 코드에게 주는 액세스권을 제어할 수 있습니다. 또, 시큐리티 경고로 표시 지난 정보를 구성하거나 JNLP API 의 확장 샌드 박스로 사용자가 어플리케이션을 실행할 수 있을지 어떨지를 설정하거나 할 수가 있습니다. 게다가 기업이 독자적인 증명서를 미리 허가해 두어, 기업 독자적인 어플리케이션으로 시큐리티 경고가 표시되지 않게 하는 일도 가능합니다.

  • 기업용의 구성:이 2 개의 배치 제품에서는, 기업 레벨의 구성 파일을 사용해 구성할 수 있게 되었습니다. 이 파일은 URL 로 액세스 되어 디폴트로서 또는 사용자가 오버라이드(override) 할 수 없게 락 된 프로퍼티으로서 구성 프로퍼티세트를 설정할 수 있습니다.

  • Java 어플리케이션이나 애플릿의 다운로드로 pack/crunch 압축의 이용을 지원 :Pack200 는, JSR-200 로 정의된, JAR 파일을 고압축 할 수 있는 새로운 형식입니다. JNLP 어플리케이션이나 Java Plug-in 애플릿의 다운로드 사이즈를 삭감할 수 있습니다. 파일 사이즈는 원의 사이즈의 대략 8 분의 1 이 됩니다. (4666040)

  • Java Plug-in 와 Java Web Start 에서의 Internet Explorer 및 Mozilla 의 브라우자키스토어의 지원:브라우자키스토어내의 증명서와 열쇠는, 서명의 검증, HTTPS 서버 인증, 및 HTTPS 클라이언트 인증에 사용됩니다. 브라우자키스토어에 공개된 스마트 카드상의 증명서와 열쇠도 인식됩니다. 자세한 것은,「브라우자키스토어」 를 참조해 주세요. (4480333, 4957907)

  • 타임 스탬프 첨부의 서명필 JAR 파일의 지원:타임 스탬프 기능이 서명 툴에 짜넣어져 Java Plug-in 및 Java Web Start 로, 서명필 JAR 파일의 검증시에 서명 시각 정보를 인식할 수 있게 되었습니다. 이것에 의해, 유효한 증명서로 서명되었음에도 관련되지 않고, 애플릿의 배치 후에 증명서가 기한 마감이 되었을 경우에 발생하는, 서명필 애플릿의 문제가 해결되었습니다. (4649690 )

  • HTTPS 에 있어서의 클라이언트 인증의 지원 :Java Plug-in 와 Java Web Start 로, HTTPS 클라이언트 인증중에 사용자가 개인용 증명서를 개인용 키스토어로부터 선택할 수 있도록(듯이), 대화형의 사용자 인터페이스가 준비되었습니다. (4802844)
