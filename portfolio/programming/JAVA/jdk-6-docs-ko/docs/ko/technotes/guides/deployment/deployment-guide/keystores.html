브라우자키스토어

개요

Java Plug-in 및 Java Web Start 는, Internet Explorer 및 Mozilla 의 브라우자키스토어를 지원합니다. 증명서와 브라우자키스토어의 키는 다음의 목적으로 사용됩니다.

  • 서명 검증
  • HTTPS 서버 인증
  • HTTPS 클라이언트 인증

스마트 카드상의 증명서와 열쇠가 브라우자키스토어에 공개되었을 경우도, Java Plug-in 및 Java Web Start 에 의해 인식됩니다.

디폴트에서는, 브라우자키스토어사포트는 유효합니다. 사용자는,Java 컨트롤 패널의 「상세」탭으로 지원를 설정할 수 있습니다.
 

Internet Explorer 키스토어사포트

Windows 에서는, 브라우자키스토어는 Internet Explorer 에 의해 지원됩니다. Internet Explorer 에서의 증명서와 키는, Java 어플리케이션 및 애플릿이 Windows 상에서 배치되었을 때에 Java Plug-in 및 Java Web Start 에 의해 자동적으로 인식됩니다.

이 기능은, Internet Explorer 5.0 이후에서만 지원됩니다.


Mozilla 키스토어사포트

Linux 및 Solaris 에서는, 브라우자키스토어는, Mozilla 에 의해 지원됩니다. Mozilla 에서의 증명서와 키는, Java 어플리케이션 및 애플릿이 Linux 및 Solaris 상에서 배치되었을 때에 Java Plug-in 및 Java Web Start 에 의해 인식됩니다.

Mozilla 키스토어의 지원에는,JSS (Network Security Services for Java)로 불리는 Mozilla 라이브러리가 필요합니다. 디폴트에서는, JSS 는 Mozilla 에 포함되지 않는 개별의 컴퍼넌트입니다. Mozilla 에 JSS 를 인스톨 하는 경우는, 배치자의 책임으로 가 주세요. Java Plug-in 및 Java Web Start 는, JSS 를 배치한 뒤에, Mozilla 키스토어사포트를 적절히 이용 가능하게 합니다.

이 기능은, Mozilla 1.4 이후 및 JSS 3.2 이후에 지원됩니다. 또 Mozilla 및 JSS 가 같은 플랫폼상에서 같은 컴파일러를 사용해 컴파일 되고 있는 것을 전제로 하고 있습니다.

  • Mozilla 에의 JSS 의 인스톨

    Windows

    1. Mozilla 의 인스톨 디렉토리아래에 jss 디렉토리를 작성한다
    2. JSS JAR 파일을 jss 디렉토리에 카피한다
    3. JSS 네이티브 라이브러리 DLL 를 Mozilla 의 인스톨 디렉토리에 카피한다

    Linux/Solaris

    1. Mozilla 의 인스톨 디렉토리아래에 jss 디렉토리를 작성한다
    2. JSS JAR 파일을 jss 디렉토리에 카피한다
    3. JSS 네이티브 라이브러리 . so 를 Mozilla 의 인스톨 디렉토리에 카피한다
    4. Mozilla 기동 스크립트내의 환경 변수 MOZILLA_HOME 를, Mozilla 의 인스톨 디렉토리로 설정한다
    5. Mozilla 기동 스크립트내의 환경 변수 LD_LIBRARY_PATH 에, Mozilla 의 인스톨 디렉토리를 포함하도록(듯이) 변경한다
