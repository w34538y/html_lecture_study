확장

어플리케이션 매니저의 JNLP URL 지원의 개선

JNLP URL 를 JAWS 어플리케이션 매니저로부터 직접 열어, 북마크할 수 있게 되었습니다. 게다가 파일을 . html 또는 . jnlp 로 할 수 있습니다 (RFE 4531885 ).

HTTPS 증명서의 동적 다운로드

Java Web Start 가 브라우저와 같은 방법으로 증명서를 동적으로 임포트 할 수 있게 되어,https 를 한층 더 활용할 수 있게 되었습니다 (RFE 4696477 ). 상세한 것에 대하여는,「Java Web Start 1.4. 2 개발자 가이드」의「HTTPS 증명서의 동적 다운로드」를 참조해 주세요.

JRE 의 자동 검출

JAWS 가 실행되면(자), 매회 컴퓨터상에 「등록」된 모든 JRE 가 검출됩니다.

JAWS 에 인식되는 JRE 에는, 등록을 마친 것과 미등록의 것의 2 개의 타입이 있습니다. 등록을 마친 JRE 는, 시스템이나 패키지의 레지스트리 (Windows 의 시스템 레지스트리, Solaris 의 패키지 데이타베이스, Linux 의 RPM 데이타베이스등)에 기록되고 있습니다. 미등록의 JRE 는 레지스트리에 기록되고 있지 않습니다 (미등록의 JRE 의 네트워크 드라이브에 마운트된 JRE 등) (RFE 4474876 ).

컨트롤 패널의 인프라스트럭쳐(infrastructure)를 통합

Java Plug-in 나 Java Web Start 등, Java 배치의 제품으로 사용하는 컨트롤 패널의 인프라스트럭쳐(infrastructure)를 통합했습니다. Java Web Start 의 캐쉬 배치, 구성 파일의 형식이나 장소등의 문서화되어 있지 않은 기존의 정보를 이용했을 경우, 이것에 의해 영향을 받는 일이 있습니다 (RFE 4723794 ).

Java Web Start 의 시큐리티 감사

Java Web Start 의 시큐리티 감사가 완료했습니다 (RFE 4645808 ).

쇼트 컷의 첫회 메세지

처음 어플리케이션을 다운로드해 캐쉬할 때에, 데스크탑에 쇼트 컷을 작성할지 어떨지를 묻는 메세지가 표시되게 되었습니다 (RFE 4737538 ).

InstallShield 7 과의 통합

Java Web Start 가 JRE 의 일부로서 InstallShield 7 으로 인스톨 되게 되었습니다. 이번으로부터, Java Web Start 의 인스톨러와는 다른 프로그램이 필요하게 될 것은 없습니다 (RFE 4718282 ).
