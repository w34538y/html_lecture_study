속성의 기능

인쇄 작업을 클라이언트로부터 프린터에 송신하면(자), 문서명등의 인쇄 데이터의 특성을 지정한 속성과 양면 인쇄나 인쇄 부수등의 인쇄 데이터의 인쇄 방법도 송신됩니다. 인쇄 작업이 복수의 인쇄 데이터로부터 구성되어 있는 경우는, 인쇄 데이터 마다 다른 처리 명령이 지정되고 있는 일이 있습니다. 예를 들어, 최초의 문서에는 8 × 11 인치의 매체를 사용해, 다음의 문서에는 11 × 17 인치의 매체를 사용한다, 라고 지정되고 있는 일이 있습니다.

프린터가 인쇄 작업의 처리를 개시하면(자), 작업 상태 (완료한 작업, 대기중의 작업등)나 인쇄 끝난 페이지수 등, 다양한 작업 정보를 취득할 수 있습니다. 이러한 정보도 속성입니다. 프린터명, 프린터의 장소, 큐내의 작업수 등, 속성을 사용해 프린터 자체를 제어할 수도 있습니다.

Java 인쇄 서비스 API 에는, Attribute 인터페이스의 다음의 5 개의 서브 인터페이스를 사용해, 다양한 속성의 기능을 정의합니다.

  • PrintRequestAttribute. 인쇄 작업 전체에 적용하는 설정 및 인쇄 방법을 지정한다. 어플리케이션으로 사용한다
  • DocAttribute. 단일 문서의 특성과 그 문서에 적용하는 인쇄 작업의 설정을 지정한다. 어플리케이션으로 사용한다
  • PrintJobAttribute. 작업의 인쇄 상태를 보고한다. 인쇄 서비스로 사용한다. 일반적으로, 이러한 값은, 요구한 속성치와 일치하고 있다. 다만, 요구한 속성치가 프린터로 지원되지 않는 경우는, 그 인쇄 작업의 속성에 프린터로 지원되는 다른 값이 설정된다
  • PrintServiceAttribute. 인쇄 서비스 상태를 보고한다. 인쇄 서비스로 사용된다.
  • SupportedValuesAttribute. 요구된 속성으로 지원되는 값의 범위를 나타낸다. 인쇄 서비스로 사용된다. 예를 들어, 특정의 프린터로, 일정 범위의 부수 밖에 인쇄할 수 없다고 한다. 인쇄 서비스는, 지원되는 부수의 범위에 관한 조회를 수신하면(자), SupportedValuesAttribute 를 구현하고 있는 CopiesSupported 객체에 이 정보를 돌려준다.
많은 경우, 속성은 복수의 기능에 속하고 있습니다. 예를 들어, 「Media」속성은, 문서, 인쇄 작업, 및 인쇄 요구의 기능에 속하고 있습니다. 「Media」속성에는, 용지의 사이즈, 트레이, 또는 종류를 기술할 수 있기 (위해)때문에입니다. 실제, 대부분의 문서 속성은, 요구 속성입니다. 또, 모든 요구 속성은, 작업 속성입니다.

다음의 마디에서는, 속성을 수집해 속성 세트에 정리하는 방법에 대해 설명합니다. 속성 세트에도 기능이 있습니다.
