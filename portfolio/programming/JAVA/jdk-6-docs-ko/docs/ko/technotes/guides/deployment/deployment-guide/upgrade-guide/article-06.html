애플릿을 실행하면(자) java.lang.ClassNotFoundException 가 com.ms. * 패키지로 throw 된다


증상

애플릿을 Sun JRE 를 사용한 브라우저로 실행하고 있는 경우,ClassNotFoundException 가 com.ms. * 패키지의 ClassLoader 로 throw 됩니다. 같은 애플릿이 Microsoft VM 에서는 실행할 수 있습니다.

원인

Microsoft VM 에서는, J/Direct, AFC, WFC 등 많은 독자적인 클래스 라이브러리가 제공되고 있습니다. 이러한 패키지는 Sun JRE 에서는 이용할 수 없기 때문에, 애플릿을 실행하면(자) ClassNotFoundException 가 throw 됩니다.

해결 방법

애플릿의 원시 코드를 Microsoft 독자적인 클래스 라이브러리를 사용한 것으로부터, Java 플랫폼에 있어서의 같은 클래스를 사용한 것으로 이행합니다.

Microsoft VM Java 플랫폼
J/Direct Java Native Interface
Application Foundation Classes Swing, Java 2D API
Windows Foundation Classes Swing, Java 2D API
