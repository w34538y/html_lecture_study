Java Web Start 테크놀러지

Java Web Start 는, Web 브라우저에 관련지어 사용하는 헬퍼 어플리케이션입니다. 사용자가 특별한 기동 파일 (JNLP 파일)에의 링크를 클릭하면(자), 브라우저에 의해 Java Web Start 가 기동됩니다. Java Web Start 는, 지정된 Java 테크놀러지 베이스의 어플리케이션을 자동적으로 다운로드, 캐쉬, 및 실행합니다. 일반적으로은, 모든 처리가 완료할 때까지, 사용자에 의한 개입은 전혀 필요 없습니다. 최초로 한 번 클릭하는 것만입니다.

Java Web Start 는, 기술면에서의 이점을 많이 갖추고 있기 (위해)때문에, 어플리케이션 배치용의 플랫폼으로서 매우 매력적입니다. 그러한 이점을 다음에 열거합니다.

  • Java Web Start 는, Java(TM) Platform Standard Edition 플랫폼전용으로 기술된 어플리케이션만을 기동하도록(듯이) 설계되고 있습니다. 따라서, 어느 단일의 어플리케이션을 특정의 Web 서버상에서 이용 가능하게 하는 것만으로, Windows 98/NT/2000/Me/XP, Linux, SolarisTM 오퍼레이팅(operating) 환경 등, 다양한 플랫폼상에 그 어플리케이션을 배치할 수 있게 됩니다. Java 플랫폼은, 매우 견뢰해 생산성이 높고, 표현력도 풍부한 개발 플랫폼인 것이 증명되고 있습니다. 따라서, Java 플랫폼을 사용하면, 개발과 테스트에 필요로 하는 코스트를 최소한으로 억제할 수가 있어 대폭적인 코스트 삭감이 가능해집니다.
  • Java Web Start 는, 복수 버젼의 Java(TM) Platform Standard Edition 를 지원하고 있습니다. 따라서, 각 어플리케이션은, Java SETM 1.5. 0 등, 자신의 동작에 필요한 특정 버젼의 플랫폼을 요구할 수가 있습니다. 몇개의 어플리케이션을 각각 다른 버젼의 플랫폼상에서 실행해도, 경합은 발생하지 않습니다.
  • 또, Java Web Start 는, 클라이언트 시스템상에 인스톨되어 있지 않은 버젼이 어플리케이션으로부터 요구되었을 경우, 그 버젼을 자동적으로 다운로드 및 인스톨 할 수 있습니다. Java Web Start 를 사용하면, Web 브라우저를 사용하지 않고 어플리케이션을 기동할 수 있습니다. 이 방법을 사용하면, 어플리케이션을 오프 라인으로 실행할 수 있습니다. 오프 라인 상태에서는 일반적으로, 브라우저에 의한 기동은 불편하거나 불가능해 있기도 합니다. 또, 데스크탑의 쇼트 컷 경유로 어플리케이션을 기동하는 일도 가능합니다. 이 방법을 사용하면, 네이티브 어플리케이션과 닮은 조작으로 Web 배치된 어플리케이션을 기동할 수 있습니다.
  • Java Web Start 는, Java 플랫폼에 내장된 시큐리티 기능을 이용합니다. 어플리케이션은 디폴트로, 로컬 디스크와 네트워크 자원에의 액세스가 제한된 보호 환경 (샌드 박스) 내에서 실행됩니다. 이 때문에, 사용자는, 신뢰할 수 없는 소스로부터 입수한 어플리케이션에서도 안전하게 실행할 수 있습니다.
  • Java Web Start 로 기동된 어플리케이션은, 로컬에 캐쉬됩니다. 따라서, 다운로드 끝난 어플리케이션은, 종래의 방법으로 인스톨 된 어플리케이션과 같은 방법으로 기동됩니다.

Java Web Start 의 기반이 되는 테크놀러지는, Java TM Network Launching Protocol & API (JNLP)입니다. 이 테크놀러지는, Java Community Process (JCP)를 통해서 개발이 진행되었습니다. Java Web Start 는, JNLP 스펙의 레퍼런스 구현 (RI)입니다. JNLP 테크놀러지에서는 다양한 표준이 정의되고 있습니다. JNLP 파일로 불리는, 어플리케이션의 기동 방법을 기술하는 파일의 표준 파일 형식도, 그 1 개(살)입니다.

시큐리티

Java Web Start 의 기반으로서 사용되고 있는 Java(TM) SE 플랫폼에는, 포괄적인 시큐리티 아키텍쳐(architecture)가 짜넣어지고 있습니다. Java Web Start 로 기동된 어플리케이션은 디폴트로, 「샌드 박스」라고 불리는 제한된 환경내에서 실행됩니다. 이 환경에서는, 파일이나 네트워크에의 액세스는 제한됩니다. 따라서, Java Web Start 를 사용해 어플리케이션을 기동하면, 시스템의 시큐리티나 무결성을 유지할 수 있습니다.

어플리케이션은 무제한의 액세스를 시스템에 요구할 수 있습니다. 어플리케이션은, 시스템에 대한 무제한의 액세스권을 요구할 수 있습니다. 그 경우, 어플리케이션의 첫회 기동시에, Java Web Start 에 의해 [시큐리티 경고] 다이얼로그가 표시됩니다. 이 시큐리티 경고 다이얼로그에는, 어플리케이션을 개발한 벤더에 관한 정보가 표시됩니다. 그 벤더를 신뢰하는 것을 선택했을 경우, 어플리케이션이 기동됩니다. 어플리케이션의 개발 바탕으로 관한 정보는, 코드의 디지털 서명에 근거하고 있습니다.

Java Web Start 의 입수처

Java Web Start 는,http://java.sun.com 로부터 입수할 수 있는 Java Runtime Environment 에 동고 되고 있습니다. JNLP 의 스펙에 대해서는,「JSR 56:Java Network Launching Protocol and API」를 참조해 주세요.

Java Web Start 소프트웨어의 사용법

Java Web Start 를 사용하면, Java 테크놀러지 베이스의 어플리케이션을 Web 로부터 직접 기동할 수 있습니다. 어플리케이션의 기동 방법에는, 다음의 3 방법이 있습니다.

  • Web 브라우저로부터 링크를 클릭하는 것으로 기동
  • 데스크탑의 아이콘 또는 「스타트」메뉴로부터 기동.
  • Java 캐쉬 viewer로부터 기동

어느 방법을 사용했을 경우도, 어플리케이션이 기동될 때마다, Java Web Start 는 Web 서버에 접속해, 어플리케이션의 갱신판이 있을지 어떨지를 체크합니다.

Web 브라우저로부터의 기동

Web 브라우저에 jnlp 어플리케이션에의 링크를 포함한 페이지를 표시해, 그 링크를 클릭합니다.

어플리케이션의 몇개인가는, 로컬 하드 디스크에의 액세스권 등, 특별한 권한을 필요로 합니다. 이러한 어플리케이션의 경우, 시큐리티 다이얼로그가 pop-up 해, 어플리케이션 코드에 누가 디지털 서명 했는지를 기초를 두어, 어플리케이션의 제공원에 관한 정보가 표시됩니다. 어플리케이션이 실행되는 것은, 그 벤더를 사용자가 신뢰한다고 결정했을 경우만입니다.

이상이 Java Web Start 의 사용법에 관한 방법이라고입니다만, 어떻게 동작하는 것일까요. 어플리케이션을 기동하는 HTML 링크는, 실제의 곳, 단순한 표준의 HTML 링크입니다. 다만, 다른 Web 페이지를 가리키는 대신에, JNLP 파일로 불리는 특수한 설정 파일을 가리키고 있습니다. Web 브라우저는, 그 파일의 파일 확장자(extension) 또는 MIME 타입, 혹은 그 양쪽 모두를 검사해, 그 파일이 Java Web Start 용의 것임을 인식합니다. 그 후에, 다운로드된 JNLP 파일을 인수로서 Java Web Start 를 기동합니다. Java Web Start 는, JNLP 파일의 지시에 따라, 어플리케이션을 다운로드, 캐쉬, 및 실행합니다.  

데스크탑의 아이콘 또는 [스타트] 메뉴로부터의 기동 (Microsoft Windows 및 GNOME 2.0+ 를 실행하는 UNIX 의 경우만)

Java Web Start 테크놀러지에서는, 데스크탑상과 [스타트] 메뉴내에, Web 배치된 Java 테크놀러지 베이스 어플리케이션의 쇼트 컷을 자동 작성할 수 있습니다. 쇼트 컷의 설정의 제어에는, Java 컨트롤 패널을 사용합니다. 쇼트 컷은, Java Web Start 캐쉬 viewer의, 인스톨 쇼트 컷 메뉴 항목을 사용해 추가하는 일도 가능합니다.

프록시 서버-/방화벽(fire wall) 환경에서의 Java Web Start 소프트웨어의 사용

Java Web Start 소프트웨어의 프록시 설정이 올바르지 않았던 경우, 방화벽(fire wall)의 외측에 있는 어플리케이션을 기동할 수 없습니다. Java Web Start 소프트웨어는, 시스템상의 디폴트 브라우저 (Microsoft Windows 상에서는 Internet Explorer 또는 NetscapeTM 브라우저, Solaris 오퍼레이팅(operating) 환경 및 Linux 상에서는 Netscape 브라우저)로부터, 프록시 설정을 자동적으로 검출하려고 합니다. Java Web Start 테크놀러지는, Web 프록시 자동 설정 스크립트의 대부분을 지원하고 있습니다. 거의 모든 환경에서 프록시 설정을 검출할 수 있습니다.

또, Java Web Start 의 컨트롤 패널을 사용해, 프록시 설정을 표시 또는 편집할 수도 있습니다. 자세한 것은,「Java 컨트롤 패널」을 참조해 주세요.

요건

데스크탑/클라이언트 요건

클라이언트 머신은, Java Runtime Environment (JRE)의 버젼 1.3 이후를 지원하고 있을 필요가 있습니다. Java Web Start 는, Windows 98/NT/2000/Me/XP, Solaris 오퍼레이팅(operating) 환경, 및 Linux 상에서 사용할 수 있습니다.

서버 요건

어플리케이션은, 임의의 표준 Web 서버로부터 배치할 수 있습니다. Java Web Start 를 사용하려면 , 새로운 MIME 타입을 지원하도록(듯이) Web 서버를 설정할 필요가 있습니다. 「Web 사이트의 설정」을 참조해 주세요.
