Java™ 제품의 버젼 관리의 스펙

저자: Roger.Riggs@East.Sun.COM

일자: 11/8/02

개방적인 분산 시스템은, 패키지간이 많은 의존관계(dependencies)에 근거해 적확하게 운용할 필요가 있습니다. 그 때문에, 갱신을 주의 깊게 관리할 필요가 있습니다. 분산 시스템내의 변경은, 사용자, 지원 조직, Web 관리자, 및 개발자에게 중대한 영향을 미칩니다. 분산 시스템내의 패키지의 조작은, 시스템 전체 상태에 관한 부분적인 특정의 지식만으로 정확하게 실시할 필요가 있습니다. 시스템의 각 패키지가 갱신되는 속도가 각각 다르기 (위해)때문에, 관리는 그 만큼 어려워집니다. 객체 지향의 설계 수법으로 개개의 패키지의 갱신을 관리해, 패키지간의 의존관계(dependencies)를 명시적으로 관리하는 것으로써, 시스템의 갱신이 가능하게 됩니다. Java 언어는, 일관한 갱신 단위에 적절한 패키지, public 인터페이스만을 공개해, 한편 다른 클래스의 public 인터페이스만을 사용하는 패키지를 정의합니다.
