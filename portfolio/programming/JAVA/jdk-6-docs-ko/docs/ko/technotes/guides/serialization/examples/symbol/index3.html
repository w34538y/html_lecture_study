직렬화와 writeReplace/readResolve 메소드

개설

이 예제에서는 writeReplace/readResolve 메소드의 사용법을 나타냅니다. readResolve 메소드는, 스트림로부터 읽어내진 객체가 호출해 바탕으로 돌려주어지기 전에, 해석 처리를 실시합니다. writeReplace 메소드를 사용하면(자), 객체의 기입을 하기 전에, 그 객체의 Serializable 클래스가 스트림내에서의 스스로의 치환을 지정할 수 있습니다.

이 예제에서는, 각 심볼 바인딩에 인스턴스가 1 개만 존재하는 심볼 클래스를 생성합니다. Symbol 클래스는 readResolve 메소드를 정의합니다. symbollookup 메소드를 사용해, 외부로부터 1 개의 심볼이 생성됩니다. symbollookup 메소드는, 심볼이 벌써 존재하는 경우는 그 심볼을 검출해 돌려주어, 심볼이 존재하지 않는 경우는 심볼을 생성합니다. 이것에 의해, 1개의 VM 내에서 심볼의 일의성이 보증됩니다. 다음에, 심볼을 읽어내기 위해서(때문에) readResolve 메소드가 불려 가면(자), 해시 테이블에 등가인 심볼 객체가 벌써 있는 경우는, 일의성의 제한을 지키기 위해서 기존의 객체로 대용됩니다. 기존의 객체가 없는 경우는, 새로운 심볼이 해시 테이블에 추가되고 나서 돌려주어집니다. 이것에 의해, 복수의 VM 가 심볼을 취급할 때의 일의성이 보증됩니다.
