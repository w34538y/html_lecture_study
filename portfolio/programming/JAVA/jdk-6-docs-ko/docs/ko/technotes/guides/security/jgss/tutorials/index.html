JAAS 및 Java GSS-API 튜토리얼의 소개



이 페이지는, JAAS (JavaTM Authentication and Authorization Service) 및 JavaTM GSS-API 의 다양한 사용 방법을 나타내는 일련의 튜토리얼에 링크하고 있습니다.

JAAS 는, 다음의 2 개의 목적으로 사용할 수 있습니다.

  • 사용자의 「인증」. Java 코드를 실행중의 사용자를, 신뢰성이 높고, 시큐리티가 확보된 방법으로 확인합니다.

  • 사용자의 「승인」. 시큐리티 관련 조작의 실행에 필요한 액세스 제어권 (액세스권)의 보관 유지를 보증합니다.

Java GSS-API 는, 통신 어플리케이션간의 「안전한 메시지 교환」에 사용됩니다. Java GSS-API 에는,RFC 2853 으로 정의된 GSS-API (Generic Security Services Application Program Interface) 용의 Java 바인딩이 포함됩니다. GSS-API 는, 어플리케이션 프로그래머에 대해, Kerberos 를 포함한 기반이 되는 시큐리티 기구상에 배치된 시큐리티 서비스에의 통일된 액세스를 제공합니다.

주:JSSE 는, 시큐리티 보호된 통신에 사용 가능한 다른 API 입니다. 2 개의 API 의 차이점의 자세한 것은,「Java GSS-API 및 JSSE 를 언제 사용할까」를 참조해 주세요.

JAAS 와 Java GSS-API 의 튜토리얼이 양쪽 모두 제공되고 있는 이유는, JAAS 인증은, 일반적으로 Java GSS-API 를 사용한 안전한 통신전에 행해지기 (위해)때문에입니다. JAAS 및 Java GSS-API 는, 관련하고 있어, 자주 병용 됩니다. 다만, 어플리케이션으로부터, Java GSS-API 없음으로 JAAS 를 사용하는 일도, JAAS 없음으로 Java GSS-API 를 사용하는 일도 가능합니다. 게다가 JAAS 자체를 인증에만 사용하는 일도, 인증과 승인의 양쪽 모두에 사용할 수도 있습니다.

이하의 튜토리얼에서는, 상술의 시나리오 모든 실례를 나타냅니다.

  1. JavaTM GSS-API 를 사용한, JAAS 프로그래밍 없음의 안전한 메시지 교환

    Java GSS-API 를 사용해, 클라이언트 어플리케이션과 서버 어플리케이션간의 안전한 메시지 교환을 실시하는 방법을 나타냅니다.

  2. 「JAAS 인증」

    JAAS 를 사용해, 어플리케이션으로부터 사용자의 인증을 실시하는 방법을 설명합니다.

  3. 「JAAS 승인」

    JAAS 를 사용해, 사용자 베이스의 액세스 제어를 실행하는 방법을 설명합니다.

  4. JAAS Login 유틸리티의 사용

    JAAS 를 사용해 사용자를 인증해, 그 사용자로 어플리케이션을 실행하는 유틸리티프로그램에 대해 설명합니다. 어플리케이션의 실행중에, 사용자 베이스의 적절한 액세스 제어를 합니다. 이 유틸리티는, 편의상, 기본적으로 JAAS 인증 및 JAAS 승인 튜토리얼로 설명한 조작을 사용자에 대신해 실행합니다. 이 때문에, JAAS 인증 및 승인을 직접 실행하는 방법을 알 필요가 없는 경우, 전의 튜토리얼을 스킵 해, 이 튜토리얼에 진행되어도 괜찮습니다.

  5. JAAS Login 유틸리티 및 Java GSS-API 를 사용한 안전한 메시지 교환

    가장 종합적인 튜토리얼입니다. Login 유틸리티는, 서비스의 사용자를 인증해, 그 사용자로 서버 어플리케이션을 기동하기 위해서 사용합니다. Login 유틸리티는, 클라이언트의 사용자를 인증해, 그 사용자로 클라이언트 어플리케이션을 기동하는 경우에도 사용합니다. 마지막으로, 클라이언트 및 서버 어플리케이션은, 인증된 클라이언트 및 서비스의 사용자로, Java GSS-API 를 사용한 안전한 메시지 교환을 실시합니다.

  6. Java GSS-API 및 JAAS 로 실행 가능한 다른 조작

    전의 튜토리얼의 서버 어플리케이션으로 클라이언트 어플리케이션과의 통신 확립 후에 실행 가능한, 다른 조작에 대해 설명합니다.

이 시리즈의 튜토리얼내의 어플리케이션은 모두, 인증 및 안전한 통신용의 기반이 되는 기술로서 Kerberos Version 5 를 사용합니다. 자세한 것은,「Kerberos 요건」을 참조해 주세요. 튜토리얼을 통해서, 「Kerberos」라고 하는 말은 Kerberos Version 5 를 가리킵니다.
