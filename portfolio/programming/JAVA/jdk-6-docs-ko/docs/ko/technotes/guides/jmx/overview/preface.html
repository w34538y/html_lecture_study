처음에

「Java Management Extensions (JMX) 테크놀러지 개요」에서는, JMX 테크놀러지를 소개합니다.

본서를 읽기 전에

「Java Management Extensions (JMX) 테크놀러지 개요」의 정보를 충분히 활용하려면 , 다음의 테크놀러지에 관한 지식이 필요합니다.

  • Java 프로그램 언어와 그 개발 환경
  • Java Platform, Standard Edition (Java SE) 소프트웨어

본개요의 일부로, 다음의 프로토콜 및 스펙을 참조하고 있는 개소가 있습니다. JMX 테크놀러지를 보다 깊게 이해하려면 , 이러한 분야에 도착해 조사해 주세요.

  • JavaBeans 모델
  • Remote Method Invocation (RMI)
  • Lightweight Directory Access Protocol (LDAP)
  • Service Location Protocol (SLP)
  • Jini 네트워크 테크놀러지
  • Java Naming and Directory Interface (JNDI) API
  • Java Secure Socket Extension (JSSE)
  • Java Authentication and Authorization Service (JAAS)
  • Simple Authentication and Security Layer (SASL)

본서의 구성

본서에서는, 이하의 장으로,JMX 테크놀러지에 대해 설명합니다.

  • 제 1 장 「JMX 테크놀러지의 소개」 에서는, JMX 테크놀러지에 대해 간단하게 설명해, 주된 이점과 기능을 소개합니다.
  • 제 2장 「JMX 아키텍쳐(architecture)」
  • 제 3 장 「JMX 테크놀러지의 자원의 계측」
  • 제 4장 「JMX 에이전트의 사용」
  • 제 5 장 「JMX 연결기를 사용한 자원의 remote administration」 에서는, JMX 테크놀러지 스펙으로 정의되고 있는 필수 연결기에 대해 설명합니다. 또, 생략 가능한 총칭 연결기에 대해서도 설명하고 있습니다. 이 장에는, 연결기의 시큐리티 기능에 관한 정보가 있습니다.
  • 제 6 장 「검출·검색 서비스」 에서는, 기존의 검출·검색 서비스를 JMX 테크놀러지로 사용하는 방법에 대해 간단하게 설명합니다.
  • 부록  A 「JMX 테크놀러지의 버젼」 에서는, 본버젼의 Java SE 플랫폼에 구현되고 있는 JMX 테크놀러지에 관련하는 Java Specification Requests (JSR)의 버젼을 기재하고 있습니다.
