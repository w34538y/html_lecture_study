복수 버젼의 JRE 및 Java Plug-in 의 지원

이 장에서는, 다음의 토픽에 대해 설명합니다.

  • 일의의 CLSID
  • 일의의 MIME 타입 및 dll
  • 일의의 Java Plug-in 레지스트리 키
  • 일의의 Java Plug-in 프로퍼티 파일
  • 일의의 Java Plug-in 트레이스 및 로그 파일
  • 일의의 Java 컨트롤 패널
  • JRE/JDK 용의 일의의 레지스트리 키
  • 브라우저 세션에서의 복수 버젼의 지원

Java Plug-in 의 모든 신규 또는 패치 릴리스에서는, 레지스트리 키,CLSID, MIME 타입, 및 다른 자원을 일의에 식별하게 되었습니다. 이것에 의해, 같은 환경에 복수의 JRE 버젼을 배치할 수 있습니다.

일의의 CLSID

Java Plug-in 의 모든 신규 또는 패치 버젼에, 일의의 CLSID 가 존재합니다. CLSID 는, Windows 상에서 가동하는 Internet Explorer 의 OBJECT 태그로 사용됩니다. 최소 버젼의 Java Plug-in 를 사용하는 경우,OBJECT 태그로 이 일의인 CLSID 를 지정해 주세요. CLSID 는, 레지스트리내의 다음의 위치에 포함됩니다.

HKEY_CLASSES_ROOT\CLSID\

일반적으로, CLSID 는 다음과 같은 형식이 됩니다.

CAFEEFAC-<major version>-<minor version>-<patch version>-ABCDEFFEDCBA

major version,minor version, 및 patch version 는, 모두 4 자리수의 16 진수입니다.

례:

예를 들어, Java Plug-in 6 의 CLSID 는, 다음과 같이 됩니다.

clsid:CAFEEFAC-0016-0000-0000-ABCDEFFEDCBA

이것은, 다음의 레지스트리에 포함됩니다.

HKEY_CLASSES_ROOT\CLSID\CAFEEFAC-0016-0000-0000-ABCDEFFEDCBA

CLSID 는, 레지스트리내의 다음의 위치에도 포함됩니다.  

HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Code Store Database\Distribution Units\

일의의 MIME 타입 및 dll

신규 또는 패치 버젼의 모든 Java Plug-in 에는, 일의의 MIME 타입 및 NP*.dll 가 존재합니다. MIME 타입은, Java Plug-in 에 의해 로드 및 초기화되는 Java 실행 가능 파일의 타입 또는 버젼을 식별합니다. MIME 타입은, 지원되고 있는 Mozilla 브라우저에서는 EMBED 태그내의 요소형에 의해, Internet Explorer 에서는 OBJECT 태그내의 <PARAM name="type"> 에 의해 지정됩니다. 지원되고 있는 Mozilla 브라우저로 Java Plug-in 의 최소 버젼을 사용하는 경우,EMBED 태그에 일의의 MIME 타입을 지정합니다.

주:

NP*.dll 는, 신규 또는 패치 버젼 마다 일의인 이름을 사용해, MIME 타입을 지원합니다. 이 때문에, 버젼이 다른 Java Plug-in 를 인스톨 해도, 지원되고 있는 Mozilla 브라우저의 Plugins 디렉토리내외의 . dll 파일은 덧쓰기되지 않습니다.

일반적으로, MIME 타입은 다음과 같은 형식이 됩니다.

application/x-java-applet;jpi-version=<version>

version 에는, 메이저, 마이너, 및 패치 버젼 번호가 포함됩니다.

MIME 타입은, 다음의 파일로 지원됩니다.

        NPJPI<modified version number>. dll

례:

Java Plug-in 1.6. 0 에서는, 일의의 MIME 타입은 다음과 같이 됩니다.

application/x-java-applet;jpi-version=1. 6.0

이것은, Mozilla 의 Plugins 디렉토리내의 다음의 파일로 지원됩니다.

NPJPI160.dll

일의의 Java Plug-in 레지스트리 키

Java Plug-in 의 신규 또는 패치 버젼 마다, 일의의 Java Plug-in 레지스트리 키가 존재합니다. 주요한 Java Plug-in 레지스트리 키를 다음에 나타냅니다.

HKEY_LOCAL_MACHINE\Software\JavaSoft\Java Plug-in\<version number>

<version number> 에는, 메이저, 마이너, 및 팟치파젼 번호가 포함됩니다.

례:

Java Plug-in 1.6. 0 에서는, 레지스트리내에 이하의 엔트리가 존재합니다.

HKEY_LOCAL_MACHINE\Software\JavaSoft\Java Plug-in\1. 6.0

일의의 Java Plug-in 프로퍼티 파일

프로퍼티 파일의 이름이,deployment.properties 가 되었습니다. 이 파일은, 다음의 디렉토리에 포함되고 있습니다.

<User Application Data Folder>\Sun\Java\Deployment

<User Application Data Folder> 는, 사용자 고유의 어플리케이션 데이터 폴더입니다.

예를 들어, Windows XP 시스템의 사용자 testuser 의 경우,<User Application Data Folder> 의 값은 C:\Documents and Settings\testuser\Application Data 가 됩니다.

deployment.properties 파일은, 버젼 1.6. 0 에 의해 사용됩니다. <User Application Data Folder> 의 값은,CSIDL_APPDATA 에서의 Win32 API 함수 호출 SHGetFolderPath() 로부터 취득됩니다.

일의의 Java Plug-in 트레이스 및 로그 파일

트레이스 파일 및 로그 파일의 디폴트의 장소는,<User Application Data Folder>\Sun\Java\Deployment\log 입니다.

<User Application Data Folder> 의 정의는,전술했던 대로입니다.

프로퍼티 javaplugin.outputfiles.path 를 사용해, 트레이스 파일 및 로그 파일의 디폴트의 장소를 오버라이드(override) 할 수 있습니다. 자세한 것은,「배치 가이드」의 트레이스 및 로그의 섹션을 참조해 주세요.

일의의 Java 컨트롤 패널

신규 또는 패치 릴리스 마다, 일의의 Java 컨트롤 패널이 존재합니다. 1.6. 0 에서는, 컨트롤 패널의 파일명은 javacpl.exe 입니다.

Java 컨트롤 패널의 최신 인스톨 버젼만이, Windows 의 컨트롤 패널로부터 이용 가능합니다.

JRE/JDK 용의 일의의 레지스트리 키

JRE/JDK 의 신규 또는 패치 릴리스 마다, 일의의 레지스트리 키가 존재합니다. 이러한 레지스트리 키는, 다음의 위치에 배치됩니다.

HKEY_LOCAL_MACHINE\Software\JavaSoft\Java Runtime Environment\<version number>
HKEY_LOCAL_MACHINE\Software\JavaSoft\Java Development Kit\<version number>

<version number> 에는, 메이저, 마이너, 및 팟치파젼 번호가 포함됩니다 (1.6. 0 등).

Java Plug-in 는, 이러한 키를 사용해, 적절한 버젼의 JRE 를 검출할 수 있습니다.

브라우저 세션에서의 복수 버젼의 지원

복수의 JRE 버젼을 Windows 플랫폼에 인스톨 할 수 있습니다. 복수의 JRE 의 애플릿이 있는 경우, MIME 타입 jpi-version 의 사용은, 다음의 옵션을 유효하게 했을 때에 마셔 Mozilla 브라우저로 기능합니다.

  • JRE 6 의「Mozilla 패밀리」옵션을 유효하게 하려면 ,「Java 컨트롤 패널」을 열어,「상세」> 「브라우저의 디폴트의 Java」> 「Mozilla 패밀리」를 클릭합니다.
  • JRE 1.5. 0 이전의「Mozilla 및 Netscape」옵션을 유효하게 하려면 ,「Java 컨트롤 패널」을 열어,「상세」> 「<Applet> 태그의 지원」> 「Mozilla 및 Netscape」를 클릭합니다.

JRE 의 특정의 버젼의 Java 컨트롤 패널을 호출하려면 , 그 버젼의 <JRE_HOME>/bin 디렉토리를 브라우즈 해, javacpl.exe 를 실행합니다.

주:

복수의 버젼의 JRE 를, 다른 브라우저 세션으로 실행할 수 있습니다. 다만, 복수의 버젼을 같은 브라우저 세션으로 실행할 수 없습니다. 실행하려고 하면(자), 사용자에게 경고가 표시되어 시행은 실패합니다.
