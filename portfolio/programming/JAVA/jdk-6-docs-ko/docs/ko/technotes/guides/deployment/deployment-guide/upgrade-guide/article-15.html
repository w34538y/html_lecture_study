. cab 파일로부터 . jar 파일에 업그레이드 한 서명필 애플릿을 Microsoft VM 로 실행하면(자) 시큐리티 예외가 throw 된다


증상

. cab 파일로부터 . jar 파일에 서명필 애플릿을 업그레이드 하면(자), 그 애플릿은 Microsoft VM 로 동작하지 않게 됩니다. Sun JRE 에서는 정상적으로 실행됩니다.

원인

이 에러는, Microsoft VM 에 서명필 . jar 파일의 지원가 없기 위해(때문에) 일어납니다. While the Microsoft VM supports unsigned . jar files through the archive attribute of the <APPLET> tag, classes and resources within signed . jar files are treated as untrusted in the Microsoft VM.

그 결과, 그 애플릿은 Microsoft VM 에 의해 신뢰되어 있지 않은 것으로서 다루어지게 되어, 애플릿 sandbox 외부에 액세스가 필요한 애플릿에 의한 액션은, 시큐리티 예외가 되는 경우가 있습니다.

해결 방법

Microsoft VM 에 의해 인식 가능한 애플릿 서명 스킴은,. cab 파일로 사용하는 Authenticode 만입니다.

회피하려면 , Sun JVM 의 경우는, 업그레이드 했다 . jar 파일을 archive 속성을 통해서 지정합니다. Microsoft VM 의 경우는 그대로,. cab 파일을 cabbase 를 통해서 지정합니다. 다음에 예를 나타냅니다.

<APPLET code="MyApplet" width=100 height=100>
<PARAM name="cabbase" value="package1.cab, package2.cab">
</APPLET>

이 코드를 다음과 같이 변경합니다.

<APPLET code="MyApplet" archive="package1.jar, package2.jar"
width=100 height=100>
<PARAM name="cabbase" value="package1.cab, package2.cab">
</APPLET>
