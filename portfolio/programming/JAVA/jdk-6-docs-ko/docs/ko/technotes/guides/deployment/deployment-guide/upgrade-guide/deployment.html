Sun JRE 의 인터넷/인트라넷 배치

인터넷에서의 배치

작성한 애플릿을 Internet Explorer 로 실행하기 위해서, 인터넷 사용자에게 Sun 의 Java Runtime Enivronment (JRE)를 다운로드받는 경우는, Web 사이트에 Java(TM) 테크놀러지의 로고를 실리는 것을 검토해 주세요. 로고에 대한 자세한 것은, 다음의 Web 사이트를 참조해 주세요.

http://logos.sun.com/logosite.jsp? Category=third#java

사용자가 몇개의 로고를 클릭하면(자), Java Plug-in 를 인스톨 해 실행하는 것을 확인하는 시큐리티 다이알로그 박스가 표시됩니다. 이 Java Plug-in 는, Sun Microsystems 에 의해 서명되어 배포되고 있습니다. 인스톨 하는 것에 동의 했을 경우는, 인스톨러가 다운로드되어 기동합니다. 몇개의 인스톨러 다이얼로그를 조작하면(자), Java Plug-in 가 정상적으로 시스템에 인스톨 됩니다. 인스톨이 성공하면(자), 브라우저에 확인의 메세지가 표시됩니다. 시스템 트레이에는 Java 아이콘이 표시되어, Sun JRE 가 정상적으로 시스템으로 실행되고 있는 것이 나타납니다.

인트라넷에서의 배치

인트라넷 사용자에게 Sun JRE 를 배포하려면 , 내부 Web 사이트에 . msi 파일을 두어, 사용자에게 msiexec 경유로 인스톨러를 기동하도록(듯이) 지시합니다. Active Directory 를 개입시켜 JRE 를 인스톨 하는 방법도 있습니다. Active Directory 를 사용하고 있는 경우는,. msi 파일을 커스터마이즈 (변환을 작성) 해야 하는 경우도 있습니다.

JRE 인스톨러용의 . msi 파일을 입수하려면 , JRE 오프 라인 Windows 인스톨러를 인스톨 합니다. 인스톨 후에, AppDataFolder\jre1. 6.0 디렉토리에 있다 . msi 파일 (jre1. 6.0.msi)를 사용할 수 있습니다. AppDataFolder 는 사용자의 「어플리케이션 데이터」폴더입니다. 일반적으로은,C:\Documents and Settings\username\Application Data 와 같이 됩니다. CLSID 는,7148F0A6-6813-11D6-A77b-00B0D0142010 와 같이 긴 캐릭터 라인으로, 릴리스에 따라서 다릅니다.
