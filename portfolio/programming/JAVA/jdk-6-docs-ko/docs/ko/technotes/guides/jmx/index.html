JMX (Java Management Extensions)

 문서의 목차
Java Management Extensions (JMX) API 는, 어플리케이션, 디바이스, 서비스, Java 가상 머신등의 자원을 관리 및 감시하기 위한 표준 API 입니다. JMX 테크놀러지는, Java Community Process (JCP)를 통해서, Java Specification Request (JSR) 3, Java Management Extensions, 및 JSR 160, JMX Remote API 로서 개발되었습니다.

JMX 테크놀러지의 일반적인 용도에는 다음과 같은 것이 있습니다.

  • 어플리케이션 설정의 참조라고 변경
  • 어플리케이션 동작에 관한 통계의 누적과 통계의 공개
  • 상태 변경과 에러 상황의 통지

JMX API 에는 원격 접근이 포함되어 있기 (위해)때문에, remote administration 프로그램이 이러한 목적으로 실행중의 어플리케이션이라고 대화하는 것이 가능합니다.

개요

  • 개요 - JMX 테크놀러지에 대해 소개합니다.

API 의 스펙

  • API 레퍼런스 - JMX 의 모든 패키지, 클래스, 및 에이전트와 Instrumentation RI 의 멤버에 관한, Javadoc 툴로 생성된 on-line document.

튜토리얼 및 프로그래밍 가이드

  • 튜토리얼 - JMX 테크놀러지의 기능을 몇개인가 예시합니다.
  • 예 - JMX 레퍼런스 구현에는, 다른 분야의 JMX 조작의 코드예를 거두고 있습니다.

여기서 가리키는 예의 그 밖에도, JDK (JDK Development Kit) 6 을 인스톨 하고 있으면, JMX API 의 실제의 구현을 나타내는 샘플 어플리케이션이 다음의 디렉토리에 들어가 있습니다.

JDK_HOME/sample/jmx/jmx-scandir

상기의 패스로,JDK_HOME 는, JDK 소프트웨어가 인스톨 되고 있는 디렉토리를 나타냅니다. jmx-scandir 의 예는, 실제의 시나리오에 있어서의 JMX API 의 고도의 개념을 나타내는 예입니다.

API 의 확장
