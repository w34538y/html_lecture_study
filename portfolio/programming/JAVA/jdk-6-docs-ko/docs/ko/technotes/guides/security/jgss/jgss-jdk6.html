Java Generic Security Services (Java GSS)

및 JavaTM 6 의 Kerberos 확장 기능

Java SE 6 의 Java GSS/Kerberos 구현에는, 다음의 확장 기능이 추가되었습니다. 자세한 것은,JGSS 확장 기능의 리스트를 참조해 주세요.

  • AES 암호화 타입의 지원

    Java GSS/Kerberos 로 AES 암호화 타입 (AES128 및 AES256)을 지원합니다. 이것에 의해, Solaris 10 이나 MIT Kerberos 등의 다른 Kerberos 구현과 Java SE Kerberos 구현의 상호 운용성이 개선되고 있습니다.

  • RC4-HMAC 암호화 타입의 지원

    Java GSS/Kerberos 로 RC4-HMAC 암호화 타입을 지원합니다. 이것에 의해, Windows, Solaris 10 이나 MIT Kerberos 등의 다른 Kerberos 구현과 Java SE Kerberos 구현의 상호 운용성이 개선되고 있습니다. Windows Active Directoiory 는, 디폴트의 Kerberos 암호화 타입으로서 RC4-HMAC 를 지원하고 있습니다.

  • Java GSS 에서의 SPNEGO 의 지원

    Java GSS 로 SPNEGO 기구를 지원합니다. 의사 시큐리티 기구인 Simple and Protected GSS-API Negotiation (SPNEGO) 기구를 사용하면(자), GSS-API 피어는, 사용하는 공통의 시큐리티 기구와 안전하게 협상을 실시할 수가 있습니다.

    HTTP 로 SPNEGO 인증 방식도 지원합니다.

  • 신규의 사전 인증 기구의 지원

    Java GSS/Kerberos 는, 최신의 Kerberos 스펙에 나타나고 있도록(듯이), 신규의 사전 인증 기구를 지원합니다.

  • 네이티브 플랫폼 GSS 통합

    이 기능을 사용하면(자), Java GSS 어플리케이션은, 플랫폼에서 사용 가능한 네이티브 GSS 구현의 기능을 이용할 수 있습니다.
