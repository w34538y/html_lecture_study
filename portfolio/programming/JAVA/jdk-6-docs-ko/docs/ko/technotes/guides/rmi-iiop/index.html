Java RMI over IIOP

 문서의 목차

Java Remote Method Invocation over Internet Inter-ORB Protocol 테크놀러지 ("RMI-IIOP")는, Java Platform, Standard Edition (Java SE)에 포함되어 있는 기능입니다. RMI 프로그래밍 모델을 사용하면(자),rmi API 를 개입시킨 CORBA 서버 및 어플리케이션의 프로그래밍이 가능하게 됩니다. Java Remote Method Protocol (JRMP)를 트랜스폴트로서 사용해 Java 프로그램 언어만으로 작업하는 일도, Internet InterORB Protocol (IIOP)를 사용해 다른 CORBA 대응 프로그램 언어를 병용 해 작업할 수도 있습니다.

RMI-IIOP 에서는 Java CORBA Object Request Broker (ORB)와 IIOP 를 이용하므로, 모든 코드를 Java 프로그램 언어로 쓸 수가 있어 어플리케이션을 임의의 CORBA 대응 언어로 쓰여진 다른 어플리케이션에 Internet InterORB Protocol (IIOP) 경유로 접속하는데 필요한 코드를 rmic 컴파일러를 사용해 생성할 수가 있습니다. 다른 언어로 쓰여진 CORBA 어플리케이션을 사용하는 경우, IDL 는,-idl 옵션을 지정한 rmic 컴파일러를 사용해 Java 프로그램 언어 인터페이스로부터 생성할 수 있습니다. IIOP Stub 및 Tie 클래스를 생성하려면 ,rmic 컴파일러를 -iiop 옵션을 지정해 사용합니다.

상세 정보

Java RMI over IIOP 테크놀러지의 홈 페이지를 참조해 주세요.

Java IDL 테크놀러지의 페이지에는,Java RMI-IIOP 테크놀러지를 사용하는 개발자에게 있어 유용한 많은 정보가 포함되어 있습니다.

OMG 는, CORBA 및 IIOP 관련의 정보 모든 공식 제공원입니다. CORBA 2.3. 1 의 스펙은,formal/99-10-07 로 전자적인 데이터로서 입수할 수 있습니다. CORBA 스펙의 URL 는 변경되는 일이 있습니다. 링크가 망가져 있는 경우는, http://www.omg.org 에 액세스 해 스펙을 검색해 주세요.

Java 플랫폼의 이 릴리스로 구현되고 있는 스펙에 대한 자세한 것은,준거에 관한 문서를 참조해 주세요.

질문이 있는 경우는, RMI-IIOP 테크놀러지의 사용자 써포트 포럼 (http://forum.java.sun.com )을 이용하십시오.
