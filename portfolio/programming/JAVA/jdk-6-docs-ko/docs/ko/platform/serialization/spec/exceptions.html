Version 6.0

객체 직렬화에 있어서의 예외




부록 B

직렬화 클래스에 의해 throw 되는 모든 예외는,IOException 의 서브 클래스인 ObjectStreamException 의 서브 클래스입니다.

예외
 설명

ObjectStreamException

모든 직렬화 예외의 슈퍼 클래스

InvalidClassException

다음의 어떤 것인가의 이유에 의해, 클래스가 객체를 복원할 수 없을 때에 throw 된다

  • 그 클래스가, 스트림에 있는 클래스의 직렬화 버젼과 일치하지 않는다

  • 그 클래스에 무효인 프리미티브(primitive) 데이터형을 가지는 필드가 포함되어 있다

  • Externalizable 클래스에, public 로 인수 없음의 생성자 이 없다

  • Serializable 클래스가, 가장 가까운 비직렬화 가능 슈퍼 클래스의 인수 없음의 생성자 에 액세스 할 수 없다

    • NotSerializableException

    readObject 이나 writeObject 메소드에 의해 throw 되어 직렬화 또는 직렬화 복원을 정지한다

    StreamCorruptedException

    다음의 경우에 throw 된다

  • 스트림 헤더가 올바르지 않다

  • 제어 정보가 발견되지 않는다

  • 제어 정보가 올바르지 않다

  • JDKTM 1.1. 5 이전의 릴리스로 PROTOCOL_VERSION_2 스트림상에 readExternal 를 호출하려고 했다

    • NotActiveException

    다음의 ObjectOutputStream 메소드내에서 writeObject 상태가 올바르지 않은 경우에 throw 된다

  • defaultWriteObject

  • putFields

  • writeFields

    • 다음의 ObjectInputStream 메소드내에서 readObject 상태가 올바르지 않은 경우에 throw 된다

  • defaultReadObject

  • readFields

  • registerValidation

    • InvalidObjectException

    복원된 객체를 유효하게 할 수 없으면 throw 된다

    OptionalDataException

    스트림에 프리미티브(primitive) 데이터가 있어, 객체가 예기 될 때 readObject 에 의해 throw 된다. 예외의 길이 필드는, 현재의 블록으로 사용 가능한 바이트수를 나타낸다

    WriteAbortedException

    스트림의 기입중에 일어난 예외에 의해 정지된 스트림을 읽어들이면(자) throw 된다
