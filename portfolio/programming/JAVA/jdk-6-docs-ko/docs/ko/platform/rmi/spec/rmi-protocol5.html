10.4 RMI 에 있어서의 HTTP POST 프로토콜

방화벽(fire wall)를 경유해 원격 메소드를 호출하기 위해서(때문에), RMI 호출의 몇개인가는 HTTP 프로토콜, 보다 구체적으로는 HTTP POST 를 사용합니다. 포스트헤더로 지정되는 URL 는 다음의 머지않아인가입니다.

http://<host>:<port>/
http://<host>:80/cgi-bin/java-rmi? forward=<port>

최초의 URL 는, 특정의 host 와port 상의 RMI 서버와 직접 통신하기 위해서 사용됩니다. 제2의 URL 형식은, 서버상의 cgi 스크립트가 호출을 특정의 port 상의 서버에 전송 하기 위해서 사용됩니다.

HttpPostHeader 는 POST 요청를 위한 표준 HTTP 헤더입니다. HttpResponseHeader 는 포스트에의 표준 HTTP 응답입니다. 응답의 상태가 200 이외의 경우는,Return 가 없다고 판단됩니다. HTTP POST 요청에 파묻히는 RMI 호출은 1 개 뿐인 것에 주의해 주세요.

HttpMessage:

HttpPostHeader Header Message

HttpReturn:

HttpResponseHeader Return

      주 - SingleOpProtocol 만이 HttpMessage 의 Header 에 표시됩니다. HttpReturn 는 프로토콜의 확인 바이트를 포함하지 않습니다.
