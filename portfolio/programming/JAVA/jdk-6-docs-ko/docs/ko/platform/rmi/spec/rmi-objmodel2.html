2.1 분산 객체 어플리케이션

RMI 어플리케이션은, 많은 경우, 서버와 클라이언트라고 하는 2 개의 프로그램으로부터 구성됩니다. 일반적인 서버 어플리케이션은, 수많은 원격 객체를 작성해, 그러한 원격 객체에의 참조를 액세스 가능하게 해, 클라이언트가 그러한 원격 객체의 메소드를 호출하는 것을 기다립니다. 일반적인 클라이언트 어플리케이션은, 서버내의 1 개 또는 복수의 원격 객체에의 원격 참조를 취득해, 그러한 객체의 메소드를 호출합니다. RMI 는, 서버 및 클라이언트간에 통신이나 정보의 수수를 실시하는 기구를 제공합니다. 이러한 어플리케이션은, 분산 객체 어플리케이션으로 불리는 일이 있습니다.

분산 객체 어플리케이션으로 필요하게 되는 처리는, 다음과 우리입니다.

  • 원격 객체의 검색

    • 어플리케이션은, 원격 객체에의 참조를 취득하기 위해서, 2 개의 기구 가운데 1 개를 사용할 수 있습니다. 어플리케이션은, 원격 객체를 RMI 의 단순한 이름부 기능인 rmiregistry 에 등록하든가, 혹은 일반적으로의 조작의 일부로서 원격 객체 참조를 건네주거나 돌려주거나 할 수가 있습니다.

  • 원격 객체와의 통신

    • 원격 객체간의 통신의 자세한 것은, RMI 에 의해 처리됩니다. 프로그래머로부터는, 원격 통신이 표준적인 메소드 호출로 보입니다.

  • 파라미터 또는 반환값으로서 건네받는 객체의 클래스 바이트 코드의 로드

    • RMI 에서는, 호출측은 원격 객체에 객체를 건네줄 수가 있기 (위해)때문에, RMI 는, 데이터 전송의 경우와 같게, 객체의 코드의 로드에도 필요한 기구를 제공합니다.

이하의 그림은, 원격 객체에의 참조의 취득에 레지스트리를 사용하는 RMI 분산 어플리케이션을 나타내고 있습니다. 서버는, 원격 객체에 이름을 관련짓기 위해서(때문에), 레지스트리를 호출합니다. 클라이언트는, 서버의 레지스트리내의 이름에 의해 원격 객체를 찾아내 그 객체의 메소드를 호출합니다. 이 그림은, RMI 시스템으로, 기존의 Web 서버를 사용해, 서버로부터 클라이언트, 및 클라이언트로부터 서버에, 필요에 따라서 객체용의 Java 언어내의 클래스의 바이트 코드를 로드하는 모습도 가리키고 있습니다. RMI 에서는, Java 플랫폼의 지원하는 임의의 URL 프로토콜 (HTTP, FTP, 파일등)을 사용해, 클래스 바이트 코드가 로드 됩니다.
