8.6 Operation 클래스

Operation 클래스는, 원격 객체용의 Java 언어에서의 메소드의 기술을 보관 유지합니다.

주 - Operation 인터페이스는, Java 2 SDK, Standard Edition, v1. 2 에서는 추천 되고 있지 않습니다. 1.2 의 Stub 프로토콜은,Operation 를 인수의 1 개로서 취하는, 이전의 RemoteRef.invoke 를 사용하지 않습니다. Java 2 SDK, Standard Edition, v1. 2 에서는, Stub는, 파라미터로서 Operation 를 필요로 하지 않는 새로운 invoke 메소드를 사용합니다. 
package java.rmi.server;

public class Operation {
	public Operation(String op) {...}
	public String getOperation() {...}
	public String toString() {...}
}
Operation 객체는, 많은 경우 메소드의 시그니챠 첨부로 생성됩니다.

getOperation 메소드는, 조작 기술자의 내용을 돌려줍니다 (초기화되었을 때의 값).

toString 메소드도 또 조작 기술자의 캐릭터 라인에 의한 표현을 돌려줍니다 (일반적으로는 메소드의 시그니챠).
