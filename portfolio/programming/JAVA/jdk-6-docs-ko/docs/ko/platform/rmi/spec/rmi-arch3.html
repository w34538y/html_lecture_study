3.2 원격 메소드 호출해에 있어서의 thread의 사용법

RMI 런타임이 원격 객체의 구현에 발송 한 메소드는 별도인 thread로 실행되는 경우와 그렇지 않은 경우가 있습니다. RMI 런타임은, 원격 객체 호출과 thread와의 매핑에 대해서는 어떤 보증도 실시하지 않습니다. 동일한 원격 객체에 대한 원격 메소드 호출은 병렬에 실행되는 일이 있기 (위해)때문에, 원격 객체의 구현은, 그 구현이 thread에 대해서 안전한 것을 확인할 필요가 있습니다.
