8.5 Skeleton 인터페이스

Skeleton 인터페이스는,rmic 컴파일러에 의해 작성된 스켈리턴의 구현 에 의해서만 사용됩니다. 원격 객체에 대한 스켈리턴이란, 실제의 원격 객체의 구현에 호출을 발송 하는, 서버측의 엔티티입니다.

주 - Skeleton 인터페이스는, Java 2 SDK, Standard Edition, v1. 2 에서는 추천 되지 않습니다. rmic Stub 컴파일러에 의해 생성된 1.1 의 각 스켈리턴 클래스 ( 및 디폴트의 rmic -vcompat 를 사용해 JDK 1.2 로 생성된, JDK 1.1 호환의 스켈리턴)는, 이 인터페이스를 구현하고 있습니다. Java 2 SDK, Standard Edition, v1. 2 호환의 버젼에서는, 원격 메소드 호출의 발송에, 스켈리턴이 필요없게 되었습니다. 1.2 이후의 버젼과 호환성이 있는 Stub를 생성하려면 , 커멘드 rmic 로 옵션 -v1.2 를 지정해 주세요. 
package java.rmi.server;

public interface Skeleton {
    void dispatch(Remote obj, RemoteCall call, int opnum, long hash)
		throws Exception;
    Operation[] getOperations();
}
dispatch 메소드는,call 객체로 취득한 입력 스트림의 모든 인수를 비정렬화해, 실제의 원격 객체의 구현인 obj 상의 메소드 (조작 번호 opnum 로 나타난다)를 호출합니다. 그리고 반환값을 스트림에 흘리기 (위해)때문에 정렬화하는지, 또는 호출해 안에 예외가 발생했을 경우에는 예외를 throw 합니다.

getOperations 메소드는, 원격 객체의 메소드에 대한 조작 기술자가 쓰여진 배열을 돌려줍니다.
