2.5 원격 인터페이스의 구현

원격 인터페이스를 구현하는 클래스의 일반적인 규약은, 다음과 같습니다.

  • 클래스는 일반적으로 java.rmi.server.UnicastRemoteObject 를 확장하는 것으로써 java.rmi.server.RemoteObject 나 java.rmi.server.RemoteServer 클래스가 제공하는 원격 동작을 상속한다
  • 클래스는 임의의 수의 원격 인터페이스를 구현할 수 있다
  • 클래스는 별도인 원격 구현 클래스를 확장할 수 있다
  • 클래스는 원격 인터페이스에는 나타나지 않는 메소드를 정의할 수도 있지만, 이러한 메소드는 로컬에 대해서만 사용할 수 있어 원격로부터는 사용할 수 없다
예를 들어, 다음에 나타내는 클래스 BankAcctImpl 는 BankAccount 원격 인터페이스를 구현해,java.rmi.server.UnicastRemoteObject 클래스를 확장하고 있습니다. 

package mypackage;

import java.rmi.RemoteException;
import java.rmi.server.UnicastRemoteObject;

public class BankAccountImpl
	extends UnicastRemoteObject
	implements BankAccount
{
	private float balance = 0.0;

	public BankAccountImpl(float initialBalance) 
		throws RemoteException 
	{
		balance = initialBalance;
	}
	public void deposit(float amount) throws RemoteException {
		...
	}
	public void withdraw(float amount) throws OverdrawnException,
		RemoteException {
		...
	}
	public float getBalance() throws RemoteException {
		...
	}
}
필요에 따라서, 원격 인터페이스를 구현하는 클래스는 java.rmi.server.UnicastRemoteObject 이외의 몇개의 클래스를 확장할 수 있습니다. 다만 구현 클래스는, 객체의 export (UnicastRemoteObject 생성자 에 의해 처리된다), 및 java.lang.Object 클래스로부터 상속되는 hashCode,equals, 및 toString 메소드의 올바른 원격 시멘틱스의 구현 (필요한 경우)에 대할 책임을 지지 않으면 안됩니다.
