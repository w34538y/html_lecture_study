2.3 분산 모델과 비분산 모델의 비교

Java SE 플랫폼의 분산 객체 모델은, Java SE 플랫폼의 객체 모델과 다음의 점으로써 자주(잘) 닮았습니다.

  • 원격 객체에의 참조를 인수로서 건네줄 수가 있어 메소드 (로컬이든, 원격여라) 호출의 결과는 반환값으로서 반환된다
  • 원격 객체는 원격 인터페이스의 구현이 지원하는 임의의 형태에 캐스트 할 수가 있다. 이 때 Java 언어에 짜넣어진 형태 변환 구문에 따른다
  • 편입 연산자 instanceof 에 의해, 원격 객체가 지원하는 원격 인터페이스를 테스트할 수가 있다
Java SE 플랫폼의 분산 객체는 Java SE 플랫폼의 객체 모델과 다음의 점으로써 다릅니다.

  • 원격 객체의 클라이언트는, 원격 인터페이스에 제의하는 것에서 만나며, 이러한 인터페이스의 구현 클래스에 제의할 것은 없다
  • 원격 메소드 호출에 대한 원격가 아닌 인수와 원격가 아닌 반환값은 참조 인도는 아니고 카피 인도가 된다. 이 이유는, 객체에의 참조는 단일의 가상 머신상에서만 유효하기 때문에
  • 원격 객체는 참조로 건네받아 실제의 원격 구현이 카피되어 건네받는 것은 아니다
  • java.lang.Object 클래스에서 정의되는 메소드의 몇개의 시멘틱스는 원격 객체용으로 특수화 되고 있다
  • 원격 객체 호출의 실패 모드는, 그 성질로서 로컬 객체 호출의 실패 모드보다 복잡하게 된다. 클라이언트는 원격 메소드 실행시에는, 일어날 수 있는 추가적인 예외 처리를 취급하지 않으면 안 된다
