1.2 시스템의 목표

Java 언어에서의 분산 객체 지원의 목표를 다음에 나타냅니다.

  • 다른 가상 머신상의 객체에의 심리스인 원격 호출을 지원한다
  • 서버로부터 애플릿에의 콜백을 지원한다
  • Java 언어의 객체 시멘틱스를 가능한 한 남기면서, 분산 객체 모델을 자연스러운 방법으로 Java 언어에 통합한다
  • 분산 객체 모델과 Java 플랫폼의 로컬인 객체의 차이를 명료하게 한다
  • 신뢰성의 높은 분산 어플리케이션을 할 수 있는 한 간명에 기술할 수 있도록(듯이) 한다
  • Java 플랫폼의 실행 환경이 가지는 형태의 안전성을 유지하도록(듯이) 한다
  • 원격 객체 참조를 위한 여러 가지의 시멘틱스의 지원. 예를 들어, 라이브 (비지속성의) 참조, 지속 참조, 및 지연 기동
  • 시큐리티 매니저 및 클래스 로더가 제공하는, 안전한 Java 플랫폼 환경의 유지
즉, 이러한 목표로 요구되고 있는 것은, RMI 모델은 단순 (사용하기 쉽게) 또한 자연 (언어에 피트한다)이 아니면 안된다고 하는 것입니다.

이 스펙의 최초의 2 살의 장에서는, Java 언어에 있어서의 분산 객체 모델과 시스템 개요에 대해 기술하고 있습니다. 그 후의 장에서는, Java SE 플랫폼의 일부인 RMI 클라이언트 및 서버 관리 API 에 대해 해설하고 있습니다.
