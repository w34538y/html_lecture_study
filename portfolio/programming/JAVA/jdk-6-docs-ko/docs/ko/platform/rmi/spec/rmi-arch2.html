3.1 Stub와 스켈리턴

RMI 에서는, 원격 객체인 Stub 및 스켈리턴과의 통신에, RPC 시스템으로 이용되는 표준의 기구가 사용됩니다. Stub와 스켈리턴원격 객체의 Stub는, 원격 객체에 대한 클라이언트의 로컬에서의 대리, 즉 프록시의 역할을 완수합니다. 호출측은, 원격 객체에 대한 메소드 호출을 실시하는 로컬 Stub상의 메소드를 호출합니다. RMI 에서는, 원격 객체의 Stub는, 원격 객체가 구현하고 있는 원격 인터페이스세트와 같은 세트를 구현합니다.

Stub의 메소드는, 불려 가면(자), 다음의 동작을 실행합니다.

  • 원격 객체를 포함한 원격 JVM 와의 통신의 개시
  • 원격 JVM 에의 파라미터의 정렬화 (기입 및 전송)
  • 메소드 호출의 결과의 대기
  • 반환값 또는 발행된 예외의 비정렬화 (읽기)
  • 호출 측에치를 돌려주는 것
Stub가, 파라미터의 직렬화 및 네트워크 레벨의 통신을 숨기는 것으로, 호출 측에 대해 심플한 호출 기구가 제공됩니다.

원격 JVM 에서는, 각 원격 객체에 대응하는 스켈리턴이 있습니다. Java 2 플랫폼만의 환경에서는, 스켈리턴은 필요 없습니다. 스켈리턴에 의해, 호출이 실제의 원격 객체의 구현에 발송 됩니다. 스켈리턴은, 메소드 호출을 받으면(자), 다음의 동작을 실행합니다.

  • 원격 메소드용 파라미터의 비정렬화 (읽기)
  • 실제의 원격 객체의 구현상에 있는 메소드의 호출
  • 호출 측에 대하는 결과 (반환값 또는 예외)의 정렬화 (기입 및 전송)
Java 2 SDK, Standard Edition, v1. 2 에서는, Stub 프로토콜이 추가되어 Java 2 플랫폼만의 환경에서는 스켈리턴이 필요없게 되었습니다. JDK 1.1 으로 스켈리턴에 의해 행해진 작업은, 총칭 코드를 사용해 실행됩니다. Stub 및 스켈리턴은,rmic 컴파일러에 의해 생성됩니다.
