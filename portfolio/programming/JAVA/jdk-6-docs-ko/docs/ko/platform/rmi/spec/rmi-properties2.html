B0. 1 서버의 프로퍼티

다음의 겉(표)는, 서버가 설정을 위해서(때문에) 사용하는 프로퍼티의 리스트입니다. 프로퍼티은 일반적으로, 애플릿으로부터 설정되면(자) 제한을 받는 것에 주의해 주세요.

프로퍼티
 설명

java.rmi.server.codebase

JVM 로부터 발생한 클래스의 codebase URL 를 나타낸다. codebase 프로퍼티은, 원격 메소드 호출로 파라미터 또는 반환값으로서 송신된 객체의 클래스가 수신 측에 로드 되도록(듯이), JVM 로부터 「발생한다」클래스의 클래스 기술자에게 정보를 부가한다

java.rmi.server.disableHttp

true 로 설정하면(자), RMI 호출은 HTTP 를 사용할 수 없게 된다. 이 상태에서는, RMI 는 방화벽(fire wall) 경유에서의 호출에 HTTP를 사용하지 않는다. 디폴트 설정은 false (HTTP 를 이용한다)

java.rmi.server.hostname

RMI 에서는, (원격 참조에 파묻힌) 서버의 위치를 지정하는데 IP 주소를 사용한다. RMI 가 로컬 JVM 에 export 된 원격 객체를 사용하기 위해서 호스트명을 사용할 필요가 있는 경우에는, 이 프로퍼티은 완전 수식된 호스트명을 지정하기 위해서 사용된다. 이 프로퍼티은, IP 주소에도 설정할 수 있다. 디폴트는 비설정

java.rmi.dgc.leaseValue

RMI 런타임으로 클라이언트가 JVM 의 원격 객체의 참조가 용서되는 최대 lease 기간을 설정한다. 디폴트 값는 10 분

java.rmi.server.logCalls

true 로 설정하면(자), 서버 호출의 로그가 남겨져 표준 에러 출력에 표시된다. 디폴트에서는 false 로 설정된다

java.rmi.server.useCodebaseOnly

true 로 설정해 두면(자), RMI 가 클래스를 로딩 할 경우에 (CLASSPATH 를 사용할 수 없으면), 프로퍼티 java.rmi.server.codebase 로 지정되는 URL 만을 참조해 로드한다

java.rmi.server.useLocalHostname

java.rmi.server.hostname 프로퍼티이 설정되지 않고, 이 프로퍼티이 설정되어 있는 경우, RMI 에서는, JVM 에 export 된 원격 객체의 (원격 참조에 파묻힌) 위치를 지정하는데 IP 주소는 사용되지 않는다. 그 대신해,java.net.InetAddress.getLocalHost 메소드에의 호출의 값이 사용된다
