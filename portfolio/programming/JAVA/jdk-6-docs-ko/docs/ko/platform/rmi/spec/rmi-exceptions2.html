A. 1 원격 객체의 export에 있어서의 예외

UnicastRemoteObject 를 확장하는 원격 객체 클래스가 작성되었을 때는, 그 객체는 export 됩니다. 이것은, 이 객체가 외부의 Java 가상 머신으로부터의 호출을 받을 수가 있어 RMI 호출에 대해 파라미터 또는 반환값으로서 건네줄 수가 있는 것을 의미합니다. 객체는, 익명 포트 또는 지정된 포트의 어느 쪽에라도 export 할 수 있습니다. UnicastRemoteObject 를 확장한 이외의 객체에서는,java.rmi.server.UnicastRemoteObject.exportObject 를 사용해 객체를 명시적으로 export 합니다.

내용

java.rmi.StubNotFoundException

Stub의 클래스가 발견되지 않는다

Stub가 다음의 어느 쪽인가의 에러를 일으켜, 같은 이름의 클래스와 이름의 충돌을 일으켰다

  • Stub가 인스턴스를 생성할 수 없다

  • Stub의 클래스가 올바르지 않다

    • codebase 가 잘못되어 있기 (위해)때문에 URL 가 올바르지 않다

  • 스켈리턴이 인스턴스를 생성할 수 없다
