A. 3 리턴중의 예외와 에러



예외
내용

java.rmi.UnmarshalException

다음의 조작시에 스트림이 파괴되어 입출력 에러 또는 프로토콜 에러가 발생했다

  • 리턴 헤더의 정렬

  • 리턴 타입의 체크

  • return code의 체크

  • 리턴의 비정렬화

    • 반환값의 클래스가 발견되지 않는다

    java.rmi.UnexpectedException

    메소드시그니챠에 기재되지 않은 예외가 발생 (실행시 예외를 제외하다). UnexpectedException 예외 객체는 서버에 의해 throw 된 기저의 예외를 포함한다

    java.rmi.ServerError

    서버가 원격 메소드를 실행중에 발생하는 에러의 어떤 것인가. ServerError 예외 객체는 서버에 의해 throw 된 기저의 예외를 포함한다

    java.rmi.ServerException

    이 예외는, 서버상에서의 호출해 처리의 실행중에 RemoteException 가 throw 되었을 경우에, 원격 메소드 호출 (인수의 비정렬화 또는 원격 메소드 자체)의 결과로서 throw 된다. 「A. 3.1 java.rmi.ServerException 가 발생하는 원인」을 참조

    java.rmi.ServerRuntimeException

    주 -이 예외는, Java 2 SDK, Standard Edition, v1. 2 에서는 추천 되고 있지 않습니다.

    Java 2 SDK, Standard Edition, v1. 2 로 호환성이 있는 버젼이 동작하고 있는 서버는, 이 예외를 throw 하지 않는다. RuntimeException 가 클라이언트에 그대로 전해진다
