1.1 배경

분산 시스템에서는, 다른 address 공간상, 즉 잠재적으로는 다른 호스트상에서, 실행되고 있는 계산 과정을 서로 통신할 수 있을 필요가 있습니다. 통신을 위한 기본적인 기구로서 JavaTM 언어는 소켓을 지원합니다. 이것은, 유연성이 풍부해 일반적인 통신에는 충분한 기능을 갖추고 있습니다. 그러나, 소켓에서는, 클라이언트와 서버는 정보 교환 메세지를 encode 및 복호화 하기 위해서 어플리케이션 레벨에서의 프로토콜에 집중하지 않으면 안되어, 이러한 프로토콜의 설계는 복잡해 에러를 부르기 쉬운 것입니다.

소켓에 바뀌는 수단으로서 RPC (Remote Procedure Call)가 있어, 이것은 통신 인터페이스를 프로시저 호출의 레벨까지 추상화 한 것입니다. 직접 소켓을 취급하는 것이 아니라, 마치 로컬인 프로시저 호출로서 프로그램 합니다만, 실제로는, 인수는 패키지화되어 호출의 대상인 원격 타겟에 보내집니다. RPC 시스템이, XDR 와 같은 외부 데이터 표현을 이용해, 인수와 반환값을 encode 합니다.

그러나, PRC는, 다른 address 공간에 존재하는 프로그램 레벨 객체간의 통신을 필요로 하는, 분산 객체 시스템에 적합시키려면 난점이 있습니다. 객체 호출의 시멘틱스를 적합시키기 위해서(때문에), 분산 객체 시스템은 RMI (Remote Method Invocation)를 필요로 합니다. 이러한 시스템에서는, 로컬인 대리 객체 (Stub)가 원격 객체 호출을 관리합니다.

이 시방서에 기술되고 있는 Java 플랫폼의 RMI 시스템은, Java 어플리케이션 환경에서 동작시키기 위해서(때문에) 특별히 설계된 것입니다. Java 언어의 RMI 시스템은, Java 가상 머신 (JVM)으로부터 구성되는 동종 환경을 상정하고 있기 (위해)때문에, 이 시스템에서는 Java 플랫폼의 객체 모델의 특성을 최대한으로 활용할 수 있습니다.
