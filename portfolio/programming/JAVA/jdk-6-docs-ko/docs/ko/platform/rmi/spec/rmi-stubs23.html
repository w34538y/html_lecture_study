8.2 RemoteCall 인터페이스

RemoteCall 인터페이스는, 원격 객체의 Stub와 스켈리턴이 원격 객체에의 호출을 실행하기 위한 추상화입니다.

주 - Java 2 SDK, Standard Edition, v1. 2 에서는,RemoteCall 인터페이스는 추천 되고 있지 않습니다. 1.2 의 Stub 프로토콜은, 이 인터페이스를 사용하지 않습니다. Java 2 SDK, Standard Edition, v1. 2 에서는, Stub는, 파라미터로서 RemoteCall 를 필요로 하지 않는 새로운 invoke 메소드를 사용합니다. 
package java.rmi.server;

import java.io. *;

public interface RemoteCall {
	ObjectOutput getOutputStream() throws IOException;
	void releaseOutputStream() throws IOException;
	ObjectInput getInputStream() throws IOException;
	void releaseInputStream() throws IOException;
	ObjectOutput getResultStream(boolean success)
		throws IOException, StreamCorruptedException;
	void executeCall() throws Exception;
	void done() throws IOException;
}
getOutputStream 메소드는, Stub가 인수를 정렬화한다, 또는 스켈리턴이 결과를 정렬화하는 앞에 출력 스트림을 돌려줍니다.

releaseOutputStream 메소드는, 출력 스트림을 해제합니다. 트랜스폴트에 따라서는, 이 메소드는 스트림 자체를 해제합니다.

getInputStream 메소드는, Stub가 결과를 비정렬화한다, 또는 스켈리턴이 인수를 비정렬화하는 앞으로부터 InputStream 를 돌려줍니다.

releaseInputStream 메소드는, 입력 스트림을 해제합니다. 이 결과 몇개의 트랜스폴트는, 접속의 입력측을 조기에 해제할 수가 있습니다.

getResultStream 메소드는, 출력 스트림을 돌려줍니다 (호출이 성공했는지의 여부를 나타내는 헤더 정보를 기입한 뒤). 결과 스트림을 얻을 수 있는 것은, 1 회의 원격 호출에 대해서 1 회 뿐입니다. success 가 true 이면, 전송이기 때문에 정렬화 되는 결과는 정상 종료이며, 그 이외의 결과는 예외가 됩니다. StreamCorruptedException 는, 그 원격 호출에 대해서의 결과 스트림을 벌써 얻을 수 있고 있는 경우에 throw 됩니다.

executeCall 메소드는, 어떠한 값이 주어져도 그 값에 의해 호출을 실행합니다.

done 메소드는, 원격 호출이 완료한 뒤의 클린 업을 허가합니다.
