5.9 RMIFailureHandler 인터페이스

java.rmi.server.RMIFailureHandler 인터페이스는, 서버 소켓의 작성에 실패했을 경우 (객체의 export의 실행중을 제외하다)에, RMI 런타임이 어떻게 대응하면 좋은가를 지정하는 수단을 제공합니다. 

package java.rmi.server;

public interface RMIFailureHandler {

	public boolean failure(Exception ex);
}
failure 메소드는, RMI 런타임이 java.net.ServerSocket 의 작성에 실패했을 때의 예외를 인수로서 불려 갑니다. 런타임이 리트라이 해야 할 경우에 메소드는 true 를 돌려주어, 그렇지 않은 경우에는 false 를 돌려줍니다.

이 메소드가 불려 가기 전에, 실패용 핸들러가 RMISocketFactory.setFailureHandler 의 호출에 의해 등록되지 않으면 안됩니다. 실패용 핸들러가 설정되어 있지 않으면, RMI 런타임은 당분간 기다린 뒤에,ServerSocket 를 재차 작성하려고 합니다.

객체의 최초의 export시에 ServerSocket 의 작성에 실패했을 경우는,RMIFailureHandler 는 불려 가지 않습니다. 작성의 실패가 ServerSocket 상에서 받아들여진 후에,ServerSocket 를 작성하려고 하면(자),RMIFailureHandler 가 불려 갑니다.
