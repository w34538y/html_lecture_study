10.1 개요

RMI 프로토콜은, 와이어상의 형식을 위해서(때문에), 그 밖에 2 개의 프로토콜을 이용합니다. Java 객체 직렬화 프로토콜과 HTTP 프로토콜입니다. 객체 직렬화 프로토콜은, 호출과 반환값의 정렬화 (스트림화)에 사용합니다. HTTP 프로토콜은, 원격 메소드 호출을 POST 해, 조건이 허락하면 반환값을 취득하기 위해서 사용합니다. 각각의 프로토콜은, 다른 문법으로서 문서화 됩니다. 생성 규칙의 비터미널 심볼은, 다른 프로토콜의 제어 규칙을 참조합니다 (객체 직렬화 또는 HTTP). 프로토콜의 경계를 넘는 경우는, 그것 이후의 생성 규칙은, 짜넣어진 프로토콜을 사용합니다.


문법 표기에 대한 주의

  • 여기서 사용하는 표기 방법은 Java 언어 스펙 (Java Language Specification)으로 사용되고 있는 것과 거의 같습니다 (JLS 의 제 2, 3 장 참조).
  • 스트림의 제어 코드는, 16 진수의 리터럴치로 표현합니다.
  • 문법상의 비터미널 심볼은, 메소드 호출로 주어지는 어플리케이션에 특유인 값인 것을 나타냅니다. 이러한 비터미널의 정의는 Java 언어의 데이터형으로부터 구성됩니다. 각각의 비터미널로부터 각각의 형태에의 테이블 매핑은, 문법 규칙에 따릅니다.
