2.7 원격 객체의 위치 결정

단순 bootstrap 네임서버-가 원격 객체에 대한 이름 첨부 참조를 보존하기 위해서 준비되어 있습니다. 원격 객체 참조는,java.rmi.Naming 클래스의 URL 를 베이스로 하는 메소드를 사용해 보존할 수가 있습니다.

클라이언트가 원격 객체의 메소드를 호출할 때는, 클라이언트는 우선 그 객체에 대한 참조를 취득할 필요가 있습니다. 원격 객체에 대한 참조는 일반적으로, 메소드 호출의 파라미터 또는 반환값으로서 얻을 수 있습니다. RMI 시스템은 단순 bootstrap 네임서버-를 제공해 지정된 호스트의 원격 객체를 얻을 수 있도록(듯이) 하고 있습니다. java.rmi.Naming 클래스는, URL (Uniform Resource Lacator) 베이스의 메소드를 제공해 특정의 호스트와 포트에 있어서의 이름과 객체의 페어의 룩 업, 바인드, 리바인드, 언바인드(unbind) 및 리스트를 가능하게 하고 있습니다.
